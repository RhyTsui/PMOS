# Persona 方法卡

- method id: `persona`
- category: `user-research`
- priority: `P0`
- status: integrated
- linked skill: `persona`

## 1. 解决什么问题

当系统只知道“有人要这个功能”，但不知道到底是谁、在什么处境下要、为什么要时，`Persona` 用来把目标对象收清楚。

## 2. 适用场景

- 用户群体不清晰
- 多类利益方诉求混杂
- 需求看起来合理，但缺用户画像支撑

## 3. 不适用场景

- 当前主要问题不是“服务谁”，而是“候选需求如何排序”

## 4. 输入

- 目标人群描述
- 业务场景
- 利益相关方信息
- 已知需求或痛点

## 5. 输出

- 角色画像
- 关键任务
- 主要痛点
- 典型使用场景
- 利益相关方差异

## 6. 产物挂点

- 用户画像资产
- 用户研究摘要
- PRD 用户章节
