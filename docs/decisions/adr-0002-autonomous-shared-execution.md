# ADR-0002: 共享打通与默认自主执行

- 日期：2026-04-11
- 状态：Accepted

## Decision

PMAIOS v0.3 后续执行采用以下协作约定：

- Workspace chat 与 Claude Code / CLI **优先走共享打通**，目标不是“能力大致相似”，而是尽量共享上下文装载、snapshot、run/trace 与会话语义。
- 对于用户已经明确表达的需求，**不再反复怀疑或要求重复确认**。
- 若实现过程中存在不确定项，但该不确定项**不会阻塞主链路**，则直接跳过，继续推进下一个更确定的任务。
- 默认把常规本地开发动作视为**已授权**：包括读文件、改代码、补测试、跑测试、继续推进下一个 v0.3 任务。
- 只有在以下场景才暂停确认：
  - 不可逆或高破坏性操作
  - 对外可见或影响共享系统的操作
  - 需要外部凭据、外部人工输入、或会改变远端状态的操作

## Consequences

- 优先推进共享上下文构建器、共享 snapshot/run 视图、共享 trace 可见性。
- 执行中若某任务受阻过久，不在该点停滞，直接切到下一个可推进的 v0.3 任务。
- 工作流将更偏向持续前进，而不是在局部不确定性上反复停顿。
- “权限默认都同意”在仓库内解释为：**常规本地实现默认可直接执行**；涉及高风险、不可逆、或外部可见动作时，仍保留必要确认。
