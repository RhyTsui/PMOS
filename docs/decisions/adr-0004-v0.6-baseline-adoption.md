# ADR-0004 v0.6 Baseline Adoption

- 日期：2026-04-17
- 状态：Accepted

## Context

仓库当前代码已经不再只是 `v0.3 Workflow OS`。

现状上已经同时存在这些闭环部件：

- workflow / orchestrator execution
- chat -> requirement ingestion
- version registry
- review / metrics / observability
- product management agent governance

但当前真源文档、workflow 元数据、chat truth-source 路径仍主要指向 `v0.3`。

这会导致：

- 当前执行基线与仓库真实能力不一致
- `v0.6` 相关分析文档很多，但没有一个当前生效版本
- 用户和系统都难以判断“现在到底按哪个版本执行”

## Decision

自 `2026-04-17` 起，PMAIOS 当前生效执行基线升级为：

**PMAIOS v0.6 = Release System / Closed-loop Product OS Kernel**

本轮 `v0.6` 的最小可执行范围收敛为：

- Execution
- Requirement
- Version
- Observability
- Product Management Governance

## Consequences

### Adopted

- `workflows/main.md` 与 `workflows/execution.md` 改按 `v0.6` 描述当前执行基线
- `docs/tasks/PMAIOS_v0.6_升级任务清单.md` 作为当前任务真源
- `docs/implementation/PMAIOS_v0.6_核心定义.md` 作为当前实现基线说明
- `src/core/workflowEngine.ts` 的工作流版本元数据升级到 `0.6.0`

### Explicitly not adopted

以下内容仍视为分析输入源或未来候选，不自动成为当前执行基线：

- `docs/implementation/pmaios plan/*`
- 旧的 `.docx` 版本分析
- 超出当前仓库能力边界的 LangGraph/Hermes/Dify/n8n/Meta Layer 完整落地方案

### Guardrails

- 保留现有 stage id，不做破坏式阶段命名迁移
- 不因为版本升级标签而假装已经完成更高版本能力
- 历史 `v0.3` 文档继续保留，用于溯源，不作为当前主真源
