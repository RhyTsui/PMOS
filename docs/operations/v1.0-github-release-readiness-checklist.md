# PMOS v1.0 GitHub 发布前检查清单

- version: v0.1
- date: 2026-05-09
- status: active
- owner: product office

## 1. 目的

这份清单用于回答：

`当前这个 standalone PMOS product repo 能不能上 GitHub。`

## 2. 仓库边界检查

发布前必须确认：

1. 不含 `subprojects/` 业务主体
2. 不含 `docs/sources/inbox/` 原始私有材料
3. 不含 `docs/memory` 下的真实 chats / runs / session 数据
4. 不含本机 `.env`
5. 不含临时日志

## 3. 配置检查

发布前必须确认：

1. 存在 `.env.example`
2. 存在 `config/providers.example.json`
3. 不存在真实 provider key
4. 不存在真实 Notion token
5. 不存在企业内部代理强绑定作为默认公开配置

## 4. 文档检查

发布前必须确认：

1. `README.md` 可读
2. `docs/deployment/first-run.md` 可读
3. `docs/deployment/environment-variables.md` 可读
4. `docs/operations/platform-truth-source-index.md` 可读
5. `docs/operations/current-version-progress.md` 可读
6. `CHANGELOG.md` 可读

## 5. 运行检查

发布前必须确认：

1. `npm run lint` 通过
2. `npm run build` 通过
3. `npm start` 可启动
4. `/api/health` 返回 `200`
5. 根路径返回 `200`

## 6. 云端知识检查

发布前必须确认：

1. `cloud-mirror/` 样例已存在
2. GitHub 被定义为 system of record
3. Notion 被定义为 operator cockpit
4. `latest-state` 字段结构已明确

## 7. 版本口径检查

发布前必须确认：

1. active product version = `v1.0`
2. landed runtime baseline = `v0.7`
3. 当前仓是首版 standalone product repo
4. README 没有把它误写成完整母仓

## 8. 当前判断模板

发布前可按以下三类结论打标：

1. `ready`
2. `conditional`
3. `blocked`

## 9. 当前状态建议

基于截至 `2026-05-09` 的现状：

1. 静态验证：`ready`
2. 启动验证：`ready`
3. 第二机验证：`conditional`
4. GitHub 发布：`conditional`

原因：

1. 本机 lint/build/start 已通过
2. 第二机部署验证尚未执行
3. 公开仓内容还需最终人工再扫一轮私有信息
