# 数据 / 广告 / 企业系统 UI 规范

- version: v1.1
- status: active
- date: 2026-05-08
- scope: 数据平台、广告投放平台、企业后台、运营工作台、报表系统

## 1. 目的

这份规范不是重新发明一套视觉，而是把已有 UI 规范、组件说明和企业系统经验整理成一份可执行真源。

它要同时解决 4 个问题：

1. 规范存在，但落地口径不统一
2. 字体、圆角、间距、状态、表格规则经常漏写
3. 同类页面各做各的，文档质量发散
4. AI 容易只看页面结构，不理解企业系统的信息密度、组件语义和状态约束

## 2. 适用范围

默认适用于以下页面类型：

- 报表页
- 数据看板页
- 列表页
- 详情页
- 配置页
- 表单页
- 素材页
- 广告投放管理页
- 权限 / 人员 / 组织管理页

默认不适用于：

- 品牌官网
- 营销 landing page
- 偏展示型活动页

## 3. 总体原则

整体设计语言应保持：

- 简洁
- 专业
- 高密度但不拥挤
- 数据优先
- 操作优先
- 状态清晰
- 组件复用优先

禁止：

- 大面积装饰性渐变
- 无依据的圆角、阴影、动效
- 一个页面出现多套按钮和表单风格
- 用“高级感”牺牲数据可读性
- 页面只好看但不可执行

## 4. 会话类产品例外规则

如果产品是 `会话 / chat / agent 对话工作区`，默认组件基线切换为：

`Ant Design X + 企业级 token 约束`

适用区域：

- 会话列表
- 消息气泡
- 输入框
- 建议词
- 推理过程
- 附件区
- 引用来源
- 弱通知

规则：

- 聊天工作区默认优先使用 `Ant Design X`
- 页面外壳、后台工作台、表格、筛选、表单、配置台，仍受本规范约束
- 不允许把聊天区组件风格直接外溢到整个企业后台
- `Notification` 只作为弱通知，不替代页面内主反馈和主状态流

## 5. 视觉基线

### 5.1 字体规范

- 中文主字体：`PingFang SC`
- 英文 / 数字辅助字体：`Inter`
- 代码 / ID / 技术字段：`JetBrains Mono` 或同类等宽字体
- fallback：`"PingFang SC", "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif`

### 5.2 字号层级

| 层级 | 建议值 | 用途 |
| --- | --- | --- |
| Page Title | `20px / 600` | 页面标题 |
| Section Title | `16px / 600` | 模块标题 |
| Card Title | `14px / 600` | 卡片标题 |
| Body | `14px / 400` | 正文、表单、列表常规文本 |
| Secondary | `12px / 400` | 辅助说明、筛选说明、时间说明 |
| Caption | `12px / 400` | 标签补充、表格弱信息 |
| Metric Large | `24px / 600` | 核心指标数字 |
| Metric Medium | `18px / 600` | 次级指标数字 |

规则：

- 同一页面不允许随意出现多个近似字号层级
- 标题、正文、辅助文字必须一眼分层
- 表格和表单以可读性优先，不追求花哨字重变化

### 5.3 颜色规范

- Primary：`#3B86F9`
- Primary Hover：`#94CAFC`
- Text Primary：`#252934`
- Text Secondary：`#6B7981`
- Border：`#E5E6EB`
- Surface Muted：`#F9FAFC`
- Hover Surface：`#FAFAFA`
- Base White：`#FFFFFF`

状态色：

- Success：稳定绿色体系
- Warning：稳定橙黄体系
- Danger：稳定红色体系
- Info：复用主色或蓝色弱化态

规则：

- 主色只给主操作、选中态、关键信息强调
- 弱说明不要和主标题抢层级
- 图表颜色必须来自固定 palette，不允许随机取色

### 5.4 圆角规范

| 场景 | 建议值 |
| --- | --- |
| Button | `6px` |
| Input / Select / DatePicker | `6px` |
| Card | `8px` |
| Modal | `10px` |
| Drawer 内卡片 | `8px` |
| Tag / Badge | `999px` 或 `12px` |

规则：

- 一个页面不要混用过多圆角体系
- 企业数据系统默认偏克制，不要大圆角泛滥
- 卡片、输入框、按钮圆角需要成组一致

### 5.5 间距规范

| 场景 | 建议值 |
| --- | --- |
| 页面外边距 | `24px` |
| 大模块间距 | `24px` |
| 卡片内边距 | `16px` 或 `20px` |
| 表单项垂直间距 | `16px` |
| 控件横向间距 | `8px` / `12px` |
| Filter Row Gap | `12px` |
| Button Group Gap | `8px` |
| Table Cell Padding | `12px 16px` |

规则：

- 页面结构按 `8pt` 体系收口
- 筛选区、表格区、操作区之间必须固定节奏
- 不允许某页非常松、某页非常挤

### 5.6 边框与阴影

- Border Width：`1px`
- 默认 Border Color：`#E5E6EB`
- 卡片阴影：轻阴影或无阴影
- 弹层阴影：中轻度阴影

规则：

- 数据系统不需要厚重阴影
- 优先靠层级、边框、间距，而不是夸张阴影区分模块

## 6. 页面级规范

### 6.1 标准页面骨架

推荐顺序：

`页头 -> 筛选区 -> 操作区 -> 主要内容区 -> 分页/底部操作`

页头默认包含：

- 页面标题
- 一行辅助说明
- 必要时的右侧全局操作

### 6.2 列表 / 报表页

必须具备：

- 页头
- Filter 区
- 主操作区
- Table 区
- 分页区
- empty / loading / error 状态

### 6.3 详情页

必须具备：

- 返回入口
- 标题与状态
- 信息分组
- 主操作区
- 相关记录 / 日志 / 关联表格

### 6.4 配置 / 表单页

必须具备：

- 表单分组
- 明确的必填和校验
- 固定底部操作区
- 成功 / 失败反馈

### 6.5 看板 / 概览页

必须具备：

- 全局筛选
- 指标卡
- 趋势图 / 分布图
- 明细表
- 图表与表格联动口径

## 7. 核心组件规范

### 7.1 按钮

- 高度建议：`32px / 36px / 40px`
- 主按钮：仅保留一个主焦点
- 次按钮：用于取消、返回、普通触发
- 危险按钮：仅用于删除、清空、移除
- 文本按钮：表格行内操作优先

必须具备状态：

- default
- hover
- active
- disabled
- loading

### 7.2 输入框 / 下拉 / 搜索

- 高度建议：`32px` 或 `36px`
- radius：`6px`
- placeholder 颜色必须弱于正文
- focus 态必须明确
- error 态必须有边框与提示

### 7.3 表格

- 标准行高：`40px`
- 图文表格行高：`60px`
- 表头支持排序、提示、列宽调整
- 维度字段左对齐
- 指标字段右对齐
- 操作列居中或右对齐，但项目内必须统一
- 空值使用 `-` 或约定占位
- 分页默认 `20`

### 7.4 筛选区

- 控件高度统一
- 查询 / 重置按钮位置统一
- 多条件筛选优先横向分组
- 条件很多时使用复合筛选器，不要硬堆一行

### 7.5 表单

- label 对齐统一
- 校验提示位置统一
- 必填标记统一
- 复杂表单必须分组

### 7.6 Modal / Drawer

- 轻量确认：`Modal`
- 详情查看 / 中等编辑：`Drawer`
- 大流程 / 大表单：新页面

### 7.7 状态组件

必须有：

- loading
- empty
- error
- no-permission
- success feedback

## 8. 数据与广告系统专项规则

### 8.1 数据系统

- 数字、时间、百分比、金额必须格式统一
- 表头说明比装饰更重要
- 图表颜色先服务区分度，再考虑美观

### 8.2 广告系统

- 账户 / 应用 / 媒体 / 团队 / 活动 / 广告 / 素材 这些对象层级要稳定
- Filter 区必须支持高频多维查询
- 表格要适配预算、消耗、转化率、CTR、CVR 等指标
- 图文表格与素材缩略图要统一规格
- 对比、趋势、留存、漏斗类图表必须统一口径

### 8.3 企业后台

- 权限信息、系统配置、组织管理不要做成营销页风格
- 信息密度可以高，但交互不能乱
- 每一屏都要有明确的主任务

## 9. 组件库对齐

### 9.1 企业后台默认映射

- 页面容器：`YkContainer`
- 栅格 / 卡片布局：`FlexGrid`
- 右侧抽屉：`YkDrawer`
- 空状态：`Empty`
- 通用筛选区：`ModCommonFilter`
- 图表：`YkCharts`
- 表格主题：`TableTheme`
- Tooltip：`YkTooltip`
- 图标：`YkIcon`
- Tabs：`YkTabs`

### 9.2 Chat 产品默认映射

当页面属于 `会话 / agent / chat workspace` 时，默认优先使用：

- `Conversations`
- `Bubble`
- `Sender`
- `Suggestion`
- `Prompts`
- `ThoughtChain` / `Think`
- `Attachments`
- `Sources`
- `Notification`

无法确认时：

- 标记为 `待补组件确认`
- 不要凭空猜一个组件名

## 10. AI 执行硬规则

AI 生成或整改企业数据系统页面时，必须先回答：

1. 这是什么页面类型
2. 它的主任务是什么
3. 该用哪类表格 / 表单 / 筛选 / 弹层
4. 字体层级是什么
5. 圆角体系是什么
6. 间距体系是什么
7. 状态如何处理
8. 是否已映射到 YKUI 或 Ant Design X 组件

如果答不出来，就不应该直接出图或改代码。

## 11. 最低验收清单

- 是否定义了字体体系
- 是否定义了字号层级
- 是否定义了圆角体系
- 是否定义了间距体系
- 是否定义了边框和阴影规则
- 是否定义了表格规则
- 是否定义了筛选区规则
- 是否定义了表单规则
- 是否定义了状态规则
- 是否定义了组件映射
- 是否说明当前页面属于企业后台还是 chat 工作区

缺少任意关键项时，这份 UI 规范只能算 `partial`，不能算完整真源。
