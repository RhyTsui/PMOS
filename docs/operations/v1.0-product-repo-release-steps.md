# PMOS v1.0 Product Repo 发布步骤

- version: v0.1
- date: 2026-05-09
- status: active
- owner: product office

## 1. 发布前最后检查

逐项确认：

1. `product-repo/README.md`
2. `product-repo/AGENTS.md`
3. `product-repo/.env.example`
4. `product-repo/docs/deployment/first-run.md`
5. `product-repo/docs/deployment/second-machine-verification-checklist.md`
6. `product-repo/docs/operations/github-release-readiness-checklist.md`
7. `product-repo/cloud-mirror/*sample*`
8. `product-repo/config/providers.example.json`

## 2. 本地发布准备

在 `product-repo/` 内确认：

1. `npm install`
2. `npm run lint`
3. `npm run build`
4. `npm start`

## 3. 清理发布目录

发布前确保以下内容不进入提交：

1. `node_modules/`
2. `dist/`
3. `.env`
4. 本地日志

## 4. 初始化仓库

如果 `product-repo/` 还不是独立 Git 仓：

```bash
git init
git checkout -b main
git add .
git commit -m "Initialize PMOS v1.0 standalone product repo"
```

## 5. 绑定远端

```bash
git remote add origin <your-new-github-repo-url>
git push -u origin main
```

## 6. 推送后验证

推送后检查：

1. GitHub README 首页渲染正常
2. 文档目录可浏览
3. 不含私有信息
4. 不含业务子项目主体

## 7. 旧仓处理

推送新仓成功后，再处理旧仓：

1. 改 README，标注历史仓
2. 加跳转到新 product repo
3. 必要时 archive

## 8. 完成判定

以下条件同时成立才算发布完成：

1. 新仓已存在
2. 新仓可公开阅读
3. 新仓能指导部署
4. 新仓不含私有残留
5. 旧仓不再伪装成正式产品入口
