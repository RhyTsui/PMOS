# Module Roadmap

## Purpose

This document records the current PMAIOS module roadmap against the active `v1.0` product version.

It answers one question:

what should be built or deepened next so the platform reaches `v1.0` product-grade acceptance.

## Recommended Execution Sequence

1. Lock the `v1.0` product denominator first:
   `identity -> decomposition -> review enforcement -> Hermes -> frontend -> document governance -> grounding -> deployment`
2. Then strengthen deeper product-runtime layers:
   `stage-agent -> ui-schema -> proof-of-work -> requirement system -> multi-project rollout`
3. Only after that expand infrastructure hardening:
   `CI/CD -> deploy hardening`

## Current Read

- The active product version is `v1.0`.
- The landed runtime baseline is `v0.7`.
- The roadmap is no longer about inventing the kernel again; it is about making the landed runtime become a real deployable product-manager Agent platform.

## Acceptance Guidance

- Do not reopen `v0.5` or `v0.6` as if they were still active versions.
- Do not keep treating `v0.7` as the outward product version narrative.
- Prefer finishing `v1.0` acceptance tracks over opening `v2.0 / v3.0` strategic branches.
