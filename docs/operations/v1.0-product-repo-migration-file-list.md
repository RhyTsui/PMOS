## 10. 首版迁仓顺序

1. 初始化新仓目录骨架
2. 迁入根目录基础文件
3. 迁入 `src / workflows / tests`
4. 迁入平台级 `docs/operations / docs/architecture / docs/deployment`
5. 迁入 `config/providers.example.json` 与 `config/product-management/*`
6. 迁入 `prompts/`
7. 迁入 `cloud-mirror/` 样例
8. 再补 sample 化的 `docs/memory` 子集
9. 最后做 README 和部署文档改写
