- version: v0.1
- date: 2026-05-09
- status: active
- owner: product office

## 1. 目的

这份清单把当前研发母仓里的实际目录，收口成首版 `PMOS product repo` 的三类判断：

1. `保留`
2. `排除`
3. `条件保留 / sample 化`

同时明确哪些规则虽然名字带 `codex`，但已经属于 `PMOS` 产品真源。

## 5. boards / prompts / skills / mcp 判断

### 5.1 boards

`boards/` 默认条件保留。

保留标准：

1. 解释平台结构
2. 对外有帮助
3. 不绑定业务子项目

### 5.2 prompts

`prompts/` 条件保留。

保留标准：

1. 属于平台默认工作流
2. 不含业务私货
3. 不依赖本机私有上下文

### 5.3 skills

`skills/` 默认不整包进入产品仓。

原因：

1. 它更像执行环境扩展
2. 不一定属于 PMOS 产品必需件

如果某个 skill 实际定义了产品行为，应抽成正式文档，而不是靠 skill 本身承载真源。

### 5.4 mcp

`mcp/` 需要逐项判断。

保留标准：

1. 属于 PMOS 产品运行时接口
2. 不是只为本机环境服务

## 7. codex 文件分类

### 7.1 应作为 PMOS 产品真源继续保留

这些内容定义了平台行为，应继续保留在产品仓：

1. `do-mode-execution-protocol.md`

3. `startup-whoami.md`
4. `codex-collaboration-migration-baseline.md`
5. `codex-local-state-sync.md`

### 7.2 需要 sample 化或重命名后保留

这些内容有产品价值，但现在名字或内容更偏本机实现：

1. `codex-collaboration-migration-manifest.json`
2. `codex-local-state-sync-status.json`
3. `codex-plugin-tool-registry.json`
4. `codex-runtime-capability-classification.json`
5. `codex-runtime-visible-skills.json`
6. `codex-local-runtime-core-classification.json`

建议后续处理：

1. 保留结构
2. 去掉本机专属字段
3. 必要时重命名为更产品化的 `agent-runtime-*` 或 `executor-*` 命名

### 7.3 只应留本机，不进产品仓

尚未在仓库内发现明显属于“纯本机 codex 私有目录”的规则文件；
当前风险主要不是“找不到”，而是“命名上仍然让人误以为它们只是 codex 私货”。
