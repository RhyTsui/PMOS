# PMAIOS v0.7 Minimum Loop Summary

- version: v0.1
- date: 2026-04-30
- status: active
- owner: product office

## Judgment

`v0.7` 当前已完成 minimum loop 收口。

这不表示 `v0.7` 的所有长期目标已经做完，  
而是表示 `v0.7` 定义的五条主线都已经从方向文档推进到了可运行、可验证、可在工作台看到的最小闭环。

## What Was Completed

### 1. SchedulerRun 自动调度最小循环

已完成：

- `scheduleRun`
- `tickRun`
- `tickDueRuns`
- cooldown-based auto resume
- current-stage / rework-stage resume
- 工作台侧边栏调度动作入口

### 2. Gate Runtime 动作化

已完成：

- continuation 面板直出 gate attention 项
- 对 review gate 的 `approve / rework`
- 对 truth/backwrite gate 的 `resume / run`
- gate 不再只是展示状态，而是能驱动最小动作

### 3. Requirement Backwrite + Design Landing Guardrails

已完成：

- 会话里新增/修正的需求必须回写到真源
- 设计图不能单独作为前端实现依据
- `functional-spec-pack / ui-schema-spec / implementation-handoff` 成为实现主依据
- Product Chief UI 明示前端一致性规则

### 4. UI Schema Productization

已完成：

- 新增 typed `UISchemaContract`
- `ui-schema-spec` 现在同时产出 `ui-schema-contract.json`
- 合同包含：
  - `pageContracts`
  - `blocks`
  - `states`
  - `actions`
  - `dataRefs`
  - `implementationRules`
- 工作台可直接打开 UI schema

### 5. Hermes Compare / Promote Minimum Loop

已完成：

- Hermes report 不再只做 freshness check
- 新增：
  - `comparisons`
  - `promotions`
- 当前最小覆盖：
  - UIUX stack baseline
  - structured design-delivery chain
  - structured runtime mainline

约束保持不变：

- Hermes 不能直接 route
- Hermes 不能直接改 DAG
- Hermes 不能直接阻塞 workflow
- Hermes 不能绕过人和真源直接写入最终决定

### 6. Proof-Of-Work Minimum Bundle

已完成：

- 新增 `proof-of-work bundle`
- 后端暴露 `/api/proof-of-work/bundle`
- 工作台侧边栏能统一看到：
  - final-state validation
  - gate summary
  - artifact paths
  - workflow review summary
  - outbox / receipt summary

这意味着当前 scope 的“是否真的做成”不再需要人工跨多个面板拼证据。
