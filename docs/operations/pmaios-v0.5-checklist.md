- version: v0.5

- status: accepted closeout

## 收尾目标

`v0.5` 这一轮不再继续发散成大而全平台，而是只做三件事：

1. 把已经成立的规则、入口、流程、最小运行能力固定下来
2. 把已经有基线但尚未完全产品化的能力明确标记为“收尾完成到什么程度”
3. 把明显属于下一阶段的大工程正式后移到 `v0.6`

## 本轮新增纳入收尾范围的能力

今天一并纳入 `v0.5` 收尾确认的内容：

- 设计交付先出逐页 `image2 prompt pack`
- 设计 / schema / frontend 跳步后自动回填前置真源
- implementation handoff 不再脱离需求真源
- `production / promo / confirmation / demo` 页面模式约束
- `production` 页面禁止混入解释性、宣传性、需求确认型正文
- 产品方法体系真源化
- PMAIOS 介绍与 prompt 文档集入口

## 为什么 v0.5 现在可以收尾

因为这一轮最重要的问题已经解决：

1. 不再只靠聊天推进产品工作
2. 不再只报完成项，开始按分母和用户场景回查
3. 不再让设计与前端脱离需求真源
4. 不再让解释性内容混进生产页
5. 不再让产品能力散落成没有版本边界的临时修补

`v0.5` 到此不再继续挂成长期开口版本。

后续如果继续升级，必须进入 `v0.6`：

- 有独立目标

- 有独立验收
