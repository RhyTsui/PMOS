# PMAIOS v0.4 -> v0.5 过渡快照

- version: v0.3
- date: 2026-04-21
- status: active transition snapshot
- purpose: 作为今天这轮迭代的正式收口文档，明确哪些已经构成 `v0.4` 的当前价值，哪些应作为 `v0.5` 的正式实施入口继续推进

## 1. 过渡原则

今天这轮不是继续发散，而是明确拆开两类内容：

- 已经对当前工作产生直接价值的部分
- 需要作为下一版本持续推进的部分

原则是：

1. `v0.4` 不再被难啃的外部阻塞项拖住。
2. 只要今天形成了直接改善当前工作方式的资产，就计入 `v0.4` 当前价值。
3. 更大的人机协同、模式架构、人阅读层 / AI 层、统一阅读面、AI 产研闭环等，统一收敛到 `v0.5`。

## 2. 什么算当前 v0.4 价值

今天已经直接改善当前工作方式的内容包括：

- 启动入口默认中文已经前移到 repo 级规则。
- `boards/index.svg` 已成为统一图板入口。
- 公司演示主图、能力图、路线图、未来探究图已经可以直接用于沟通。
- `startup-whoami`、`current-version-progress`、`pmaios-version-plan` 等关键真源已经重新收紧。
- `{do}` 实验模式已经具备：
  - 架构文档
  - 执行协议
  - task-start 契约
  - 模板任务
  - SVG 图板
- backend 已经提供最小 human-reading surface。
- workspace 已经暴露共享 mode / tasks / checkpoints / events，并支持模式切换。

这些内容虽然服务于 `v0.5`，但已经直接提升了当前的恢复速度、阅读效率和协作可见性，因此应计作 `v0.4` 当前价值。

## 3. 什么是正式的 v0.5 入口

从今天开始，`v0.5` 的正式实施入口包括：

- `docs/operations/pmaios-v0.5-checklist.md`
- `docs/operations/v0.5-research-and-execution-backlog.md`
- `docs/operations/v0.5-implementation-index.md`

这三层分别承担：

- 清单层：定义收敛目标
- backlog 层：定义研究与交付方向
- implementation index 层：承接模板、协议、图板和样板

## 5. 下一步 carry-over 方向

接下来不再反复重讲方向，而是沿着这几条继续推进：

1. 把 `v0.4` 能快速收口的项继续收掉。
2. 把外部阻塞项明确标成 carry-over。
3. 让 `v0.5 implementation index` 承接更多真实模板、样板和项目级落地例子。
4. 让人阅读层、AI 层、知识库、统一访问面从规则走向实际接入。
5. 让长目标分母式进度和用户场景回查变成默认汇报方式。
