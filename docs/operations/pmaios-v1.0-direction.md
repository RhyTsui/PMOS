# PMOS v1.0 Direction

## 一句话定义

`v1.0 = 一个能部署、能协作、能交付、能持续同步知识的 product-manager Agent 平台。`

## 这一版必须回答什么

1. PMOS 能不能从内部母仓里的运行时，升级成真实可交付产品
2. 它能不能被部署到其他设备，而不是只在当前机器有效
3. 它能不能服务产品、设计、前后端、测试、运营等多角色协作
4. 它能不能把交付证据、评审、状态同步到云端知识层，减少复制粘贴上下文

## 当前固定主轨

1. product-manager Agent 身份与统一入口
2. 多角色工作承接与差异化输出
3. `requirement -> function -> API -> task` 深拆链
4. specialist review 与 Hermes 治理闭环
5. 交付级前端生产与浏览器验证
6. 文档生命周期治理
7. 知识 grounding 与系统状态上下文
8. 部署 / 运维 / 发布 / GitHub 共享
9. 云端知识同步层
10. PMOS 与 codex 规格边界

## 本版的关键判断

`v1.0` 不再回答“runtime baseline 是否存在”，而回答：

`PMOS 能不能作为一个独立产品被部署、使用、验证和持续演进。`

## 当前不做什么

1. 不把业务子项目主体混进产品仓
2. 不把本地私有配置、真实状态、个人上下文直接发到 GitHub
3. 不为了看起来大而重开新的 `2.0 / 3.0` 叙事
