- date: 2026-05-09

## Judgment

这 8 条不是历史废案，而是 `v1.0` 仍未完全收口的能力债。

按现状判断：

- 已有正式壳层但未完全 runtime 化：`6`
- 仅有零散能力、缺统一机制：`2`
- 完整 solved：`0`

## Priority Re-entry

建议重新按下面优先级挂回当前主线：

1. `Kernel Context`
2. `Task SSOT`
3. `Gate Runtime`
4. `Asset Backwrite Contract`
5. `Autonomous Scheduler`
6. `External Sync Outbox`
7. `Agent Collaboration Levels`
8. `Pipeline Launcher`

## Why This Matters

这 8 条共同决定了一件事：

`PMOS 到底是在“有很多能力模块”，还是已经进入“可持续、自治理、可恢复、可回写”的真正运行时阶段。`

如果它们继续被拆散处理，会出现：

1. 有 review，但 gate 不是统一 runtime
2. 有 scheduler，但长链任务仍靠人盯
3. 有 Task SSOT，但不是唯一真源
4. 有连接器，但没有统一 outbox
5. 有 prompts / agents，但协作等级没有正式协议
