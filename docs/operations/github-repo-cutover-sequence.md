# GitHub 仓库切换顺序

## 推荐策略

推荐按这个顺序推进：

1. 先整理出干净的独立 `PMOS product repo`
2. 先发布这个独立仓
3. 再处理旧母仓、历史镜像仓和外部说明

## 为什么这样做

1. 风险更低，不会把产品仓和业务仓继续混在一起
2. 外部读者更容易理解哪个仓是当前主仓
3. 更容易逐步处理历史文档和旧链接

## 发布后要同步的地方

1. 旧仓 README
2. 旧仓发布说明
3. 对外入口文档
4. 云端知识层里的仓库指向

## 不建议的做法

1. 还没整理边界就直接把母仓公开成产品仓
2. 产品仓、业务仓、试验仓共用同一份对外入口
3. 先删旧仓再发新仓
