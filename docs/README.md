# PMAIOS Docs Guide

这份文档是 `PMAIOS` 当前文档体系的总入口。

如果你现在只记一件事，记这个：

**你新的人工输入文件，统一先放到 `docs/sources/inbox/`。**

---

## 1. 唯一人工输入入口

当前唯一人工投递入口：

- `docs/sources/inbox/`

适合直接放进去的内容：

- 会议纪要
- 版本分析
- 需求草稿
- 架构思考
- 外部摘录
- 任何“还没定稿，但希望 PMAIOS 帮你读取、提炼、搬运”的原始材料

处理原则：

- 这里是 `输入口`，不是最终归档位置
- 我读取后，会按内容搬运到正式目录
- 如果内容是规则，会提炼到全局规则
- 如果内容是需求，会考虑沉淀到需求池
- 如果内容是决策候选，不会直接视为最终生效决策

如果你不知道放哪里，就先放这里，不要自己分发。

---

## 2. 文档目录怎么理解

### 平台输入源

- `docs/sources/`
  - 输入源文档区
  - 可草稿、可冲突、可并存多个版本
  - 默认不是最终执行真源

### 决策与规则

- `docs/decisions/`
  - 已确认的决策
  - 应有变更记录
  - 不应与分析草稿混放

### 执行基线

- `workflows/`
  - 工作流和角色定义
- `docs/tasks/`
  - 执行任务和升级清单
- `docs/implementation/`
  - 已确认的实现基线与阶段性说明
  - 不再继续混放输入草稿

### 长期记忆

- `docs/memory/`
  - 运行记忆、需求池、版本库、聊天记录、全局规则

### 模板

- `docs/templates/`
  - 需求卡、版本记录、handoff、调用说明等模板

## 3. 目录入口和角色默认值

后续对话和材料处理，按这个模型理解：

- 从仓库根目录 `ai-os/` 进入
  - 默认视为 `真实产品主管`
  - 默认是 `平台作用域`
- 从 `subprojects/<id>/` 进入
  - 默认视为 `该项目产品经理`
  - 默认是 `子项目作用域`
- 涉及多个项目联动
  - 进入 `multi-subproject` 作用域
  - 主记录优先放平台层
  - 子项目侧保留引用和分发结果

注意：

- `目录入口` 只是默认推断
- 真正生效的是 `角色 + 作用域 + 关联项目`

## 5. 你现在最该怎么用

你的使用方式可以简化成下面这条：

1. 有新材料就丢到 `docs/sources/inbox/`
2. 告诉我“读取 inbox，提炼并搬运”
3. 需要平台规则时，从根目录语义处理
4. 需要具体项目时，明确项目 id
5. 需要跨项目联动时，直接说出涉及项目

如果后面你还是不确定文件放哪，默认答案仍然是：

**先放 `docs/sources/inbox/`。**
