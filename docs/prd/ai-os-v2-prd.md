# AI OS v2 产品需求文档（PRD）

- 版本：v0.1
- 日期：2026-04-07
- 状态：Draft

---

## 1. 产品目标

AI OS v2 是一个本地优先、文件驱动、可持续运行的 AI 产品研发工作流系统。它负责驱动完整的软件研发闭环：

- 需求
- 行业分析
- PRD
- 架构设计
- Task 拆解
- 开发实现
- Review 委员会评审
- 多模态交付
- 迭代优化

### 核心目标

1. 将产品设计和软件研发阶段化、文件化、可追踪化
2. 让 AI 能力嵌入研发流程，而非停留在对话层
3. 对接 Claude Code、MCP、未来 AI Studio、OpenSpec、Superpowers 等外部能力
4. 通过评审委员会机制提升交付质量与治理能力
5. 提供本地 dashboard 作为工作流观测与执行界面

---

## 2. 用户场景

### 场景 A：从 0 到 1 初始化新产品
用户输入一个新产品方向，系统完成行业分析、PRD、架构、任务与 demo 实现。

### 场景 B：已有项目持续迭代
用户已有产品结构，系统围绕已有 artifacts 做增量分析、重构建议与迭代流程。

### 场景 C：多角色协作
产品、架构、前后端、数据与评审角色共享同一套文件化资产和工作流。

### 场景 D：治理驱动的交付
在进入开发与部署前，通过 review committee 对风险、可维护性、数据与产品一致性做统一审查。

---

## 3. 核心功能

### 3.1 行业分析
- 调研行业最佳实践
- 对标成熟产品
- 输出结构化竞品 / 模式 / 优缺点 / 可借鉴点

### 3.2 PRD 生成
- 输出产品目标
- 用户场景
- 核心功能
- 非功能需求
- MVP 范围

### 3.3 架构设计
- 输出分层架构
- 输出 Mermaid 图
- 定义模块边界
- 定义数据流与调用链

### 3.4 Task 拆解
- 将需求与架构拆成阶段任务
- 明确产出、验收标准、依赖关系

### 3.5 开发实现
- 支持前后端骨架
- 支持 mock API
- 支持 dashboard 与 workflow viewer

### 3.6 Review 委员会
- 架构评审
- 产品评审
- 技术评审
- 数据评审
- 风险评审

### 3.7 多模态产物
- Markdown 文档
- Mermaid 架构图
- UI 结构说明
- 可执行代码
- 视频脚本 / 分镜说明
- 设计工具对接说明

### 3.8 Memory 与知识积累
- 长期项目记忆
- 运行记录
- 可扩展知识库

### 3.9 外部能力接入
- MCP 工具接入
- Provider abstraction
- AI Studio 预留接入
- OpenSpec / Superpowers 兼容策略

---

## 4. 非功能需求

- 文件驱动
- 可扩展
- 可审计
- 本地可运行
- 配置与密钥分离
- 支持阶段化返工
- 支持前端可视化展示

---

## 5. MVP 边界

MVP 必须包含：
- 行业分析文档
- PRD
- 架构文档
- Task 文档
- Workflow 定义
- Prompts
- 前后端最小运行系统
- Review 委员会模板与聚合输出
- Memory 文件结构
- Provider / MCP 接入抽象层

---

## 6. 验收标准

- 仓库结构初始化完成
- 文档可直接阅读和演进
- workflow 可被程序读取
- 前端可展示 workflow 状态与 artifacts
- 后端可返回 workflow 与 review 数据
- review 输出满足结构化要求
- 未重复实现 OpenSpec / Superpowers
