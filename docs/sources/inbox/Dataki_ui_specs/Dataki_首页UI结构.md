# Dataki 首页 UI 结构（3 屏完整稿）

## 1. 首页定位
Dataki 首页不是企业官网式宣传页，也不是后台仪表盘首页。  
它应该像一个“可直接开始工作的智能入口”。

首页的任务只有三个：
1. 告诉用户 Dataki 是什么
2. 让用户马上开始提问
3. 展示 Dataki 如何把数据与知识连接起来

---

## 3. 第一屏 Hero（最关键）

### 目标
- 让用户第一眼知道 Dataki 的品牌内核
- 让用户立刻开始提问
- 让首页有“AI系统感”，而不是营销页感

### 文案
**主标题**
Dataki

**副标题**
让知识与数据相遇

**说明文案**
面向业务洞察的智能系统，把数据、文档、口径与经验连接起来，让问题被理解，让结论被生成，让知识被持续沉淀。

**主按钮**
- 立即体验
- 查看能力

**输入框占位**
试着问 Dataki：为什么本周 ROI 下滑？

**建议问题 chips**
- 这个指标的业务口径是什么？
- 帮我解释本周投放波动
- 把这次分析沉淀成 Wiki
- 今天有哪些关键异常值得关注？

### 布局
#### Desktop
- 左侧：标题、文案、按钮
- 右侧：HeroSearch + 一张抽象可视化（蓝点 + 紫块 + 渐变相遇）

#### Mobile
- 上：标题与文案
- 下：输入框 + 示例问题
- 去掉复杂视觉装饰，保留品牌渐变背景

### 视觉要求
- 背景不要纯白，可用很浅的 `surfaceSubtle`
- 中间或右侧要有一块“数据与知识相遇”的抽象视觉
- HeroSearch 要成为整页焦点

---

## 4. 第二屏 Value（价值说明区）

### 目标
让用户知道 Dataki 不是：
- 单纯知识库
- 单纯 BI
- 单纯问答机器人

而是一个让数据与知识共同参与推理的系统。

### 标题
不止看数据，而是理解业务

### 三张价值卡
#### 卡片1：数据解释
**标题**
自动理解指标变化

**说明**
Dataki 结合数据、上下文与历史规则，自动解释异常波动、核心原因与影响范围。

#### 卡片2：知识沉淀
**标题**
让分析自动沉淀为知识

**说明**
每一次解读、复盘与讨论，都可以被结构化为 Wiki、规则卡和可复用经验。

#### 卡片3：智能推理
**标题**
让问题得到更接近决策的答案

**说明**
不只是返回信息，而是给出更接近业务语境的解释、归因和下一步建议。

### 布局
- Desktop：三列卡片
- Tablet：2 + 1
- Mobile：单列

### 视觉要求
- 卡片之间层次统一
- 每张卡有独立图标
- 数据类图标偏蓝
- 知识类图标偏紫
- 推理类图标可用蓝紫渐变

---

## 5. 第三屏 Flow（能力链路区）

### 目标
把 Dataki 的产品机制讲清楚，让用户相信它不是一句 slogan，而是有完整链路。

### 标题
从问题到洞察，形成可复用闭环

### 链路文案
问题输入 → 数据检索 → 知识检索 → LLM 推理 → 洞察生成 → AutoWiki 沉淀 → Agent 执行 / 回写

### 展示方式
#### 左侧：能力流程图
建议做横向步骤流或弯曲流程图：
1. Ask
2. Retrieve Data
3. Retrieve Knowledge
4. Reason
5. Generate Insight
6. Write Back Wiki
7. Trigger Agent

#### 右侧：结果预览区
由两张预览卡组成：

##### 预览卡A：洞察卡
- 标题：ROI 下滑原因分析
- 结论：主要由渠道成本上升与转化率下滑共同导致
- 标签：数据 / 知识 / 综合

##### 预览卡B：Wiki 卡
- 标题：2026Q2 ROI 异常复盘
- 摘要：自动沉淀本次分析结论、指标口径与建议动作

### 视觉要求
- 流程线条不要太复杂
- 强调“闭环”
- 洞察卡和 Wiki 卡风格要明显不同

## 8. 首页设计建议（给设计师）
1. 第一屏绝对不能太像普通官网 Banner
2. 搜索输入框必须成为视觉中心
3. 用“数据蓝”和“知识紫”建立稳定语义
4. 不要堆太多图标，要让结构比装饰更强
5. 首页只讲三件事：是什么、为什么、怎么形成闭环

## 10. 开发优先级
### P0
- HomeView
- AppHeader
- HeroSearch
- ValueFeatureCard

### P1
- CapabilityFlow
- InsightCard
- KnowledgePanelPreview

### P2
- 动效
- 暗色模式细化
- 自适应优化

Dataki 首页不应先追求“内容很多”，而应先追求“入口清晰、气质统一、核心机制讲透”。
