# 广告联调自动化执行器 - 快速启动指南

## 一、环境准备（约10分钟）

### 1.1 Python环境
```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境（Windows）
.\venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 安装Playwright浏览器
playwright install chromium
```

### 1.2 ADB工具
1. 下载 Android SDK Platform Tools
2. 解压到本地目录（如 `D:\tools\platform-tools`）
3. 记录 `adb.exe` 完整路径

### 1.3 scrcpy投屏
1. 下载 scrcpy v3.3.4+
2. 解压到本地目录
3. 运行时需将窗口标题改为 `MobileAgent`

### 1.4 MiniMax API
1. 访问 MiniMax 开发者平台
2. 创建应用并获取 API Key
3. 确保账户有余额

---

## 二、配置文件（约5分钟）

### 2.1 创建配置文件
```bash
# 从模板复制
cp 03_配置模板_env_secrets.json configs/env_secrets.json
```

### 2.2 填写必填项
打开 `configs/env_secrets.json`，填写以下内容：

| 配置项 | 获取方式 |
|--------|----------|
| `username` | 公司分配的巨量引擎账号 |
| `password` | 对应密码 |
| `account_id` | 登录后台后从URL获取 |
| `api_key` | MiniMax开发者平台 |
| `device_id` | 执行 `adb devices` |
| `adb_path` | ADB安装路径 |

---

## 三、设备连接（约5分钟）

### 3.1 手机设置
1. 开启「开发者选项」
2. 启用「USB调试」
3. 连接USB后选择「文件传输」模式

### 3.2 验证连接
```bash
# 检查设备
adb devices

# 应显示类似：
# List of devices attached
# XXXXXXXX    device
```

### 3.3 启动投屏
```bash
# 启动scrcpy（窗口标题设为MobileAgent）
scrcpy --window-title=MobileAgent
```

---

## 四、执行测试

### 4.1 启动执行器
```bash
cd Ad_Agent_Stable_V6
.\venv\Scripts\activate
python scripts/executor.py
```

### 4.2 执行前检查清单
- [ ] 手机已连接，`adb devices` 显示 `device` 状态
- [ ] scrcpy 投屏窗口已打开（标题: MobileAgent）
- [ ] 手机已解锁，抖音已安装
- [ ] MiniMax API Key 有效且有余额
- [ ] 巨量引擎账号可正常登录
- [ ] 联调广告已在后台配置

## 五、常见问题快速排查

### 问题1: 登录失败
**症状**: 点击登录无反应  
**原因**: 协议checkbox未勾选  
**解决**: 检查视觉兜底逻辑，确保向左偏移25px定位checkbox

### 问题2: ADB连接失败
**症状**: `adb devices` 显示空或offline  
**解决**:
```bash
adb kill-server
adb start-server
adb devices
```

### 问题3: 刷不到广告
**症状**: 8次滑动后仍未找到广告  
**排查**:
1. 确认联调二维码已扫描成功
2. 检查 `mobile_feed_keyword` 配置
3. 手动刷视频确认广告是否存在

### 问题4: 视觉识别失败
**症状**: MiniMax返回错误或坐标偏移  
**排查**:
1. 检查 API Key 是否有效
2. 检查账户余额
3. 查看返回的原始响应
