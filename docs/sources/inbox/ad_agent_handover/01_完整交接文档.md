# 第一部分：项目概述

## 1.1 一句话描述
**自动化完成广告归因联调测试：Web端登录巨量引擎生成二维码 → 手机端扫码刷视频找广告 → 点击启动游戏 → 验证数据回传**

## 1.2 业务背景
广告投放后需要验证"归因链路"是否打通：
```
用户看到广告 → 点击广告 → 下载/打开APP → 激活 → 注册 → 付费
     ↓                                              ↓
  媒体记录点击                                    数据回传给媒体
```
手动联调需要：
1. 登录巨量引擎后台生成联调二维码
2. 用手机扫码进入联调模式
3. 在抖音刷视频直到刷到测试广告
4. 点击广告启动游戏
5. 等待后台显示"联调成功"

**本工具实现以上全流程自动化**

## 1.3 技术选型
| 层面 | 技术 | 选型原因 |
|------|------|----------|
| Web自动化 | Playwright | 异步、稳定、支持现代前端 |
| 视觉识别 | MiniMax VL-01 | 多模态大模型、支持坐标定位 |
| 手机控制 | ADB | 标准Android调试桥 |
| 手机投屏 | scrcpy | 开源、低延迟 |
| 桌面截图 | pyautogui | 轻量、跨平台 |

## 4.1 MiniMaxVision 类（视觉识别）
```python
class MiniMaxVision:
    """MiniMax 视觉识别接口"""
    
    def __init__(self, api_key: str, base_url: str = "https://api.minimax.chat/v1"):
        self.api_key = api_key
        self.base_url = base_url
        self.model_name = "MiniMax-VL-01"
    
    def find_element_center(self, image_path: str, target_keyword: str) -> Optional[Tuple[int, int]]:
        """
        在图片中查找包含关键词的元素，返回中心坐标
        
        Args:
            image_path: 截图路径
            target_keyword: 要查找的关键词
            
        Returns:
            (center_x, center_y) 或 None
        """
        import httpx
        import base64
        import re
        
        with open(image_path, 'rb') as f:
            image_base64 = base64.b64encode(f.read()).decode('utf-8')
        
        prompt = f"""请在这张图片中找到包含"{target_keyword}"的元素。
如果找到，请返回该元素的边界框坐标，格式为: x1,y1,x2,y2
其中 x1,y1 是左上角坐标，x2,y2 是右下角坐标。
如果找不到，返回 NOT_FOUND"""
        
        payload = {
            "model": self.model_name,
            "messages": [{
                "role": "user",
                "content": [
                    {"type": "text", "text": prompt},
                    {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{image_base64}"}}
                ]
            }],
            "max_tokens": 256
        }
        
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        
        response = httpx.post(
            f"{self.base_url}/chat/completions",
            json=payload,
            headers=headers,
            timeout=30.0
        )
        
        if response.status_code == 200:
            result = response.json()
            content = result['choices'][0]['message']['content']
            
            # 解析坐标
            match = re.search(r'(\d+),(\d+),(\d+),(\d+)', content)
            if match:
                x1, y1, x2, y2 = map(int, match.groups())
                return ((x1 + x2) // 2, (y1 + y2) // 2)
        
        return None
    
    def detect_keyword_on_screen(self, image_path: str, keyword: str) -> bool:
        """
        检测屏幕上是否存在包含关键词的元素
        
        Returns:
            True 存在, False 不存在
        """
        import httpx
        import base64
        
        with open(image_path, 'rb') as f:
            image_base64 = base64.b64encode(f.read()).decode('utf-8')
        
        prompt = f"""请仔细检查这张图片中是否包含"{keyword}"字样的内容。
如果存在，请返回 FOUND
如果不存在，请返回 NOT_FOUND"""
        
        payload = {
            "model": self.model_name,
            "messages": [{
                "role": "user",
                "content": [
                    {"type": "text", "text": prompt},
                    {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{image_base64}"}}
                ]
            }],
            "max_tokens": 64
        }
        
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        
        response = httpx.post(
            f"{self.base_url}/chat/completions",
            json=payload,
            headers=headers,
            timeout=30.0
        )
        
        if response.status_code == 200:
            result = response.json()
            content = result['choices'][0]['message']['content']
            return "FOUND" in content.upper()
        
        return False
```

## 4.2 ADBTools 类（手机控制）
```python
class ADBTools:
    """ADB 工具类"""
    
    def __init__(self, device_id: str = None, adb_path: str = "adb"):
        self.device_id = device_id
        self.adb_path = adb_path
    
    def _run_adb(self, command: List[str]) -> str:
        """执行 ADB 命令"""
        import subprocess
        
        cmd = [self.adb_path]
        if self.device_id:
            cmd.extend(['-s', self.device_id])
        cmd.extend(command)
        
        result = subprocess.run(
            cmd,
            capture_output=True,
            text=True,
            encoding='utf-8',
            errors='replace'
        )
        return result.stdout + result.stderr
    
    def tap(self, x: int, y: int) -> bool:
        """点击坐标"""
        self._run_adb(['shell', 'input', 'tap', str(x), str(y)])
        return True
    
    def swipe(self, x1: int, y1: int, x2: int, y2: int, duration: int = 300) -> bool:
        """滑动"""
        self._run_adb(['shell', 'input', 'swipe', str(x1), str(y1), str(x2), str(y2), str(duration)])
        return True
    
    def screenshot(self, save_path: str) -> bool:
        """截图并保存到本地"""
        import os
        
        phone_path = '/sdcard/screenshot_temp.png'
        self._run_adb(['shell', 'screencap', '-p', phone_path])
        self._run_adb(['pull', phone_path, save_path])
        self._run_adb(['shell', 'rm', phone_path])
        
        return os.path.exists(save_path)
    
    def push_file(self, local_path: str, remote_path: str) -> bool:
        """推送文件到手机"""
        result = self._run_adb(['push', local_path, remote_path])
        return "pushed" in result.lower()
    
    def refresh_media_library(self, file_path: str) -> bool:
        """刷新媒体库"""
        self._run_adb([
            'shell', 'am', 'broadcast',
            '-a', 'android.intent.action.MEDIA_SCANNER_SCAN_FILE',
            '-d', f'file://{file_path}'
        ])
        return True
    
    def start_activity(self, package: str, activity: str) -> bool:
        """启动 Activity"""
        self._run_adb(['shell', 'am', 'start', '-n', f'{package}/{activity}'])
        return True
```

```python
async def step_login(self) -> bool:
    """登录巨量引擎"""
    
    # 导航到登录页
    await self.page.goto("https://business.oceanengine.com/login", wait_until='networkidle')
    await self.page.wait_for_timeout(2000)
    
    # 填写账号密码
    await self.page.fill('input[type="text"]', self.username)
    await self.page.fill('input[type="password"]', self.password)
    
    # ===== 协议勾选逻辑 =====
    checkbox_clicked = False
    
    # 方法1: Playwright关联定位
    try:
        checkbox = await self.page.wait_for_selector(
            'span:has-text("我已阅读并同意") >> xpath=preceding::span[1]',
            timeout=3000
        )
        if checkbox:
            await checkbox.click()
            checkbox_clicked = True
    except:
        pass
    
    # 方法2: 视觉兜底（向左偏移25像素）
    if not checkbox_clicked:
        temp_screenshot = "temp_login.png"
        await self.page.screenshot(path=temp_screenshot)
        
        text_pos = self.vision.find_element_center(temp_screenshot, "我已阅读并同意")
        if text_pos:
            checkbox_x = text_pos[0] - 25  # 向左偏移找到checkbox
            checkbox_y = text_pos[1]
            await self.page.mouse.click(checkbox_x, checkbox_y)
    
    # 点击登录按钮
    await self.page.locator('button:has-text("登录")').click()
    
    # 等待跳转
    await self.page.wait_for_url("**/business.oceanengine.com/main/**", timeout=10000)
    
    return True
```

**现象**: 点击登录按钮无反应  
**原因**: 协议checkbox未勾选  
**排查**:
1. 检查 Playwright 定位器是否正确
2. 查看截图确认checkbox状态
3. 尝试视觉兜底方案

**解决代码**:
```python
# 视觉定位"我已阅读并同意"，向左偏移25像素点击checkbox
text_pos = vision.find_element_center(screenshot, "我已阅读并同意")
checkbox_x = text_pos[0] - 25
page.mouse.click(checkbox_x, text_pos[1])
```

### 问题2: 弹窗阻断流程
**现象**: 页面出现意外弹窗导致后续步骤失败  
**原因**: 新手引导、协议确认等弹窗  
**排查**:
1. 检查截图中是否有弹窗
2. 添加防御性检测代码

**解决代码**:
```python
# 防御性弹窗处理（超时则跳过）
try:
    popup = page.get_by_text("退出引导")
    await popup.wait_for(state="visible", timeout=3000)
    await popup.click()
except:
    pass  # 无弹窗，继续执行
```

### 问题3: 二维码抓取失败
**现象**: 二维码截图为空或不完整  
**原因**: 二维码元素定位失败  
**排查**:
1. 检查页面是否完全加载
2. 检查二维码元素选择器

**解决方案**:
```python
# 兜底：全屏截图
try:
    qr_element.screenshot(path=qr_path)
except:
    page.screenshot(path=qr_path, full_page=True)
```

## 6.2 手机端卡点

### 问题4: ADB连接失败
**现象**: `adb devices` 无输出或显示 offline  
**原因**: USB调试未开启、驱动问题  
**排查**:
1. 检查手机USB调试是否开启
2. 检查USB连接模式是否为"文件传输"
3. 重启ADB服务: `adb kill-server && adb start-server`

### 问题5: 8次滑动仍找不到广告
**现象**: 滑动8次后返回False  
**原因**: 
- 关键词配置错误
- 广告未投放
- 联调环境未生效

**排查**:
1. 检查 `mobile_feed_keyword` 配置
2. 确认联调二维码已扫描成功
3. 手动刷视频确认广告是否存在

### 问题6: 视觉识别返回错误坐标
**现象**: 点击位置偏移  
**原因**: 屏幕分辨率差异、MiniMax识别误差  
**排查**:
1. 检查手机屏幕分辨率（通常1080x1920）
2. 查看MiniMax返回的原始坐标
3. 调整坐标偏移量

## 6.3 API卡点

### 问题7: MiniMax 401错误
**现象**: `API 返回 401 错误`  
**原因**: API Key无效或过期  
**排查**:
1. 检查 `env_secrets.json` 中的 api_key
2. 登录MiniMax平台确认余额

### 问题8: 请求超时
**现象**: `httpx.TimeoutException`  
**原因**: 网络问题或API响应慢  
**解决**: 增加超时时间
```python
response = httpx.post(url, timeout=60.0)  # 默认30秒
```
