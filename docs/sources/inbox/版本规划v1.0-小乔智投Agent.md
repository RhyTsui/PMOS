## 一、项目背景与战略必要性

### 1.1 业务现状与核心矛盾

我司游戏广告投放业务面临三大效率瓶颈，已严重影响投放规模扩张和团队人效：

|  痛点领域  |  具体表现  |  业务影响  |
| --- | --- | --- |
|  **广告对接与联调**  |  需求流程不规范，开发者、测试账号、游戏包体等前置条件常后置。隐性知识（媒体文档理解、配置规则）未沉淀，新人上手慢。每次对接依赖大量人工配置和手动联调。  |  单次新渠道对接周期长达5人天，交付时间紧张，冲突频发。  |
|  **数据异常排查**  |  激活/付费数据Gap、回传失败等问题定位链路长，依赖专家经验逐层排查SDK、点击、归因、回传日志。排查一次平均耗时30分钟以上。  |  数据信任度低，优化师不敢依据数据决策。  |
|  **数据分析响应**  |  业务侧新增指标、聚合对比、多维下钻需求旺盛，但受限于安全策略无法下载明细到本地。内部BI工具操作受限，大量需求依赖数分手动SQL配报表。分析师被临时取数淹没，用户满意度持续走低。  |  需求响应周期长，数据价值未能释放。  |
|  **素材数据割裂**  |  国内广告系统从2.0升级至3.0，素材模块尚未升级，新旧系统数据断裂。海外投放中同一视频上传不同媒体后MD5变化、文件名被修改，导致素材表现数据无法聚合，形成数据孤岛。  |  素材效果评估失准，创意经验无法沉淀复用。  |
|  **媒体规则静默变更**  |  巨量、腾讯等媒体不定期、无通知更新API字段、归因逻辑或宏参数规则，系统无感知，直到出现数据断流才被动排查。  |  系统稳定性受外部因素威胁，故障排查成本高。  |

### 1.2 AI技术窗口与战略机遇

大语言模型（LLM）与AI Agent技术的成熟，为解决上述问题提供了全新范式：

*   **自然语言交互**：可替代复杂BI操作，让用户以对话方式完成数据查询与分析。
    
*   **决策树推理**：结合MCP工具调用，可自动化执行全链路数据排查，输出结构化诊断结论。
    
*   **浏览器/手机自动化**：Playwright + Appium等工具可模拟人工完成媒体后台操作和扫码联调。
    
*   **语义理解与向量化**：可对视频素材进行脚本级语义解析，打破基于MD5或文件名的物理匹配限制。
    

**结论**：当前是构建广告业务AI操作系统的最佳时机，以AI Agent矩阵解决效率瓶颈，将团队从重复劳动中解放，聚焦高价值策略。

## 二、产品定位与北极星指标

### 2.1 产品定位

**小乔 · Ad OS** 是一个面向游戏广告业务的**智能接入、诊断与对话式分析工作台**。它不是一个重BI看板，也不是一个简单的ChatBot，而是一个以 **“自动化执行”与“因果推理”** 为核心能力的AI协同系统。

**一句话价值主张**：**让业务问得出，让系统接得住，让问题看得清。**

### 12.2 第一阶段北极星指标

**用户自助解决率**：衡量有多少分析需求和异常排查需求，不再需要提工单给数据开发或手动下载Excel。

具体目标：

*   **对接联调效率提升60%**：单次新渠道对接周期从 **5人天** 降至 **2人天**。
    
*   **异常排查效率提升90%**：单次数据Gap定位时间从 **30分钟** 降至 **3分钟**。
    
*   **临时取数需求减少50%**：分析师被手动SQL配报表占用的时间显著释放。

## 四、核心模块详解

### 4.1 智能对接与联调 (Integration Hub)

**核心职责**：将广告媒体对接流程从“口头需求+手动配置+人工联调”升级为“结构化工单+自动配置+自动化联调”。

**工作流程**：

1.  **结构化需求提交**：投放同学在Web表单选择媒体、应用、事件类型（激活/付费/关键行为）及预期时间。
    
2.  **自动预检与工时预估**：引擎校验前置条件（测试账号是否共享、应用包体是否已上传等），若缺失则阻断并提示。通过后，基于历史工单数据自动给出**工作量预估**（如“预估2天”），倒逼业务明确交付预期。
    
3.  **MCP自动配置**：工单生成后，系统调用媒体API MCP工具，自动完成监测链接组创建、事件资产添加等可自动化配置项。
    
4.  **一键联调**：开发人员点击“开始联调”，系统触发n8n工作流，依次调用Playwright（浏览器自动化登录巨量/腾讯后台）和Appium（手机自动化扫码、刷广告、验证回传），实时推送日志流。
    
5.  **报告生成**：联调完成后，自动生成包含截图、日志比对、数据流验证结果的联调报告，归档至工单并飞书通知相关方。
    

**用户价值**：新渠道对接周期从5人天降至2人天，流程规范化，隐性知识系统化。

---

### 4.2 诊断 Agent (Diagnose Agent)

**核心职责**：全链路数据异常的一站式排查，输出带有证据链的根因结论和修复建议。

**技术内核**：**因果推理引擎** = 决策树（预定义排查路径） + MCP工具调用 + LLM证据总结。

**支持的诊断场景（Phase 1）**：

*   **激活数据Gap**（媒体侧激活数 ≠ BI激活数）
    
*   **付费数据Gap**（媒体侧付费金额 ≠ BI付费金额）
    
*   **回传失败/异常**
    

**排查路径示例（激活Gap）**：

text

    用户输入：“为什么昨天巨量激活比BI少30%？”
        ↓
    意图识别：激活Gap诊断
        ↓
    Step 1: MCP查询SDK日志（activation事件） → 有数据/无数据
        ↓ (无数据) → 结论：SDK未上报
        ↓ (有数据)
    Step 2: MCP查询点击日志匹配度 → 未匹配/匹配
        ↓ (未匹配) → 结论：非归因流量或超出回溯窗口
        ↓ (匹配)
    Step 3: MCP查询归因表 → 归因成功/失败
        ↓ (失败) → 结论：归因逻辑或风控拦截
        ↓ (成功)
    Step 4: MCP查询媒体回传日志 → HTTP状态、响应体
        ↓ (4xx/5xx) → 结论：媒体接口异常或参数错误（结合知识底座文档变更记录给出具体原因）
        ↓ (200)
    Step 5: 结论：数据延迟或BI聚合问题

**输出格式**：

*   **一句话结论**（如“因媒体昨日将回调URL时间戳单位从秒改为毫秒，导致回传失败”）
    
*   **证据链**（可折叠，每一条关联具体日志或截图）
    
*   **置信度**（基于决策树路径确定性评估）
    
*   **操作建议**（如“请更新回传URL中的\_\_TIMESTAMP\_\_宏参数格式”）
    

**用户价值**：异常排查时间从30分钟降至3分钟，且不依赖专家经验，新人亦可快速定位问题。

---

### 4.3 对话式分析 (Insight on Demand)

**核心职责**：以自然语言交互方式，满足用户高频的聚合对比、多维下钻和新增指标查询需求，替代数分手动SQL配报表的传统模式。

**设计原则**：

*   **不生成SQL**：LLM仅做意图识别和槽位填充（维度、指标、时间范围），调用**预置的MCP分析模板**执行查询，确保数据准确。
    
*   **渐进式交互**：支持在已有结果上追加指标、过滤、排序，模拟分析师与业务方确认需求的对话过程。
    
*   **数据不落地**：结果在Web页面以表格/卡片展示，支持一键分享链接，需审批后可导出CSV。
    

**MCP分析模板库（示例）**：

|  模板名称  |  参数  |  说明  |
| --- | --- | --- |
|  `compare_dimensions`  |  维度、指标、时间范围、对比维度值  |  如：对比A/B优化师在巨量和腾讯的消耗与ROI  |
|  `top_n_analysis`  |  维度、指标、时间范围、N  |  如：消耗TOP 10素材的付费成本  |
|  `cohort_analysis`  |  用户分群逻辑、观察指标、观察周期  |  如：巨量分包新用户7日付费次数  |
|  `trend_analysis`  |  指标、时间粒度、时间范围  |  如：最近7天分小时激活数趋势  |

**用户故事**：

> 优化师输入：“帮我对比一下‘传奇素材包’和‘仙侠素材包’上周在巨量的消耗和首日ROI。”  
系统返回对比表格，差异项用红绿箭头标注。  
优化师追问：“再加一个‘付费用户次留率’指标。”  
系统调用指标计算MCP，在原结果上追加新列，无需重新查询。

**用户价值**：临时分析需求响应时间从天级/小时级降至秒级，分析师从取数工作中释放，用户满意度显著提升。

---

### 4.4 素材 Agent (Creative Agent)

**核心职责**：基于语义理解，实现跨媒体、跨平台的素材数据聚合，打破因MD5变化或文件名修改导致的数据孤岛，同时为3.0系统补齐素材认知能力。

**技术实现路径**：

1.  **脚本生成**：调用多模态大模型，将视频素材转译为结构化脚本文本（包含Hook、冲突、反转、结尾等元素）。
    
2.  **语义向量化**：使用Embedding模型将脚本转化为向量，存入向量数据库（如Milvus）。
    
3.  **聚类与相似度计算**：通过BERTopic等算法对向量进行聚类，自动发现爆款脚本结构簇。
    
4.  **数据关联**：将素材的跨媒体表现数据（消耗、展示、点击、转化）按聚类结果重新聚合，生成**创意级报表**。
    

**产品形态**：

*   **素材库同步**：支持从本地盘自动同步上传，或从媒体后台采集转存。
    
*   **创意报表**：按脚本簇（如“强冲突-情绪反转型”）聚合展示全球多媒体投放表现，支持相似度阈值调节。
    
*   **相似素材推荐**：为指定素材找出语义相似的其他素材，辅助创意迭代。
    

**用户价值**：首次实现海外多媒体素材数据的统一分析，为国内3.0系统补齐素材维度，将创意经验从“这个素材好”沉淀为“这类结构好”的可复用知识。

---

### 4.5 知识底座与主动监控 (Knowledge Base & Media Doc Monitor)

**核心职责**：作为所有上层应用的认知基础，沉淀隐性知识，并主动感知外部规则变化。

**构成**：

*   **静态知识库**：媒体API文档的结构化抽取、内部开发规范、历史工单方案库、指标与归因口径字典。
    
*   **动态监控模块**：定时爬取目标媒体官方文档页，使用大模型对比新旧版本差异，识别字段变更、枚举值增删、规则描述修改等。生成影响分析报告（如“该变更影响我们所有巨量激活回传”），并主动推送至飞书，关联相关模块负责人。
    
*   **MCP工具模板库**：沉淀高频SQL查询、自动化操作序列，供上层Agent调用。
    

**关键价值**：

*   让新人无需反复踩坑，系统直接提供上下文。
    
*   媒体规则变更从“故障驱动排查”变为“主动感知预警”，大幅提升系统稳定性。

## 五、核心用户故事（闭环验证）

### 故事一：新渠道对接（投放 → 开发）

1.  投放同学登录小乔工作台，点击“新建媒体对接”，在表单中选择“巨量引擎”、“王牌\*\*\*.apk”、“激活+付费事件”。
    
2.  系统自动预检，弹窗提示：“检测到该应用未在巨量后台共享测试账号，请先完成。预计开发资源空闲，预估工时：2天。”
    
3.  投放同学完成账号共享后点击确认，系统自动生成标准化工单，飞书通知开发。
    
4.  开发同学打开工单，系统已完成监测链接的自动创建。点击“开始联调”，页面展示实时自动化流程：
    
    *   Playwright登录巨量后台 → 创建资产 → 生成二维码
        
    *   Appium控制测试机扫码 → 刷广告 → 模拟付费
        
    *   系统比对两端数据 → 生成联调成功报告
        
5.  工单自动关闭，报告归档。全程实际耗时2小时。
    

### 故事二：数据异常排查（优化师）

1.  优化师在首页对话框输入：“昨天下午巨量激活比BI少了30%，帮我看下。”
    
2.  诊断Agent启动，页面展示决策树执行路径和实时日志。
    
3.  1分钟后，系统返回诊断卡片：
    
    *   **结论**：媒体昨日将回调URL中的`__TIMESTAMP__`单位从秒改为毫秒，导致我们回传的时间戳校验失败。
        
    *   **证据链**：①SDK上报量1200；②归因成功量1200；③媒体回传接口返回`error: invalid timestamp`；④知识底座显示昨日该文档字段发生变更。
        
    *   **建议**：更新回传URL模板，将`__TIMESTAMP__`乘以1000。
        
4.  优化师点击“采纳建议”，系统自动生成修复工单派发后端。
    

### 故事三：临时分析需求（优化师）

1.  优化师想分析新素材的表现，输入：“帮我看下最近3天，消耗TOP 5素材的点击率、转化率和首日ROI。”
    
2.  系统调用`top_n_analysis`模板，返回表格。
    
3.  优化师追问：“再加一个‘付费用户次留率’，按ROI降序排。”
    
4.  系统在原结果上追加指标并重排序，无需重新跑数。
    
5.  优化师点击“保存为我的分析卡片”，生成链接分享给团队。

## 九、总结

**小乔 · Ad OS** 不是又一个BI系统，也不是一个简单的AI聊天机器人。它是一个直面游戏广告业务核心痛点的**AI协同工作台**。

我们砍掉了宏大叙事的监控大屏和不确定性高的预测模块，将所有火力聚焦于三个最痛的效率瓶颈：

*   **对接慢？** → 智能对接与联调，2天变2小时。
    
*   **排查难？** → 诊断Agent，30分钟变3分钟。
    
*   **取数烦？** → 对话式分析，一句话替代手动SQL。
    

配合素材Agent打破数据孤岛，知识底座沉淀经验并主动监控外部变更，我们构建的是一套 **“可解释、可执行、可进化”** 的广告业务AI操作系统。
