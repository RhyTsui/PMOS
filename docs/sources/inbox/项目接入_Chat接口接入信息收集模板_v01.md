# 项目接入：Chat 接口接入信息收集模板

版本：v0.1  
适用范围：项目接入模块中非 Trace 关联链路  
使用对象：准备接入评测平台的外部 Chat / Agent / 问答类项目  
文档目的：由接入项目方填写接口信息，评测平台据此完成项目创建、接口配置、测试连通、实验执行和结果解析

## 1. 接入方需提供的资料清单

| 序号 | 资料 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 1 | 项目基础信息 | 是 | 项目名称、负责人、环境、业务说明 |
| 2 | 接口环境信息 | 是 | 预发布、生产环境 Base URL |
| 3 | 接口调用方式 | 是 | Method、Path、Content-Type、是否流式 |
| 4 | 鉴权信息 | 是 | Header、Token、网关参数、密钥申请方式 |
| 5 | 请求体说明 | 是 | 请求 Body 字段定义、必填项、类型、枚举、默认值 |
| 6 | 请求体示例 | 是 | 可直接用于联调的完整 JSON |
| 7 | 返回体说明 | 是 | 成功、失败、流式分片或一次性 JSON 的结构定义 |
| 8 | 成功返回体示例 | 是 | 完整原始返回体，包含最终回答 |
| 9 | 失败返回体示例 | 是 | 鉴权失败、参数错误、服务异常等 |
| 10 | 响应解析规则 | 是 | answer、错误信息、工具调用等字段位置 |
| 11 | 可运行 curl | 是 | 至少提供预发布环境 curl |

## 2. 请求体与返回体提供要求

项目接入必须明确请求体和返回体。评测平台依赖这两部分完成请求组装、字段映射、测试连通、实验执行和结果解析。

### 2.1 请求体必须提供的信息

| 信息 | 是否必填 | 说明 |
| --- | --- | --- |
| 完整请求 Body 示例 | 是 | 一份能直接联调的 JSON |
| 字段路径 | 是 | 例如 `input_query`、`extra.app_id` |
| 字段类型 | 是 | string / number / boolean / object / array |
| 是否必填 | 是 | 缺失后接口是否报错 |
| 示例值 | 是 | 联调用真实或脱敏值 |
| 字段来源 | 是 | 项目固定值 / 评测集字段 / 系统生成 / 鉴权密钥 |
| 默认值 | 是 | 没有映射值时是否允许使用默认值 |
| 枚举值 | 如有则必填 | 例如 `module`、`scoops`、`app_region` |
| 字段约束 | 如有则必填 | 长度、格式、时间格式、数组大小等 |
| 嵌套结构 | 如有则必填 | object / array 内部字段也要展开说明 |

### 2.2 返回体必须提供的信息

| 信息 | 是否必填 | 说明 |
| --- | --- | --- |
| 成功返回体原文 | 是 | 真实接口返回内容，不使用截图替代 |
| 失败返回体原文 | 是 | 至少包含参数错误、鉴权失败、服务异常 |
| 响应协议 | 是 | JSON / SSE / NDJSON / WebSocket |
| 最终回答字段 | 是 | answer 字段路径或 response 分片拼接规则 |
| 错误信息字段 | 是 | error / error_message / code 等字段路径 |
| 流式结束标识 | 流式接口必填 | 例如 `[DONE]` 或连接自然结束 |
| 工具调用字段 | 如有则必填 | 工具名、工具参数、工具结果字段位置 |
| trace_id / request_id | 如有则必填 | 返回中的链路标识字段 |
| 空回答规则 | 是 | 无 answer / 无 response 时如何表达 |
| Content-Type | 是 | 响应头中的内容类型 |

### 2.3 请求体与返回体交付格式

接入项目方需要提供以下内容：

```text
1. request_body_schema.md 或文档章节
2. request_body_example.json
3. success_response_raw.txt
4. error_response_raw.txt
5. runnable_pre_curl.sh
```

## 15. 响应解析规则

由接入项目方明确评测平台应该如何解析。

| 目标字段 | 解析来源 | 解析规则 |
| --- | --- | --- |
| 最终回答 `final_answer` |  |  |
| 原始回答 `response_text` |  |  |
| 工具调用 `tool_calls` |  |  |
| 工具参数 `tool_calls[].arguments` |  |  |
| 错误信息 `error_message` |  |  |
| 请求链路 ID `trace_id` |  |  |
| 是否成功 `status` |  |  |

示例：

| 目标字段 | 解析来源 | 解析规则 |
| --- | --- | --- |
| `final_answer` | SSE 中所有 `response` | 按顺序拼接 |
| `tool_calls[].tool_name` | SSE 中 `tools` | 每次出现生成一条工具调用 |
| `tool_calls[].arguments` | SSE 中 `tools-args` | JSON 字符串反序列化 |
| `error_message` | `error_message` / `error` | 任一存在则视为失败 |

## 16. 标准化输出格式

  "content_type": "text/event-stream",

  ],

}

## 20. 测试连通验收标准

| 验收项 | 通过标准 |
| --- | --- |
| URL 可访问 | 请求返回 HTTP 2xx |
| 鉴权可用 | 携带鉴权信息后不返回 401 / 403 |
| 请求体可识别 | 不返回参数错误 |
| 响应可解析 | 能解析出 answer 或 response |
| 流式可读取 | SSE / NDJSON 能逐段读取 |
| 错误可识别 | 失败响应能解析出错误信息 |
| 原始响应可保存 | 能保存 raw response |
