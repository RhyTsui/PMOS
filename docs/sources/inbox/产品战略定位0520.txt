行业最佳实践判断

从行业最佳实践看，成熟 AI 产品正在从“聊天机器人”转向“AI Workspace / Agent Platform”。

OpenAI 的企业产品已经强调 shared workspace、内部工具连接、data analysis、shared projects、tasks、custom workspace GPTs、workspace agents，并且 Workspace Agents 可以跨连接应用执行重复任务和业务流程，支持模板、发布、共享和定时运行。

Microsoft Copilot Studio 的方向也不是单纯聊天，而是构建、测试、发布企业 Agent；它把 Agent 定义为可以自动化和执行业务流程的专家系统，并支持连接器、MCP、工作流、人工介入、版本和治理。

Notion 也已经把自身定位成 AI workspace，而不是 AI chat；它把 Docs、Projects、Knowledge Base、Enterprise Search、Connections、Custom Agents 放在同一个工作空间内，并强调 Agent 可以基于 Notion、连接应用和 Web 的上下文完成复杂多步任务。

Intercom Fin 的最佳实践更接近你们的“小乔”：它不是泛用聊天机器人，而是特定领域的服务 Agent。它强调 query refinement、retrieval、relevance scoring、answer validation、workflow triggers 和 human escalation，并明确把回答建立在知识库、文档、对话历史、结构化片段和第三方系统之上。

# 产品方向变更：小乔智投从 Chat 产品升级为 AI 数据BP工作台

## 1. 背景

当前产品原点不是“做一个 AI 聊天工具”，而是解决数据部门对发行/投放团队的能力支持效率问题。

过去数据部门需要人工支持发行/投放团队完成：
- 查数
- 指标解释
- 异常排查
- 归因/回传逻辑解释
- 流程验收
- 报表拼接
- 新需求理解
- 媒体对接
- 联调排查
- 行业动态和投放情报支持

现在要把数据团队长期沉淀的经验、SOP、口径、排查方法、工具调用能力、知识库和沟通方式，蒸馏成一个虚拟 Agent。

这个 Agent 名称为：小乔。

小乔不是普通聊天机器人。
小乔是数据支持团队能力蒸馏出来的 AI 数据BP / AI 投放运营搭档。

## 2. 产品定位
