### 规范目的

*   全局统一产品的设计风格，统一用户体验，形成设计标准与规范
    
*   帮助设计、开发团队提高工作效率，减少沟通成本
    
*   支持模块化设计，加快产品开发迭代速度，减轻开发成本
    

### ![image](https://docs-body.xiaoshanim.com/res/NyxQqz4BkrE18ogm/img/ab8ec397-663e-45b1-a9da-13adec52e484.jpg?Expires=1778059654&OSSAccessKeyId=file-s3-gateway-ak&Signature=sYpIEP3OE%2BCjF10qzkE8og8Ayn0%3D)

### 文档使用

*   **设计师：**设计师可以快速上手工作，保持相同风格，规范组件标准
    
*   **开发人员：**提供开发标准，形成模块化，减少沟通成本，组件复用，提高开发效率
    
*   **产品经理：**了解设计师和开发标准，使产品形成统一
