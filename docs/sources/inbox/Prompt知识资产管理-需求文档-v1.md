# Prompt 知识资产管理需求文档 v1

日期：2026-04-28

## 1. 背景

当前系统已逐步沉淀知识库、业务 Chat、AI 助手与外部 Agent 能力，但 Prompt 仍分散在个人文档、群消息或临时配置中，存在以下问题：

- Prompt 无统一入口，难以检索和复用
- Prompt 缺少版本说明与状态管理，容易误用旧版本
- Prompt 的适用场景、变量说明、负责人不清晰
- Prompt 被挂载到主对话框、业务 Chat 或外部 Agent 时，缺少统一绑定与权限校验

因此需要在 `知识库` 菜单下新增 `Prompt` 作为一种正式知识资产类型，纳入 Dataki 资产体系统一管理。

## 2. 目标

在不改变原有系统整体布局和导航范式的前提下，补齐 Prompt 资产的：

- 列表管理
- 筛选检索
- 详情查看
- 新增与编辑
- 启停状态管理
- 版本说明
- 绑定关系展示
- 权限控制与调用留痕

## 3. 范围

### 3.1 v1.1 必做范围

- 在 `知识库` 菜单下增加 `Prompt` 管理页面
- 支持 Prompt 资产列表、筛选、详情、新增、编辑
- 支持 Prompt 启用 / 停用状态
- 支持版本说明与最近更新时间展示
- 支持挂载关系查看
- 支持基础权限控制

### 3.2 本期不做

- 不单独新建一套独立 Prompt 平台
- 不改变原有系统一级导航
- 不做复杂 Prompt 编排或多版本对比编辑器
- 不做可视化流程式 Prompt Orchestration

## 4. 信息架构

### 4.1 菜单位置

基于原系统布局，在 `知识库` 菜单下增加二级菜单：

- 知识文档
- Prompt 资产
- 知识绑定

本次重点页面为：`Prompt 资产`

### 4.2 页面定位

`Prompt 资产` 是知识资产管理页，不是聊天页，也不是模型调试页。

页面重点是：

- 管理 Prompt 资产本身
- 明确用途、变量、归属和状态
- 明确它被谁使用、挂到哪里
- 明确谁可以看、谁可以改、谁可以发、谁可以绑定调用

## 5. 核心对象

### 5.1 Prompt 资产基础字段

每条 Prompt 资产至少包含：

- 名称
- 用途说明
- 正文内容
- 变量说明
- 适用模型或场景
- 归属范围
- 负责人
- 状态
- 版本说明
- 最近更新时间

### 5.2 状态

建议最小状态集：

- 草稿
- 已启用
- 已停用

### 5.3 权限

沿用知识资产权限模型，至少区分：

- 查看
- 编辑
- 发布
- 绑定 / 调用

并记录调用主体。

## 6. 页面需求

## 6.1 列表页

列表页应沿用原系统后台表格布局，不新增独立站点式 UI。

应包含：

- 顶部页面标题
- 顶部操作按钮：
  - 新增 Prompt
  - 导出列表
- 筛选区：
  - 名称 / 关键词搜索
  - 状态
  - 适用场景
  - 归属范围
  - 负责人
- 列表表格字段至少包含：
  - 名称
  - 用途说明
  - 适用模型 / 场景
  - 归属范围
  - 负责人
  - 状态
  - 版本说明
  - 最近更新时间

## 6.2 详情页

详情页建议沿用系统现有右侧详情面板或详情页结构，不额外引入新布局体系。

详情至少包含：

- 基础信息
- Prompt 正文
- 变量说明
- 适用场景
- 绑定关系
- 权限信息
- 版本说明
- 最近更新记录

## 6.3 新增 / 编辑

新增和编辑应使用系统已有表单交互模式。

表单至少包含：

- 名称
- 用途说明
- 正文内容
- 变量说明
- 适用模型 / 场景
- 归属范围
- 负责人
- 状态
- 版本说明

## 7. 绑定关系

Prompt 资产需支持挂载到：

- 主对话框
- 指定业务 Chat
- 外部 Agent

调用前必须完成：

- 绑定关系校验
- 权限校验

并记录：

- 调用主体
- 调用时间
- 调用目标

## 8. 设计约束

本次设计必须遵守：

- 基于原系统界面布局出图
- 不新增新的一级导航
- 不做新平台式首页
- 不做聊天主界面
- 不增加额外复杂功能模块
- 保持原系统后台风格、表格风格、详情面板风格

## 9. 成功标志

- Prompt 资产可以统一被检索、查看和维护
- Prompt 不再散落在线下文档与个人记录中
- Prompt 的用途、变量、负责人、状态可被清晰理解
- Prompt 的绑定和调用有权限控制与留痕
