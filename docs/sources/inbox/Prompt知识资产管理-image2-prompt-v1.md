# Prompt 知识资产管理 image2 设计图说明 v1

日期：2026-04-28

## 主 Prompt

```text
请生成一张高保真中文 B 端后台产品设计图，桌面端 16:9 比例，风格为成熟企业内部数据平台，要求真实可落地、信息密度适中、界面清晰。

设计主题：
在原有系统界面布局基础上，为“知识库”模块新增“Prompt 资产”管理页面。

非常重要：
- 必须基于原系统的布局结构和视觉语言出图
- 不要新设计一套站点
- 不要增加新的一级导航
- 不要加入多余的新模块
- 不要做成 AI 聊天界面
- 不要做成独立 Prompt 平台首页

界面框架要求：
1. 顶部保留原系统深蓝色导航栏
2. 左侧保留原系统菜单布局
3. 当前一级菜单为：知识库
4. 左侧二级菜单中高亮：Prompt 资产

页面定位：
这是一个知识资产管理页面，用于统一管理 Prompt 作为知识资产，不是聊天页，不是调试页，不是流程编排页。

页面结构要求：

一、页面头部
- 标题：Prompt 资产
- 副标题：统一管理 Prompt 作为知识资产的内容、状态、版本和绑定关系
- 右上角按钮：
  - 新增 Prompt
  - 导出列表

二、顶部筛选区
筛选项包括：
- 名称 / 关键词搜索
- 状态
- 适用场景
- 归属范围
- 负责人
- 搜索按钮
- 重置按钮

三、主体列表区
采用原系统常见表格布局
表格字段至少包括：
- 名称
- 用途说明
- 适用模型 / 场景
- 归属范围
- 负责人
- 状态
- 版本说明
- 最近更新时间

列表内容要像真实系统数据，例如：
- 验收问题归因总结 Prompt
- 埋点需求标准化改写 Prompt
- 字段冲突识别 Prompt
- 广告验收专项解释 Prompt
- 版本差异摘要 Prompt

四、右侧详情面板
沿用原系统右侧详情区风格，不要重造交互
详情页签包括：
- 基础信息
- Prompt 正文
- 变量说明
- 绑定关系
- 权限信息
- 版本说明

详情内容至少展示：
- 名称
- 用途说明
- 正文内容
- 变量说明
- 适用模型或场景
- 归属范围
- 负责人
- 状态
- 最近更新时间
- 已绑定目标，例如：
  - 主对话框
  - 验收业务 Chat
  - 外部 Agent

五、界面表达重点
- 这是“知识资产管理”
- 不是“Prompt 创作工具”
- 不是“AI 对话页”
- 不是“独立新平台”
- 要体现 Prompt 属于知识库资产体系的一部分
- 要体现状态、版本、权限、绑定关系是重点

视觉要求：
- 保持原系统后台风格
- 深蓝顶栏
- 浅色内容区
- 表格和右侧详情面板清晰
- 白底、浅灰分隔、蓝色主强调
- 不要紫色
- 不要营销官网风格
- 不要赛博风
- 不要额外插画
- 不要出现无关新模块
```
