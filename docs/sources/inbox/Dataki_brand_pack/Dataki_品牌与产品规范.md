# Dataki 品牌与产品规范（商用版）

## 1. 品牌定稿
- **产品名**：Dataki
- **Slogan**：让知识与数据相遇
- **品牌定义**：Dataki 是一个让知识与数据共同参与推理的智能系统，面向业务洞察、自动 Wiki、知识沉淀与决策支持。

## 2. 品牌核心表达
Dataki 不是传统知识库，也不是传统 BI。  
它强调的是：
- 数据提供“发生了什么”
- 知识提供“为什么”
- LLM + RAG + Agent 让两者在系统中相遇，并形成可复用的洞察

## 3. Logo 设计结构
1. **左侧点阵**：表示数据  
2. **中间相遇点**：表示融合、语义对齐、推理启动  
3. **右侧块阵与弧面**：表示知识、结构化、体系化、可复用  

## 4. 用色规范（建议与产品系统统一）
建议把品牌色直接纳入产品 Design Token：
- `brand.navy = #0B132B`
- `brand.blue = #2563EB`
- `brand.cyan = #38BDF8`
- `brand.violet = #7C3AED`
- `brand.violetLight = #B388FF`
- `bg.default = #F8FAFC`

### 4.1 产品 UI 对应关系
- 数据相关模块：蓝 / 青
- 知识相关模块：紫 / 浅紫
- 推理 / 相遇 / 洞察：蓝紫渐变 + 中央亮点
- 深色模式：主背景优先使用 `brand.navy`

## 5. 字体规范
- 英文品牌字：Inter / Segoe UI / Arial
- 中文界面字：思源黑体 / Noto Sans SC / 微软雅黑
- 标题建议字重：700
- 正文字重：400 或 500

## 6. Logo 使用规范
### 6.1 标准版
- 使用文件：`dataki_logo_primary.svg`
- 场景：官网首屏、PPT 封面、产品启动页、宣传页

### 6.2 图标版
- 使用文件：`dataki_icon_light.svg` / `dataki_icon_dark.svg`
- 场景：App 图标、Favicon、左上角产品入口、菜单图标

### 6.3 最小尺寸
- 横版 Logo：宽度不低于 180px
- Icon：不低于 24px
- 若低于 24px，请移除过细发光效果，保留左点右块中间留白结构

### 6.4 安全边距
- 以 Logo 中心发光椭圆短轴长度作为 `x`
- Logo 四周至少保留 `1x` 空白

### 6.5 禁止事项
- 不要替换成无关色系
- 不要打散左右结构单独使用
- 不要随意改变 slogan 顺序
- 不要把 Dataki 写成 DATAKI / dataKi

## 8. 产品架构说明
Dataki 的标准架构分为四层：
1. 交互层  
2. 智能编排层  
3. 推理与知识层  
4. 连接与执行层  

## 9. 与 BI / 数据中台差异
### 传统 BI
- 核心是报表与可视化
- 解决“看数”
- 知识通常游离在系统外

### 数据中台
- 核心是数据资产、模型、服务化供数
- 解决“供数”
- 仍不负责把知识真正带入解释过程

### Dataki
- 核心是知识 + 数据 + 推理
- 解决“理解、沉淀、复用、决策”
- 每次分析都能成为下一次洞察的基础
