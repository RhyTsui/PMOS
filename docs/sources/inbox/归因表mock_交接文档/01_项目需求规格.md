# Ad-Expert V2 MCP项目需求规格

## 一、项目目标

创建一个基于 **FastMCP** 框架的广告归因诊断MCP服务，让AI Agent能够查询归因相关数据表并进行问题诊断。

---

## 二、技术要求

### 2.1 技术栈
- **MCP框架**: FastMCP (`pip install fastmcp`)
- **数据处理**: Pandas
- **可选HTTP服务**: FastAPI + Uvicorn（用于Web看板联调）

### 2.2 核心依赖
```bash
pip install fastmcp pandas
# 可选（如需HTTP API和Web看板）
pip install fastapi uvicorn
```

### 4.3 诊断Tool规格 (4个)

#### diagnose_full_funnel
```
功能: 全链路归因诊断
描述: 按"点击→激活→注册→付费→回推"漏斗进行时序校验
参数:
  - device_id: str (必填)
  - app_id: str (必填)

诊断逻辑:
1. 通过device_id查设备表获取idfa
2. 通过idfa查点击表
3. 检查各环节是否存在数据
4. 校验时序: 点击时间 < 激活时间 < 注册时间 < 付费时间
5. 检查回推状态

返回诊断码:
- FULL_SUCCESS: 全链路成功
- NO_DATA: 无任何数据
- CLICK_NO_ACTIVATION: 有点击无激活
- NATURAL: 自然量用户
- ACTIVATION_NO_REGISTER: 有激活无注册
- REGISTER_NO_PAY: 有注册无付费
- TIME_SEQUENCE_ERROR: 时序异常
```

#### diagnose_h5_app_link
```
功能: H5+APP归因专项诊断
描述: 检查H5跳转APP链路中的指纹归因、设备ID获取问题
参数:
  - device_id: str (必填)
  - app_id: str (必填)
```

#### diagnose_pc_tracking
```
功能: PC端归因专项诊断
描述: 检查web_device_id生成、追踪文件写入读取问题
参数:
  - device_id: str (必填)
  - app_id: str (必填)
```

#### diagnose_ios_fusion
```
功能: iOS融合归因专项诊断
描述: 检查融合归因版本、时间戳裁决问题
参数:
  - device_id: str (必填)
  - app_id: str (必填)
```

## 五、Mock数据规格

### 5.1 数据量
- 总计 **20条** 完整归因链路数据
- 15条正常链路 (SUCCESS_000 ~ SUCCESS_014)
- 5条异常链路 (ABNORMAL_000 ~ ABNORMAL_004)

### 5.2 归因链路约束

**时序约束（必须满足）：**
```
点击时间 < 激活时间 < 注册时间 < 付费时间
```

**ID关联约束：**
- 同一case_id的数据在各表中必须能通过idfa/device_id关联
- 点击表通过idfa关联到设备表的device_id

### 5.3 异常用例设计

| case_id | 异常类型 | 说明 |
|---------|---------|------|
| ABNORMAL_000 | 时序倒挂 | 激活时间早于点击时间 |
| ABNORMAL_001 | 自然量 | 只有激活，无点击记录 |
| ABNORMAL_002 | 转化中断 | 有激活无注册 |
| ABNORMAL_003 | 付费流失 | 有注册无付费 |
| ABNORMAL_004 | 回推失败 | 回推状态为WAITING |

### 5.4 设备类型覆盖

- iOS设备：使用idfa
- Android设备：使用oaid + android_id
- 鸿蒙设备：使用idfa

## 七、可选功能

### 7.1 HTTP API服务 (FastAPI)

如需Web联调看板，可创建FastAPI服务：
- 为每个MCP Tool创建对应的POST路由
- 路由格式：`/api/v1/{db_type}/{table_name}`
- 配置CORS允许跨域
- 服务端口：8000

### 7.2 联调看板 (HTML)

单页面HTML，功能：
- 左侧导航列出所有表
- 右侧动态渲染查询表单
- 发送请求并展示JSON结果
