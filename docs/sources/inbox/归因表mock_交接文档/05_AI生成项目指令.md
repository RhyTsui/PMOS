# AI生成项目指令

将以下内容发送给AI，让其生成完整的MCP项目。

---

## 指令文本

```
【任务】：基于以下规格，生成一个广告归因诊断MCP服务项目。

一、项目结构
请在当前目录创建以下结构：
```
ad_mcp_v2/
├── ad_mcp_v2.py           # MCP服务主文件
├── api_server.py          # FastAPI HTTP服务（可选）
├── gen_mock_data.py       # Mock数据生成脚本
├── data/                  # 数据目录
│   └── *.csv              # Mock数据文件
└── web/
    └── dashboard.html     # 联调看板（可选）
```

二、技术栈
- MCP框架：FastMCP (`pip install fastmcp`)
- 数据处理：Pandas
- 可选HTTP服务：FastAPI + Uvicorn

三、MCP Tools要求

1. **命名规范**：`{db_type}_{table_name}_query`

2. **查询Tools（10个）**：
   - es_attr_click_query：媒体点击表（**无device_id**，用idfa/oaid/android_id查询）
   - es_attr_device_query：全量设备表（有device_id）
   - es_attr_user_query：全量用户表
   - es_attr_res_activation_query：激活归因结果表
   - es_attr_res_register_query：注册归因结果表
   - es_attr_res_pay_query：付费归因结果表
   - sr_ods_uba_base_event_rt_query：基础原始事件表
   - sr_ods_biz_campaign_query：监测活动表
   - sr_ods_uba_attr_event_rt_query：点击归因表
   - mysql_retry_query：回推失败重试表

3. **诊断Tools（4个）**：
   - diagnose_full_funnel：全链路诊断（点击→激活→注册→付费→回推）
   - diagnose_h5_app_link：H5+APP归因诊断
   - diagnose_pc_tracking：PC端归因诊断
   - diagnose_ios_fusion：iOS融合归因诊断

4. **辅助Tools（2个）**：
   - list_test_cases：列出测试用例
   - get_test_case：获取测试用例详情

四、Mock数据要求

1. **数据量**：20条归因链路
   - 15条正常链路（SUCCESS_000~014）
   - 5条异常链路（ABNORMAL_000~004）

2. **时序约束**：点击时间 < 激活时间 < 注册时间 < 付费时间

3. **异常用例**：
   - ABNORMAL_000：时序倒挂（激活早于点击）
   - ABNORMAL_001：自然量（无点击）
   - ABNORMAL_002：转化中断（有激活无注册）
   - ABNORMAL_003：付费流失（有注册无付费）
   - ABNORMAL_004：回推失败（retry_status=WAITING）

4. **ID关联**：
   - 每条链路通过case_id贯穿所有表
   - 点击表.idfa = 设备表.idfa
   - 设备表.device_id关联其他表

五、重要注意事项

1. **attr_click表没有device_id字段**，必须通过idfa/oaid/android_id查询
2. **diagnose_full_funnel诊断逻辑**：
   - 先通过device_id查设备表获取idfa
   - 再通过idfa查点击表
3. 所有Tool返回统一格式：
   ```json
   {"success": true, "data": [...], "total": n, "returned": n}
   ```

六、交付要求
1. 生成完整可运行的代码
2. 生成Mock数据CSV文件
3. 数据之间ID关联正确
4. 时序约束正确
```
