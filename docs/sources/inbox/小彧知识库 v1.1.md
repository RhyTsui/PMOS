## 一、需求/功能点

1.  知识范围：个人 / 部门 / 项目 / 公共四类知识范围
    
2.  首个用户注册后作为超管，其他用户通过小闪登录
    
3.  个人知识库的知识默认私有，其他人不可见，除非个人知识、个人知识库分享给其他人和空间。并可挂主对话框
    
4.  部门内成员可共建部门知识库，由部门管理员维护，发布后可供部门成员和指定业务 Chat 使用。
    
5.  公司知识库，公共支持库由指定负责人维护，发布后可供公司内部用户检索和指定 AI 应用挂载。
    
6.  项目成员创建 `project` 范围资产，项目内协作维护，项目结束后支持转交或回收。
    
7.  登录后默认进入主对话框
    
8.  模型只支持超管配置，用户默认使用系统内置模型，页面只展示模型名称
    
9.  知识库默认全选，可切换
    
10.  默认内置数据部MCP服务，可MCP外挂管理
    
11.  挂载提示词资产
    
12.  仅超管有高级配置权限
    
13.  个人默认私有
    
14.  项目范围隔离
    
15.  部门范围隔离
    
16.  公共范围受控开放
    
17.  AI 应用单独作为绑定主体
    
18.  小闪用户管理，包括所属小组/部门/公司/集团多级组织结构。
    
19.  知识权限包括：可控制、可见、检索、引用、编辑、绑定
    
20.  支持资产范围类型：`scope_type`：`personal`、`project`、`department`、`public`
    
21.  统一消费接口，不允许业务 Chat 直接绕过平台层访问底座知识。最小能力：权限\-指定资产范围\-指定资产类型，返回引用片段\-来源信息\-绑定信息
    

### 二、开发约束点

|  **模块**  |  **说明**  |
| --- | --- |
|  ### 与 WeKnora 的边界1  |  权限中心只作为上游鉴权输入，不替代平台权限控制面  |
|  ### 与 WeKnora 的边界2  |  平台层负责：资产模型、权限模型、消费绑定、页面信息架构、业务入口表达  |
|  权限控制  |  非 v1.1 阻塞项，可先用适配导入结果替代  |
|  来源追踪  |  严禁把 v1.2 及后续能力提前塞入  |

### 三、测试验收点

|  **非功能需求**  |  **验收标准**  |
| --- | --- |
|  权限控制  |  100% 按策略过滤，包含检索、引用、绑定  |
|  来源追踪  |  100% 返回来源信息，主对话框、业务 Chat 均适用  |
|  可扩展性  |  支持新增资产类型与消费端，不改底层主结构  |
|  Demo跑通  |  本地 Demo 可联通  |
|  架构边界  |  平台层与底座层职责明确不允许职责混杂  |
|  项目  |  验收标准  |
|  资产模型  |  个人 / 项目 / 部门 / 公共统一建模  |
|  权限模型  |  知识权限与工具权限分离  |
|  消费模型  |  主对话框与业务 Chat 共享统一消费壳  |
|  产品结构  |  已不再是单一知识库产品结构  |
|  底座边界  |  WeKnora 底座职责未被重写  |
|  **Demo**   |  **验收标准**  |
|  个人知识使用  |  个人资产默认私有并可挂主对话框  |
|  部门知识服务  |  数据部资产可供指定小闪 cherry Chat 使用  |
|  项目知识协作  |  指间山海资产可被项目成员共建和消费  |
|  公共知识服务  |  公共资产可被受控检索和挂载  |
|  主对话框消费  |  可联合多个知识范围与提示词资产分析  |
