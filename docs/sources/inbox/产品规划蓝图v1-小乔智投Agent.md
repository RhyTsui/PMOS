### 第2页：我们面临的核心效率瓶颈

**页面标题**：广告业务规模扩张，但人效瓶颈日益突出

**表格**：

|  痛点  |  现状  |  影响  |
| --- | --- | --- |
|  对接联调太慢  |  单次新渠道对接耗时5人天，依赖人工配置和手动联调  |  业务等不起，交付时间冲突频发  |
|  问题排查太难  |  数据Gap定位平均30分钟，依赖专家逐层查日志  |  数据信任度低，不敢依数据决策  |
|  取数分析太烦  |  大量需求靠数分手动SQL配报表，用户不能下载明细  |  分析师被临时需求淹没，满意度走低  |
|  素材数据断裂  |  海外MD5变化、国内3.0升级断档，素材无法聚合分析  |  创意效果评估失准，经验无法沉淀  |
|  外部规则静默变更  |  媒体文档无通知更新，系统被动发现故障  |  排查成本高，稳定性受制于人  |

**备注话术**：

> 我们看一下现状。随着投放规模扩大，五个核心痛点越来越突出。对接联调一次要5人天，排查一个数据Gap平均30分钟，临时取数需求全靠数分手写SQL，海外素材因为MD5变化根本对不齐，媒体规则还经常静默更新让我们被动。这些问题每天都在发生，直接拖累了人效和业务满意度。

### 第3页：AI时代的新解法

**页面标题**：大模型与Agent技术，为解决效率瓶颈提供全新范式

**对比表**：

|  场景  |  传统模式  |  AI Agent模式  |
| --- | --- | --- |
|  媒体对接  |  人工读文档 → 手动配置 → 联调验证  |  知识底座理解 → MCP自动配置 → 自动化联调  |
|  数据排查  |  跨系统查日志 → 人肉对比 → 经验判断  |  决策树执行 → MCP并行查询 → 证据链输出  |

> 我们不是做BI升级或RPA工具，而是构建一个**能理解广告业务、能自动执行任务的AI操作系统**。

**备注话术**：

> AI大模型和Agent技术的成熟，给了我们全新的解题思路。左边是传统人肉模式，右边是我们规划的AI Agent模式。核心变化在于：用知识底座替代人脑记忆，用MCP工具替代手动操作，用决策树替代经验判断。我们的目标不是做又一个BI看板，而是做一个真正能理解和执行广告任务的AI操作系统。

### 第6页：核心能力详解（一）—— 智能对接与联调

**页面标题**：把“5天”压缩到“2小时”的自动化对接引擎

**工作流程图**：

text

    投放提交结构化需求
            ↓
    系统自动预检（调用数据质量监控MCP校验埋点/账号/包体）
            ↓
    自动生成工时预估（倒逼业务明确交付预期）
            ↓
    MCP自动配置监测链接（调用媒体API）
            ↓
    一键触发Playwright+Appium自动化联调
            ↓
    实时日志推送 + 5分钟生成联调报告

**协同亮点**：联调前调用数据开发平台的**埋点验收AI**，确保游戏侧埋点已就绪，避免无效联调。

**预期效果**：

*   对接联调耗时：5人天 → 2小时（效率提升 **94%**）
    
*   60%+配置工作由AI自动完成
    
*   流程规范化，隐性知识系统化
    

**备注话术**：

> 第一个核心能力是智能对接与联调。传统模式下一个新渠道对接要5人天，中间大量时间花在来回沟通需求、手动配置后台、人工扫码联调上。我们的方案是：投放填一张结构化表单，系统自动校验埋点、账号、包体是否就绪，给出工时预估，然后自动调用媒体API完成监测链接配置，最后Playwright和Appium自动执行联调。整个流程从5天压缩到2小时，效率提升94%。

### 第7页：核心能力详解（二）—— 诊断Agent

**页面标题**：把“30分钟”缩短到“3分钟”的因果推理引擎

**技术内核**：决策树 + MCP工具调用 + LLM证据总结

**排查路径图（激活Gap示例）**：

text

    SDK日志检查 → 有无数据？ → 无 → 结论：SDK未上报
                    ↓有
             点击日志匹配 → 是否匹配？ → 否 → 结论：非归因流量/超出窗口
                    ↓是
             归因状态检查 → 是否成功？ → 否 → 结论：归因逻辑/风控拦截
                    ↓是
             回传日志验证 → HTTP状态？ → 4xx/5xx → 结论：媒体接口异常/参数错误
                    ↓200
             结论：数据延迟/BI聚合问题

**协同亮点**：

*   调用**数据质量监控MCP**快速判断是否为数据侧问题
    
*   调用**多维分析系统**查询明细日志，与**现有BI**数据交叉对比
    
*   调用**算法知识库**检索近期媒体文档变更，主动发现外部规则问题
    

**输出格式**：一句话结论 + 可下钻证据链 + 置信度 + 操作建议

**预期效果**：

*   问题定位时间：30分钟 → 3分钟（效率提升 **90%**）
    
*   新人无需培训即可排查80%常见问题
    

**备注话术**：

> 第二个能力是诊断Agent。传统排查一个激活Gap，需要人工登录多个系统查日志、做对比，平均30分钟。我们的诊断Agent内置了决策树逻辑，自动调用MCP工具并行查询SDK、点击、归因、回传四个环节，结合知识库的媒体文档变更记录，快速定位根因并输出证据链。从30分钟降到3分钟，而且新人也能用。

### 第8页：核心能力详解（三）—— 对话式分析与素材Agent

**页面标题**：把分析师从“手动SQL”中解放，让素材数据“看得全”

**对话式分析对比**：

|  用户说  |  系统做  |  替代旧模式  |
| --- | --- | --- |
|  “对比A/B优化师上周ROI”  |  调用分析模板，返回对比卡片  |  数分手写SQL→跑数→贴Excel  |
|  “再加一个次留率指标”  |  在原结果上追加计算，无需重跑  |  数分改SQL重新执行  |
|  “按素材组下钻”  |  执行预置下钻查询  |  数分手工分组聚合  |

**素材Agent技术路径**：

text

    视频素材 → 大模型生成结构化脚本 → 语义向量化 → 存入算法知识库 
        → BERTopic聚类 → 按“内容”而非“文件名”聚合 → 关联广告BI数据 → 创意报表

**协同亮点**：

*   对话式分析直接翻译用户自然语言为**多维分析系统**的查询指令
    
*   素材聚类结果存入**算法知识库**，与**广告BI**消耗/转化数据关联
    

**预期效果**：

*   临时取数需求响应：天级 → 秒级，数分人力节省 **60%**
    
*   海外素材跨媒体聚合分析：从“不可能”变为“分钟级”
    

**备注话术**：

> 第三个是对话式分析和素材Agent。对话式分析让用户用自然语言就能做聚合对比，系统直接调用多维分析引擎返回结果，替代了数分手写SQL。素材Agent则是用大模型把视频转成结构化的脚本文本，再向量化聚类，这样同一个创意不管在哪个媒体、MD5怎么变，都能按内容聚合起来，关联BI数据后就能看到全球素材的真实表现。这两个能力一个解放了数分人力，一个打破了素材数据孤岛。

|  **设计Agent**  |  媒体素材自动采集、脚本智能评分、脚本生成、视频生成  |  素材从“人工生产”变为“AI原生”，抓住广告最大变量  |
|  **投手Agent**  |  对话建分包CLI、建短链MCP、媒体投放MCP封装、素材组合测试、广告托管  |  投放操作从“GUI点点点”升级为“对话式指令”，投放规模不再受人力限制  |

**未来开放生态**：

> MCP/Skill/Agent 本身是开放可组合的。小乔的MCP工具可被公司其他系统调用（如运营分析Chat集成广告能力），外部系统（AI素材生产平台、AI投放平台、游戏研发中台、SDK客户端）未来也可通过MCP协议无缝接入，形成**广告业务AI生态**。

**一句话总结**：

> Phase 1让我们**跑得快**，Phase 2/3让我们**跑得远**。最终形态是一个覆盖“接入→诊断→分析→策略→创意→执行”全链路的**广告业务AI操作系统**。
