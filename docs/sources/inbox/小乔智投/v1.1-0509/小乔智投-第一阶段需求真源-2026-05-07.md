## 1. 需求问题定义

### 1.1 原始问题

广告投放支持当前高度依赖极少数专家人力，主要工作包括：

1. 使用帮助与指标解释
2. 需求沟通与对接配置
3. 数据异常排查与归因解释
4. 广告联调与回传验证
5. 长驻监控与问题感知
6. 素材内容分析与沉淀
7. 广告效果预测与测算

这些工作长期依赖人工沟通、人工判断、人工串联多个系统完成。

### 1.2 当前最核心损失

1. 人效低
2. 判断慢
3. 风险高
4. 经验无法稳定复用
5. 新老成员都要反复接受同类解释
6. 关键人力被低效沟通和重复排查长期占用

---

## 2. 第一阶段需求目标

### 2.1 总目标

先把广告支持体系中最典型、最高频、最依赖专家经验的支持工作，收敛到一个统一产品入口内，形成可演示、可扩展、可继续接能力的前端工作台。

### 2.2 第一阶段产品目标

第一阶段不是要一次性完成广告 AI 全链路，而是要完成以下三件事：

1. 把高频任务统一收到一个产品入口内
2. 把主要工作从人工口头推进，转换成结构化任务推进
3. 为后续分析、执行、创意能力接入预留稳定前端骨架

### 2.3 第一阶段用户目标

让用户能够在一个统一入口内完成：

1. 获取使用帮助
2. 提交和澄清需求
3. 发起和查看问题排查
4. 发起和查看广告联调
5. 查看系统监控状态
6. 查看素材分析结果
7. 查看广告预测结果

---

## 3. 第一阶段需求范围

### 3.1 主工作台范围

主工作台必须覆盖四类高频主任务：

1. 使用帮助
2. 需求沟通
3. 问题排查
4. 广告联调

### 3.2 专项页范围

第一阶段同时纳入三类专项能力页：

1. 监控大屏页
2. 素材分析页
3. 广告预测页

### 3.3 后续已知接入节奏

1. 2026 年 6 月：广告 BI 数据分析 MCP 接入
2. 2026 年 8 月：广告执行 Agent 接入，包括自动搭建、自动暂停等动作
3. 2026 年 9 月：基于创意脚本衍生视频的素材生产能力接入

### 3.4 应用支撑层范围

第一阶段除了业务能力层，还需要纳入应用支撑层需求。

#### 3.4.1 已确认的支撑能力

1. 个人知识库挂载
2. 跨部门 MCP 挂载
3. 定时任务
4. 小闪通知
5. 全局记忆
6. 网络搜索
7. 快捷短语
8. 技能
9. 常规设置
10. 图片 / 文档上传

#### 3.4.2 需求判断

这些能力不一定都要在第一阶段做成完整独立系统，但必须进入产品结构与前端原型范围。

#### 3.4.3 外部平台边界

以下能力由外部平台承接，不在小乔智投内重做：

1. Dataki：知识库维护、知识资产、提示词资产管理与归档
2. 连弩测试平台：AI 应用评测、调试与效果验证

以下平台不作为当前主流程核心编排前提：

1. n8n：不作为小乔智投主流程核心编排平台，仅保留为未来可选外围自动化工具

小乔智投内部仍必须保留：

1. 业务会话记录
2. 任务上下文
3. 结果记录
4. 历史任务
5. Case 沉淀

## 4. Agent 需求边界

### 4.1 意图 Agent

负责识别当前用户属于哪类任务：

1. 使用帮助
2. 需求沟通
3. 问题排查
4. 广告联调

### 4.2 客服 / 使用帮助 Agent

负责：

1. 知识库问答
2. 示例参考
3. 系统跳转
4. 指标解释
5. 术语与逻辑说明

### 4.3 需求沟通 Agent

负责：

1. 结合知识库理解诉求
2. 调用媒体文档爬虫获取规则上下文
3. 调用需求池 MCP 写入或读取需求
4. 生成标准需求文档
5. 调用媒体对接配置 MCP
6. 通过追问补齐缺失字段
7. 获取需求依赖
8. 输出需求反馈
9. 解释指标与逻辑

### 4.4 问题排查 Agent

负责：

1. 调取数据上报
2. 调取预处理数据
3. 调取归因结果
4. 调取回推结果
5. 调取媒体广告数据
6. 调取游戏客户端回推记录 MCP
7. 输出证据链、结论、置信度和下一步建议

### 4.5 联调 Agent

负责：

1. 读取默认投放账号
2. 读取白名单设备
3. 执行浏览器自动化
4. 执行 App 自动化控制
5. 联动问题排查
6. 联动大屏监控
7. 输出联调日志和联调报告

### 4.6 大屏监控 Agent

负责：

1. 运维 Agent 接入
2. 操作监控
3. 游戏上报监控
4. 回推监控
5. 归因监控
6. 报表监控
7. 页面监控

### 4.7 素材分析 Agent

负责：

1. 创意脚本解析
2. 创意脚本相似度匹配
3. 创意脚本 ID 生成
4. 创意报表生成
5. 创意数据分析
6. 优质创意沉淀

### 4.8 广告预测 Agent

负责：

1. 大盘预测
2. 媒体预测
3. 应用类型预测
4. 广告预测
5. 素材预测
6. LTV 预测
7. ROI 预测
8. 回本测算
9. 首日 ROI 倒推

## 5. 应用支撑层需求

### 5.1 业务上下文与历史

小乔智投必须有自己的业务上下文系统，至少记录：

1. 用户与 Agent 的会话内容
2. 当前任务类型
3. 当前任务关联的媒体、应用、计划、设备、时间窗
4. 已生成的结构化结果
5. 历史任务与 Case 沉淀

### 5.2 配置与挂载

小乔智投需要能展示或配置：

1. 个人知识库挂载状态
2. 跨部门 MCP 挂载状态
3. 技能启用状态
4. 常规设置
5. 快捷短语

### 5.3 自动化与通知

小乔智投需要支持：

1. 定时任务
2. 小闪通知
3. 结果完成后的外部提醒

这些能力的主实现方向应优先落在小乔智投自有流程与业务运行时上，而不是默认依赖 n8n。

### 5.4 非结构化输入

小乔智投需要支持：

1. 图片上传
2. 文档上传
3. 基于上传内容启动帮助、需求、排查或联调任务
