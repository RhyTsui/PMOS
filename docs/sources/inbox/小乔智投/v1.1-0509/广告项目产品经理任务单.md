- 状态：Active

- 评审人：虚拟产品主管 Agent

---

## 一、任务目标

你当前不是继续扩写长期广告平台方案。

你当前只负责把广告项目收敛成一套可执行的月底 Demo 方案输入，并把后续 MVP 输入清楚交出来。

你的职责是：

1. 把已有 PRD、蓝图、Demo 重规划统一收口
2. 形成月底 Demo 的执行拆解
3. 主动同步进展

---

## 二、必须交付的 4 份文档

### 1. `月底 Demo 执行任务拆解`

必须回答：

1. 产品需要完成哪些任务
2. 前端需要完成哪些任务
3. 后端需要完成哪些 Mock API 任务
4. 数据/内容需要准备什么
5. 哪些任务是 P0，哪些是 P1

### 2. `统一工作台与核心场景用户流`

必须包含：

1. 统一工作台入口结构
2. `诊断 Agent Demo` 用户流
3. `对话式分析 Demo` 用户流
4. 状态流转
5. 关键结果卡片结构

### 3. `Demo 验收清单`

至少包含：

1. 演示前要检查什么
2. 演示时必须能看到什么
3. 什么情况算月底 Demo 达标
4. 遗留问题如何记录

### 4. `Demo 后 MVP 输入清单`

必须说明：

1. 月底 Demo 验证后哪些能力进入 MVP
2. 哪些能力继续延后
3. 哪些问题需要主管拍板

---

## 三、提交格式

每份文档必须包含：

1. 核心结论
2. 依据来源
3. 当前范围
4. 不确定项
5. 需要主管决策的问题

如果缺少以上字段，视为未提交完成。

---

## 四、主动同步要求

你必须主动同步，不允许只埋头做而不汇报。

同步频率：

1. 接到任务后先同步计划
2. 每完成一份首稿同步一次
3. 若 30 分钟内仍未形成首稿，也要同步卡点

每次同步至少写清：

1. 已产出什么
2. 当前在写什么
3. 还缺什么
4. 当前卡点是什么

没有同步的内容，不视为主管已收到。

---

## 五、禁止事项

当前阶段禁止：

1. 继续扩写“大而全 Ad OS”
2. 把真实媒体接入、素材 Agent、知识库治理等后续能力塞进月底 Demo
3. 只给方向，不给任务拆解
4. 不写不确定项，直接下定案

---

## 六、评审标准

你的提交会按以下标准评审：

1. 是否忠实承接当前 `Demo First` 方向
2. 是否真的收敛范围，而不是继续发散
3. 是否能直接支撑月底 Demo 执行
4. 是否写清楚了任务、验收和后续 MVP 输入
5. 是否按要求主动同步

主管评审结论只会是：

1. `accepted`
2. `accepted-with-constraints`
3. `needs-more-clarity`
4. `blocked-by-scope`
5. `rejected`

---

## 七、一句话要求

先把月底 Demo 跑通，再把后续 MVP 说清；先交执行拆解，不要继续扩大战线。
