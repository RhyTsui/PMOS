# 广告项目业务视角补充

更新日期：2026-04-20

## 1. 当前业务位置

当前产品设计必须从广告支持业务视角出发，而不是从抽象 AI 平台视角出发。

需求提出人与协作链路如下：

### 1.1 前端需求侧

1. 投放
2. 设计师
3. 投放主管
4. 财务

### 1.2 中间协同侧

1. 广告支持
2. SDK
3. QA
4. 商务
5. 数据分析师
6. 开发

### 1.3 外部生态侧

1. 第三方合作方
2. 创量
3. 热云 ADX
4. 其他媒体、归因与广告生态合作方

## 2. 核心出发点

产品不是先做一个泛化广告 AI 平台，也不是先做自动投放系统。  
第一阶段必须服务当前广告支持业务，并优先解决投放对支持团队的依赖问题。

核心判断：

1. 先实现支持侧自动化。
2. 先赋能投放团队，而不是一开始替代投放团队。
3. 中长期再逐步实现投放自动化。
4. 最终目标确实指向减少投放人力，但不能跨阶段跳做。

## 3. 组织目标与阶段路线

### 3.1 短期目标

先把广告支持业务中的高重复、强规则、低创造性劳动自动化。

这部分直接对应：

1. 需求接入
2. 标准联调
3. 异常排查
4. 对话式分析
5. 知识复用
6. Case 自动归档

短期目标不是“自动投放”，而是：

1. 减少支持侧重复劳动
2. 缩短投放响应时间
3. 提高投放满意度
4. 让投放更少依赖人找人

### 3.2 中期目标

逐步把投放培养成系统用户，让投放具备自助能力。

包括：

1. 自助分析
2. 自助排查
3. 自助联调发起
4. 自助规则配置
5. 自助指标调用
6. 自助素材洞察

中期目标本质上是让投放从“提需求的人”变成“操作系统的人”。

### 3.3 长期目标

逐步推进投放自动化，服务老板长期减少投放人力的目标。

长期能力包括：

1. 自动预算建议
2. 自动出价建议
3. 自动停启建议
4. 自动素材投放策略
5. 自动实验组合
6. 自动归因反馈闭环

但这部分必须建立在前两阶段已验证的支持自动化与投放自助化基础上。

## 4. 当前最合理的产品定位

当前阶段最准确的产品定位不是：

1. 广告 AI 平台
2. 自动投放系统
3. 通用广告 AI 中台

而是：

**广告支持与投放协同自动化工作台**

这个定位强调：

1. 从广告支持业务出发
2. 以投放协同提效为第一价值
3. 先自动化支持侧，再逐步走向投放自动化

## 5. 自动化边界

### 5.1 可直接自动化

特点：

1. 规则明确
2. 输入结构化
3. 错误成本可控
4. 不需要承担最终商业责任

优先候选：

1. 需求收集
2. 表单补齐
3. 标准联调
4. 日志归集
5. 指标查询
6. 常见诊断
7. 素材标签化初筛

### 5.2 适合人机协作

特点：

1. 系统先做 70%
2. 人做最后确认
3. 存在业务判断，但不完全依赖创意或资源博弈

优先候选：

1. 异常根因判断
2. 联调问题解释
3. 分析结论确认
4. 需求可行性判断
5. 投放建议
6. 素材方向总结

### 5.3 现阶段必须由人负责

特点：

1. 高风险
2. 高博弈
3. 高商业判断
4. 高责任归属

包括：

1. 投放预算决策
2. 重大活动排期
3. 财务协调
4. 商务策略
5. 品牌与合规风险判断
6. 跨团队资源协调

## 6. 两条主线与当前优先级

### 6.1 主线 A：从素材切

适合解决：

1. 素材聚合
2. 素材标签化
3. 素材脚本理解
4. 相似素材推荐
5. 创意方向辅助

价值高，但路径长、依赖多、月底难以形成完整闭环。

判断：

中长期主战场，但不应抢占当前主线。

### 6.2 主线 B：从数据和广告支持切

适合解决：

1. 需求接入
2. 自动联调
3. 异常诊断
4. 对话式分析
5. 指标解释
6. Case 归档
7. 知识复用

判断：

这是当前第一优先级主线，因为：

1. 规则更明确
2. 依赖更可控
3. 更容易先替代支持人力
4. 更容易先赋能投放

## 7. 第一阶段的核心价值点

第一阶段建议聚焦以下四个价值点：

1. 投放需求自动接入
2. 标准联调自动化
3. 异常排查自动化
4. 对话式分析

这四个能力共同构成“先替代支持侧重复劳动，再逐步训练投放使用系统”的最小闭环。

## 8. 设计原则

后续所有需求拆解、方案设计与 Demo 规划必须遵守以下原则：

1. 先服务广告支持业务，不先追求宏大平台叙事。
2. 先让投放少依赖人，不先承诺完全替代投放。
3. 先自动化支持侧，再推进投放自助化，最后才讨论投放自动化。
4. 优先选择规则明确、可沉淀、可演示、可替代人工的场景。
5. 需求讨论必须同时考虑“人、流程、自动化边界”，不能只看功能点。

## 9. 对后续需求分析的约束

后续逐个需求进行头脑风暴、拆解与可行性分析时，必须增加以下分析维度：

1. 这个需求服务哪个角色群体。
2. 这个需求是在替代支持侧劳动，还是在赋能投放侧自助。
3. 这个需求属于可直接自动化、人机协作、还是必须由人负责。
4. 这个需求对长期投放自动化路线的价值是什么。
5. 这个需求是否会偏离当前阶段的主线。

## 10. 当前共识

当前阶段的正确路线是：

**先把广告支持业务流程自动化，并把投放逐步训练成系统用户，而不是人力依赖者。**
