## 0. 说明

本文档不是替代需求池，而是把小乔智投当前已确认的规划内容，先压成符合需求池字段结构的首批候选条目，便于后续逐条写入正式 `需求池.xlsx`。

当前原则：

1. 一条需求只解决一个明确问题
2. 每条需求都要可衡量、可验收
3. 每条需求都要有对应解决方案
4. 先写 P0 主链路，再写专项页和支撑能力

## 4. 当前建议的入池顺序

建议不要一次性把所有规划都写进需求池，而是先按主链路入池：

1. 自由问答统一入口
2. 工作/非工作隐式判断
3. 使用帮助工作流
4. 问题排查工作流
5. 需求沟通工作流
6. 广告联调工作流
7. 会话与任务沉淀
8. 单对话工作台原型

这样做的原因是：

1. 先锁定主链路
2. 先保证每条需求都能明确验收
3. 避免把专项页和支撑能力先写成大而散的规划词
