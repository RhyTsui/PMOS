## 一、规范定位

本规范是基于现有设计规范和组件使用说明整理出的“落地执行版 UI 规范”。  
它不是重新设计一套视觉体系，而是将已有规范转化为前端、AI 工具和页面整改时可直接执行的标准。

本规范主要用于：

1. 统一当前项目页面视觉风格；
2. 统一组件使用方式；
3. 统一页面布局、字体、颜色、间距、交互状态；
4. 指导 AI / Codex / Figma Make 按规范生成或修改页面；
5. 帮助前端判断当前实现是否符合 UI 规范。

---

## 二、设计目标

项目 UI 需满足以下目标：

1. **统一性**  
   页面、组件、交互、状态保持一致，避免不同页面各自一套样式。

2. **规范性**  
   设计和开发均以现有 UI 设计规范、Pixso 组件库和前端通用组件库为依据。

3. **效率性**  
   产品侧可直接使用 Pixso 组件搭建原型，前端侧可直接复用通用组件，减少重复标注和沟通。

4. **可维护性**  
   将颜色、字体、间距、按钮、表格、表单等沉淀为可复用规则，便于后续版本迭代。

5. **业务适配性**  
   规范服务于 B 端后台、报表、配置、数据分析、素材管理等业务场景，不做过度装饰。

## 三、设计语言

整体设计语言为：

- 简洁
- 清晰
- 专业
- 统一
- 高效
- 数据友好
- 适合后台管理与业务分析场景

页面设计需避免：

1. 无依据的大面积渐变；
2. 无规范来源的阴影和圆角；
3. 多套按钮样式混用；
4. 表格、表单、筛选区各页面样式不统一；
5. 字号、颜色、间距随意变化；
6. 为追求视觉效果而影响数据阅读效率。

## 四、组件库使用原则

### 4.1 产品侧组件库

产品侧在 Pixso 中使用设计组件搭建原型，确保原型阶段即符合设计系统。  
使用方式：

1. 打开项目设计文件；
2. 进入左上角「组件」查看组件；
3. 如无组件，进入「资源」；
4. 打开团队资源库中的「UI设计规范」；
5. 在组件模块中选择分类；
6. 进入二级分类后应用对应实例到画布中。

### 4.2 前端侧组件库

前端侧优先调用内部通用组件库。  
组件类型包括：

| 类型 | 包含内容 |
|---|---|
| 主题组件 | 输入框主题、表格主题、日期主题 |
| 业务组件 | 抽屉提示页、富文本编辑器、空状态占位、通用筛选器 |
| 功能组件 | 倒计时时钟、防抖输入框、多选组件、数字输入框、刷新按钮、历史搜索、文本可编辑、Tooltip、树形穿梭框 |
| 创意组件 | 带进度条的按钮 |
| UI 组件 | 逻辑运算符、按钮、卡片、复选框、描述列表、全局提示、分页、单选框、分段控制器、选择器、加载中、统计数值、开关、标签页、文字提示 |

### 4.3 组件复用要求

1. 页面中所有通用组件必须优先复用组件库；
2. 不允许为单个页面私自写一套按钮、表格、输入框样式；
3. 如业务确实需要扩展，需在现有组件基础上扩展，不得破坏基础规范；
4. 同类组件在不同页面中需保持一致；
5. 组件状态必须完整，包括默认、悬停、点击、选中、禁用、加载、错误、空状态。

## 五、色彩规范

### 5.1 色彩体系

项目色彩包含：

1. 主题色 / 品牌色；
2. 辅助色 / 状态色；
3. 中性色 / 文字色；
4. 背景色；
5. 边框色；
6. 阴影色；
7. 可视化图表色。

### 5.2 已提取的常用色值

| 色值 | 使用建议 |
|---|---|
| `#3B86F9` | 主色、页面标题、选中态、重点操作、编辑操作 |
| `#94CAFC` | 主色悬停或弱化状态 |
| `#6B7981` | 辅助说明文字 |
| `#252934` | 重要文字、主要信息 |
| `#E7E7E7` | 默认筛选边框或弱边界 |
| `#F9FAFC` | 表格汇总行、浅灰背景、禁用背景 |
| `#FAFAFA` | 表格行 hover 背景 |
| `#FAFAFC` | 下拉悬停背景 |
| `#FFFFFF` | 卡片、表格默认底色 |
| `#E5E6EB` | 常规边框/分割线 |

### 5.3 色彩使用要求

1. 主操作和选中态优先使用主色 `#3B86F9`；
2. 辅助文字使用弱化色，不与主标题抢层级；
3. 表格、卡片、筛选区背景保持简洁；
4. 错误、警告、成功状态需使用规范状态色；
5. 图表颜色需使用规范中的可视化色值，不得随机取色；
6. 禁止页面中出现与规范不一致的自定义强色。

## 六、字体规范

### 6.1 字体使用原则

1. 使用系统默认中文字体；
2. 字号、字重需按信息层级使用；
3. 不允许页面内随意改变字体大小；
4. 数据、标题、正文、辅助说明需形成明显层级；
5. 表格和表单内容以易读为优先。

### 6.2 常用字号层级

| 类型 | 建议规范 | 使用场景 |
|---|---|---|
| 页面名称 | 18px / 主色 | 页面顶部标题 |
| 辅助说明 | 12px / `#6B7981` | 页面说明、弱提示 |
| 模块标题 | 加粗 | 卡片标题、模块标题 |
| 正文内容 | 常规字号 | 表单、表格、正文 |
| 表格内容 | 常规字号 | 表格单元格 |
| 统计数值 | 强调字号 | 指标卡片、数据概览 |

### 6.3 页眉文字规范

| 内容 | 规范 |
|---|---|
| 页面名称 | 18px，颜色 `#3B86F9` |
| 辅助说明 | 12px，颜色 `#6B7981` |
| 默认页眉高度 | 最小 60px |
| 有筛选项页眉高度 | 最大 110px |

## 七、布局与间距规范

### 7.1 页面结构

常规页面建议使用以下结构：

1. 页面页眉；
2. 页面说明 / 页面切换；
3. 筛选区；
4. 操作区；
5. 内容区；
6. 表格 / 图表 / 表单；
7. 分页 / 底部操作区。

### 7.2 布局要求

1. 页面内容不得贴边；
2. 页面标题、筛选区、表格区之间间距统一；
3. 卡片上下左右内边距统一；
4. 操作按钮成组排列，间距一致；
5. 筛选项横向排列时需对齐；
6. 表单 label 与输入框需统一对齐；
7. 卡片页签与卡片上下左间距建议为 20px；
8. 卡片页签图标与模块名称间距建议为 5px。

---

## 八、按钮规范

### 8.1 按钮类型

| 类型 | 使用场景 |
|---|---|
| 主按钮 | 新增、保存、提交、确认、查询等主要操作 |
| 次按钮 | 取消、返回、重置等次要操作 |
| 文字按钮 | 表格内查看、编辑、复制、详情等轻量操作 |
| 危险按钮 | 删除、移除、清空等高风险操作 |
| 带进度按钮 | 有明确进度反馈的特殊操作 |

### 8.2 按钮状态

按钮需包含以下状态：

1. 默认；
2. 悬停；
3. 点击；
4. 禁用；
5. 加载中；
6. 成功反馈；
7. 失败反馈。

### 8.3 按钮使用规则

1. 页面主要操作使用一个主按钮，避免多个主按钮抢焦点；
2. 编辑类操作默认使用主色；
3. 编辑按钮默认高亮色可使用 `#3B86F9`；
4. 编辑按钮悬停色可使用 `#94CAFC`；
5. 删除类操作必须二次确认；
6. 表格行内操作按钮之间间距统一，建议 10px；
7. 禁止使用无规范来源的自定义按钮颜色。

## 九、输入类组件规范

### 9.1 文本输入框

适用场景：名称、备注、搜索、编号等短文本输入。

状态要求：

| 状态 | 规范 |
|---|---|
| 默认 | 灰色边框，显示占位符或已输入内容 |
| 聚焦 | 边框高亮，显示光标 |
| 输入中 | 实时显示内容，支持清空按钮 |
| 错误 | 边框变红，下方显示错误提示 |
| 禁用 | 置灰，不可点击，不可输入 |
| 只读 | 内容可复制，不可修改 |

交互要求：

1. 支持前后缀图标；
2. 支持最大长度限制；
3. 支持回车提交或搜索；
4. 失焦触发校验。

### 9.2 密码输入框

1. 默认密文展示；
2. 支持明文 / 密文切换；
3. 右侧眼睛图标随状态变化；
4. 支持长度限制；
5. 支持密码强度或格式校验；
6. 错误时显示错误提示。

### 9.3 数字输入框

1. 仅允许输入数字、小数点、负号；
2. 支持最大值、最小值；
3. 支持步长配置；
4. 支持小数精度；
5. 支持千分位格式化；
6. 超出范围时显示错误状态。

### 9.4 多行文本输入框

1. 支持换行；
2. 支持字数统计；
3. 支持固定高度或自动增高；
4. 接近字数上限时预警；
5. 超出限制时提示错误；
6. 回车只换行，不触发表单提交。

### 9.5 单选 / 复选 / 开关

| 组件 | 使用要求 |
|---|---|
| 单选框 | 同组内只能选一个，点击文字也可选中 |
| 复选框 | 支持多选、取消、全选、半选 |
| 开关 | 用于启用/禁用、显示/隐藏等二元状态 |
| 滑动选择器 | 用于连续数值、区间、权重等场景 |
| 键值对输入 | 用于扩展参数、自定义字段、规则配置 |

## 十、日期组件规范

### 10.1 日期类型

| 类型 | 使用场景 | 格式 |
|---|---|---|
| 单日期-年月日 | 查询某一天数据 | `YYYY-MM-DD` |
| 日期区间-年月日 | 查询时间范围 | `YYYY-MM-DD ~ YYYY-MM-DD` |
| 日期时间-时分秒 | 日志、实时监控 | `YYYY-MM-DD HH:MM:SS` |
| 日期区间-时分秒 | 事件分析、漏斗分析 | `YYYY-MM-DD HH:MM:SS ~ YYYY-MM-DD HH:MM:SS` |
| 日期快捷筛选 | 今天、昨天、近7天、近30天等 | 按业务配置 |
| 日期对比 | 同比、环比、自定义对比 | 按业务配置 |
| 项目节点 | 版本发布时间、活动开始日 | 节点名称 + 日期 |

### 10.2 日期交互

1. 点击输入框弹出日期选择面板；
2. 面板包含日历、年月切换、快捷选项和底部按钮；
3. 选择完成后触发查询；
4. 关联图表和表格需同步刷新；
5. 点击弹窗外或按 ESC 可关闭，不保存本次操作；
6. 选中日期需高亮；
7. 快捷筛选需展示选中状态。

## 十一、筛选区规范

### 11.1 筛选类型

| 类型 | 适用场景 |
|---|---|
| 顶部全局筛选 | 全局报表页面，筛选项少于 5 个 |
| 复合搜索筛选 | 输入框 + 下拉 + 日期等组合查询 |
| 多维度查询筛选 | 多条件、多字段、多维度查询 |
| 历史条件搜索 | 顶部全局搜索、报表名称搜索、最近访问 |

### 11.2 筛选交互

1. 条件之间默认使用“且”逻辑；
2. 点击“查询”按钮执行检索；
3. 点击“重置”按钮清空条件并恢复初始状态；
4. 输入关键词后支持回车查询；
5. 条件未填全时，查询按钮置灰；
6. 查询失败时展示红色错误提示；
7. 查询结果较多时联动分页；
8. 默认每页 20 条。

## 十二、表格规范

### 12.1 表格类型

| 类型 | 使用场景 |
|---|---|
| 基础表格 | 简单文本、数字数据展示 |
| 操作型表格 | 配置后台、权限管理、行内编辑/删除/详情 |
| 图文表格 | 素材库、素材报表、图片/视频缩略图 |
| 多维透视表 | 广告分析、用户行为分析、多维度钻取 |
| 矩阵表/热力表 | 留存分析、LTV、漏斗数据 |
| 汇总表 | 核心指标看板、数据概览 |
| 可编辑表格 | 抽屉配置表单、操作表单 |
| 看板表格 | 任务管理、分析看板、卡片拖拽排序 |

### 12.2 基础表格规范

| 项目 | 规范 |
|---|---|
| 行高 | 固定 40px，包括边框 |
| 默认背景 | 白色 |
| 汇总行背景 | `#F9FAFC` |
| 行悬停背景 | `#FAFAFA` |
| 表头 Tips | 默认无问号，悬停弹出提示 |
| 列宽 | 支持拖动调整，宽度缓存到浏览器 |
| 表头长度 | 默认一行 6 个中文字符，超出换行，最多 2 行 |
| 排序 | 点击排序图标切换升序/降序/取消排序 |
| 表头固定 | 表格滚动超过一屏时自动吸顶 |
| 单元格 | 文本超出隐藏并显示省略号 |
| 数字 | 千分位分隔符，保留两位小数 |
| 空数据 | 使用 “-” 或 “0” 占位 |
| 分页 | 默认 20 条，支持 20 / 50 / 100 |
| 分页位置 | 表格底部右侧 |
| 导出 | 表格右上角，导出时展示 loading |

### 12.3 对齐规范

| 字段类型 | 对齐方式 |
|---|---|
| 维度字段 | 左对齐 |
| 日期字段 | 左对齐 |
| 指标字段 | 右对齐 |
| 操作列 | 居中对齐 |

### 12.4 操作型表格规范

1. 继承基础表格全部规范；
2. 操作列固定在最右侧；
3. 操作列宽度按按钮数量自适应；
4. 操作按钮间距 10px；
5. 编辑操作弹出抽屉或模态框；
6. 删除操作弹出二次确认；
7. 详情操作使用右侧抽屉；
8. 复制成功后展示“复制成功”轻提示；
9. 支持批量选择和批量操作栏。

### 12.5 图文表格规范

1. 行高固定 60px；
2. 缩略图尺寸 75×48px；
3. 多图缩略图右上角叠加数量标识；
4. 视频缩略图中部增加播放图标；
5. 素材名称支持点击复制。

### 12.6 可编辑表格规范

1. 默认展示只读文本；
2. 悬停展示编辑图标；
3. 点击后变为输入框、下拉框或日期选择器；
4. 回车保存，ESC 取消；
5. 输入内容实时校验；
6. 错误时边框变红并展示提示；
7. 禁用编辑时置灰且不可响应。

## 十三、表单规范

### 13.1 表单承载方式

| 类型 | 使用场景 |
|---|---|
| 中部弹框 | 字段少、操作简单、无需参考主界面 |
| 右侧抽屉弹框 | 查看/编辑单行详情，需要参考列表上下文 |
| 新页面打开 | 流程复杂、字段多、信息量大 |

### 13.2 基础表单要求

1. label 对齐方式统一；
2. 输入框宽度统一；
3. 表单项上下间距统一；
4. 必填项需明确标识；
5. 错误提示显示在对应输入项下方；
6. 帮助说明使用弱提示样式；
7. 保存、取消、重置按钮位置统一；
8. 提交前进行校验；
9. 提交成功/失败需有反馈。

### 13.3 复杂表单要求

1. 使用模块化分组；
2. 每组需有明确标题；
3. 字段过多时可分步骤或分区展示；
4. 涉及表格、键值对、穿梭框、开关等组件时，调用对应组件规范；
5. 保证表单主操作始终清晰可见。

---

## 十四、弹框 / 抽屉规范

### 14.1 中部弹框

适用于：

1. 删除确认；
2. 简单新增；
3. 简单编辑；
4. 轻量提示；
5. 少量字段表单。

要求：

1. 标题明确；
2. 内容简洁；
3. 底部按钮顺序统一；
4. 危险操作突出风险；
5. 点击确认后给出反馈。

### 14.2 右侧抽屉

适用于：

1. 行数据详情；
2. 复杂编辑；
3. 需要参考主列表上下文；
4. 字段较多但不需要跳转新页面。

要求：

1. 抽屉标题清晰；
2. 字段分组展示；
3. 关闭后返回原页面状态；
4. 编辑成功后刷新当前行；
5. 详情可配置只读或编辑。

---

## 十五、空状态 / 加载 / 错误状态规范

### 15.1 空状态

1. 容器内水平 + 垂直居中；
2. 元素垂直排列：插图 → 文案 → 按钮；
3. 元素间距均等；
4. 提示文案简洁明确；
5. 如有必要，提供新增或刷新按钮。

### 15.2 加载状态

1. 页面加载、表格加载、导出加载需明确展示；
2. 加载状态不应导致页面布局跳动；
3. 异步操作完成后自动恢复。

### 15.3 错误状态

1. 查询失败展示错误提示；
2. 操作失败展示具体失败原因；
3. 网络异常提示用户重试；
4. 错误提示使用规范错误色；
5. 不使用含糊文案。

---

## 十六、页面类型规范

### 16.1 列表页 / 报表页

必须包含：

1. 页眉；
2. 辅助说明；
3. 筛选区；
4. 操作区；
5. 表格区；
6. 分页；
7. 空状态 / 加载 / 错误状态。

重点检查：

1. 筛选项是否规范；
2. 表格行高是否统一；
3. 操作列是否固定；
4. 分页是否支持 20/50/100；
5. 导出、新增、批量操作是否位置统一。

### 16.2 详情页

必须包含：

1. 页面标题；
2. 返回入口；
3. 状态标签；
4. 信息分组；
5. 主操作区；
6. 必要表格或日志。

重点检查：

1. 字段展示是否清晰；
2. 描述列表是否统一；
3. 操作按钮是否规范；
4. 模块间距是否一致。

### 16.3 表单页 / 配置页

必须包含：

1. 表单标题；
2. 分组区域；
3. 输入组件；
4. 校验提示；
5. 底部操作按钮；
6. 提交反馈。

重点检查：

1. label 是否对齐；
2. 输入框样式是否统一；
3. 错误提示是否规范；
4. 保存/取消按钮是否固定。

### 16.4 看板页 / 数据概览页

必须包含：

1. 顶部全局筛选；
2. 指标卡片；
3. 图表；
4. 表格；
5. 数据状态反馈。

重点检查：

1. 图表颜色是否来自规范；
2. 指标卡片层级是否统一；
3. 图表与表格是否联动；
4. 空数据状态是否补齐。

## 十七、当前项目 UI 整改重点

针对“当前页面与设计规范不一致”的问题，建议优先整改以下内容：

| 优先级 | 整改项 | 说明 |
|---|---|---|
| P0 | 组件复用 | 所有通用组件优先替换为组件库组件 |
| P0 | 表格规范 | 统一行高、表头、分页、操作列、空数据 |
| P0 | 筛选区规范 | 统一筛选项高度、间距、查询/重置按钮 |
| P1 | 字体层级 | 统一页面标题、模块标题、正文、辅助说明 |
| P1 | 颜色使用 | 清除非规范色值，统一主色和状态色 |
| P1 | 表单规范 | 统一 label、输入框、错误提示、按钮区 |
| P1 | 弹框/抽屉 | 新增、编辑、详情、删除确认按场景使用 |
| P2 | 空状态 | 补齐无数据、无权限、加载、错误状态 |
| P2 | 图表规范 | 图表色值与视觉层级统一 |
| P2 | 代码维护 | 抽离变量，减少页面级重复样式 |
