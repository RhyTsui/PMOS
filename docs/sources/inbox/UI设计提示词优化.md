> 适用项目：当前后台/业务系统页面 UI 规范化优化  
> 适用对象：AI 编码工具、Codex、Figma Make、前端开发、UI 设计复核  
> 依据文档：  
> - 《【UI设计规范文档】-设计规范.md》

---

## 一、文档目的

当前项目页面已完成基础功能实现，但页面视觉表现与既有 UI 设计规范存在不一致问题。  
本文件用于约束后续 AI 生成页面、前端页面优化、组件调用与设计还原过程，确保页面严格遵循项目已有设计规范，不再出现样式自由发挥、组件不统一、间距混乱、字体层级不一致等问题。

本文件重点解决以下问题：

1. 页面实现与设计规范不一致；
2. 同类组件在不同页面样式不统一；
3. AI 生成页面时没有明确设计约束；
4. 前端开发缺少统一的页面还原检查标准；
5. 组件库已有能力未被充分复用。

---

## 二、核心执行原则

后续所有页面优化、页面重构、页面生成，必须遵循以下原则：

1. **不重新创造视觉风格**  
   严格使用现有 UI 设计规范，不新增无依据的颜色、圆角、阴影、字体、按钮、表格样式。

2. **优先复用组件库**  
   页面中出现按钮、输入框、表格、筛选器、日期选择器、表单、弹框、抽屉、分页、标签页、空状态等内容时，优先调用已有通用组件或按已有组件规范实现。

3. **业务结构不变**  
   UI 优化不得改变现有业务逻辑、接口字段、页面流程、菜单层级和数据关系。

4. **统一视觉层级**  
   页面标题、模块标题、正文、辅助说明、表格内容、按钮文字需保持统一的字号、字重和颜色层级。

5. **统一页面布局**  
   页面边距、卡片间距、筛选区间距、表格间距、按钮组间距必须符合统一规范，避免局部拥挤或松散。

6. **统一交互状态**  
   所有组件需具备默认、悬停、点击、选中、禁用、加载、错误、空状态等基础状态。

7. **先分析，再修改**  
   AI 或前端在修改页面前，需要先输出当前页面与规范的差异点，再执行优化。

## 三、全局 UI 优化总提示词

以下提示词可作为后续 AI / Codex / Figma Make 处理页面的总入口提示词。

```text
你是一个资深 B 端 UI 设计还原工程师和前端页面规范化专家。

现在需要对当前项目页面进行 UI 规范化优化。项目已经有完整的 UI 设计规范和组件使用说明，页面优化必须严格遵循现有规范，不允许自由发挥新的视觉风格。

请按以下要求执行：

1. 保持现有业务功能、页面结构、接口逻辑、字段含义不变；
2. 严格使用项目现有 UI 设计规范中的色彩、字体、图标、布局、组件和交互规则；
3. 优先复用已有组件库，包括按钮、输入框、选择器、日期组件、表格、分页、卡片、表单、弹框、抽屉、标签页、全局提示、空状态等；
4. 统一页面中的字体层级、颜色层级、组件尺寸、圆角、间距、表格行高、筛选区布局和操作区样式；
5. 不允许同一类型组件在不同位置出现多套样式；
6. 不允许新增无规范依据的颜色、渐变、阴影、边框、动效；
7. 表格类页面必须遵循基础表格、操作型表格、图文表格、可编辑表格等对应规范；
8. 表单类页面必须遵循基础表单、复杂表单、中部弹框、右侧抽屉、新页面打开等对应场景规范；
9. 筛选类区域必须遵循页面筛选、顶部全局筛选、复合搜索筛选、多维度查询筛选等规范；
10. 输出前请先列出当前页面与 UI 规范的差异，再给出优化方案或代码修改。

请按以下格式输出：

一、当前页面问题诊断
- 页面布局问题
- 字体层级问题
- 颜色使用问题
- 组件使用问题
- 表格/表单/筛选区问题
- 交互状态问题

二、规范化修改方案
- 布局如何调整
- 组件如何替换或复用
- 样式变量如何统一
- 交互状态如何补齐

三、最终修改结果
- 输出可执行代码或明确的修改清单
- 不改变原有业务逻辑
- 保证后续页面可复用、可维护
```

## 四、页面整体优化提示词

适用于首页、列表页、详情页、配置页、报表页等整体页面优化。

```text
请基于项目已有 UI 设计规范，对当前页面进行整体 UI 优化。

优化目标：
1. 页面视觉风格与项目设计规范保持一致；
2. 页面结构清晰，信息层级明确；
3. 组件样式统一，避免同类组件多套样式；
4. 保持业务逻辑不变，只优化 UI 表现和组件规范性。

具体要求：

【页面布局】
- 页面采用统一的内容容器宽度和内边距；
- 顶部页眉区域最小高度按规范控制，有筛选项时高度按规范扩展；
- 页面标题、辅助说明、筛选区、内容区、操作区之间保持统一间距；
- 页面元素需左对齐或按设计规范对齐，不允许出现随机偏移；
- 内容区域避免贴边、拥挤、错位。

【视觉层级】
- 页面名称使用规范字号和主色；
- 辅助说明使用较弱文字颜色；
- 模块标题、正文、表格文字、备注信息需形成清晰层级；
- 不得随意加粗、放大或改变文字颜色。

【组件复用】
- 按钮、输入框、下拉选择、日期选择、表格、分页、弹框、抽屉、标签页等必须使用已有组件规范；
- 页面不得单独写一套与组件库不一致的样式；
- 相同组件在不同页面中表现必须一致。

【交互状态】
- 所有可点击元素需具备 hover、active、disabled 状态；
- 查询、重置、保存、删除、导出等操作需有明确反馈；
- 删除、批量删除等危险操作需二次确认；
- 加载、空数据、错误状态需按规范展示。

请输出：
1. 当前页面不符合规范的问题清单；
2. 对应的规范化修改建议；
3. 可直接执行的代码修改或样式调整方案。
```

## 五、组件级优化提示词

### 5.1 按钮组件优化提示词

```text
请检查并优化当前页面所有按钮组件。

要求：
1. 主按钮、次按钮、文字按钮、危险按钮需区分清楚；
2. 页面核心操作使用主按钮，不得多个主按钮抢占视觉焦点；
3. 编辑类操作使用项目主色；
4. 删除类操作使用危险操作样式，并增加二次确认；
5. 按钮尺寸、高度、圆角、文字字号、间距保持一致；
6. 按钮需具备默认、悬停、点击、禁用、加载状态；
7. 表格内操作按钮使用文字按钮，按钮之间保持统一间距；
8. 不允许使用自定义颜色按钮替代组件库按钮。

请输出当前按钮问题和优化后的按钮实现。
```

---

### 5.2 输入框 / 表单组件优化提示词

```text
请检查并优化当前页面中的输入类组件和表单区域。

要求：
1. 文本输入、密码输入、数字输入、多行文本、单选、复选、开关、滑动选择器等组件需按项目规范实现；
2. 输入框需包含默认、聚焦、输入、错误、禁用、只读状态；
3. 输入框高度、边框、圆角、占位符、清空按钮、错误提示需统一；
4. 表单 label 对齐方式、输入框宽度、表单项间距需统一；
5. 必填项、错误提示、帮助说明需位置一致；
6. 数字输入需支持最大值、最小值、小数精度、千分位格式；
7. 多行文本需支持字数统计和高度控制；
8. 表单底部按钮区域位置固定，保存/取消顺序统一。

请输出当前表单问题和优化后的表单规范化方案。
```

---

### 5.3 表格组件优化提示词

```text
请检查并优化当前页面中的表格组件。

要求：
1. 根据业务场景选择正确表格类型：基础表格、操作型表格、图文表格、多维透视表、矩阵表/热力表、汇总表、可编辑表格或看板表格；
2. 基础表格行高固定为 40px；
3. 图文表格行高固定为 60px，缩略图尺寸按 75×48px 控制；
4. 表头支持排序、提示、列宽拖拽、表头吸顶等规范能力；
5. 维度/日期字段左对齐，指标字段右对齐，操作列居中对齐；
6. 普通文本超出隐藏并显示省略号；
7. 数字需使用千分位分隔符，保留两位小数；
8. 空数据使用 “-” 或 “0” 占位；
9. 分页默认每页 20 条，支持 20 / 50 / 100 条切换；
10. 操作列固定在最右侧，行内按钮间距 10px；
11. 删除操作需二次确认，成功/失败需有反馈提示；
12. 如有汇总行，放在首行，背景使用浅灰，字体加粗；
13. 表格右上角如有导出按钮，导出时需有 loading 状态。

请输出表格问题清单，并按规范给出优化方案。
```

---

### 5.4 筛选区组件优化提示词

```text
请检查并优化当前页面筛选区。

要求：
1. 筛选区根据场景选择顶部全局筛选、复合搜索筛选或多维度查询筛选；
2. 筛选项少于 5 个时，优先使用顶部全局筛选布局；
3. 输入框、下拉选择、日期选择器、设备切换等筛选组件需统一高度和间距；
4. 查询按钮和重置按钮位置统一；
5. 条件未填全时，查询按钮按规范置灰；
6. 查询失败时展示错误提示；
7. 查询结果较多时与分页联动，默认每页 20 条；
8. 支持回车查询或点击查询按钮触发查询；
9. 重置按钮需清空所有条件并恢复初始状态。

请输出筛选区问题清单和优化方案。
```

---

### 5.5 日期组件优化提示词

```text
请检查并优化当前页面日期选择组件。

要求：
1. 根据业务场景选择单日期、日期区间、日期时间、日期对比、快捷范围或项目节点日期组件；
2. 日期格式统一使用 YYYY-MM-DD；
3. 日期时间格式统一使用 YYYY-MM-DD HH:MM:SS；
4. 点击输入框弹出日期选择面板；
5. 日期面板需包含年月切换、日历面板、可选快捷选项、底部确定/取消按钮；
6. 选择完成后按所选日期范围提交查询；
7. 所有关联报表、图表、表格需同步刷新；
8. 点击弹窗外或按 ESC 关闭弹窗，不保存本次操作；
9. 选中日期需高亮展示。

请输出当前日期组件问题和规范化方案。
```

---

### 5.6 弹框 / 抽屉 / 表单承载方式优化提示词

```text
请检查当前页面中的新增、编辑、详情、确认提示等操作承载方式。

要求：
1. 字段少、操作简单、无需参考主页面内容时，使用中部弹框；
2. 需要参考主页面上下文、查看或编辑单行数据详情时，使用右侧抽屉；
3. 流程复杂、字段较多、页面信息量大时，使用新页面打开；
4. 编辑操作需回填原有数据；
5. 删除操作需使用确认弹框；
6. 操作成功后刷新当前数据；
7. 操作失败时展示明确错误原因；
8. 抽屉关闭后需返回原列表状态。

请输出当前弹框/抽屉使用问题和优化建议。
```

---

### 5.7 空状态 / 加载 / 错误状态优化提示词

```text
请补齐当前页面中的空状态、加载状态和错误状态。

要求：
1. 无数据时展示统一空状态插图、提示文案和必要操作按钮；
2. 无权限时展示居中布局，插图、文案、按钮垂直排列；
3. 加载数据时展示 loading 状态；
4. 查询失败或接口异常时展示红色错误提示；
5. 删除、保存、导出等异步操作需展示处理中状态；
6. 状态提示文案需简短明确，不使用口语化表达；
7. 状态组件需保持页面视觉平衡，不得破坏布局。

请输出需要补齐的状态和对应优化方案。
```

## 七、设计规范映射表

| 页面元素 | 必须参考的规范模块 | 优化重点 |
|---|---|---|
| 页面标题 | 字体规范 / 页眉规范 | 页面名称字号、颜色、辅助说明 |
| 主操作按钮 | 按钮类 | 类型、状态、尺寸、颜色 |
| 输入框 | 输入类 / 文本输入 | 默认、聚焦、错误、禁用 |
| 下拉选择 | 页面筛选 / 下拉筛选组件 | 高度、选中态、悬停态 |
| 日期选择 | 日期组件 | 格式、弹窗、快捷筛选、提交逻辑 |
| 筛选区域 | 页面筛选 / 通用筛选组件 | 筛选项布局、查询、重置 |
| 表格 | 表格组件 | 行高、表头、分页、操作列 |
| 图文表格 | 图文列表 | 60px 行高、75×48px 缩略图 |
| 可编辑表格 | 可编辑表格 | 编辑状态、校验、保存 |
| 表单 | 表单使用 / 表单类型 | label、输入框、按钮区 |
| 弹框 | 中部弹框 | 确认提示、轻量表单 |
| 抽屉 | 右侧抽屉弹框 | 详情、编辑、上下文查看 |
| 空状态 | 空状态占位符 | 居中布局、提示文案、操作按钮 |
| 全局提示 | 全局提示 | 成功、失败、警告、反馈 |
| 分页 | 分页组件 | 20/50/100 条切换 |

AI 或前端在每次优化页面前，需先按以下模板输出问题：

```md

| 模块 | 当前表现 | 规范要求 | 问题类型 | 优先级 | 修改建议 |
|---|---|---|---|---|---|
| 页面布局 |  |  | 布局/间距 | 高 |  |
```
