# PMOS 第二设备验证清单

## 目的

这份清单用于验证独立 `PMOS product repo` 是否真的可以在另一台设备上部署，而不是依赖原作者机器上的隐含上下文。

## 必过项

1. 克隆仓库成功
2. `npm install` 成功
3. 可以从 `.env.example` 创建 `.env`
4. `npm run lint` 成功
5. `npm run build` 成功
6. `npm start` 成功
7. `/api/health` 返回 `200`
8. 浏览器访问根路径返回 `200`
9. 操作者无需追问原作者，也能读懂当前真源文档和启动步骤

## 建议补充项

1. 能打开统一入口页面
2. 能读取当前版本真源
3. 能跑通一条最小 workflow
4. 能识别外部连接器哪些已接、哪些未接
5. 能按文档定位下一步该补什么，而不是只能靠口头交接

## 失败条件

满足以下任意一项，都不能判定为第二设备验证通过：

1. 仍依赖母仓文件或隐藏目录
2. 仍依赖本地私有 Codex 配置才能启动
3. 启动过程依赖机器专属路径
4. 关键真源文档不可读或口径冲突
5. 操作者无法区分产品仓边界和业务仓边界

## 验证输出要求

第二设备验证结束后，至少要留下：

1. 验证机器环境说明
2. 实际运行的命令
3. 通过项与失败项
4. 遇到的问题和修复方法
5. 是否允许进入 GitHub 正式发布
