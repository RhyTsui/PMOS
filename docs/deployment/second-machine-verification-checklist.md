# PMOS 第二机部署验证清单

- version: v0.1
- date: 2026-05-09
- status: active

## 1. 目的

这份清单用于验证独立 `PMOS product repo` 是否真的可以脱离当前开发机，在另一台机器上完成部署、启动和最小使用。

## 2. 验证前提

验证机器必须：

1. 不依赖当前机器的隐藏文件
2. 不依赖当前会话上下文
3. 不依赖当前机器已有 node_modules
4. 不依赖私有本地路径

## 3. 环境检查

在第二机先确认：

1. 已安装 `Node.js 22+`
2. 已安装 `npm`
3. 已安装 `Git`
4. 可创建本地 `.env`

## 4. 拉仓与安装

必须能完成：

1. clone 独立 `PMOS product repo`
2. `npm install`
3. 从 `.env.example` 复制出 `.env`
4. 不填私有集成凭证时，仍可完成最小本地启动

## 5. 静态验证

第二机必须能通过：

1. `npm run lint`
2. `npm run build`

## 6. 启动验证

第二机必须能完成：

1. `npm start`
2. `GET /api/health` 返回 `200`
3. 根路径返回 `200`

## 7. 真源验证

第二机操作者必须能在不问原作者的情况下读懂：

1. 当前产品版本是 `v1.0`
2. 运行基线是 `v0.7`
3. 当前仓是独立产品仓，不是母仓
4. GitHub / Notion / cloud-mirror 的分层关系

## 8. 配置验证

第二机必须确认：

1. 仓内没有真实密钥
2. provider 使用样例配置而不是内部代理死绑定
3. 本机路径不写死在公开配置里

## 9. 通过标准

以下条件全部成立，第二机验证才算通过：

1. 可安装
2. 可构建
3. 可启动
4. 可读懂当前版本真源
5. 不依赖当前开发者隐式上下文

## 10. 不通过判定

以下任一成立，即为不通过：

1. 只能靠当前机器已有依赖才能跑
2. 只能靠母仓隐藏文件才能跑
3. README / first-run 仍不足以指导新操作者启动
4. 关键产品行为规则仍只存在于 codex 私有环境
