|  **运维**  |  提供服务器地址、备用服务器、服务器操作。  |
|  **魔方**  |  提供区服数据分析。  |

# 三、功能介绍

### 业务概念

项目组通过渠道号管理游戏包，不同的游戏包的渠道号不同，不同的渠道号对应的区服表不同，不同的区服表将玩家隔离成不同的专服。

  **官服与渠道服**

  官服不用多说，就是游戏官方运营的服务器，有时候又称买量服。

  除了自己运营，还会把游戏代理给华为、小米等手机厂商以及B站、头条、抖音这种也有游戏运营业务的公司，这些运营的就叫梦幻西游渠道服。

  渠道服是官网和其他APP应用一起合作运营的服务器，特点是帐号和密码是和其他APP关联的，比如你下载的是小米客户端，登陆帐号就是小米帐号。你下九游客户端，就要用九游的帐号登陆。官服的好处是帐号比较独立，以后帐号要变卖或者转让比较方便。渠道服的特点是登陆方便，不用输帐号密码，而且游戏充值有优惠，会有充值返利之类的。

  上面我们提到渠道服，渠道服又分为专服和混服。

  **混服与专服**

  最早专服和混服的定义是指游戏的开服方式。

  混服玩家很多，厂家就会为联运平台共同搭建一个服务器，不同平台的玩家互通在一起。

  专服则是指官方为某些大的渠道、比如手机应用商店、B站、头条等，包括九游这样的渠道，因为用户多，流量大，厂家会为它们专门搭设一个服务器，游戏内玩家均为该渠道下的玩家。

  另外，像大家在市面上看到的由一些小工作室推出的所谓专服，严格意义上来讲它不属于专服，而是一种包服的形式。顾名思义，包服就是一个代理商承包一个游戏里的一个区服，或者量比较大的代理商，可以承包整个游戏

  **渠道组**

  相同专混开服方式的渠道合集，便于批量控制和管理。

  **区服ID** 开服系统提供项目唯一ID,项目组如果有自己的区服ID，则运营需要在开服时映射项目组的区服ID。

  另外，区服ID和服务器ID在同时期一一对应，也存在服务器清空后再次使用到新区服上，或有服务器迁移的情况。

  **服务器实例ID**

  运维部署的服务器ID。

# 四、常见问题

问：首次使用，缺少备用服务器？

答：首次使用需联系陈旭初始化，将该项目的服务器同步到开服系统。

问：创建开服时需要准备哪些信息？

答：需研发提供服务器地址（没有统一网关的游戏）

问：一个区服能同时绑定多个渠道组么？比如又是提审服又是正式服

答：不行

问：能否删除已开服的区服

答：不行，配置的时候，最好把开服时间往后挪，等测试通过了以后再改开服时间

问：停服了以后，相同的服务器地址还能重新用么

答：可以

问：某个区服怎么设置成测试服还是提审服

答：渠道组设置
