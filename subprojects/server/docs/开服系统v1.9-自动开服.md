### 需求初步结论：

注册人数达到 n 个或开服时长达到 n 小时

1、选一个自动开服的区服，启动该区服的服务器,失败重试 3 次。

2、服务器启动成功后再将这个区服的区服状态改为已开服。

3、原区服的注册状态改为“关闭”。

4、原区服的角标状态改为“火爆”。

5、主动通知区服信息更新。

6、推动自动开服记录到小闪（包括启动失败）。

7、 可同时自动开多个区服。

**异常处理说明：**

服务器没启动，不关注册不改角标，不开新服

没有新的区服，关注册改角标，只是不开新服。

**期望实现效果：**

首开服预设是开了N个服，假设N=6，后续需要固定开M个服。M=2。

即：首开服是6个服，当其中1个服满了以后，开1个服务器；当其中2个服满了以后，开第二个服务器，后续是保持2个服务器同时开启。（所以N和M是可以配置控制，而非固定某个数字）
:::
