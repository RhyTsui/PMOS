# ChoKoNu 首版总规划

- 项目：`连弩-AI测试平台 / ChoKoNu`
- 文档层级：`internal-detailed`
- 状态：`active draft`

## 1. 规划摘要

ChoKoNu 首版不是再做一个泛化 AI 平台，也不是继续停留在单点埋点验收工具。

首版要做的是一个：

`面向 AI 应用测试、评测、自动化回归与结果治理的工作台`

核心目标是把下面这条主链收成稳定闭环：

`标准 -> 样本 -> 评测集 -> 执行 -> 评估 -> Trace -> 报告 -> 回归`

## 2. 首版目标

首版必须解决 5 个问题：

1. AI 应用测试与评测没有统一主链
2. 样本、评测集、评估器、报告之间没有稳定回路
3. 自动化测试结果无法自然回挂到 Trace 和版本判断
4. 过程文档和下游交付文档没有清晰分层
5. 产品动作和 skill 支撑没有一一对应

## 3. 首版边界

### 首版要做

1. 测试标准与规则定义
2. 样本资产沉淀
3. 评测集组织与运行入口
4. 评估器分类与执行
5. Trace、报告与结果归集
6. 回归集沉淀
7. 版本评审输入

### 首版不做

1. 通用 AI Agent 开发平台
2. 大而全的企业中台资源门户
3. 复杂的跨系统实时治理闭环
4. 无边界扩张的对象中心后台

## 5. 五层要素设计

### 第一层：业务目标层

回答：

- 首版解决什么问题
- 哪些场景最值得先收

要素：

- AI 应用质量闭环
- 自动化测试与版本判断
- 企业内部治理

### 第二层：工作对象层

回答：

- 用户实际围绕什么对象工作

要素：

- `Case`
- `Suite`
- `Run`
- `Trace`
- `Report`
- `Regression Set`

### 第三层：动作主链层

回答：

- 用户到底按什么动作推进工作

要素：

- 定义标准
- 沉淀样本
- 组织评测集
- 配置执行任务
- 运行评估器
- 归集结果与 Trace
- 生成报告
- 沉淀回归集

### 第四层：能力与 skill 层

回答：

- 每个动作靠什么 skill 支撑

要素：

- 产品经理 skill
- 测试/测开 skill
- 评估与报告 skill
- 决策支持 skill

### 第五层：治理与交付层

回答：

- 怎么进入需求、版本、评审和交付

要素：

- 需求清单
- 版本评审
- 人读版交付文档
- 开发对接文档

## 6. 首版动作主链与 skill 对应

| 动作 | 目标 | 关键输入 | 关键输出 | 必备 skill |
| --- | --- | --- | --- | --- |
| 定义测试标准 | 明确什么算通过 | 场景、规则、质量要求 | 标准定义 | 规则定义、边界收敛 |
| 沉淀测试样本 | 固化可复用输入 | 历史案例、失败样本、会议纪要 | 样本库 | 输入归档、样本沉淀 |
| 组织评测集 | 把样本编组成运行资产 | 样本、标准、场景分组 | 评测集 | 评测集组织 |
| 配置执行任务 | 明确本轮运行内容 | 数据集、版本、对象范围 | 运行任务 | 执行任务配置 |
| 运行评估器 | 执行规则与判断 | 任务、评估器配置 | 原始评估结果 | 评估器执行 |
| 归集结果与 Trace | 对失败和异常做挂接 | 运行结果、链路信息 | Trace 与异常归档 | Trace归集、风险显化 |
| 生成测试报告 | 输出可读结果 | 结果、Trace、结论 | 报告 | 报告生成、文档编排 |
| 沉淀回归集 | 为后续版本复用 | 报告、失败样本、结论 | 回归集 | 回归沉淀、决策支持 |

## 7. 产品经理专项 skill

当前产品经理必须掌握的 skill：

1. `输入归档`
2. `研究提炼`
3. `边界收敛`
4. `动作链规划`
5. `架构吸收`
6. `风险显化`
7. `文档编排`
8. `主动同步`
9. `决策支持`

## 8. 测试/测开专项 skill

当前测试/测开侧必须具备的 skill：

1. `规则定义`
2. `样本沉淀`
3. `评测集组织`
4. `执行任务配置`
5. `评估器执行`
6. `Trace归集`
7. `报告生成`
8. `回归沉淀`

## 9. 首版交付文档结构

首版默认交付两层：

### 过程层

- 首版总规划
- 需求清单详细版
- 风险与待拍板项
- 版本评审输入稿

### 交付层

- 需求清单人读版
- 开发对接精简版
- 版本评审人读版

2. 版本评审是否首版就纳入必做链路
3. 当前 legacy 中哪些方法稿要升格为现行真源
