# 需求池明细版

## P0：一期核心需求池

### P0-1 责任与角色治理

- 明确文档主导方、确认方、查看方
- 明确研发、QA、策划、运营、数据BP在验收流程中的职责
- 明确谁能触发验收、谁能最终确认、谁只读
- 角色权限稳定化

### P0-2 验收流程主链路

- 定义统一状态流转
- 串起文档确认、开发完成、数据触发、自动校验、人工确认、最终通过
- 支持最终状态沉淀

### P0-3 文档与规范基线

- 统一承载标准事件和自定义事件
- 明确事件说明、字段说明、是否必填、字段类型、可选项、后果说明
- 支持版本差异说明
- 支持 AI 生成 / 改写 / 补全

### P0-4 验收规则治理

- 通用字段规则
- 标准事件规则
- 自定义事件规则
- 枚举值 / 格式 / 必填校验
- 规则冲突识别与人工确认

### P0-5 数据查询与验收主路径

- 按 ID / DID / 时间范围快速核验
- 事件筛选与关键字段精确匹配
- 标准事件 / 自定义事件结构化展示
- 错误高亮和原因说明

### P0-6 自动校验 + 人工确认闭环

- 自动校验结果输出
- 自动校验通过后进入人工确认
- 异常项沉淀
- 最终状态回写

### P0-7 协作提效与结果透明

- 验收进度透明
- 异常结果共享
- 报告导出和推送
- 角色化结果展示

### P0-8 AI 提效切入点

- AI 辅助文档标准化
- AI 辅助规则生成
- AI 辅助纪要提炼
- AI 辅助错误总结

## P1：紧随一期推进

### P1-1 项目、版本、渠道、包体、节点管理

- 项目概览信息
- 测试节点和版本节点
- 渠道、平台、环境信息
- 验收 ddl 和关键时间点管理

### P1-2 知识与排查支撑

- 常见链路问题知识
- 报错跳转排查指引
- 透传字段问题说明
- 历史经验复用

### P1-3 广告与特殊场景治理

- 广告场景规则沉淀
- 广告角色参与提醒
- 特殊场景接入检查
- 必接项和可选项分级

### P1-4 历史文档导入与版本继承

- 历史 Excel / 文档导入
- 新版本继承旧版本
- 差异管理

## P2：后续增强

- 深度动态逻辑校验
- 更完整的 AI 规则冲突识别与协作总结
- 更完整的持续监控
