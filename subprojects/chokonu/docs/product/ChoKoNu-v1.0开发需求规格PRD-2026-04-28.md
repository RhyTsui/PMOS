# 连弩-AI自动化质量控制平台 v1.0 开发需求规格 PRD

- 项目名称：`连弩-AI自动化质量控制平台`
- 英文标识：`ChoKoNu`
- 文档版本：`v1.0`
- 文档日期：`2026-04-28`
- 文档定位：面向前端、后端、测试、联调的开发实现规格真源

## 1. 文档目标

本文件不是产品规划说明，而是直接给开发实现使用的需求规格文档。

它要回答：

1. `v1.0` 必须实现哪些模块
2. 前端需要交付什么
3. 后端需要交付什么
4. 登录、权限、三系统整合各自要落哪些能力
5. 哪些是 P0，哪些可以后置

## 2. 开发范围总述

`v1.0` 开发范围固定为四大块：

1. `统一工作台前端`
2. `后端基础服务`
3. `小闪登录接入`
4. `基础权限管理`

业务能力范围固定为：

1. `Web UI 自动化`
2. `MCP / Agent AI 应用评测`
3. `扣子罗盘 AI 评测模块内容吸收`

明确不进入 `v1.0` 实现主线：

1. App 自动化
2. Skill
3. 知识库
4. 接口测试

## 3. P0 交付要求

### 3.1 登录与权限

必须交付：

1. 接入 `小闪登录`
2. 登录后拿到平台可识别的用户身份
3. 建立基础权限管理能力

基础权限管理首版至少支持：

1. 登录用户识别
2. 页面访问控制
3. 项目级访问控制
4. 基础角色区分

建议首版角色：

1. `管理员`
2. `产品/运营`
3. `测试/测开`
4. `研发/平台工程`
5. `发布决策`

### 3.2 统一工作台

必须交付以下页面：

1. `登录页`
2. `总览`
3. `项目台`
4. `需求台`
5. `用例台`
6. `执行台`
7. `Trace 台`
8. `报告台`
9. `复核台`
10. `门禁台`

### 3.3 后端对象服务

必须交付基础对象服务：

1. `Project`
2. `Requirement`
3. `Case`
4. `Run`
5. `Trace`
6. `Report`
7. `Review`
8. `Gate`

### 3.4 三系统能力整合

必须完成三类能力收口：

1. `ai-auto-test`
   - Web UI 自动化能力接入

2. `chat_mcp_qa_platform`
   - MCP / Agent AI 应用评测能力接入

3. `coze-studio`
   - 扣子罗盘 AI 评测模块内容吸收与页面表达落位

## 4. 前端开发需求

### 4.1 登录页

必须实现：

1. `小闪登录` 接入位
2. 登录态识别
3. 登录成功后跳转工作台
4. 权限不足时的提示与申请入口

不允许继续停留在纯展示态扫码页。

### 4.2 工作台通用框架

必须实现：

1. 左侧导航
2. 顶部全局栏
3. 项目切换
4. 环境切换
5. 版本切换
6. 右侧详情栏
7. 移动端抽屉

### 4.3 项目台

必须实现：

1. 项目列表页
2. 项目接入抽屉/表单
3. 仓库地址、分支、环境、安装命令、启动命令录入
4. 能力映射展示
5. 接入校验结果展示
6. 从项目进入需求台

### 4.4 需求台

必须实现：

1. 需求列表
2. 来源标识：`PRD / Diff / AI Coding / 人工`
3. 需求详情
4. AI 理解摘要
5. 从需求进入用例台

### 4.5 用例台

`v1.0` 只支持：

1. `ui`
2. `agent`
3. `mcp`

必须实现：

1. 用例列表
2. case_type 标签
3. 执行配置与断言信息
4. Trace 引用
5. 从用例进入执行台

### 4.6 执行台

必须实现：

1. 执行任务列表
2. 执行器展示
3. 触发来源展示
4. 任务状态展示
5. 步骤与失败定位
6. 从执行进入 Trace 台

### 4.7 Trace 台

必须实现：

1. 步骤轨迹
2. 日志
3. 截图
4. 工具调用
5. AI 分析
6. 从 Trace 进入报告台

### 4.8 报告台

必须实现：

1. 报告列表
2. 风险等级
3. 结果分布
4. 证据引用
5. AI 分析
6. 从报告进入复核台

### 4.9 复核台

必须实现：

1. AI 结论
2. 规则判定
3. 人工复核
4. 争议记录
5. 从复核进入门禁台

### 4.10 门禁台

必须实现：

1. 版本门禁结论
2. 阻断项
3. 放行建议
4. 决策记录

## 5. 后端开发需求

### 5.1 基础服务能力

必须实现：

1. 用户会话与登录态承接
2. 基础权限校验
3. 统一对象存储与查询能力
4. 页面所需列表与详情接口

### 5.2 对象接口

每个对象至少需要：

1. 列表接口
2. 详情接口
3. 基础创建/更新接口

对象包括：

1. `Project`
2. `Requirement`
3. `Case`
4. `Run`
5. `Trace`
6. `Report`
7. `Review`
8. `Gate`

### 5.3 项目接入服务

必须实现：

1. 项目基础信息存储
2. 仓库信息存储
3. 环境信息存储
4. 能力映射存储
5. 接入校验结果存储

### 5.4 三系统能力接入服务

#### `ai-auto-test`

至少需要：

1. Web UI 自动化任务记录接入
2. 截图 / 日志 / 结果回传结构

#### `chat_mcp_qa_platform`

至少需要：

1. MCP / Agent 评测任务记录接入
2. 工具调用、Judge、结果回传结构

#### `coze-studio`

首版至少需要：

1. AI 评测模块对象与结果表达的后端对象映射
2. 与报告、复核相关的结构化数据承接

## 6. 登录与权限规格

### 6.1 登录

必须实现：

1. 接入小闪登录
2. 登录成功后建立平台会话
3. 前端可获取当前登录用户信息

### 6.2 权限

首版基础权限至少实现：

1. 页面访问权限
2. 项目访问权限
3. 核心操作权限

核心操作至少包括：

1. 项目接入
2. 发起执行
3. 生成报告
4. 提交复核
5. 生成门禁结论

## 7. 页面与接口映射要求

### 7.1 项目台

前端需要：

1. 项目列表
2. 项目详情
3. 项目接入表单

后端需要：

1. 项目列表接口
2. 项目详情接口
3. 项目创建/更新接口

### 7.2 需求台

前端需要：

1. 需求列表
2. 需求详情

后端需要：

1. 需求列表接口
2. 需求详情接口
3. 需求创建/更新接口

### 7.3 用例台

前端需要：

1. 用例列表
2. 用例详情

后端需要：

1. 用例列表接口
2. 用例详情接口
3. 用例创建/更新接口

### 7.4 执行台

前端需要：

1. 执行列表
2. 执行详情

后端需要：

1. 执行列表接口
2. 执行详情接口
3. 执行发起接口

### 7.5 Trace / 报告 / 复核 / 门禁

前端需要：

1. 列表
2. 详情
3. 链路跳转

后端需要：

1. 各对象列表接口
2. 各对象详情接口
3. 关键状态更新接口

## 8. 数据与状态要求

`v1.0` 不能只停留在纯 mock 页面。

至少需要做到：

1. 登录态真实化
2. 项目对象真实化
3. 主链对象可由后端接口返回
4. 列表与详情能读取真实服务数据

允许首版阶段性保留：

1. 局部模拟数据
2. 局部演示态链路

但不允许：

1. 整个平台只有静态前端

## 9. 非功能要求

### 9.1 可用性

1. PC 优先完整可用
2. 移动端至少保证可浏览与详情抽屉可用

### 9.2 性能

1. 工作台页面可正常加载
2. 不允许因无控制的大包导致首页不可用

### 9.3 稳定性

1. 登录页可用
2. 核心页面可访问
3. 主链路可演示

## 10. 联调要求

`v1.0` 必须安排真实联调，不再仅以前端壳为止。

联调优先级：

1. 登录与用户信息联调
2. 项目台联调
3. 需求 / 用例 / 执行 / Trace 主链联调
4. 报告 / 复核 / 门禁联调

## 11. 验收标准

### 11.1 产品验收

1. 三系统能力已在统一工作台落位
2. 页面结构与产品主链一致

### 11.2 前端验收

1. 10 个核心页面可用
2. 详情链路可点通
3. 项目接入表单可用

### 11.3 后端验收

1. 基础对象接口存在
2. 项目接入、主链对象、报告门禁可联调

### 11.4 登录与权限验收

1. 小闪登录可用
2. 基础权限可控

## 12. 开发优先级

### P0

1. 小闪登录
2. 基础权限
3. 项目台
4. 需求台
5. 用例台
6. 执行台
7. Trace 台
8. 报告台
9. 复核台
10. 门禁台
11. 后端基础对象服务

### P1

1. 页面体验优化
2. 更细的状态机
3. 更完整的异常提示

### P2

1. 进入 `v1.1 / v1.2` 的预留能力

## 13. 当前结论

开发实现必须基于以下前提推进：

1. `v1.0` 不再是纯规划或纯前端 Demo
2. `v1.0` 需要包含 `小闪登录`、`基础权限管理`、`真实后端服务`
3. `v1.0` 聚焦三个系统现有能力整合
4. App 自动化、Skill、知识库、接口测试暂不进入当前主实现

后续研发、联调、测试应以本文件作为开发实现真源。
