# 工作台架构与导航方案

## 1. 目标

这份文档只回答一个问题：

`连弩当前版首页和主导航到底应该怎么搭。`

结论是：

不做传统首页，不做大而全总览，不做对象分类门户。

应该做成：

`Workspace Shell + 左侧工作台导航 + 中间主画布 + 右侧上下文面板`

## 2. 一级导航

一级导航固定为 6 个：

1. `Case Studio`
2. `Suite Studio`
3. `Run Studio`
4. `Trace Studio`
5. `Report Studio`
6. `Eval Studio`

这 6 个是产品真正的一层骨架。

## 3. Workspace Shell

Shell 顶部固定承接：

1. 当前工作区名称
2. 环境切换
3. 搜索
4. 创建 run 快捷入口
5. 最近版本上下文

Shell 不负责承接详细业务数据。

## 4. 左侧导航设计

左侧导航应该更像 Studio，而不是菜单树。

建议规则：

1. 一级只放 6 个工作台
2. 不展开多层树
3. 不放大量计数和状态
4. 激活态强调当前工作区，而不是系统结构

## 5. 中间主画布设计

中间区域是当前工作台主内容。

每个工作台都应该以“当前最重要的工作动作”为第一屏核心：

### Case Studio

第一屏是：

1. Case 列表
2. 筛选
3. 当前选中 case 摘要

### Suite Studio

第一屏是：

1. Suite 类型切换
2. Suite 列表
3. 当前 suite 概况

### Run Studio

第一屏是：

1. 新建 run 卡片
2. 最近运行
3. 当前运行态摘要

### Trace Studio

第一屏是：

1. 失败结果列表
2. 当前 trace 详情
3. review 入口

### Report Studio

第一屏是：

1. 当前 run 结论
2. compare 结果
3. 放行建议

### Eval Studio

第一屏是：

1. evaluator 列表
2. 当前规则配置
3. 红线项摘要

## 6. 右侧上下文面板

右侧面板是关键，它决定页面有没有 Studio 感。

右侧只承接上下文，不承接主操作。

建议固定承接：

1. 当前对象摘要
2. 当前版本上下文
3. 当前风险状态
4. 当前关联资产

这样页面才会更像一个连续工作空间。

## 7. 当前不该独立成页的东西

以下内容不应再做成一级页面：

1. 全局首页大盘
2. 对象类型门户页
3. 纯配置页集合
4. 独立 compare 中心

这些都应该嵌回工作台和上下文面板。

## 8. 页面气质约束

界面气质上要靠近 `Coze Studio / Google AI Studio` 的工作台，而不是企业后台：

1. 少做总览面板拼盘
2. 多做当前工作区
3. 少做“统计大盘”
4. 多做“对象 + 状态 + 下一步动作”

## 9. 一句话结论

连弩的页面组织应该从“后台模块导航”切换成“Studio 工作台导航”。只有这样，后续页面才有可能做得像扣子，而不是像一个测试后台。
