## 1. 拆分目标

当前 `ChoKoNu-v1.0开发需求规格PRD-2026-04-28.md` 已经能回答首版做什么，但还不能直接支撑按模块研发排期、交互落图、接口细化和测试用例编写。

因此需要把总 PRD 拆成模块级详细 PRD，每份文档至少回答：

1. 该模块解决什么用户问题
2. 该模块在主链中的位置
3. 页面与交互结构是什么
4. 前端、后端、数据、状态分别要交付什么
5. 与其他模块的依赖和回流关系是什么
6. 模块级验收标准是什么

## 2. 拆分原则

1. 先拆承载层，再拆业务工作台。
2. 先拆 P0 主链模块，再拆次级支撑模块。
3. 每份 PRD 同时覆盖用户需求层和产品实现层，不能只写信息架构。
4. 页面壳层、对象模型、动作链路、权限约束、验收标准必须写实，不能只停留在概念描述。
5. 共性规范只在承载层定义一次，业务模块引用，不重复发明。
