- 项目：`连弩-AI自动化质量控制平台 / ChoKoNu`
- 模块：`工作台`
- 版本：`v1.0`
- 日期：`2026-04-28`
- 状态：`active`
- 文档类型：`模块详细 PRD`
- 前置分析：`docs/analysis/modules/ChoKoNu-v1.0-工作台模块分析-2026-04-28.md`

## 1. 文档定位

本文不是工作台分析文档，也不是视觉稿。

本文只回答一件事：

`在工作台模块分析结论已经确认后，v1.0 的工作台到底要交付什么。`

它将作为后续：

1. 设计图
2. 前端实现
3. 前后端联调规格

的直接真源。

## 2. 前置继承结论

本文不重新讨论以下问题，直接继承分析文档结论：

1. 工作台必须先于业务工作台独立成模块。
2. 工作台本质是统一承载层，不是单独业务页。
3. `6 Studio` 是形态原则，不是 `v1.0` 交付菜单。
4. `9 页面` 是 `v1.0` 交付分解，不是传统后台模块拼盘。
5. `总览` 只能做轻量工作入口，不可做 KPI 大盘。
6. 工作台模块只负责壳层、导航、上下文、路由、状态和跨模块跳转，不负责业务对象字段本体。

## 3. 模块目标

### 3.1 用户目标

工作台模块必须让用户做到：

1. 登录后直接进入统一质量工作空间。
2. 在项目、需求、用例、执行、Trace、报告、复核、门禁之间连续工作。
3. 在切换阶段时不丢项目、版本、环境和对象上下文。
4. 优先看到当前工作对象和下一步动作，而不是先看统计大盘。

### 3.2 产品目标

工作台模块必须提供：

1. 统一壳层
2. 统一一级导航
3. 统一顶部全局区
4. 统一上下文承载机制
5. 统一页面骨架
6. 统一路由守卫
7. 统一通用状态表达
8. 统一跨模块跳转上下文保留

### 3.3 v1.0 实施原则

`v1.0` 对工作台模块的要求不是一次把所有展示层做满，而是：

`骨架一次做对，表现层不过度做满。`

具体规则：

1. 结构层必须首版定住。
2. 主链交互必须首版打通。
3. 增强展示层允许首版弱实现。

## 4. 非目标

本模块不负责：

1. 业务对象字段设计
2. 业务列表字段定义
3. 业务详情页签细节
4. 业务执行编排逻辑
5. 登录鉴权实现细节
6. 具体视觉风格定稿

## 5. 模块职责边界

### 5.1 模块内

1. `Workspace Shell`
2. 顶部全局区
3. 左侧一级导航
4. 中间主画布承载区
5. 右侧上下文面板机制
6. 全局上下文状态
7. 通用状态页
8. 工作台间跳转规则
9. 响应式退化行为

### 5.2 模块外

1. `项目台` 负责项目接入表单和项目对象明细
2. `需求台` 负责需求对象与需求流转
3. `用例台` 负责 case/suite 资产
4. `执行台` 负责 run 编排和执行状态
5. `Trace 台` 负责 trace 证据归集
6. `报告台` 负责结果汇总和 compare
7. `复核台` 负责人工复核与争议处理
8. `门禁台` 负责版本放行与阻断判断

## 6. 信息架构

### 6.1 一级导航

`v1.0` 一级导航固定为：

1. `总览`
2. `项目台`
3. `需求台`
4. `用例台`
5. `执行台`
6. `Trace 台`
7. `报告台`
8. `复核台`
9. `门禁台`

### 6.2 导航原则

1. 只显示一级，不做多层树状展开。
2. 显示当前激活工作台。
3. 允许折叠为 icon-only。
4. 不在一级导航上堆叠大规模数字看板。
5. 允许保留弱提示态，例如待复核、阻断风险。

### 6.3 形态原则

虽然 `v1.0` 展示为 9 个一级入口，但整体形态必须遵守 `Studio` 原则：

1. 工作空间优先
2. 当前对象优先
3. 当前动作优先
4. 右侧上下文补充
5. 少统计拼盘，多连续操作

## 7. 页面壳层结构

全平台登录后统一采用四段式：

1. `顶部全局区`
2. `左侧导航区`
3. `中间主画布`
4. `右侧上下文区`

### 7.1 顶部全局区

必须承接：

1. 平台标识
2. 当前项目
3. 当前环境
4. 当前版本范围
5. 全局搜索
6. 最近任务入口
7. 待复核摘要
8. 门禁阻断摘要
9. 当前登录用户

### 7.2 左侧导航区

必须支持：

1. 一级工作台切换
2. 折叠
3. 窄屏抽屉
4. 当前页高亮

### 7.3 中间主画布

必须支持：

1. 当前工作台主内容渲染
2. 顶部摘要区
3. 筛选与动作区
4. 列表区或主内容区
5. 详情区或详情抽屉

### 7.4 右侧上下文区

必须承接：

1. 当前对象摘要
2. 当前版本上下文
3. 当前风险状态
4. 当前关联资产
5. 当前推荐下一步动作

约束：

1. 不承接重表单主操作
2. 不与中间详情区重复堆字段
3. 可折叠

`v1.0` 收口规则：

1. 必须先把上下文承载方式定住。
2. 首版不要求右侧信息量做满。
3. 若实现成本过高，允许首版把部分上下文并入详情区，但必须保留后续回归右侧面板的结构位置。

## 8. 默认进入与恢复规则

### 8.1 登录后默认进入

1. 若存在最近活跃项目与最近工作台，则恢复最近上下文。
2. 若不存在最近上下文，则进入 `总览`。
3. 若用户没有任何工作台权限，则进入无权限页。

### 8.2 上下文恢复

恢复至少包含：

1. 最近项目
2. 最近环境
3. 最近版本范围
4. 最近工作台
5. 最近对象上下文

### 8.3 恢复失败回退

若恢复对象失效或无权限：

1. 先保留用户身份
2. 回退到可访问项目
3. 再回退到 `总览`

## 9. 工作台页面骨架规范

除登录页外，每个一级工作台默认采用统一骨架：

1. `顶部摘要区`
2. `筛选与动作区`
3. `列表区`
4. `详情区`

允许的变体：

1. 列表 + 详情并排
2. 列表 + 抽屉详情
3. 主内容 + 右侧上下文

不允许的变体：

1. 每页独立发明新壳层
2. 某页没有筛选与动作语义只剩静态展示
3. 详情区完全替代右侧上下文面板

## 10. 总览页约束

`总览` 在工作台模块中只定义承载规则，不定义业务指标字段细节。

### 10.1 总览定位

1. 作为轻量工作入口
2. 暴露最近上下文
3. 暴露当前风险
4. 暴露待办动作

### 10.2 总览禁止项

1. 不做全平台 BI 大盘
2. 不做长列表统计墙
3. 不做与业务工作台重复的信息堆叠

### 10.3 总览首版收口

首版总览只要求稳定承接：

1. 最近工作上下文
2. 当前待办动作
3. 重点风险提醒

首版不要求：

1. 复杂指标矩阵
2. 多层健康度看板
3. 重图表拼盘

## 11. 全局状态模型

工作台模块至少统一管理以下状态：

1. `currentUser`
2. `currentRole`
3. `currentProject`
4. `currentEnvironment`
5. `currentVersionRange`
6. `activeWorkspace`
7. `activeObjectSummary`
8. `pendingReviewCount`
9. `blockedGateCount`
10. `recentTasks`

## 12. 路由要求

建议统一路由为：

1. `/login`
2. `/workspace/overview`
3. `/workspace/projects`
4. `/workspace/requirements`
5. `/workspace/cases`
6. `/workspace/runs`
7. `/workspace/traces`
8. `/workspace/reports`
9. `/workspace/reviews`
10. `/workspace/gates`

## 13. 跨模块跳转要求

必须支持以下标准跳转链：

1. `项目台 -> 需求台`
2. `需求台 -> 用例台`
3. `用例台 -> 执行台`
4. `执行台 -> Trace 台`
5. `Trace 台 -> 报告台`
6. `报告台 -> 复核台`
7. `复核台 -> 门禁台`

跳转时必须尽量保留：

1. `projectId`
2. `environment`
3. `versionRange`
4. `sourceObjectId`
5. `sourceFilterState`

## 14. 通用状态要求

每个工作台必须统一支持：

1. `loading`
2. `empty`
3. `error`
4. `forbidden`
5. `partial data`

定义要求：

1. `loading` 不能整页白屏
2. `empty` 必须给下一步动作
3. `error` 必须可重试
4. `forbidden` 必须说明权限缺口
5. `partial data` 必须明确哪些区域仍为占位

## 15. 响应式与设备约束

### 15.1 PC 优先

`v1.0` 以 PC 完整可用为主。

### 15.2 窄屏退化

必须保证：

1. 左侧导航可抽屉化
2. 右侧上下文可折叠
3. 详情区可切抽屉
4. 主内容仍可浏览和操作

### 15.3 不要求

1. 不要求移动端全功能优化
2. 不要求复杂拖拽布局

## 16. 前端交付要求

前端必须交付：

1. 工作台壳层布局
2. 顶部全局区组件
3. 左侧导航组件
4. 右侧上下文面板组件
5. 主画布承载区
6. 路由守卫
7. 通用状态页
8. 上下文恢复逻辑
9. 跨模块跳转上下文保留
10. 响应式折叠逻辑

### 16.1 首版必须做对

1. 壳层布局
2. 顶部全局区
3. 左侧导航
4. 中间主画布的摘要/筛选/列表/详情关系
5. 路由守卫
6. 上下文恢复
7. 跨模块跳转保留
8. 通用状态页

### 16.2 首版允许弱实现

1. 右侧上下文面板的信息密度
2. 待复核与门禁阻断的弱提示样式
3. 总览页的聚合信息丰富度
4. 窄屏下的高级交互细节

## 17. 后端支撑要求

后端至少提供以下壳层级支撑接口：

1. 当前用户信息
2. 当前用户工作台权限
3. 当前用户可访问项目列表
4. 最近任务摘要
5. 待复核摘要
6. 门禁阻断摘要
7. 最近上下文恢复

要求：

1. 字段稳定
2. 只返回摘要，不耦合完整业务详情
3. 数据必须带项目、环境、版本归属

说明：

1. 首版后端目标是支撑工作台稳定工作。
2. 不要求首版一次提供全量聚合分析能力。

## 18. 权限要求

### 18.1 页面级权限

1. 无权限工作台入口不可见或不可进入
2. 直接访问无权限路由必须返回无权限页

### 18.2 动作级权限

工作台壳层至少要支持动作可见性控制：

1. 新建项目
2. 新建需求
3. 发起执行
4. 提交复核
5. 生成门禁结论

## 19. 异常与边界场景

1. 用户已登录但没有任何项目
2. 用户有项目但没有工作台权限
3. 最近上下文指向已删除对象
4. 壳层摘要接口成功但业务页接口失败
5. 右侧上下文对象与中间详情对象不一致
6. 窄屏下左右面板冲突遮挡

## 20. 验收标准

### 20.1 产品验收

1. 登录后进入统一工作台，而不是来源系统入口集合
2. 用户在主链工作台间切换时上下文连续
3. 整体气质是工作空间，不是后台菜单树
4. 首版没有因为追求展示做满而破坏主链可用性

### 20.2 前端验收

1. 9 个一级工作台入口可访问
2. 左侧导航折叠与抽屉正常
3. 顶部全局区信息完整
4. 右侧上下文面板可用
5. 通用状态齐全

### 20.3 后端验收

1. 壳层摘要接口可联调
2. 权限接口可控制入口与路由
3. 上下文恢复接口可处理失效回退

### 20.4 用户需求判定

- 产品要求层：`solved`
  工作台模块的交付边界、结构、状态、接口支撑和验收规则已经明确。
- 用户需求层：`partial`
  只有当设计图、前端和后续业务模块都基于本 PRD 落地后，用户的连续工作体验才算真正解决。

## 22. 当前结论

工作台模块是 `v1.0` 的承载基础设施，不是装饰层。

它一旦不先收住，后续所有业务模块都会继续出现：

1. 导航不一致
2. 页面骨架不一致
3. 状态与权限语义不一致
4. 研发与设计重复造壳

因此，工作台模块的下一阶段应该直接进入：

`设计图`

而不是再回到产品定义阶段重复讨论。
