# ChoKoNu v1 产品规格书

- 项目：`连弩-AI测试平台 / ChoKoNu`
- 版本：`v1.0-rc1`
- 日期：`2026-04-28`
- 规格类型：`产品规格书`

## 1. 产品定位

ChoKoNu v1 的定位不是通用 AI 开发平台，也不是单点测试工具。

它是一个：

`面向 AI 应用测试、评测、自动化回归、结果报告与质量治理闭环的统一工作台`

## 2. 产品目标

v1 重点解决四类问题：

1. 多类测试对象没有统一入口
2. 项目接入、需求、用例、执行、Trace、报告、复核、门禁之间缺少主链
3. AI Coding 上游变更与质量主链衔接不顺
4. 当前测试结论无法自然沉淀成版本判断

## 3. 目标用户

1. 产品经理
2. 测试 / 测开
3. 平台工程 / 研发
4. 质量负责人
5. 版本评审与发布角色

## 4. v1 核心业务对象

1. 项目
2. 需求
3. 用例
4. 执行任务
5. Trace
6. 报告
7. 复核项
8. 门禁项

## 5. v1 核心业务主链

`项目接入 -> 需求 -> 用例 -> 执行 -> Trace -> 报告 -> 复核 -> 门禁`

说明：

- `项目接入` 是新增前置层
- `AI Coding` 不是独立主导航，而是贯穿多页面的上游衔接能力

## 6. v1 信息架构

一级页面固定为：

1. `总览`
2. `项目台`
3. `需求台`
4. `用例台`
5. `执行台`
6. `Trace 台`
7. `报告台`
8. `复核台`
9. `门禁台`

## 7. 页面级产品定义

### 7.1 登录页

目标：

- 建立品牌感
- 承接统一登录
- 为真实小闪登录组件预留接入位

### 7.2 总览

目标：

- 提供平台总控视角
- 展示项目健康、AI Coding 衔接、高风险项、最近执行

### 7.3 项目台

目标：

- 承接项目接入
- 建立仓库、环境、能力、AI Coding 衔接的统一入口

### 7.4 需求台

目标：

- 接收 PRD、Diff、AI Coding、人工补录产生的需求
- 为生成用例提供输入

### 7.5 用例台

目标：

- 统一沉淀测试资产
- 支持按 `case_type`、执行器、断言方式管理

### 7.6 执行台

目标：

- 成为统一执行入口
- 明确区分平台手动、AI Coding、分支、MCP、CI/CD 等触发来源

### 7.7 Trace 台

目标：

- 成为统一证据中心
- 提供日志、截图、工具调用、AI 分析的聚合视图

### 7.8 报告台

目标：

- 输出质量结论
- 为研发和版本评审提供可读报告

### 7.9 复核台

目标：

- 承接 AI Judge 结果与人工复核
- 管理争议项与结论确认

### 7.10 门禁台

目标：

- 沉淀版本放行结论
- 给出阻断、可放行、例外放行判断

## 8. v1 非目标

1. 通用 AI Agent 开发平台
2. 通用企业级中后台门户
3. 全量后端能力平台化
4. 复杂实时跨系统治理引擎
5. 生产级多租户权限体系

## 9. v1 成功标准

1. 用户能通过登录页进入统一工作台
2. 用户能按项目切换上下文
3. 用户能完整浏览主链页面
4. 用户能通过桌面侧栏和移动端抽屉查看详情
5. 页面结构足够支撑后续接真实接口和真实登录
