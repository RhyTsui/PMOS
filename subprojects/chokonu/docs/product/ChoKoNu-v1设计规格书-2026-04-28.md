# ChoKoNu v1 设计规格书

- 项目：`连弩-AI测试平台 / ChoKoNu`
- 版本：`v1.0-rc1`
- 日期：`2026-04-28`
- 规格类型：`设计规格书`

## 1. 设计目标

v1 设计目标不是做营销官网，而是建立一套：

`企业级、可操作、可扩展、可承接复杂质量流程的工作台设计语言`

## 2. 品牌基调

品牌名称：

- 中文：`连弩`
- 英文：`ChoKoNu`

品牌气质：

1. 冷静
2. 可控
3. 可信
4. 平台化
5. 有工程感但不老旧

## 3. 色彩规范

主色方向：

1. 蓝色：品牌主导色
2. 绿色：执行 / 健康 / 正向状态
3. 紫色：智能 / 分析 / AI 能力点缀
4. 浅灰：页面背景与容器层次
5. 红橙：风险、阻断、待处理

## 4. 布局规范

### 4.1 桌面端

固定采用三段式：

1. 左侧导航
2. 顶部全局栏
3. 主工作区

### 4.2 移动端

固定采用：

1. 顶部栏
2. 折叠菜单抽屉
3. 详情抽屉

说明：

- 桌面端右侧详情栏在移动端不横向保留
- 统一改成右侧全屏抽屉

## 5. 页面结构规范

工作台页统一结构：

1. 页面标题区
2. 摘要指标区
3. 筛选与动作区
4. 列表区
5. 右侧详情区

详情统一优先使用页签结构。

## 6. 交互规范

### 6.1 桌面端

1. 列表点击后在右侧详情区展开
2. 右侧详情栏允许收起
3. 尽量避免整页跳转

### 6.2 移动端

1. 点击列表项后打开详情抽屉
2. 详情抽屉全屏展示
3. 导航通过左侧抽屉展开

## 7. 登录页设计规格

登录页布局：

1. 左侧品牌叙事区
2. 右侧登录卡片

登录卡片要求：

1. 保留“小闪登录”语义
2. 预留真实组件接入位
3. 当前占位表单不改变未来替换结构

## 8. 工作台设计规格

### 8.1 总览

重点模块：

1. 平台摘要指标
2. 多项目健康度
3. AI Coding 衔接
4. 高风险关注
5. 最近执行

### 8.2 项目台

重点模块：

1. 项目列表
2. 仓库 / 分支 / 环境信息
3. 能力映射
4. AI Coding 衔接
5. 接入校验

### 8.3 需求台

重点模块：

1. 需求来源
2. 优先级
3. 关联用例
4. AI 理解摘要

### 8.4 用例台

重点模块：

1. `case_type`
2. 执行器
3. 断言方式
4. Trace 入口

### 8.5 执行台

重点模块：

1. 触发来源
2. 执行状态
3. 步骤链
4. 失败定位

### 8.6 Trace 台

重点模块：

1. 步骤轨迹
2. 日志
3. 截图
4. 工具调用
5. AI 分析

### 8.7 报告台

重点模块：

1. 结论
2. 风险等级
3. 结果分布
4. 证据引用

### 8.8 复核台

重点模块：

1. AI 结论
2. 规则判定
3. 人工复核
4. 争议记录

### 8.9 门禁台

重点模块：

1. 放行状态
2. 阻断项
3. 放行建议
4. 决策记录

## 9. 组件与样式约束

1. 统一圆角卡片容器
2. 统一状态标签
3. 统一按钮层级
4. 统一表格头样式
5. 统一详情区结构
6. 统一响应式切换规则

## 10. 设计交付边界

当前已落地的是：

1. 品牌方向
2. 页面骨架
3. 响应式规则
4. 桌面 / 移动详情切换方式

当前未落地的是：

1. 最终图标系统
2. 真实小闪登录组件视觉一致性
3. 设计 token 抽离
4. 统一组件库变量化
