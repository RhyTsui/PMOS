# 问题优先级与边界切割

## 1. P0：必须先解决

- 验收责任与归属链条不清
- 买点验收全流程缺少统一标准流程
- 文档与规范不统一、不结构化
- 验收标准和规则体系不稳定
- 多角色协作成本过高
- 当前文档和规则体系不支持后续 AI 研发方式
- 团队被低效协作和重复劳动占满带宽

## 2. P1：紧随一期推进

- 项目、版本、渠道、包体、节点管理混乱
- 数据查询与验收路径低效
- 自动校验与人工确认没有形成闭环
- 知识、经验和排查指引没有进入验收现场
- 广告和特殊场景反复漏项

## 3. 一期明确不先做

- 点位触发时机是否正确
- 复杂研发逻辑正确性判断
- 深度动态业务逻辑校验
- 完整根因分析引擎
- 全量自动化测试平台
- 底层链路内部治理

## 4. 一句话边界

一期不解决所有数据接入和研发正确性问题，而是先把买点验收场景中最卡效率的责任、流程、文档、规则、查询和协作问题收住。
