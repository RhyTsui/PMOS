## 1. 文档定位

本文不是 PRD，也不是前端实现说明。

本文只负责把工作台模块 PRD 翻译成：

1. 可出图的页面壳层结构
2. 设计语言约束
3. 区块优先级
4. 响应式退化方式
5. 前端实现前不应再变动的布局规则

## 2. 设计目标

工作台模块设计必须同时满足四个目标：

1. 看起来像连续工作空间，而不是后台菜单树
2. 能承接 `9 页面 v1.0 交付分解`
3. 保持 `Studio` 式工作流气质
4. 给后续业务工作台提供统一壳层

## 2.1 v1.0 设计执行原则

设计阶段采用：

`骨架一次做对，表现层不过度做满`

因此：

1. 壳层关系、导航关系、主画布关系必须定住。
2. 首版信息密度可以收住。
3. 右侧上下文、总览聚合、弱提示样式允许先做轻量表达。

## 3. 设计结论

### 3.1 壳层采用四段式

统一为：

1. `顶部全局区`
2. `左侧导航区`
3. `中间主画布`
4. `右侧上下文面板`

### 3.2 布局重点

1. 左侧导航是工作阶段切换，不是对象目录。
2. 中间主画布永远承接当前最重要动作。
3. 右侧上下文永远只做补充，不做主表单。
4. 顶部全局区只放全局语义，不放业务明细。
5. `v1.0` 优先保证可工作，不优先追求信息满载。

### 3.3 总览页处理

`总览` 必须看起来像入口页，不像大盘页：

1. 可以有摘要
2. 可以有待办
3. 可以有风险聚焦
4. 不能成为 BI 看板墙

首版建议：

1. 总览先做轻入口页
2. 聚焦最近工作、待办、风险
3. 不为了“完整感”堆大量指标模块

## 4. 视觉语言

### 4.1 设计气质

工作台应呈现：

1. 冷静
2. 平台化
3. 可信
4. 工程感
5. 非旧式企业后台

### 4.2 推荐视觉方向

1. 用大面积浅底与轻阴影卡片构造空间层次
2. 用低饱和青绿/蓝灰作为主系统色
3. 用少量风险色强调阻断、待复核、异常
4. 用清晰留白区分导航、主内容、上下文

### 4.3 禁止项

1. 不做高密度表格后台感首页
2. 不做花哨渐变营销页
3. 不做大面积 KPI 仪表盘拼接
4. 不让右侧面板变成第二个详情页

## 5. 区块设计说明

### 5.1 顶部全局区

建议从左到右分为三段：

1. 品牌与当前页面语义
2. 当前项目 / 环境 / 版本范围
3. 搜索 / 最近任务 / 待复核 / 门禁阻断 / 用户

设计要求：

1. 高度固定
2. 不随业务台内容抖动
3. 全局摘要优先 badge 式弱提示，不做重卡片

### 5.2 左侧导航区

设计要求：

1. 一级导航仅保留 9 个入口
2. 当前激活态必须强于 hover 态
3. 折叠后仍可识别当前页面
4. 导航项允许少量弱提示，如待复核点数

### 5.3 中间主画布

设计要求：

1. 第一视觉焦点必须是当前动作，而不是说明文字
2. 默认分为摘要区、筛选动作区、列表区、详情区
3. 详情与列表关系优先采用并排布局
4. 当内容宽度不足时详情退化为抽屉

### 5.4 右侧上下文面板

设计要求：

1. 固定宽度
2. 卡片式承接对象摘要、版本上下文、风险状态、关联资产、下一步动作
3. 与中间详情信息错位表达，避免重复
4. 默认展开，可由用户手动折叠

首版收口：

1. 可以先减少卡片数量
2. 可以先弱化视觉层次
3. 但必须保留“它是上下文区，不是第二详情区”的边界

## 6. 宽屏布局建议

建议桌面端宽屏布局比例：

1. 左侧导航：`88-104px` 折叠态，`220-248px` 展开态
2. 中间主画布：主可伸缩区域
3. 右侧上下文：`320-360px`

原则：

1. 先保证主画布宽度
2. 再保证右侧上下文完整
3. 左侧导航不应过宽

## 7. 窄屏与退化策略

### 7.1 平板与窄桌面

1. 左侧导航切抽屉
2. 右侧上下文可折叠为侧滑面板
3. 中间详情区优先保留

### 7.2 手机宽度

1. 仅保证浏览与基础操作
2. 列表与详情改为单列
3. 上下文通过底部抽屉或右侧面板替代

### 7.3 退化优先级

1. 先折叠右侧上下文
2. 再抽屉化左侧导航
3. 再把详情区抽屉化

## 8. 交互热点说明

### 8.1 导航切换

1. 切换工作台时保留全局项目、环境、版本范围
2. 若来源对象存在，则带来源跳转状态

### 8.2 列表到详情

1. 列表点击后默认在中间详情区或抽屉展开
2. 右侧上下文同步刷新，但不抢主内容焦点

### 8.3 风险提示

1. 待复核、门禁阻断、异常应采用统一风险语义颜色
2. 风险提示应是行动入口，不是纯数字装饰

## 8.4 首版不过度展开项

以下内容在 `v1.0` 不要求做满：

1. 复杂总览信息拼盘
2. 过多全局状态块
3. 右侧上下文的满配卡片体系
4. 高级个性化布局

## 9. 设计图使用说明

对应设计图不是 SVG，而是统一通过 `image2` 生成的工作台模块设计稿。

它主要供：

1. 产品确认设计方向
2. 设计师开始细化页面皮肤
3. 前端确认壳层结构和区块边界

使用原则：

1. 后续业务台都必须沿用这个壳层
2. 不允许后续某个工作台自行改掉壳层结构
3. 视觉风格可细化，但区块关系不应推翻
