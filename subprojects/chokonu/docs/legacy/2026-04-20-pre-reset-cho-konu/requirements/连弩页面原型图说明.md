# 连弩页面原型图说明

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

本文件不是视觉稿，也不是交互稿，而是供产品、设计、研发共用的页面原型图说明书。

它回答三件事：

1. 每个页面原型图应该画出哪些区域
2. 每个区域重点承载什么信息和动作
3. 哪些区域是首版必画，哪些是后续增强

首版固定覆盖 7 个页面：

1. `首页概览`
2. `样本中心`
3. `评测集中心`
4. `执行中心`
5. `结果与Trace`
6. `报告与版本对比`
7. `规则与评估器`

## 2. 原型图统一绘制规则

### 2.1 页面骨架统一

每张页面原型图统一拆成：

1. `页头区`
2. `筛选/操作区`
3. `主列表或主内容区`
4. `详情区或抽屉区`
5. `状态反馈区`

### 2.2 视觉标注统一

原型图中需要明确标注：

1. 一级标题
2. 关键按钮
3. 核心筛选项
4. 主信息块
5. 空状态
6. 错误状态

### 2.3 首版裁剪规则

原型图不画以下内容：

1. 通用 Agent Builder
2. 复杂 Workflow 设计器
3. 组织级 BI 分析中心
4. 自动发布阻断流程

## 3. 首页概览原型图说明

### 必画区域

1. `顶部导航`
   包含页面标题、环境切换、全局搜索、新建 Run
2. `运行总览卡片区`
   展示今日 run 数、失败数、红线命中数、待复核数
3. `最近运行列表区`
   展示最近运行名称、对象类型、状态、通过率
4. `高风险问题区`
   展示红线问题摘要和跳转入口
5. `最新版本结论区`
   展示最新 compare 结果和放行建议

### 重点标注

1. 哪些卡片可点击跳转
2. 红线命中时的高优先级视觉提醒
3. 无运行记录时的空状态入口

## 4. 样本中心原型图说明

### 必画区域

1. `顶部操作栏`
   包含新建样本、导入、导出、批量操作
2. `左侧筛选栏`
   按对象类型、场景、风险等级、回归样本筛选
3. `样本列表区`
   展示样本标题、对象类型、场景、风险、最近引用
4. `样本详情抽屉`
   展示输入、预期、评估器绑定、红线项、历史运行引用

### 重点标注

1. 列表与详情联动关系
2. 新建样本入口位置
3. 筛选后无结果状态

## 5. 评测集中心原型图说明

### 必画区域

1. `顶部操作栏`
   包含新建 suite、复制 suite、从样本生成 suite
2. `suite 分类切换区`
   `Baseline / Regression / Scenario`
3. `suite 列表区`
   展示 suite 名称、类型、样本数、版本标签、最近运行结果
4. `suite 详情区`
   展示成员样本、绑定对象、版本信息、最近运行摘要

### 重点标注

1. suite 类型切换方式
2. 成员样本查看入口
3. 无 baseline 的提示区域

## 6. 执行中心原型图说明

### 必画区域

1. `顶部操作栏`
   包含新建 Run、重跑失败样本、查看队列
2. `run 列表区`
   展示 run 名称、suite、对象类型、环境、状态、通过率
3. `新建 Run 面板`
   包含 suite、环境、evaluator、trial 参数
4. `run 详情区`
   展示执行进度、结果摘要、Trace 入口

### 重点标注

1. 新建 Run 是侧边抽屉还是独立页面
2. 执行中与执行失败的状态差异
3. 重跑入口的作用范围

## 7. 结果与Trace原型图说明

### 必画区域

1. `顶部筛选栏`
   支持 `pass / fail / review`、对象类型、failure stage 筛选
2. `结果列表区`
   展示 case 结果、红线状态、review 状态
3. `Trace 列表区`
   展示 trace 摘要、failure stage
4. `Trace 详情区`
   展示请求/响应摘要、evaluator 结果、样本和 run 引用

### 重点标注

1. 失败结果与 Trace 的联动
2. review-needed 的突出方式
3. Trace 缺失时的替代展示

## 8. 报告与版本对比原型图说明

### 必画区域

1. `顶部操作栏`
   包含导出报告、选择 baseline、查看红线问题
2. `run 报告区`
   展示本次 run 总结论
3. `指标摘要区`
   展示分对象指标摘要
4. `版本对比区`
   展示 baseline 与 current 的对比
5. `放行建议区`
   展示红线命中、恶化样本、结论建议

### 重点标注

1. baseline 选择控件
2. 恶化样本列表位置
3. 红线命中和放行建议的关系

## 9. 规则与评估器原型图说明

### 必画区域

1. `顶部操作栏`
   包含新建 evaluator 配置、阈值配置、绑定 suite
2. `评估器列表区`
   展示评估器类型、适用对象、启用状态
3. `阈值配置区`
   展示主指标阈值、红线阈值
4. `规则详情区`
   展示判定方式、绑定 suite、历史命中情况

### 重点标注

1. evaluator 与 rule 的区分
2. 阈值冲突提示位置
3. 红线项影响范围展示

## 10. 交付建议

原型图交付顺序建议为：

1. 首页概览
2. 样本中心
3. 评测集中心
4. 执行中心
5. 结果与Trace
6. 报告与版本对比
7. 规则与评估器

每个页面至少产出两张图：

1. `默认状态`
2. `关键异常状态`

## 11. 一句话结论

页面原型图说明的目标，是让设计师知道每张图必须画什么、让产品知道每张图重点表达什么、让研发知道每张图最终对应什么能力闭环。
