# 连弩首版页面原型框架

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

连弩首版页面原型框架不追求高保真视觉，而是把首版工作台压到可直接出线框图的粒度。

首版页面固定为 7 个：

1. `首页概览`
2. `样本中心`
3. `评测集中心`
4. `执行中心`
5. `结果与Trace`
6. `报告与版本对比`
7. `规则与评估器`

页面设计统一遵守三条：

1. 以工作台承接对象差异，不为 `MCP / Chat / Agent / 知识检索 / Skill / Workflow` 各做一套独立产品区
2. 页面必须服务主链：`样本 -> suite -> run -> evaluator -> trace -> report -> version compare`
3. `Skill / Workflow` 只做轻接入，不升格为一级导航

## 2. 首页概览

### 页面目标

给产品、测试、研发和主管一个统一入口，快速看到最近运行、高风险问题、待复核项和最新版本判断。

### 目标用户

1. 项目负责人
2. 产品经理
3. 测试负责人
4. AI 应用研发

### 页面结构分区

1. 顶部导航区
2. 今日概览区
3. 最近运行区
4. 高风险问题区
5. 待复核区
6. 最新版本对比摘要区

### 顶部操作区

1. 全局搜索
2. 环境切换
3. 新建 Run
4. 进入样本中心

### 主内容区

1. 今日运行数、失败数、红线命中数、待复核数
2. 最近 5 次运行摘要
3. 失败最多对象类型排行
4. 最新一次版本对比结论

### 关键筛选项

1. 时间范围
2. 环境
3. 对象类型

### 关键状态

1. 空状态：尚无运行记录，提示从样本中心或评测集中心开始
2. 加载状态：骨架屏
3. 失败状态：概览加载失败，允许重试
4. 红线命中状态：显著展示高风险问题入口

### 不做项

1. 大而全 BI 看板
2. 复杂趋势分析中心

## 9. 一句话结论

连弩首版页面原型框架的目标，是让设计和研发都能围绕 7 个固定页面直接展开线框图、交互和实现，而不是继续在对象分类和功能发散上浪费决策成本。
