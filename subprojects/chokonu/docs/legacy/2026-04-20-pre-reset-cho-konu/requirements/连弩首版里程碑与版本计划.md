# 连弩首版里程碑与版本计划

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

连弩首版里程碑不能按“页面做完几个”来定义，而要按“能力闭环是否成立”来定义。

首版版本计划固定为三段：

1. `M1：主链路可运行`
2. `M2：四类主战场对象可判断`
3. `M3：版本对比与基础门禁可决策`

## 2. 里程碑定义

### M1：主链路可运行

目标：

打通统一测试主链：

`样本 -> suite -> run -> evaluator -> trace -> report`

必须完成：

1. 样本管理
2. 评测集管理
3. 执行编排
4. 评估器执行
5. 结果与 Trace 关联
6. 单次 run 报告

验收标准：

1. 至少一套样本可被组织成 suite
2. suite 可发起 run
3. run 能返回 case result
4. 失败样本能看到 trace
5. run 能生成正式报告

### M2：四类主战场对象可判断

目标：

让四类核心对象具备首版可判断能力：

1. MCP
2. Chat
3. Agent
4. 知识检索

必须完成：

1. MCP：发现 / 契约 / 参数映射 / 错误处理
2. Chat：单轮 / 多轮 / 会话解决率
3. Agent：任务完成 / 工具路径 / 多 trial
4. 知识检索：检索-生成拆分 / 证据支撑 / 版本回归

验收标准：

1. 四类对象都至少各有一套 suite
2. 四类对象都能输出核心指标
3. 四类对象都能形成对象级报告结论

### M3：版本对比与基础门禁可决策

目标：

让连弩不只会“跑一次测试”，而是能支持版本判断。

必须完成：

1. 基础版本对比
2. 红线项判断
3. 主指标判断
4. release suggestion

验收标准：

1. 两次 run 可做 compare
2. 红线问题可被识别
3. 报告中可输出放行建议

## 3. 版本规划建议

### v0.1

范围：

1. M1 全量完成
2. M2 启动并至少完成四类对象的基本能力

重点：

1. 主链路打实
2. 四类对象各能跑通一轮

### v0.2

范围：

1. M2 完整完成
2. M3 基础完成
3. Skill / Workflow 轻接入

重点：

1. 四类对象能力稳定
2. 版本对比与基础门禁可用

### v1.0

范围：

1. Skill / Workflow 能力增强
2. 长期线上观察位
3. 更成熟的治理闭环

重点：

1. 不是更多页面，而是更稳定的版本决策能力

## 4. 不进入首版里程碑的事项

以下不作为首版里程碑条件：

1. 多 Agent 协同
2. Workflow 全量分支覆盖
3. 自动发布阻断
4. 全量线上监控告警中心
5. 通用 Agent Builder

## 5. 风险与裁剪策略

若研发资源不足，按以下顺序裁剪：

1. 不裁 `M1`
2. `M2` 中保 MCP / Chat / Agent / 知识检索的最小能力
3. `Skill / Workflow` 允许延后为方法预埋
4. `M3` 中先保 compare，再保完整门禁

## 6. 与其他文档的关系

这份计划直接继承：

## 7. 需要主管决策的问题

1. `版本对比` 是否必须在 v0.1 就进入承诺
2. Skill / Workflow 轻接入是否放到 v0.2
3. M3 是否只做“建议放行”，不接自动阻断
