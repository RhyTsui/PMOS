# Chat专项测试方法草案

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

Chat 测试不能只测单轮回答是否“像样”，而要测整段会话是否真正解决用户问题。

因此，Chat 专项测试首版应分成两层：

1. `turn-level`：单轮回答质量
2. `thread-level`：会话级任务完成质量

## 2. 测试对象边界

Chat 专项测试覆盖：

1. 单轮问答
2. 多轮上下文承接
3. 业务口径一致性
4. 用户问题解决程度
5. 该答、该拒绝、该升级的边界

不直接覆盖：

1. MCP 契约与 schema
2. 多工具长任务编排
3. 检索层召回质量拆分

## 3. 主要风险

1. 表面流畅但事实错误
2. 业务口径偏移
3. 多轮上下文丢失
4. 重复追问与无效循环
5. 幻觉与伪引用
6. 应拒绝时不拒绝
7. 应转人工或升级时不升级

## 4. 测试分层

### 4.1 单轮质量

测什么：

1. 正确性
2. 完整性
3. 相关性
4. groundedness

### 4.2 多轮会话

测什么：

1. 上下文记忆
2. 追问承接
3. 任务推进
4. 会话结束质量

### 4.3 边界行为

测什么：

1. 拒答
2. 安全边界
3. 升级人工
4. 不确定场景处理

## 5. 首版测试方法

### 方法 1：单轮语义问答测试

目标：

1. 检查回答本身是否正确且完整

样本设计：

1. 标准问题
2. 高相似问题
3. 模糊表达问题
4. 反问式问题

判定：

1. 正确率
2. 关键点覆盖率
3. 幻觉率

### 方法 2：多轮上下文测试

目标：

1. 检查模型是否记住并正确使用历史上下文

样本设计：

1. 信息追问
2. 条件补充
3. 纠错重述
4. 历史引用

判定：

1. 上下文丢失率
2. 多轮一致性
3. 重复问答率

### 方法 3：会话解决率测试

目标：

1. 检查一段对话是否真正解决用户目标

样本设计：

1. 一次可解决任务
2. 需要 2-4 轮澄清的任务
3. 明确需要升级人工的任务

判定：

1. 会话解决率
2. 用户意图满足率
3. 错误结束率

### 方法 4：拒答与升级测试

目标：

1. 检查边界处理是否正确

样本设计：

1. 超出能力范围问题
2. 高风险问题
3. 缺少关键信息问题

判定：

1. 拒答正确率
2. 升级正确率
3. 误答率

### 方法 5：业务口径稳定性测试

目标：

1. 检查不同问法下是否保持同一业务口径

样本设计：

1. 同义改写
2. 长短句变体
3. 带噪声表达

判定：

1. 口径一致率
2. 关键结论一致率

## 6. 量化指标

首版建议固定：

1. 回答正确率
2. 关键点覆盖率
3. 幻觉率
4. 相关性
5. groundedness
6. 会话解决率
7. 上下文丢失率
8. 重复问答率
9. 拒答正确率
10. 升级正确率

## 7. 结果结构建议

每条结果至少保留：

1. `thread_id`
2. `turn_id`
3. `user_goal`
4. `final_answer`
5. `expected_keypoints`
6. `judge_result`
7. `review_needed`
8. `trace_ref`

## 8. 门禁建议

### 红线项

1. 严重幻觉
2. 高风险问题误答
3. 应升级未升级

### 主指标项

1. 会话解决率
2. 回答正确率
3. 关键点覆盖率

### 回归项

1. 多轮一致性是否下降
2. 幻觉率是否上升
3. 高风险场景误答是否上升

## 9. 不确定项

1. 首版是否接入用户模拟器
2. 会话解决率是否全部依赖 LLM judge
3. Chat 与 Agent 的分界线在首版中是否严格切开

## 10. 需要主管决策的问题

1. Chat 首版是否必须支持 thread-level 指标
2. 高风险拒答是否作为红线项
3. 首版是否要求业务口径一致性进入回归集
