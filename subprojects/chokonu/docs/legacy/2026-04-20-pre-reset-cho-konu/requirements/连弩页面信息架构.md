# 连弩页面信息架构

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

连弩首版页面信息架构必须服务于测试主链，不服务于“看起来完整的平台导航”。

所以首版 IA 固定为：

1. 一级导航按工作台分
2. 二级导航按对象、结果和动作分
3. 不单独给每个测试对象做一套独立产品区

## 2. 一级信息架构

首版一级导航固定为 6 个：

1. `样本中心`
2. `评测集中心`
3. `执行中心`
4. `结果与Trace`
5. `报告与版本对比`
6. `规则与评估器`

辅助入口：

1. `首页概览`
2. `全局搜索`
3. `运行目标/环境切换`

## 4. 首页概览

页面目标：

1. 给主管、产品、测试一个统一入口
2. 快速看到最近运行和高风险问题

展示内容固定为：

1. 最近运行摘要
2. 各对象最新通过率
3. 红线问题数
4. 待复核样本数
5. 最近版本对比结论

不做：

1. 大而全 BI 首页
2. 大量趋势图表

## 5. 样本中心 IA

### 一级目标

管理所有测试对象的样本资产。

### 二级结构

1. `全部样本`
2. `按对象过滤`
3. `按场景过滤`
4. `按风险过滤`
5. `样本详情`

### 样本详情固定信息块

1. 基本信息
2. 输入
3. 预期
4. 绑定评估器
5. 红线项
6. 历史运行引用

## 6. 评测集中心 IA

### 一级目标

组织和维护 baseline / regression / scenario suites。

### 二级结构

1. `全部Suites`
2. `Baseline`
3. `Regression`
4. `Scenario`
5. `Suite详情`

### Suite详情固定信息块

1. 基本信息
2. 成员样本
3. 绑定对象
4. 绑定版本
5. 最近运行

## 7. 执行中心 IA

### 一级目标

发起运行、管理运行、查看运行状态。

### 二级结构

1. `Run列表`
2. `新建Run`
3. `Run详情`
4. `重跑入口`

### Run详情固定信息块

1. 基本配置
2. suite 与对象
3. evaluator 组合
4. 运行状态
5. 结果摘要
6. Trace 入口

## 8. 结果与Trace IA

### 一级目标

让失败样本可下钻、可定位、可复盘。

### 二级结构

1. `全部结果`
2. `失败结果`
3. `待复核`
4. `Trace列表`
5. `Trace详情`

### Trace详情固定信息块

1. 请求/响应摘要
2. failure stage
3. evaluator 结果
4. case 引用
5. run 引用

## 9. 报告与版本对比 IA

### 一级目标

输出可读结论，支撑版本判断。

### 二级结构

1. `Run报告`
2. `聚合报告`
3. `版本对比`
4. `放行建议`

### 版本对比页面固定信息块

1. 比较对象
2. 主指标对比
3. 红线问题对比
4. 恶化样本
5. 建议结论

## 10. 规则与评估器 IA

### 一级目标

统一管理规则、阈值、评估器。

### 二级结构

1. `评估器列表`
2. `规则库`
3. `阈值配置`
4. `红线项管理`

### 详情固定信息块

1. 适用对象
2. 判定方式
3. 阈值
4. 绑定 suite
5. 历史命中情况

## 11. IA 约束

页面信息架构必须遵守：

1. 不为 `MCP / Chat / Agent / 知识检索 / Skill / Workflow` 各做一套独立产品导航
2. 以工作台和动作为主，而不是以技术名词堆导航
3. 所有页面必须能映射回对象-能力-指标矩阵
4. `Skill / Workflow` 只通过样本、suite、run、result 进入，不做独立对象专区

## 12. 首版实现顺序建议

信息架构对应实现顺序固定为：

1. 首页概览
2. 样本中心
3. 评测集中心
4. 执行中心
5. 结果与Trace
6. 报告与版本对比
7. 规则与评估器

## 13. 一句话结论

连弩首版页面信息架构应该让用户沿着“定义样本 -> 组织评测集 -> 发起运行 -> 查看结果与Trace -> 看报告与版本对比”的路径走，而不是在平台里迷失在对象分类里。
