# MCP专项测试方法草案

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

MCP 测试的核心不是“模型有没有回答”，而是：

1. 能力发现是否正确
2. tool/schema 契约是否正确
3. 参数映射与请求视图是否正确
4. 鉴权、确认、权限边界是否正确
5. 调用失败能否定位到具体阶段

因此，MCP 专项测试首版应以 `契约测试 + 参数映射测试 + 鉴权确认测试 + Trace 分阶段测试` 为主，而不是以语义打分为主。

## 2. 测试对象边界

MCP 专项测试覆盖：

1. tool discovery
2. tool schema
3. 参数解析与映射
4. tool call
5. result / error 返回
6. 鉴权与 human-in-the-loop 确认

不直接覆盖：

1. 多轮聊天质量
2. 最终自然语言回答美观度
3. 复杂 Agent 任务编排质量

## 3. 主要风险

1. 工具发现不完整
2. schema 与真实可执行能力不一致
3. 参数名正确但业务含义错误
4. request view 与实际请求体不一致
5. 错误码与错误类型不稳定
6. 高风险工具被误触发
7. 鉴权流程缺失或确认机制失效

## 4. 测试分层

### 4.1 发现层

测什么：

1. 工具是否可发现
2. 工具列表是否完整
3. tool description 是否足够清晰

### 4.2 契约层

测什么：

1. schema 是否可解析
2. 必填参数、类型、枚举是否一致
3. 返回结构是否符合契约

### 4.3 执行层

测什么：

1. 参数映射是否正确
2. 请求体是否正确
3. tool call 是否成功
4. 错误是否正确暴露

### 4.4 治理层

测什么：

1. 鉴权是否正确
2. 高风险工具是否需要确认
3. 越权调用是否被阻止

## 5. 首版测试方法

### 方法 1：工具发现测试

目标：

1. 确认工具可被稳定发现
2. 确认工具清单与预期一致

样本设计：

1. 正常发现样本
2. 缺失工具样本
3. 多工具歧义样本

判定：

1. 是否发现到目标工具
2. 是否误选相邻工具

### 方法 2：schema 契约测试

目标：

1. 检查 schema 定义与执行能力是否一致

样本设计：

1. happy path
2. missing required
3. wrong enum
4. wrong type
5. extra field

判定：

1. 是否按契约接受/拒绝
2. 错误类型是否正确

### 方法 3：参数映射测试

目标：

1. 检查 canonical 输入到目标请求的映射是否正确

样本设计：

1. 单参数映射
2. 多参数组合
3. 默认值回填
4. 枚举转换

判定：

1. expected_mcp_args 与 actual_mcp_args 是否一致
2. request view 差异是否在允许范围内

### 方法 4：错误处理测试

目标：

1. 检查错误是否可解释、可分层

样本设计：

1. schema 错误
2. call 错误
3. timeout
4. downstream 业务错误

判定：

1. failure_stage 是否正确
2. error_type / error_message 是否正确

### 方法 5：鉴权与确认测试

目标：

1. 检查高风险操作是否具备授权与确认

样本设计：

1. 正常鉴权
2. 失效 token
3. 无权限访问
4. 需人工确认工具
5. 误触发高风险工具

判定：

1. 鉴权是否通过/拒绝正确
2. 是否触发确认
3. 是否阻止误操作

## 6. 结果结构建议

每条结果至少保留：

1. `tool_name`
2. `schema_version`
3. `expected_mcp_args`
4. `actual_mcp_args`
5. `expected_request_view`
6. `actual_request_view`
7. `failure_stage`
8. `error_type`
9. `trace_ref`
10. `decision`

## 7. 量化指标

首版建议固定以下指标：

1. 工具发现成功率
2. 契约通过率
3. 参数命中率
4. request view 一致率
5. 工具调用成功率
6. 鉴权通过率
7. 错误码正确率
8. 高风险工具误触发率

## 8. 门禁建议

MCP 首版门禁建议：

### 红线项

1. 越权调用
2. 高风险工具误触发
3. 契约严重漂移

### 主指标项

1. 契约通过率
2. 参数命中率
3. 调用成功率

### 回归项

1. 新版本 schema 差异率
2. 高风险工具误触发率是否升高

## 9. 不确定项

1. 首版是否覆盖 stdio 与 HTTP 两类 MCP 传输方式
2. 首版是否要测 OAuth / metadata discovery 全链路
3. request view 是否定义为统一平台对象

## 10. 需要主管决策的问题

1. MCP 首版是否把鉴权与人工确认纳入必测范围
2. MCP 首版是否优先围绕高风险工具建立回归集
3. MCP 结果是否必须落 trace 并保留 failure_stage
