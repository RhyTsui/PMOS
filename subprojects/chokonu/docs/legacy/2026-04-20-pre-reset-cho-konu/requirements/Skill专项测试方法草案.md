# Skill专项测试方法草案

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

Skill 不是普通问答，也不是完整 Agent。它更像带触发条件、边界输入和明确产出的能力组件。

因此，Skill 测试的关键不是“输出看起来不错”，而是：

1. 该触发时能触发
2. 不该触发时不触发
3. 触发后能按定义完成任务
4. 输出格式和行为稳定

## 2. 测试对象边界

Skill 专项测试覆盖：

1. 触发条件
2. 输入边界
3. 产出结构
4. 任务完成质量
5. 异常与降级行为

不直接覆盖：

1. 多工具长链路任务编排
2. MCP 协议契约
3. 完整 Workflow 节点协同

## 3. 主要风险

1. 该触发时未触发
2. 不该触发时误触发
3. 输出格式波动
4. 复杂输入下性能或质量急剧下降
5. 单 Skill 指标正常，但放进真实场景后不稳定
6. 异常场景没有降级策略

## 4. 测试分层

### 4.1 触发层

测什么：

1. 显式触发
2. 隐式触发
3. 上下文触发
4. 负样本不触发

### 4.2 能力层

测什么：

1. 任务完成质量
2. 核心行为是否满足 definition of done

### 4.3 结构层

测什么：

1. 输出格式
2. 字段完整性
3. 结构化结果稳定性

### 4.4 降级层

测什么：

1. 异常处理
2. 超时/失败后的恢复或降级

## 5. 首版测试方法

### 方法 1：触发精度测试

目标：

1. 检查 Skill 触发是否准确

样本设计：

1. 显式触发样本
2. 隐式触发样本
3. 上下文触发样本
4. 负样本

判定：

1. 触发准确率
2. 漏触发率
3. 误触发率

### 方法 2：能力边界测试

目标：

1. 检查 Skill 在边界输入下是否仍可完成任务

样本设计：

1. 标准输入
2. 缺失字段输入
3. 噪声输入
4. 极长输入

判定：

1. 任务完成率
2. 边界退化率

### 方法 3：输出结构测试

目标：

1. 检查结果是否稳定符合预期格式

样本设计：

1. 正常结构样本
2. 可选字段样本
3. 嵌套结构样本

判定：

1. 输出格式一致率
2. 字段完整率

### 方法 4：异常与降级测试

目标：

1. 检查异常情况下是否能正确降级

样本设计：

1. 上游输入缺失
2. 下游依赖失败
3. 超时样本

判定：

1. 异常恢复率
2. 降级成功率
3. 错误外泄率

## 6. 量化指标

首版建议固定：

1. 触发准确率
2. 漏触发率
3. 误触发率
4. 任务完成率
5. 输出格式一致率
6. 字段完整率
7. 异常恢复率
8. 边界退化率

## 7. 结果结构建议

每条结果至少保留：

1. `skill_id`
2. `trigger_type`
3. `input_variant`
4. `output_payload`
5. `expected_structure`
6. `decision`
7. `trace_ref`

## 8. 门禁建议

### 红线项

1. 高风险场景误触发
2. 输出结构严重错乱

### 主指标项

1. 触发准确率
2. 任务完成率
3. 输出格式一致率

### 回归项

1. 误触发率是否上升
2. 边界退化率是否恶化

## 9. 不确定项

1. 首版是否要求 Skill 全部输出结构化结果
2. 首版是否引入 should_trigger 标签体系
3. Skill 与 Agent 的边界是否用“是否自主规划”来定义

## 10. 需要主管决策的问题

1. Skill 首版是否必须覆盖误触发/漏触发
2. 输出格式稳定性是否作为主指标
3. Skill 是否单独建回归集
