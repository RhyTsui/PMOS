# 知识检索专项测试方法草案

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

知识检索测试不能只看最终回答是否“像对的”，必须拆成：

1. 检索层
2. 生成层
3. 边界层

否则无法判断问题到底出在召回、证据使用还是最终生成。

## 2. 测试对象边界

知识检索专项测试覆盖：

1. query -> retrieval
2. retrieved context -> answer
3. citations / evidence usage
4. knowledge version regression

不直接覆盖：

1. 纯聊天闲聊
2. MCP tool 契约
3. workflow 分支工程逻辑

## 3. 主要风险

1. 召回不到关键信息
2. 召回太多噪声
3. 回答未被证据支撑
4. 引用了错误证据
5. 知识更新后版本退化
6. 知识库外问题误答

## 4. 测试分层

### 4.1 检索层

测什么：

1. 检索相关性
2. Top-K 命中
3. 召回覆盖度

### 4.2 生成层

测什么：

1. 回答正确性
2. 回答完整性
3. groundedness / faithfulness

### 4.3 边界层

测什么：

1. 知识库外问题处理
2. 证据不足时的拒答
3. 新旧知识切换回归

## 5. 首版测试方法

### 方法 1：检索相关性测试

目标：

1. 检查是否检索到正确文档

样本设计：

1. 单文档可回答问题
2. 多文档综合问题
3. 干扰文档场景

判定：

1. Recall@K
2. Precision@K
3. context relevance

### 方法 2：证据支撑测试

目标：

1. 检查回答是否真的由证据支撑

样本设计：

1. 证据充分问题
2. 证据部分充分问题
3. 证据冲突问题

判定：

1. 证据命中率
2. 回答可支撑率
3. groundedness

### 方法 3：生成正确性测试

目标：

1. 检查最终回答是否正确且完整

样本设计：

1. 事实型问题
2. 归纳型问题
3. 对比型问题

判定：

1. 正确率
2. 完整性
3. 关键点覆盖率

### 方法 4：知识版本回归测试

目标：

1. 检查知识库更新后是否退化

样本设计：

1. 稳定知识样本
2. 新增知识样本
3. 删除/废弃知识样本

判定：

1. 知识版本回归率
2. 关键问题恶化率

### 方法 5：知识库外问题拒答测试

目标：

1. 检查系统是否正确拒答或标记不确定

样本设计：

1. 知识库外问题
2. 伪相似问题
3. 高风险误导问题

判定：

1. 拒答正确率
2. 误答率

## 6. 量化指标

首版建议固定：

1. Recall@K
2. Precision@K
3. context relevance
4. 证据命中率
5. 回答可支撑率
6. groundedness / faithfulness
7. 回答正确率
8. 知识版本回归率
9. 拒答正确率

## 7. 结果结构建议

每条结果至少保留：

1. `query`
2. `retrieved_docs`
3. `top_k`
4. `citations`
5. `final_answer`
6. `knowledge_version`
7. `decision`
8. `trace_ref`

## 8. 门禁建议

### 红线项

1. 高风险知识误答
2. 明显无证据支撑仍强答

### 主指标项

1. 回答可支撑率
2. Recall@K
3. 回答正确率

### 回归项

1. 知识版本回归率
2. groundedness 是否下降
3. 知识库外误答率是否上升

## 9. 不确定项

1. 首版 citation 是否要求强绑定片段级证据
2. knowledge retrieval 与 RAG generation 是否在实现层拆分
3. 是否先只做文本知识库

## 10. 需要主管决策的问题

1. 知识检索首版是否必须拆检索层与生成层结果
2. 拒答正确率是否作为红线或主指标
3. 知识版本回归是否进入首版必做范围
