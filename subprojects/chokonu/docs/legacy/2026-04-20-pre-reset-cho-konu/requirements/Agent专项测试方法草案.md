# Agent专项测试方法草案

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

Agent 的核心不是“说对一句话”，而是“在环境中完成任务”。

因此，Agent 测试首版必须同时看：

1. 最终任务结果
2. 工具路径与执行过程
3. 多次试验下的稳定性

## 2. 测试对象边界

Agent 专项测试覆盖：

1. 任务规划
2. 工具选择
3. 任务执行
4. 结果交付
5. 失败恢复

不直接覆盖：

1. 单纯对话型 Chat 问答
2. 单节点 workflow 工程逻辑
3. 底层检索召回细节

## 3. 主要风险

1. 会调用工具但完不成任务
2. 工具选错
3. 步数过多
4. 循环或卡死
5. 结果完成但过程越权
6. 同任务结果随机波动大

## 4. 测试分层

### 4.1 任务结果层

测什么：

1. 任务是否完成
2. 结果是否可用

### 4.2 行为过程层

测什么：

1. 工具路径
2. 步骤顺序
3. 无效步骤

### 4.3 稳定性层

测什么：

1. 同任务多 trial 一致性
2. 边界输入下波动

### 4.4 治理层

测什么：

1. 审批点遵循
2. 高风险操作确认
3. 异常恢复

## 5. 首版测试方法

### 方法 1：任务完成度测试

目标：

1. 检查 agent 是否完成用户目标

样本设计：

1. 单步骤任务
2. 多步骤任务
3. 带约束任务

判定：

1. 任务完成率
2. 最终结果可用率

### 方法 2：工具路径测试

目标：

1. 检查 agent 是否选择了合理工具路线

样本设计：

1. 单工具场景
2. 多工具候选场景
3. 需避免某工具场景

判定：

1. 工具路径命中率
2. 禁用工具误用率

### 方法 3：计划-执行一致性测试

目标：

1. 检查 agent 的计划是否与执行一致

样本设计：

1. 先规划后执行任务
2. 执行中修正计划任务

判定：

1. 计划一致率
2. 临时偏航率

### 方法 4：多 trial 稳定性测试

目标：

1. 降低“偶然跑通”带来的误判

样本设计：

1. 对同一任务运行多次
2. 对关键场景固定多 trial

判定：

1. 多 trial 一致性
2. pass^k

### 方法 5：异常恢复测试

目标：

1. 检查 agent 在失败时是否能正确中止、重试或回退

样本设计：

1. 工具超时
2. 权限拒绝
3. 中间步骤失败

判定：

1. 恢复成功率
2. 错误循环率

## 6. 量化指标

首版建议固定：

1. 任务完成率
2. 最终结果可用率
3. 工具路径命中率
4. 平均完成步数
5. 无效步骤占比
6. 多 trial 一致性
7. 恢复成功率
8. 越权率

## 7. 结果结构建议

每条结果至少保留：

1. `task_id`
2. `trial_id`
3. `goal`
4. `tool_path`
5. `step_count`
6. `final_artifact`
7. `decision`
8. `trace_ref`

## 8. 门禁建议

### 红线项

1. 越权操作
2. 高风险操作未确认
3. 错误循环失控

### 主指标项

1. 任务完成率
2. 工具路径命中率
3. 最终结果可用率

### 回归项

1. 多 trial 一致性是否下降
2. 平均步数是否显著膨胀
3. 错误循环率是否上升

## 9. 不确定项

1. 首版是否提供沙箱环境
2. pass^k 是否作为首版正式指标
3. 多 Agent 场景是否进入首版

## 10. 需要主管决策的问题

1. Agent 首版是否必须多 trial
2. 首版是否先只覆盖单 Agent 任务
3. 审批与越权是否纳入红线门禁
