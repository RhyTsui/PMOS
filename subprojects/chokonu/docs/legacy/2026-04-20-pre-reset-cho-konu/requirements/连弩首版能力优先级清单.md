# 连弩首版能力优先级清单

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

连弩首版不能把六类对象的所有能力一起做满，否则会直接失焦。

这份清单的作用是把“方法学上应该有的能力”压缩成“首版真正要交付的能力”。

首版优先级建议采用 `Must / Should / Could / Not-now` 四层：

1. `Must`
   没有就不构成首版闭环
2. `Should`
   首版强烈建议做，但可轻量实现
3. `Could`
   可做加分项，不进一期承诺
4. `Not-now`
   明确不在首版范围

## 2. 首版总原则

首版只优先做两类东西：

1. 能形成统一主链路的共性能力
2. 能拉开差异化的四类核心对象能力

首版主链路必须打通：

`样本 -> suite -> run -> evaluator -> trace -> report -> version compare`

## 3. 共性平台能力优先级

| 能力 | 优先级 | 说明 |
|---|---|---|
| 样本管理 | Must | 所有测试对象都依赖样本资产 |
| 评测集管理 | Must | 没有 suite 就无法回归与批量执行 |
| 执行编排 | Must | 没有 run 就没有平台运行能力 |
| 评估器执行 | Must | 没有 evaluator 就无法形成测试结论 |
| Trace 关联 | Must | 没有 trace 就无法复盘和定位 |
| 报告输出 | Must | 没有正式报告就无法支撑研发和主管决策 |
| 版本对比 | Should | 首版强烈建议具备基础版本对比 |
| 基础门禁规则 | Should | 首版至少要有红线+主指标判断 |
| 人工复核挂点 | Should | 先保留兜底，不要求工作台复杂 |
| 长期线上监控与告警 | Could | 重要，但首版不宜做深 |
| 自动发布阻断 | Not-now | 先预留，不直接做完整发布编排 |

## 4. 测试对象能力优先级

### 4.1 MCP

| 能力 | 优先级 | 说明 |
|---|---|---|
| 工具发现测试 | Must | MCP 测试最基础入口 |
| schema 契约测试 | Must | 直接决定工具是否可用 |
| 参数映射测试 | Must | 避免“调用成功但业务错” |
| 错误处理测试 | Must | 需要 failure stage 才能定位 |
| 鉴权与确认测试 | Should | 高风险工具场景很重要 |
| OAuth/metadata discovery 全链路 | Could | 首版不必全部做深 |
| 跨 server 组合测试 | Not-now | 范围过大 |

### 4.2 Chat

| 能力 | 优先级 | 说明 |
|---|---|---|
| 单轮语义问答测试 | Must | 最基础的回答质量判断 |
| 多轮上下文测试 | Must | Chat 不做多轮就不成立 |
| 会话解决率测试 | Must | 要看“解决问题”而不是“会说话” |
| 拒答与升级测试 | Should | 直接影响风险控制 |
| 业务口径稳定性测试 | Should | 对业务场景很关键 |
| 用户模拟器 | Could | 可显著增强，但可后置 |
| 多 persona 长期稳定性 | Not-now | 首版先不做深 |

### 4.3 Agent

| 能力 | 优先级 | 说明 |
|---|---|---|
| 任务完成度测试 | Must | Agent 的核心指标 |
| 工具路径测试 | Must | 不只看结果，还要看过程 |
| 多 trial 稳定性测试 | Must | 避免“偶然跑通” |
| 异常恢复测试 | Should | 首版应有基础兜底 |
| 计划-执行一致性测试 | Should | 有助于定位行为偏航 |
| 审批与越权检查 | Should | 对高风险 Agent 必要 |
| 多 Agent 协同测试 | Not-now | 首版先不碰高复杂度 |

### 4.4 知识检索

| 能力 | 优先级 | 说明 |
|---|---|---|
| 检索层/生成层拆分 | Must | 不拆就无法定位退化 |
| 检索相关性测试 | Must | 知识检索基本盘 |
| 证据支撑测试 | Must | 决定回答是否可信 |
| 知识版本回归测试 | Must | 知识更新是高频场景 |
| 知识库外拒答测试 | Should | 风险防线 |
| citation 片段级对齐 | Could | 有价值，但首版可轻 |
| 混合检索策略优化实验 | Not-now | 更像二期优化能力 |

### 4.5 Skill

| 能力 | 优先级 | 说明 |
|---|---|---|
| 触发精度测试 | Should | 首版建议轻接入 |
| 漏触发/误触发统计 | Should | Skill 的核心风险 |
| 输出结构测试 | Should | 组件可复用的基础 |
| 能力边界测试 | Could | 首版可选做部分 |
| 复杂触发策略测试 | Not-now | 先不扩展 |

### 4.6 Workflow

| 能力 | 优先级 | 说明 |
|---|---|---|
| 关键路径回放测试 | Should | 首版建议轻接入 |
| 节点契约测试 | Should | 工程稳定性的基础 |
| 状态一致性测试 | Should | workflow 的关键价值点 |
| 补偿/回退测试 | Could | 有价值，但可先轻量化 |
| 全量分支覆盖 | Not-now | 首版不做重型工程能力 |
| 可视回放 | Not-now | 先保留方向，不进入承诺 |

## 5. 首版真正承诺交付的能力

### Must 能力包

1. 样本管理
2. 评测集管理
3. 执行编排
4. 评估器执行
5. Trace 关联
6. 报告输出
7. MCP：发现 / 契约 / 参数映射 / 错误处理
8. Chat：单轮 / 多轮 / 会话解决率
9. Agent：任务完成 / 工具路径 / 多 trial
10. 知识检索：检索-生成拆分 / 证据支撑 / 版本回归

### Should 能力包

1. 版本对比
2. 基础门禁规则
3. 人工复核挂点
4. MCP 鉴权确认
5. Chat 拒答升级
6. Agent 异常恢复
7. Skill 触发精度
8. Workflow 关键路径回放

## 6. 首版不承诺的能力

以下明确不纳入首版承诺：

1. 多 Agent 协同测试
2. Workflow 全量分支覆盖
3. 通用 Agent Builder
4. 复杂低代码 Workflow 设计器
5. 全量线上监控告警中心
6. 自动发布阻断编排
7. 跨 server 复杂 MCP 组合测试

## 7. 首版裁剪原则

若资源不足，按以下顺序裁剪：

1. 先保 `Must` 共性平台能力
2. 再保 `Must` 四类主战场对象能力
3. `Should` 中优先保版本对比和基础门禁
4. `Skill` 和 `Workflow` 允许只保留方法与轻接入能力
