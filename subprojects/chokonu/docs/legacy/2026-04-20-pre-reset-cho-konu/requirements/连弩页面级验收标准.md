# 连弩页面级验收标准

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

连弩首版页面验收不以视觉完成度为主，而以“是否支撑首版能力闭环”为准。

页面级验收统一看四件事：

1. 是否能完成本页核心动作
2. 是否能进入下一个主链节点
3. 是否能看到本页最关键结果
4. 是否正确处理异常状态

## 2. 首页概览验收标准

1. 能展示最近运行摘要、红线问题、待复核数和版本结论
2. 能从概览跳到 run 详情、失败结果、版本对比和新建 run
3. 红线命中时必须显著提示
4. 无运行记录时必须给出进入样本中心或评测集中心的入口

## 3. 样本中心验收标准

1. 能创建、编辑、筛选和导出样本
2. 样本详情能展示输入、预期、风险标签、绑定评估器和历史引用
3. 能把样本纳入回归资产
4. 空状态必须给出创建样本入口
5. 无筛选结果时不能出现死页面

## 4. 评测集中心验收标准

1. 能创建 suite 并添加、移除样本
2. 能区分 `Baseline / Regression / Scenario`
3. 能查看 suite 最近运行摘要
4. 无 baseline 时必须明确提示
5. suite 详情能承接进入执行中心的入口

## 5. 执行中心验收标准

1. 能基于 suite 发起 run
2. 能配置环境、评估器组合和 trial 参数
3. 能查看 run 状态、进度和结果摘要
4. 能对失败样本发起重跑
5. 执行失败时必须给出进入结果与Trace的入口

## 6. 结果与Trace验收标准

1. 能按 `pass / fail / review`、对象类型、failure stage 过滤结果
2. 能查看 trace 详情并看到 evaluator 判定
3. 能识别红线命中和待复核项
4. trace 缺失时必须给出可理解的替代说明
5. 能从失败结果跳回样本或规则修订入口

## 7. 报告与版本对比验收标准

1. 能查看 run 报告和聚合报告
2. 能选择 baseline 做版本对比
3. 能展示红线问题、恶化样本和基础放行建议
4. 无 baseline 时必须明确提示，不允许给出虚假结论
5. 能导出正式报告

## 8. 规则与评估器验收标准

1. 能查看和配置 evaluator、threshold、redline
2. 能查看规则适用对象和绑定 suite
3. 阈值冲突时必须给出明确提示
4. 红线规则变更时必须显示影响范围
5. 不能扩成低代码规则引擎入口

## 9. 整体链路验收标准

1. 用户能从样本中心创建样本，并最终走到报告与版本对比
2. `样本 -> suite -> run -> evaluator -> trace -> report` 整条链路可在页面层完成闭环
3. `Skill / Workflow` 虽为轻接入，但在样本、suite、run、result 中必须可被正确承接
4. 所有页面都必须严格服从首版边界，不偷渡 `Not-now` 能力

## 10. 一句话结论

连弩首版页面级验收的标准，不是“页面像不像一个完整平台”，而是“页面能不能支撑 AI 应用测试主链稳定跑通，并让版本判断有据可依”。
