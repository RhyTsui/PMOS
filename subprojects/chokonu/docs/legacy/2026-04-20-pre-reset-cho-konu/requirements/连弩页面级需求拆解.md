# 连弩页面级需求拆解

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

本文件把首版需求从“能力项”下压到“页面必须承接哪些输入、操作和结果”的粒度，目的是让页面设计和研发拆分可以直接承接。

页面级需求只承接当前页面 IA 和首版需求清单里的内容，不新增范围。

## 2. 首页概览

### 服务对象

1. 项目负责人
2. 产品经理
3. 测试负责人
4. AI 应用研发

### 页面目标

快速识别当前质量状态，并跳转到下一个最合适的工作台。

### 必备功能

1. 展示最近运行摘要
2. 展示红线问题数和入口
3. 展示待复核样本数和入口
4. 展示最新版本对比结论
5. 提供新建 Run 快捷入口

### 必备信息块

1. 运行总览
2. 最近运行列表
3. 高风险问题卡片
4. 待复核卡片
5. 版本结论卡片

### 页面级输入

1. 最近 run 记录
2. 红线问题汇总
3. review-needed 样本汇总
4. 最新版本对比结果

### 页面级输出

1. 进入某个 run 详情
2. 进入失败结果列表
3. 进入版本对比页
4. 发起新 run

### 页面级约束

1. 不承接复杂趋势分析
2. 不承接组织级 BI 视图

1. run 配置
2. run 状态
3. 执行进度
4. 结果摘要
5. Trace 入口

### 页面级输入

1. suite
2. runner config
3. target environment
4. evaluator set
5. trial 参数

### 页面级输出

1. run 记录

3. 初步结果摘要

## 8. 规则与评估器

### 服务对象

2. 产品经理
3. 平台维护者

### 页面目标

统一管理评估器、阈值、红线项和 review hook，使不同对象的判定逻辑可配置、可追溯。

### 必备功能

1. 查看评估器列表
2. 配置阈值
3. 绑定 suite
4. 管理红线项
5. 配置 review hook
6. 查询历史命中情况

### 必备信息块

1. evaluator 类型
2. 适用对象
3. 判定方式
4. 阈值配置
5. 绑定 suite
6. 历史命中记录

### 页面级输入

1. evaluator definitions
2. thresholds
3. 对象类型
4. suite 绑定关系

### 页面级输出

1. evaluator 配置
2. 阈值配置
3. 红线规则
4. review hook 配置

### 页面级约束

1. 不承接低代码规则编排
2. 不承接复杂规则执行引擎

## 9. 一句话结论

页面级需求拆解的目标，是确保每个页面都明确知道自己承接什么、输入什么、产出什么，以及哪些能力明确不在首版页面里实现。
