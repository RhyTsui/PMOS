## 1. 这份文档解决什么问题

连弩不只需要测开 skill，也需要产品经理 skill。

因为这个项目不是纯工程工具，而是一个：

`要靠研究 -> 规划 -> 收敛 -> 同步 -> 决策支持`

逐步长起来的平台。

所以产品经理在这里不能只会写 PRD，而要能承担：

1. 吸收输入
2. 建立认知
3. 画清边界
4. 组织规划
5. 主动同步
6. 给主管提供决策支持

## 2. 产品经理 Skill 目标

产品经理 skill 的目标不是替代主管拍板，而是保证下面的人能先把高质量输入和中间产物做出来。

一句话：

`产品经理负责把混乱输入变成可收敛规划。`

## 3. 必备 Skill 列表

### Skill 1：输入归档

目标：

把 `inbox`、会议纪要、访谈、外部研究、历史文档归档成统一输入面。

能力点：

1. 输入分类
2. 输入去重
3. 来源标注
4. 历史输入与当前输入区分

产物：

1. 输入清单
2. 来源对照表
3. 当前有效输入范围

### Skill 2：研究提炼

目标：

把行业调研、竞品、最佳实践从资料堆里提炼成结构化结论。

能力点：

1. 最佳实践抽取
2. 竞品能力拆解
3. 共性模式归纳
4. 适用性判断

产物：

1. 研究摘要
2. 最佳实践约束
3. 候选方向说明

### Skill 3：边界收敛

目标：

把“大而全方向”压缩成首版边界。

能力点：

1. 首版/非首版拆分
2. P0/P1/P2 切分
3. 平台能力与项目能力拆分
4. 历史遗留与当前主目标拆分

产物：

1. 首版边界文档
2. 非首版范围说明
3. 优先级表

### Skill 4：动作链规划

目标：

把平台从概念图变成动作主链。

能力点：

1. 主链路拆分
2. 动作顺序设计
3. 动作之间依赖关系识别
4. 动作与模块映射

产物：

1. 动作主链
2. 动作清单
3. 模块与动作映射

### Skill 5：架构吸收

目标：

把技术架构草案翻译成产品能用的边界和分层。

能力点：

1. 架构图阅读
2. 领域对象抽取
3. 数据主链理解
4. 技术约束转产品边界

产物：

1. 产品可读架构摘要
2. 模块边界说明
3. 数据对象主链

### Skill 6：风险显化

目标：

把“看起来能做”的方案里真正的风险和未知显出来。

能力点：

1. 不确定项识别
2. 风险分级
3. 待验证假设提炼
4. 待主管拍板问题抽取

产物：

1. 风险清单
2. 不确定项清单
3. 待主管决策问题单

### Skill 7：文档编排

目标：

把研究、规划、架构、方法、需求串成一致文档体系。

能力点：

1. 文档分层
2. 真源文档识别
3. 草稿与正式稿区分
4. 文档间引用关系组织

产物：

1. 规划文档
2. 研究文档
3. 对齐摘要
4. 文档索引

### Skill 8：主动同步

目标：

不让主管靠追问才知道进展。

能力点：

1. 阶段同步
2. 首稿同步
3. 卡点同步
4. 决策请求同步

同步格式：

1. `本轮目标`
2. `已完成`
3. `当前在写`
4. `卡点`
5. `需要主管决策`

产物：

1. 进度同步
2. 卡点同步
3. 决策请求

### Skill 9：决策支持

目标：

不替主管拍板，但把拍板前需要看到的内容补齐。

能力点：

1. 候选方案对比
2. 收益/成本/风险表达
3. 决策点前置
4. 决策后影响范围说明

产物：

1. 候选方案对比表
2. 决策支持摘要
3. 决策后影响说明

### Skill 10：结构化问话

目标：

在关键收敛前，主动把需要确认的点组织成一轮高质量问话，而不是边做边猜。

能力点：

1. 目标确认问题整理
2. 边界确认问题整理
3. 优先级确认问题整理
4. 非目标确认问题整理
5. 拍板问题压缩

产物：

1. 问题单
2. 问答记录
3. 回写后的规划修订

## 4. 产品经理 Skill 分层

为了避免产品经理 skill 也变成散点，这里分成 4 层。

### 第一层：输入层

1. `输入归档`
2. `研究提炼`

### 第二层：收敛层

1. `边界收敛`
2. `动作链规划`
3. `架构吸收`

### 第三层：风险层

1. `风险显化`
2. `决策支持`

### 第四层：协作层

1. `文档编排`
2. `主动同步`
3. `结构化问话`

## 5. 产品经理与测试/测开 Skill 的分工

产品经理 skill 主要负责：

1. 研究
2. 规划
3. 边界
4. 决策支持
5. 同步

测试/测开 skill 主要负责：

1. 规则实现
2. 样本构造
3. 执行编排
4. 评估器
5. Trace
6. 报告生成

也就是说：

- 产品经理负责“把事情想清楚并组织起来”
- 测开负责“把事情跑起来并量化出来”

## 6. 当前最该优先补强的产品经理 Skill

针对连弩当前阶段，优先级建议是：

### P0

1. `研究提炼`
2. `边界收敛`
3. `动作链规划`
4. `主动同步`

### P1

1. `架构吸收`
2. `风险显化`
3. `决策支持`
4. `结构化问话`

### P2

1. `文档编排`
2. `输入归档`

## 7. 一句话结论

连弩的产品经理 skill 不是“写需求”这么窄，而是：

`把输入吸收掉，把边界收住，把动作链画清楚，把风险和决策点提出来，再稳定同步给主管。`
