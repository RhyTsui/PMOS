# 连弩页面组件级拆解

- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

本文件把页面进一步拆到组件级，目标不是定义 UI 库细节，而是明确：

1. 每个页面由哪些组件构成
2. 每个组件承接什么数据和动作
3. 组件之间如何联动
4. 哪些组件是全局复用组件，哪些是页面专属组件

## 2. 全局复用组件

### 2.1 页面框架组件

1. `TopNav`
   承接页面标题、环境切换、全局搜索、快捷入口
2. `PageHeader`
   承接页面标题、副标题、主要操作按钮
3. `FilterBar`
   承接页面筛选条件
4. `DataTable`
   承接主列表数据展示
5. `DetailDrawer`
   承接详情抽屉内容
6. `StatusBanner`
   承接红线、错误、无 baseline 等统一提示

### 2.2 状态组件

1. `EmptyState`
2. `LoadingSkeleton`
3. `ErrorState`
4. `ReviewFlag`
5. `RedlineFlag`

### 2.3 结果展示组件

1. `MetricCard`
2. `ResultBadge`
3. `CompareDelta`
4. `TraceSummaryCard`
5. `ReleaseSuggestionCard`

## 3. 首页概览组件拆解

### 页面组件清单

1. `OverviewMetricRow`
2. `RecentRunList`
3. `RiskAlertPanel`
4. `PendingReviewPanel`
5. `LatestCompareSummary`

### 组件职责

1. `OverviewMetricRow`
   展示 run 数、失败数、红线命中数、待复核数
2. `RecentRunList`
   展示最近运行及跳转
3. `RiskAlertPanel`
   展示高风险问题和红线入口
4. `PendingReviewPanel`
   展示待人工复核样本
5. `LatestCompareSummary`
   展示最新版本对比结果和放行建议

### 关键联动

1. 点击最近运行跳转 run 详情
2. 点击红线问题跳转结果与Trace
3. 点击版本结论跳转报告与版本对比

1. `CaseFilterSidebar`
2. `CaseTable`
3. `CaseDetailDrawer`
4. `CaseEditorForm`
5. `CaseImportDialog`

### 组件职责

### 页面组件清单

1. `ResultFilterBar`
2. `CaseResultTable`
3. `TraceTimelineList`
4. `TraceDetailPanel`
5. `ReviewActionPanel`

### 组件职责

1. `ResultFilterBar`
   承接 `pass / fail / review`、对象类型、failure stage 筛选
2. `CaseResultTable`
   展示 case 结果、红线、review 状态
3. `TraceTimelineList`

   展示请求/响应、evaluator 结果、引用关系

2. 人工复核后回写结果状态
3. 失败结果可回跳样本或规则页

### 关键联动

1. 切换 baseline 触发 compare 结果刷新
2. 点击恶化样本跳转结果与Trace
3. 放行建议受红线命中和主指标共同驱动

## 9. 规则与评估器组件拆解

### 页面组件清单

1. `EvaluatorTable`
2. `EvaluatorConfigDrawer`
3. `ThresholdConfigPanel`
4. `RuleBindingPanel`
5. `RedlineManager`

### 组件职责

1. `EvaluatorTable`
   展示 evaluator 列表和状态
2. `EvaluatorConfigDrawer`
   维护 evaluator 配置
3. `ThresholdConfigPanel`
   管理主指标和红线阈值
4. `RuleBindingPanel`
   绑定对象类型和 suite
5. `RedlineManager`
   维护红线规则并展示影响范围

### 数据输入

1. evaluator 定义
2. threshold 配置
3. suite 绑定关系
4. 命中记录摘要

### 关键联动

1. 调整阈值后影响 compare 结论
2. 修改红线规则后提示受影响 suite
3. 绑定关系更新后影响 run 可选项

## 10. 组件与数据层映射

首版建议按下面粒度对接数据：

1. `列表类组件`
   对接分页列表接口
2. `详情类组件`
   对接详情接口
3. `配置类组件`
   对接表单提交和配置保存接口
4. `结果类组件`
   对接聚合结果接口
5. `状态类组件`
   统一使用页面级状态字段驱动

## 12. 一句话结论

页面组件级拆解的目标，是把“页面该有什么”继续压成“每块区域最终要实现成什么组件、拿什么数据、支持什么动作”，这样后面设计、前端、后端都可以沿同一份真源继续推进。
