- 版本：v0.1
- 日期：2026-04-20
- 状态：Draft
- 项目：连弩-AI测试平台 / ChoKoNu

## 1. 核心结论

这份矩阵是连弩首版后续页面规划、需求清单和版本计划的真源。

它把六类测试对象统一压到同一张表里，回答：

1. 首版测什么对象
2. 每个对象要落哪些能力
3. 用什么指标判断结果
4. 哪些是红线项
5. 这些能力最终落到平台哪个模块

## 3. 平台共性模块映射

所有首版承诺能力必须回到统一主链：

`样本 -> suite -> run -> evaluator -> trace -> report -> version compare`

平台模块和承接范围固定如下：

1. `样本中心`
   承接 case、标签、预期、红线标记、版本快照引用
2. `评测集中心`
   承接 baseline suite、regression suite、scenario suite
3. `执行中心`
   承接 run、runner、目标环境、重跑、多 trial
4. `规则/评估器中心`
   承接规则评估、契约评估、语义评估、人工复核挂点
5. `结果与Trace中心`
   承接 case result、failure stage、raw/normalized trace
6. `报告与版本对比中心`
   承接 run report、aggregate report、version compare、基础门禁

## 4. 首版范围判断

### 首版承诺对象

1. `MCP`
2. `Chat`
3. `Agent`
4. `知识检索`

### 首版轻接入对象

1. `Skill`
2. `Workflow`

### 首版不承诺

1. 多 Agent 协同
2. Workflow 全量分支覆盖
3. 通用 Agent Builder
4. 自动发布阻断编排

## 5. 使用规则

后续所有文档必须服从这张矩阵：

1. 页面规划不能新增矩阵外对象
2. 需求清单不能承诺矩阵外能力
3. 里程碑不能以矩阵外能力作为首版验收条件

## 6. 需要主管决策的问题

1. Skill 与 Workflow 是否只按 `Should` 轻接入，不进入 Must 承诺
2. `版本对比` 是否和四类主战场对象同时进入首版验收
3. MCP 鉴权确认、Chat 拒答升级、Agent 异常恢复是否全部进入 `Should`
