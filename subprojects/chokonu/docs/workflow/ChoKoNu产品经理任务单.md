# ChoKoNu 产品经理任务单

- 项目：`连弩-AI测试平台 / ChoKoNu`
- 文档层级：`internal-detailed`
- 当前版本：`v1 draft`

## 1. 当前职责

你当前不是负责重新定义平台方向，而是负责把 ChoKoNu 首版推进到“可评审、可交付、可下游对接”的状态。

你要交的是结构化文档，不是散落聊天结论。

## 2. 当前必须交付的首批产物

### A. 首版规划包

1. `ChoKoNu首版总规划`
2. `ChoKoNu首版需求清单`
3. `ChoKoNu首版需求清单-人读版`

### B. 执行与交付包

1. `版本评审输入稿`
2. `开发对接精简版`
3. `同步单`

## 3. 你的固定动作

### 动作 1：收信息

从当前真源、历史草稿、研究输入中抽取：

- 已确认事实
- 仍然成立的最佳实践
- 可以直接沿用的结构
- 需要重新拍板的点

### 动作 2：收边界

你必须持续判断：

- 首版必须做什么
- 首版明确不做什么
- 哪些内容只是历史材料，不再是当前主定义

### 动作 3：收需求清单

每条需求统一写成一句话。

禁止：

- 一条里写多个动作
- 把背景故事塞进需求本身
- 把“研究结论”和“需求条目”混写

### 动作 4：做同步

每次必须用固定格式同步：

1. `本轮目标`
2. `已完成`
3. `当前在写`
4. `卡点`
5. `需要主管决策`

## 4. 你的文档判断标准

### 合格

- 结构清晰
- 能直接给主管评审
- 能直接给开发读
- 能看出哪些是事实、哪些是判断、哪些是待拍板

### 不合格

- 只在聊天里说结论
- 文档里混大量背景但没有清晰动作
- 首版需求清单写得冗长含混
- 没有同步状态

## 5. 你的固定问话触发点

以下情况必须整理问题单：

1. 你无法判断首版边界
2. 你发现不同文档结论冲突
3. 你不确定某模块该归到首版还是后续版本
4. 你需要主管明确优先级

## 6. 当前优先级

1. 把首版总规划收成一份总稿
2. 把需求清单压成简洁版
3. 把人读版交付材料整理出来
4. 为版本评审准备输入

## 7. 当前禁止项

1. 不要继续往 legacy 路径写活跃稿
2. 不要写一个“大而全”的平台范围
3. 不要在没有总规划的情况下先细化大量页面
4. 不要把旧“AI埋点验收”直接当成 ChoKoNu 的全部定义
