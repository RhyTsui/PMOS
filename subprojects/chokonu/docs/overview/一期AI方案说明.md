# AI 埋点验收一期 AI 方案说明

## 1. 方案定位

一期 AI 方案不是要做一个大而全 AI 平台，也不是让 AI 替代全部人工，而是做一条：

**AI 主提效 + 人工做确认 + 系统承接闭环**

的买点验收主链路。

## 2. 主链路

一期统一主链路定义为：

`原始输入 -> AI 提炼 -> 文档基线 -> 规则生成 -> 验收执行 -> 异常总结 -> 人工确认 -> 结果沉淀`

## 3. AI 负责什么

- 输入提炼
- 文档标准化
- 规则草稿生成
- 异常总结
- 协作输出

## 4. 人工负责什么

- 确认文档基线
- 确认规则灰区
- 做最终验收结论
- 拍板争议项

## 5. 系统负责什么

- 权限
- 流程
- 状态
- 版本
- 查询
- 校验执行
- 结果沉淀

## 6. 一期明确不做

- AI 全自动替代最终验收拍板
- 复杂研发正确性判断
- 完整根因分析引擎
- 重型多 Agent 编排平台
- 大而全的全链路自动测试闭环

## 7. 一期成功标志

- 文档整理时间明显下降
- 规则维护成本明显下降
- 验收路径明显缩短
- 自动校验能承担第一轮判断
- 人工主要处理灰区和最终确认
- 多角色重复沟通明显减少
- 文档和规则开始具备 AI-friendly 特征
