# AI 质量控制平台页面级原型框架 2026-04-27

- 项目：`连弩-AI测试平台 / ChoKoNu`
- 状态：`draft`
- 作用：把统一信息架构进一步细化为可直接出图的页面级原型框架，约束首页、7 个工作台、详情页签和关键动作

## 1. 本文定位

本文不是视觉稿，也不是前端实现说明。

本文回答的问题是：

`统一平台每个页面应该长什么结构，用户进入后先看什么、再点什么、详情承接什么。`

后续设计图、SVG 方案、前端原型和代码整合，都应以本文作为页面骨架真源。

## 2. 总体页面原则

1. 平台以工作台结构呈现，不以营销首页结构呈现。
2. 一级页面优先服务质量流程，不优先展示系统来源。
3. 每个工作台默认采用：
   - 顶部摘要区
   - 筛选与动作区
   - 列表区
   - 详情区
4. 复杂对象统一使用页签式详情。
5. 详情页必须能承接：
   - 基本信息
   - 资产关系
   - 执行记录
   - Trace / 证据
   - 分析 / 复核
   - 审计

## 4. 总览页

### 4.1 页面目标

让用户在进入平台后，第一眼知道：

1. 当前有哪些需求在推进
2. 当前有哪些执行在跑
3. 当前哪些结果有风险
4. 当前哪些项卡在复核和门禁

### 4.2 页面结构

1. 顶部指标条
   - 需求总数
   - 本周新增用例
   - 运行中任务
   - 失败任务
   - 待复核项
   - 阻断版本数

2. 中部左区：主链状态板
   - 需求
   - 用例
   - 执行
   - 报告
   - 复核
   - 门禁

3. 中部右区：高风险关注
   - 最近失败用例
   - 高风险版本
   - 高频异常对象类型

4. 下部左区：最近执行
   - 最近 10 次 Run

5. 下部右区：待办动作
   - 待生成用例需求
   - 待人工复核项
   - 待门禁决策版本

### 4.3 核心动作

1. 新建需求
2. 从需求生成用例
3. 发起执行
4. 查看失败 Trace
5. 进入复核
6. 查看门禁结论

## 12. 通用详情页签规范

为了保证三个系统整合后的页面一致性，复杂对象详情统一优先使用以下页签集合：

1. `基本信息`
2. `资产关系`
3. `执行记录`
4. `Trace / 证据`
5. `分析 / Judge`
6. `人工复核`
7. `审计`

并根据不同工作台做增减。

## 13. 关键跨页动作链

页面设计必须显式支持以下跨页动作链：

1. `需求台 -> 用例台`
   - 从需求生成用例

2. `用例台 -> 执行台`
   - 从用例或测试集发起执行

3. `执行台 -> Trace 台`
   - 从失败任务进入证据链

4. `Trace 台 -> 报告台`
   - 从证据汇总到报告

5. `报告台 -> 复核台`
   - 从结果进入复核

6. `复核台 -> 门禁台`
   - 从复核结果生成门禁结论
