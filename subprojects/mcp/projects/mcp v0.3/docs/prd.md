#### MCP 元数据定义

每个 MCP 工具需定义以下元数据：

```yaml
metadata:
  name: <工具名称>
  description: <工具描述>
  version: v0.3
  data_source: <数据源>
  
fields:
  - name: <字段名>
    type: <字段类型>
    description: <字段描述>
    aggregation: <聚合方式：sum/max/min/avg>

  - <维度 1>
  - <维度 2>

  - name: <指标名>
    aggregation: <聚合方式>
```

### 需求三：向上聚合计算

**核心能力**: MCP 发布时支持预制聚合 SQL，Chat 问数时按维度属性向上聚合计算。

#### 聚合层级

```
小时 → 日 → 周 → 月 → 季度 → 年

广告 → 优化师 → 团队 → 账户
```

#### 聚合规则（基于 Excel 示例）

| 维度类型 | 聚合方式 | 说明 |
|----------|----------|------|
| 时间维度 | groupby | 按时间分组 |
| 聚合维度 | groupby | 按维度分组（如优化师） |
| 消耗类指标 | sum | 求和 |
| 展示类指标 | sum | 求和 |
| 点击类指标 | sum | 求和 |
| 成本类指标 | sum | 求和 |
| 转化类指标 | sum | 求和 |
| ROI 类指标 | 计算 | (总收入/总成本) |

#### 聚合计算公式

| 指标类型 | 聚合方式 | 公式 |
|----------|----------|------|
| 消耗 | sum | SUM(cost) |
| 展示 | sum | SUM(impression) |
| 点击 | sum | SUM(click) |
| 转化 | sum | SUM(conversion) |
| ROI | 计算 | SUM(revenue) / SUM(cost) |
| CTR | 计算 | SUM(click) / SUM(impression) |
| CVR | 计算 | SUM(conversion) / SUM(click) |

#### 实现方式

1. **预制聚合 SQL**: 每个 MCP 工具配置预制的聚合 SQL 模板
2. **维度驱动**: 根据用户询问的维度自动选择聚合层级
3. **SQL 生成**: 动态生成向上聚合的 SQL
4. **指标聚合**: 根据指标类型自动选择 sum/max/min/avg

#### 示例

```sql
-- 优化师聚合
SELECT 
    optimizer_id,
    SUM(impression) as total_impression,
    SUM(click) as total_click,
    SUM(cost) as total_cost,
    SUM(click) * 1.0 / SUM(impression) as ctr
FROM ad_creative_report
WHERE date BETWEEN '2026-04-01' AND '2026-04-14'
GROUP BY optimizer_id

-- 团队聚合（向上聚合）
SELECT 
    team_id,
    SUM(impression) as total_impression,
    SUM(click) as total_click,
    SUM(cost) as total_cost,
    SUM(revenue) / SUM(cost) as roi
FROM ad_creative_report
WHERE date BETWEEN '2026-04-01' AND '2026-04-14'
GROUP BY team_id
```

## 风险与备注

1. **数据源接入**: 需确认各数据源的接入方式（直连 DB / API / 文件）
2. **数据推送**: 需确认推送目标（Hive/其他）、认证方式、表结构
3. **性能要求**: 聚合查询需考虑数据量级和响应时间
4. **用户手册**: 标签字典需与发行、数分确认使用场景
5. **嵌套字段**: 数据表结构需支持嵌套字段，不影响聚合计算
