## 0. 文档目标

这份文档聚焦“问题排查”专项，目标是把广告异常排查从“聊天判断”升级为“证据化排查任务”。

---

## 1. 专项范围

### 1.1 当前专项解决的问题

1. 异常排查跨系统、跨角色、跨日志
2. 结论和证据长期分离
3. 置信度和下一步动作没有被结构化

### 1.2 第一批覆盖的异常方向

1. 激活异常
2. 付费异常
3. 回传异常
4. 归因异常
5. BI / 数仓结果不一致

---

## 2. 专项需求

### 2.1 目标用户

1. 广告技术支持
2. 数据分析
3. 异常排查角色

### 2.2 核心需求

#### R-DIAG-01 异常问题任务化

输入：

1. 异常类型
2. 媒体
3. 应用
4. 计划对象
5. 时间窗
6. 异常现象
7. 可选截图 / 日志

输出：

1. 正式排查任务
2. 缺失信息追问

#### R-DIAG-02 证据链沉淀

必须输出：

1. 一句话结论
2. 证据项列表
3. 置信度
4. 下一步动作

#### R-DIAG-03 不完整结论表达

当证据不足时，系统必须明确：

1. 当前只能到什么判断层级
2. 缺什么证据
3. 由谁补证据

---

## 3. 专项功能

### 3.1 功能分组

#### F-DIAG-01 排查任务创建

1. 异常分类
2. 上下文收集
3. 排查任务创建

#### F-DIAG-02 缺失证据追问

1. 识别缺失证据
2. 发起追问
3. 更新任务上下文

#### F-DIAG-03 证据链结果生成

1. 结论生成
2. 证据项整理
3. 置信度生成
4. 下一步建议生成

#### F-DIAG-04 结果展示与回看

1. 结果区展示
2. 历史回看
3. 继续追问

### 3.2 后端承接对象

1. `DiagnosisTask`
2. `DiagnosisContext`
3. `DiagnosisEvidence`
4. `DiagnosisResult`

### 3.3 结果结构建议

`DiagnosisResult` 建议字段：

1. `summary_conclusion`
2. `confidence_level`
3. `evidence_items`
4. `missing_evidence_items`
5. `next_actions`
6. `owner_roles`

---

## 4. 页面与设计承接

### 4.1 主工作台结果区

必须展示：

1. 一句话结论
2. 证据链列表
3. 置信度
4. 缺证据说明
5. 下一步动作

### 4.2 历史任务区

应显示：

1. 异常类型
2. 媒体 / 应用
3. 当前结论状态
4. 是否仍待补证据

### 4.3 管理页承接

后续可支持：

1. 异常类型模板
2. 证据模板
3. 结果模板

---

## 5. 实施顺序

1. 先定义诊断结果 mock schema
2. 先做证据链与缺证据结构
3. 再做结果区正式渲染
4. 最后再补模板化管理
