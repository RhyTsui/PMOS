# 连弩评测：智投chat接口接入文档

版本：v0.2  
日期：2026-05-14  
适用对象：连弩测试平台接入智投chat进行问答、意图路由、需求、排查、联调类能力评测  
服务地址：`http://<host>:8002`

## 1. 项目基础信息

| 字段 | 内容 |
| --- | --- |
| 项目名称 | 智投chat |
| 项目类型 | Chat / Agent |
| 业务域 | 广告投放支持、数据排查、媒体对接、自动联调 |
| 接入环境 | 本地/预发布演示环境 |
| 是否返回流式响应 | 是 |
| 是否返回工具调用过程 | 是，Chat SSE 中包含 `thinking_step`、`tool_call`、`tool_result`、`trace_data` |
| 评测适配接口 | 已提供非流式评测适配接口，便于连弩直接跑用例 |

## 2. 推荐接入方式

连弩优先使用评测适配接口：

| 用途 | Method | Path |
| --- | --- | --- |
| 健康检查 | `GET` | `/api/xiaoqiao/evaluation/health` |
| 获取内置用例 | `GET` | `/api/xiaoqiao/evaluation/cases` |
| 执行单条评测 | `POST` | `/api/xiaoqiao/evaluation/run` |
| 单独验证意图路由 | `POST` | `/api/xiaoqiao/agents/intent-route` |

原因：

- 返回结构稳定，适合自动评分和批量回归。
- 已标准化输出 `answer`、`process_events`、`sources`、`scores`。
- 不依赖浏览器会话态。

## 3. 评测执行接口

### 3.1 请求协议

| 字段 | 内容 |
| --- | --- |
| Method | `POST` |
| Content-Type | `application/json` |
| Path | `/api/xiaoqiao/evaluation/run` |
| 是否流式 | 否 |
| 超时建议 | 首包 10 秒，整体 60 秒 |
| 是否可重试 | 5xx、429 可重试；4xx 不重试 |

### 3.2 请求 Body

| 字段路径 | 类型 | 必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `case_id` | string | 否 | `ztchat-demand-001` | 使用内置评测用例时填写 |
| `input` | string | 条件必填 | `我们要新增一个媒体对接` | 未传 `case_id` 时必填 |
| `scenario` | string | 否 | `demand` | 可选：`help`、`demand`、`diagnosis`、`debugging` |
| `expected_keywords` | string[] | 否 | `["对接文档","需求池"]` | 连弩自定义关键词 |
| `metadata` | object | 否 | `{ "suite": "lianu" }` | 评测批次、项目、实验信息 |

### 3.3 请求示例

```bash
curl --location 'http://127.0.0.1:8002/api/xiaoqiao/evaluation/run' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "input": "我们要新增一个媒体对接，需要创建监测链接和回传。",
    "scenario": "demand",
    "expected_keywords": ["对接文档", "结构化表单", "需求池", "监测链接"],
    "metadata": {
      "suite": "lianu-regression",
      "case_owner": "qa"
    }
  }'
```

## 4. 响应解析规则

| 目标字段 | 解析来源 | 规则 |
| --- | --- | --- |
| `final_answer` | `answer` | 直接读取 |
| `response_text` | `answer` | 直接读取 |
| `scenario` | `scenario` | 判断意图路由是否命中 |
| `tool_calls` | `process_events` | 读取 `type` 为 `knowledge_search`、`model_step`、后续真实工具事件 |
| `sources` | `sources` | 用于来源披露评测 |
| `trace_id` | `trace_id` | 作为单次评测链路 ID |
| `status` | HTTP status + body | HTTP 2xx 且无 `error` 视为成功 |
| `score` | `scores.overall` | 0-1 分 |

## 5. Chat 流式接口

如果连弩需要验证真实 Chat SSE，可使用：

| 字段 | 内容 |
| --- | --- |
| Method | `POST` |
| Path | `/api/chat` |
| Content-Type | `application/json` |
| Accept | `text/event-stream` |
| Header | 可选：`x-conversation-id` |

请求示例：

```bash
curl --location -N 'http://127.0.0.1:8002/api/chat' \
  --header 'Content-Type: application/json' \
  --header 'x-conversation-id: conv-lianu-001' \
  --data-raw '{
    "message": "今天巨量回传激活数和BI不一致，帮我排查原因。",
    "intent": "diagnosis",
    "history": []
  }'
```

SSE 事件类型：

| `type` | 说明 |
| --- | --- |
| `phase` | 当前阶段 |
| `route` | 意图路由 |
| `thinking_step` | 思考步骤，可用于过程可视化 |
| `tool_call` | 工具调用开始 |
| `tool_result` | 工具调用结果 |
| `content` | 回答文本分片 |
| `done` | 结束事件，可能包含结构化 `result` |
| `error` | 错误或降级说明 |
| `trace_data` | 调用链数据 |

流式结束规则：连接自然关闭，或收到 `done` 事件后结束读取。

## 5.1 意图路由 Agent 接口

用于连弩单独评测“用户输入应该进入哪个 Agent 和哪条可跟踪流程”。

```bash
curl --location 'http://127.0.0.1:8002/api/xiaoqiao/agents/intent-route' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "input": "我们要新增一个媒体对接，需要创建监测链接和回传"
  }'
```

响应字段：

| 字段 | 说明 |
| --- | --- |
| `intent_type` | `help`、`demand`、`diagnosis`、`debugging`、`monitor`、`material-analysis`、`forecast`、`general` |
| `agent` | 前端/后端应进入的 Agent |
| `workflow_level` | `light` 或 `heavy` |
| `required_slots` | 必须补齐的信息 |
| `suggested_actions` | 推荐下一步动作 |
| `tracking_target` | 可跟踪对象，如需求表单录入、排查记录、联调记录 |

当前路由覆盖：

| 能力 | 路由结果 | 可跟踪对象 |
| --- | --- | --- |
| 解答 | `help` | 来源与追问 |
| 排查 | `diagnosis` | 排查记录 |
| 对接 | `demand` | 需求表单录入、对接进度 |
| 联调 | `debugging` | 联调记录 |
| 报表 | `help` + 报表任务动作 | 报表定时任务 |
| 分析 | `material-analysis` | 分析记录 |
| 监控 | `monitor` | 监控任务告警 |
| 预测 | `forecast` | 预测记录 |

## 6. 内置用例

`GET /api/xiaoqiao/evaluation/cases` 当前包含：

| case_id | 场景 | 目标 |
| --- | --- | --- |
| `ztchat-diagnosis-001` | `diagnosis` | 回传数据不一致排查 |
| `ztchat-demand-001` | `demand` | 新增媒体对接需求收集 |
| `ztchat-debugging-001` | `debugging` | 老媒体自动联调准备 |

## 7. 验收标准

| 验收项 | 通过标准 |
| --- | --- |
| URL 可访问 | 健康检查返回 HTTP 2xx |
| 请求体可识别 | `run` 接口传 `input` 后不返回参数错误 |
| 场景可路由 | `scenario` 与预期场景一致 |
| 回答可解析 | `answer` 非空，包含可读 Markdown |
| 过程可披露 | `process_events` 非空 |
| 来源可披露 | `sources` 非空 |
| 自动评分可读 | `scores.overall` 为 0-1 数值 |
| 失败可识别 | 缺少 `input` 时返回明确错误 |

## 8. 当前边界

- 评测适配接口当前是稳定 mock + 业务规则适配，不等同于真实大模型全链路结果。
- `/api/chat` 才是会话流式链路，依赖模型与知识库配置。
- 后续 agent 服务完全落地后，需要将 `process_events` 接到真实需求路由、排查、联调工具链事件。
