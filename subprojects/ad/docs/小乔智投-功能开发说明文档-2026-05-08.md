## 1. 当前实现基线

### 1.1 当前已有正式接口

1. `POST /api/v1/xiaoqiao/conversations`
2. `GET /api/v1/xiaoqiao/conversations`
3. `GET /api/v1/xiaoqiao/conversations/{conversation_id}`
4. `POST /api/v1/xiaoqiao/conversations/{conversation_id}/messages`
5. `GET /api/v1/xiaoqiao/tasks`
6. `GET /api/v1/xiaoqiao/tasks/{task_id}`
7. `GET /api/v1/xiaoqiao/tasks/{task_id}/results`
8. `GET /api/v1/xiaoqiao/workspace`

### 1.2 当前已有正式对象

1. `Conversation`
2. `Message`
3. `RoutingDecision`
4. `Task`
5. `TaskContext`
6. `WorkflowResult`

### 1.3 当前最需要继续深化的 contract

1. `TaskContext.missing_fields_json`
2. `WorkflowResult.structured_payload_json`
3. 四类业务流结果 schema
4. 联调状态与日志结构
5. Prompt 管理相关对象

### 1.4 当前前端实现分层基线

前端默认按 `docs/小乔智投-前端分层与开源基座方案-2026-05-09.md` 执行：

1. `Ant Design + Ant Design X` 负责工作台和会话表现层
2. `CopilotKit` 负责 agent 驱动交互编排
3. `Zustand` 负责本地 UI 状态
4. `React Query` 负责服务端数据同步

当前前端最高优先线不是更换聊天框架，而是把 `F-33` 结果区推进为插件式 `Structured Result Renderer`
同时默认遵循 `docs/小乔智投-Ant-Design-X默认规范-2026-05-09.md`

补充治理说明：

1. 这些分层只约束实现栈，不直接定义首页信息架构
2. 任何会让首页同时平铺“会话、任务、结果、运行验证、治理说明”的实现，都视为偏离本功能文档

## 3. 核心功能清单

### 3.1 统一入口与路由

#### F-03 首条消息创建与发送

- 来源需求：`R-01`
- 前端承接：主工作台输入区
- 当前现状：已有会话与消息接口
- 依赖对象：`Conversation` `Message`
- 关键返回：`message` `routing` `task` `assistant_reply`

#### F-05 路由判断

- 来源需求：`R-02`
- 前端承接：结果区类型切换、追问提示
- 当前现状：已有 `RoutingDecision`
- 关键字段：
  - `is_business_related`
  - `intent_type`
  - `workflow_level`
  - `clarification_needed`
  - `decision_reason`

#### F-08 追问判断

- 来源需求：`R-02`
- 前端承接：缺字段追问提示
- 当前缺口：还需要把“缺什么字段”显式挂到 `TaskContext`

### 3.2 使用帮助工作流

#### F-09 帮助任务创建

- 来源需求：`R-03`
- 当前现状：可借助消息发送时生成 `Task`
- 目标：帮助类请求进入 `help` 任务

#### F-10 帮助结果生成

- 来源需求：`R-03`
- 当前现状：已有 `WorkflowResult`
- 当前缺口：帮助结果 schema 过泛
- 正式结果结构方向：
  - `definition`
  - `system_path`
  - `reference_sources`
  - `confidence`
  - `follow_up_questions`

#### F-11 帮助结果面板展示

- 来源需求：`R-03`
- 前端承接：结果区帮助卡组
- 当前缺口：需要按正式 schema 渲染

### 3.3 需求沟通工作流

#### F-13 需求任务创建

- 来源需求：`R-04`
- 当前目标：将需求类输入沉淀为 `demand` 任务

#### F-14 缺失字段识别

- 来源需求：`R-04`
- 当前现状：已有 `missing_fields_json`
- 当前缺口：字段只是字符串列表，不够支撑页面与追问
- 正式结构方向：
  - `field_key`
  - `field_label`
  - `why_required`
  - `suggested_question`

#### F-15 结构化需求单生成

- 来源需求：`R-04`
- 当前目标：让结果区展示正式需求单
- 正式结构方向：
  - `demand_type`
  - `media`
  - `app`
  - `target_object`
  - `target_date`
  - `acceptance_method`
  - `dependencies`
  - `owner_candidates`

### 3.4 问题排查工作流

### 3.6 运行时沉淀

#### F-25 ~ F-30

- 来源需求：`R-07`
- 目标：稳定承接会话、消息、路由、任务、结果、历史回看
- 当前状态：`solved` 级最小骨架已具备

### 3.7 前端工作台承接

#### F-31 顶部状态展示

- 来源需求：`R-08`
- 前端承接：顶部身份与状态区
- 依赖接口：`GET /workspace`
- 页面约束：只显示当前工作必需状态，不展开治理说明

#### F-32 主对话区展示

- 来源需求：`R-08`
- 前端承接：消息流 + Sender + Attachments
- 当前目标：从静态壳改成 API 驱动
- 页面约束：作为默认首屏主操作区，不与运行验证区平级并排竞争

#### F-33 结果区展示

- 来源需求：`R-08`
- 前端承接：右侧结果区
- 当前关键：绑定四类正式结果 schema
- 页面约束：结果区是当前任务检查器，不承接长段说明文案

#### F-34 历史任务展示

- 来源需求：`R-08`
- 前端承接：历史任务列表
- 依赖接口：`GET /tasks`
- 页面约束：优先以抽屉、侧板或二级展开承接，不做首页常驻平铺主模块

### 3.8 管理页与 Prompt 治理

### 3.9 会话支持能力

#### F-42 缺失字段补录
