## 0. 文档目标

这份需求文档的目标不是描述几个页面，而是把小乔智投第一阶段需求压到能够支撑：

1. 功能拆解
2. 设计文档
3. 前端页面
4. 数据表
5. 后端接口

它必须回答：

1. 解决谁的问题
2. 在什么业务流里发生
3. 最低要收哪些输入
4. 系统要交付什么正式结果物
5. 这条需求会拆成哪些功能

---

## 1. 需求拆解原则

### 1.1 拆解标准

每条需求至少回答：

1. 触发角色
2. 业务上下文
3. 最低输入
4. 正式输出
5. 下游功能映射

### 1.2 当前硬约束

1. 使用正式 mock contract 承接需求，不等数据库直连
2. 前端页面必须是交付级动态页面
3. 需求不能只写“页面模块”，必须写真实业务对象和结果物

## 3. 正式需求清单

### R-01 统一对话入口

- 类型：产品需求
- 优先级：P0

用户问题：

1. 用户不知道该先找哪个系统
2. 用户不知道问题是否属于帮助、需求、排查还是联调

最低输入：

1. 自然语言问题
2. 可选附件
3. 可选补充上下文

系统必须输出：

1. 可直接承接首轮输入的对话入口
2. 首轮系统反馈
3. 必要时进入后续任务流

对应功能：

1. `F-01` 首页输入区
2. `F-02` 推荐起手问题
3. `F-03` 首条消息创建与发送
4. `F-04` 首轮系统反馈

### R-02 工作流路由与追问

- 类型：产品需求
- 优先级：P0

用户问题：

1. 统一入口不能只接住问题，还要知道该走哪条业务流
2. 信息不足时不能假装理解，必须追问

最低输入：

1. 用户问题文本
2. 当前会话上下文
3. 可选附件

系统必须输出：

1. 是否属于业务工作
2. 属于哪类业务流
3. 是否需要追问
4. 缺失字段方向

对应功能：

1. `F-05` 路由判断
2. `F-06` 业务相关性识别
3. `F-07` 意图类型识别
4. `F-08` 追问判断

### R-03 使用帮助结果区

- 类型：业务需求
- 优先级：P0

典型问题：

1. 指标是什么意思
2. 某个结果在哪个系统看
3. 某条规则怎么理解
4. 常见广告技术问题如何处理

最低输入：

1. 问题类型
2. 涉及对象
3. 是否需要系统路径
4. 是否需要引用来源

系统必须输出：

1. 概念或规则解释
2. 系统路径
3. 引用来源或知识依据
4. 可继续追问的方向

对应功能：

1. `F-09` 帮助任务创建
2. `F-10` 帮助结果生成
3. `F-11` 帮助结果面板展示
4. `F-12` 帮助追问承接

### R-04 需求沟通结构化

- 类型：业务需求
- 优先级：P0

业务问题：

1. 需求长期口头化
2. 字段缺失
3. 依赖项遗漏
4. 无法直接进入下游开发或协作

第一阶段覆盖方向：

1. 媒体回传接入
2. 事件映射
3. 埋点/归因/配置类需求
4. 跨角色协作型需求

最低字段包：

1. 需求类型
2. 媒体
3. 应用/包体
4. 目标对象
5. 目标时间
6. 验收方式
7. 依赖系统或依赖方

系统必须输出：

1. 结构化需求单
2. 缺失字段清单
3. 依赖项清单
4. 下一步协作动作

对应功能：

1. `F-13` 需求任务创建
2. `F-14` 缺失字段识别
3. `F-15` 结构化需求单生成
4. `F-16` 需求结果展示

### R-05 问题排查证据区

- 类型：业务需求
- 优先级：P0

第一阶段重点异常：

1. 激活异常
2. 付费异常
3. 回传异常
4. 归因异常
5. BI/数仓结果不一致

最低字段包：

1. 异常类型
2. 媒体
3. 应用
4. 计划或对象
5. 时间窗
6. 指标或异常现象
7. 可选截图/日志

系统必须输出：

1. 一句话结论
2. 证据链
3. 置信度
4. 下一步建议

对应功能：

1. `F-17` 排查任务创建
2. `F-18` 排查上下文记录
3. `F-19` 排查结果生成
4. `F-20` 排查结果面板展示

### R-06 广告联调任务区

- 类型：业务需求
- 优先级：P0

最低字段包：

1. 联调类型
2. 账号
3. 媒体
4. 应用/包体
5. 测试设备
6. 入口方式
7. 环境
8. 当前卡点

系统必须输出：

1. 联调准备项
2. 联调执行状态
3. 执行日志或摘要
4. 联调结果报告

对应功能：

1. `F-21` 联调任务创建
2. `F-22` 联调准备项与状态展示
3. `F-23` 联调日志展示
4. `F-24` 联调结果展示

### R-07 会话与任务沉淀

- 类型：技术需求
- 优先级：P0

系统必须沉淀：

1. Conversation
2. Message
3. RoutingDecision
4. Task
5. TaskContext
6. WorkflowResult

对应功能：

1. `F-25` 会话创建与读取
2. `F-26` 消息记录
3. `F-27` 路由记录
4. `F-28` 任务记录
5. `F-29` 结果记录
6. `F-30` 历史任务回看

### R-08 单对话工作台视图

- 类型：前端需求
- 优先级：P0

页面必须呈现：

1. 顶部身份与状态区
2. 主对话工作区
3. 结果区
4. 历史任务区
5. 专项页入口

补充约束：

1. 默认首屏只围绕“当前问题推进”这一主操作展开
2. `历史任务区` 可以用抽屉、侧板或二级展开承接，不要求首页常驻平铺
3. `专项页入口` 只保留入口能力，不要求把专项内容投射回首页
4. 运行验证、压测证据、治理记录不属于本需求的首页常驻内容

验收要求：

1. 页面是工作台，不是静态介绍页
2. 对话区与结果区同时存在
3. 历史任务可回看
4. 具备真实动态交互
5. 用户进入后先能做事，而不是先阅读模块说明

对应功能：

1. `F-31` 顶部状态展示
2. `F-32` 主对话区展示
3. `F-33` 结果区展示
4. `F-34` 历史任务展示
5. `F-35` 专项页导航

### R-09 管理页面与 Prompt 管理
