# 智投 Chat Skill 体系设计

## 定位

Skill 不是功能菜单，而是可复用、可评测、可自动执行的广告业务流程单元。

在智投 Chat 中：

- Agent 负责理解用户意图、追问和服务路由。
- Skill 负责执行稳定流程。
- MCP 负责连接配置、报表、监测、归因、知识库等系统能力。
- 结果组件负责展示结论、证据、表格、图表、表单、任务和报告。

判断一个能力是否适合 Skill 化，优先看五个条件：

1. 高频出现。
2. 有固定步骤。
3. 输入输出相对稳定。
4. 需要调用多个 MCP、工具或知识库。
5. 可以评测和持续优化。

## 优先级

| 优先级 | Skill | 适用场景 | 不做成普通 Agent 推理的原因 |
| --- | --- | --- | --- |
| P0 | 自然语言查数 | 查消耗、ROI、激活、注册、付费、维度下钻 | 指标、维度、时间、筛选、口径可以结构化 |
| P0 | 媒体消耗排查 | 消耗对不上、报表无数、采集延迟 | 检查链路稳定：采集、调度、权限、口径 |
| P0 | 回传链路巡检 | 激活、注册、付费异常或回传失败 | 检查链路稳定：事件、归因、回传、入库 |
| P0 | 自动联调 | 媒体联调、事件联调、监测链接验证 | 步骤标准、结果可报告、过程可追踪 |
| P0 | 日报生成 | 定时日报、周报、投放复盘 | 固定拉数、分析、生成、发送 |
| P1 | ROI 异常监控 | ROI 下跌、消耗突增、回收异常 | 适合定时执行和阈值告警 |
| P1 | 项目健康巡检 | 项目配置、媒体配置、监测状态巡检 | 多系统检查项稳定，可输出健康度 |
| P1 | 新媒体对接 | 新媒体、渠道、事件、报表接入 | 有固定表单、依赖校验、验收清单 |
| P1 | Campaign 分析 | campaign 贡献、预算、素材、ROI 分析 | 可形成固定拆解框架 |
| P1 | 素材洞察 | 素材表现、标签、创意归因 | 输入输出可标准化，但依赖素材能力成熟度 |

## 不适合优先 Skill 化

以下能力暂时交给 Agent 动态推理：

- 开放式经营策略分析，例如“这个项目是不是该转混变现”。
- 高层跨域归因，例如“季度利润为什么下降”。
- 多轮探索式分析，例如先看 ROI，再看 campaign，再看素材，再看国家。

这些问题需要 Planner Agent 动态拆解，过程中可调用多个 Skill，但不应直接写成单个固定 Skill。

## 标准 Skill Contract

每个 Skill 至少需要定义：

| 字段 | 说明 |
| --- | --- |
| `skill_id` | 稳定 ID |
| `name` | 用户可理解的名称 |
| `intent_triggers` | 触发意图和关键词 |
| `input_schema` | 必填和可选字段 |
| `clarification_schema` | 缺字段时如何追问 |
| `workflow_steps` | 固定执行步骤 |
| `tool_bindings` | 需要调用的 MCP / API / 知识库 |
| `output_schema` | 结果结构 |
| `ui_components` | 表单、思维链、来源、表格、图表、报告等组件 |
| `evaluation_cases` | 测试用例和预期结果 |
| `risk_guardrails` | 权限、口径、敏感数据和操作审批规则 |

## P0 Skill 细化

### 1. 自然语言查数

输入：

- 指标：消耗、激活、注册、付费、ROI、CPA 等。
- 时间：今天、昨天、近 7 天、自定义区间。
- 维度：项目、媒体、账户、campaign、素材。
- 筛选：APPID、媒体 ID、账户 ID、项目名。

输出：

- 查询结果表。
- 指标口径说明。
- 来源报表和更新时间。
- 下一步下钻建议。

### 2. 媒体消耗排查

适用：消耗异常、媒体后台和智投报表对不上。

固定检查链路：

1. 明确对比数据源。
2. 查询智投报表实际值。
3. 检查媒体采集状态。
4. 检查报表调度时间和最后更新时间。
5. 检查用户权限和账户可见范围。
6. 输出差异、原因和处理建议。

注意：消耗排查不默认检查监测回传、包名、渠道号、分包 ID。只有激活、注册、付费等转化指标异常时才进入回传链路检查。

### 3. 回传链路巡检

适用：激活、注册、付费、归因、回传异常。

固定检查链路：

1. 检查监测链接和归因参数。
2. 检查客户端事件触发。
3. 检查服务端接收。
4. 检查媒体回传。
5. 检查 BI 入库和报表聚合。
6. 输出失败节点、证据和修复建议。

### 4. 自动联调

输入：

- 媒体。
- 项目。
- 事件。
- 联调地址或监测链接。
- 测试设备信息。

输出：

- 联调步骤。
- 当前进度。
- 每一步观测结果。
- 失败节点和重试建议。
- 联调报告。

### 5. 日报生成

输入：

- 项目或账户范围。
- 日期。
- 指标。
- 发送对象。
- 发送时间。

输出：

- 日报正文。
- 异常摘要。
- 关键指标表。
- 重点风险。
- 建议动作。

## 与思维链组件的关系

Skill 执行过程必须在消息区保留可查的过程组件，但不展示模型隐私推理。推荐展示：

- 已识别意图。
- 已补充条件。
- 已调用的数据源。
- 已执行检查项。
- 工具调用状态。
- 耗时。
- 结果来源。
- 失败或跳过原因。

这样用户看到的是“服务过程”和“证据链”，不是不可验证的内部思维。
