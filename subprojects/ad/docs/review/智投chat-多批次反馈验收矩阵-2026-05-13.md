# 智投chat 多批次反馈验收矩阵 - 2026-05-13

## 1. 复核口径

本矩阵用于避免“多批次反馈遗漏”和“每条浅层实现”。状态只按证据判定：

- `已完成`：代码有明确实现，且已有构建/浏览器/功能验证记录。
- `需验收`：代码有实现，但缺少本轮实机验收或用户确认。
- `部分完成`：有前端形态或局部逻辑，但没有完整产品闭环。
- `未完成`：当前没有可靠实现证据。

重要原则：

- 没有证据不标已完成。
- 只有静态 mock 不标已完成。
- 只改 UI 但没有状态流、数据流、验收流的，最多标部分完成。

## 4. 必须回头补的高风险项

### P0-1 思维链协议化，不再浅层展示

当前问题：

- 已有 ThoughtChain UI，但事件协议未统一。
- 没有 `AgentProcessEvent` 类型落地。
- 没有普通用户、管理员、审计三层披露。

完成标准：

- 后端统一输出 `process_event`。
- 前端用 `process_events` 渲染 ThoughtChain。
- 旧 `thinking_step/tool_call` 保持兼容。

### P0-2 自动联调接真实状态流

当前问题：

- 前端工作台有了，但 API 仍是 demo 数据。

完成标准：

- 接入后端真实任务接口。
- 状态变化能驱动前端步骤。
- 失败、暂停、接管、结果归档都有真实数据。

### P0-3 多批次反馈回归验收

当前问题：

- 多数 UI 细节“代码有实现”，但缺少最新浏览器复查。

完成标准：

- 用 8002 跑一次桌面和窄屏。
- 逐项覆盖侧边栏、欢迎区、消息操作、来源、资产、联调。
- 生成截图和 JSON 记录。

### P1-1 我的资产真实链路

当前问题：

- 资产仍是本地 mock 列表。
- 缩略图生成只是产品说明，不是真实调用。
- 定位会话锚点证据不足。

完成标准：

- 上传/生成时写入资产对象。
- 图片/视频触发多模态缩略图生成。
- 定位会话能打开目标会话并滚到对应消息。
