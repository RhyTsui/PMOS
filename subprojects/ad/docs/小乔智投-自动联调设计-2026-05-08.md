## 2. 专项需求

### 2.1 目标用户

1. 广告技术支持
2. 联调执行人员
3. 投放协作人员
4. 问题排查人员

### 2.2 核心问题

当前广告联调存在四个典型问题：

1. 准备项分散，靠人肉检查
2. 执行链路长，跨 Web、手机、广告流、App
3. 中间步骤难留痕，失败后很难复盘
4. 成功与失败判断缺少结构化回收

### 2.3 用户侧需求

#### R-AUTO-01 自动联调任务创建

用户希望在小乔中直接创建联调专项任务，而不是跳出系统后再口头推进。

输入最小字段包：

1. 媒体
2. 联调类型
3. 账号
4. 应用 / 包体
5. 测试设备
6. 环境
7. 当前卡点

结果物：

1. 自动联调任务卡
2. 准备项清单
3. 可执行状态

#### R-AUTO-02 执行阶段可视化

用户希望看到自动联调当前执行到哪一步，而不是只知道“正在跑”。

必须展示：

1. 当前阶段
2. 当前步骤
3. 最近截图
4. 最近日志
5. 是否卡住

#### R-AUTO-03 失败可回退、可接管

用户希望当自动执行不稳定时，系统能明确告诉他失败在哪，并允许人工继续。

结果物：

1. 失败原因
2. 失败步骤
3. 失败截图
4. 人工接管入口
5. 下一步建议

#### R-AUTO-04 联调结果结构化沉淀

用户希望联调结束后留下一份正式结果，而不是只看一遍执行过程。

结果物：

1. 是否成功
2. 成功判断依据
3. 执行日志摘要
4. 关键截图
5. 最终报告

### 2.4 管理侧需求

#### R-AUTO-05 自动联调模板与配置管理

管理侧需要管理：

1. 专项模板
2. 关键词
3. 超时参数
4. 执行开关
5. 适用媒体 / 终端 / 环境

#### R-AUTO-06 执行依赖状态查看

管理侧需要看到专项依赖是否就绪，例如：

1. 设备是否可用
2. 执行器是否可用
3. 视觉识别能力是否可用
4. 配置是否完整

## 3. 专项功能

### 3.1 功能分组

#### F-AUTO-01 任务创建与校验

1. 创建联调专项任务
2. 校验最小字段包
3. 生成准备项清单

#### F-AUTO-02 状态机驱动执行

1. 任务进入待确认
2. 启动自动执行
3. 按阶段流转状态
4. 记录当前步骤

#### F-AUTO-03 Web 侧准备执行

1. 登录媒体后台
2. 进入联调工具
3. 生成二维码
4. 推送到手机端

#### F-AUTO-04 Mobile 侧执行

1. 启动 App
2. 扫码
3. 进入授权测试
4. 刷广告
5. 点击广告
6. 拉起游戏

#### F-AUTO-05 成功轮询与结果判断

1. 轮询联调成功标记
2. 记录轮询结果
3. 判断成功 / 失败

#### F-AUTO-06 留痕与结果沉淀

1. 步骤截图
2. 错误截图
3. 产物留存
4. 日志摘要
5. 最终结果报告

#### F-AUTO-07 人工接管与重试

1. 人工接管
2. 重试当前步骤
3. 重新发起任务

#### F-AUTO-08 模板与参数管理

1. 自动联调模板管理
2. 关键词配置
3. 超时参数配置
4. 启停开关

### 3.2 后端承接对象

建议对象：

1. `DebugAutomationTask`
2. `DebugAutomationConfig`
3. `DebugExecutionStep`
4. `DebugExecutionResult`

### 3.3 接口承接

建议接口：

1. `POST /api/v1/xiaoqiao/debug-automation/tasks`
2. `POST /api/v1/xiaoqiao/debug-automation/tasks/{task_id}/start`
3. `POST /api/v1/xiaoqiao/debug-automation/tasks/{task_id}/pause`
4. `POST /api/v1/xiaoqiao/debug-automation/tasks/{task_id}/resume`
5. `POST /api/v1/xiaoqiao/debug-automation/tasks/{task_id}/takeover`
6. `GET /api/v1/xiaoqiao/debug-automation/tasks`
7. `GET /api/v1/xiaoqiao/debug-automation/tasks/{task_id}`
8. `GET /api/v1/xiaoqiao/debug-automation/tasks/{task_id}/steps`
9. `GET /api/v1/xiaoqiao/debug-automation/tasks/{task_id}/result`
10. `GET /api/v1/xiaoqiao/admin/debug-automation/configs`
11. `POST /api/v1/xiaoqiao/admin/debug-automation/configs`
12. `PUT /api/v1/xiaoqiao/admin/debug-automation/configs/{config_id}`

## 4. 专项设计

### 4.1 主工作台承接

主工作台中的联调结果区应新增自动联调专项面板，最少包含：

1. 专项标题
2. 当前阶段
3. 当前步骤
4. 状态时间线
5. 最近截图
6. 最近日志
7. 成功判断区
8. 失败原因区
9. 人工接管按钮
10. 最终报告入口

### 4.2 历史任务区承接

历史任务区需要能识别“自动联调专项任务”，并显示：

1. 媒体
2. 应用
3. 当前状态
4. 最近更新时间
5. 是否需要人工接管

### 4.3 管理页面承接

管理页面新增：

`自动联调配置页`

页面结构建议：

1. 左侧过滤区
   - 媒体
   - 终端
   - 环境
   - 状态

2. 中间列表区
   - 模板名
   - 适用范围
   - 执行器类型
   - 状态

3. 右侧详情 / 编辑区
   - 关键词配置
   - 超时配置
   - 说明文案
   - 启停开关

### 4.4 状态设计

自动联调专项状态建议：

1. `created`
2. `waiting_confirm`
3. `running_web_prepare`
4. `running_mobile_scan`
5. `running_mobile_find_ad`
6. `running_mobile_launch`
7. `running_success_poll`
8. `success`
9. `failed`
10. `manual_takeover`

### 4.5 关键交互流

#### 用户侧

1. 提交联调问题
2. 小乔识别可进入自动联调专项
3. 创建任务
4. 用户确认输入
5. 启动执行
6. 查看过程
7. 成功结束或人工接管

#### 管理侧

1. 进入自动联调配置页
2. 选择模板
3. 编辑配置
4. 发布生效
