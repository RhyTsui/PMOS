# 智投chat 设计任务与开发计划

版本日期：2026-05-15  
关联设计：`docs/智投chat-最终功能设计v1-2026-05-15.md`

## 1. 当前工作方式

后续不再把产品设计、提示词、前端样式、后端实现、MCP 对接和评测混在同一轮对话里直接改。

标准流程：

1. 需求归纳。
2. 功能设计。
3. 设计问答确认。
4. 开发计划。
5. task 拆解。
6. 实现。
7. 浏览器和连弩评测验收。
8. 经验沉淀。

## 2. 设计任务

### D1 运行时协议设计确认

目标：确认 `AgentProcessEvent` 是唯一过程真源。

待确认：

- 事件类型是否覆盖思维链、Skill、MCP、来源、确认表单、组件、评测。
- 是否所有服务都必须输出 `process_events`。
- 刷新恢复是否按事件恢复，而不是前端临时状态。
- 连弩评测是否直接消费事件链。

输出：

- 运行时协议最终版。
- 事件字段表。
- 前端渲染映射。
- 评测映射。

### D2 Agent 路由设计确认

目标：确认路由规则，避免关键词误判。

待确认：

- 宽泛异常是否先走监控巡检。
- 指标差异是否先走报表/MCP 实际值查询。
- 自动报表是否与监控任务彻底分开。
- 对接是否作为联调前置，而不是独立大模块。
- P2/P3 是否只记录不主交付。

输出：

- 路由规则表。
- 误判规避表。
- 评测用例。

### D3 P0 服务闭环设计确认

目标：打透质量保障。

范围：

- 投放前质量保障。
- 自动联调。
- 监控巡检。
- 指标差异排查。

输出：

- 每个服务的端到端流程。
- Skill contract。
- MCP 依赖。
- 前端组件。
- 验收用例。

监控巡检需要补充确认的系统观测层：

- 媒体报表指标资产与口径上下文。
- 媒体报表采集状态。
- 媒体点击日志采集状态。
- 自归因结果采集状态。
- 运维调度系统状态。
- 平台数据上报报告。
- 数据质量监控结果。

### D4 P1 服务闭环设计确认

目标：打透高频直接交付。

范围：

- 疑问解答。
- 指标解释器。
- 监测链接交付。
- 自动报表。

输出：

- 每个服务的端到端流程。
- Skill contract。
- MCP / 知识库依赖。
- 前端组件。
- 验收用例。

### D5 后台配置与管理设计确认

目标：后台成为运行时配置中心。

范围：

- Skill 管理。
- Prompt 管理。
- MCP 管理。
- 联调配置。
- 指标解释资料。
- 监控规则。
- 记忆管理。
- 连弩评测准入状态追踪。

输出：

- 后台信息架构。
- 每个页面字段。
- 查看/编辑状态。
- 持久化要求。

### D6 评测与验收设计确认

目标：把连弩评测变成上线准入。

范围：

- 意图路由。
- Skill 调用。
- MCP 调用。
- 确认表单。
- 来源披露。
- 动态组件。
- 结果闭环。

输出：

- 评测字段。
- 用例集。
- 通过标准。
- 失败原因分类。

## 3. 已确认设计决策

1. v1 只聚焦 P0 + P1，不提前做洞察、预测、决策主流程。
2. `AgentProcessEvent` 作为前端、后端、后台、评测的唯一过程真源。
3. P0 先打透监控巡检、指标差异排查、自动联调、投放前质量保障。
4. P1 先打透疑问解答、指标解释器、监测链接交付、自动报表。
5. 对接作为投放前质量保障和自动联调前置，不单独做成大模块。
6. 后台 v1 只做配置和追踪；评测由连弩负责，不先做复杂审批流。
7. 连弩评测作为 Skill 上线准入。
8. 监控巡检 7 层对象需要新增采集 MCP。
9. 自动联调 v1 先打透巨量。
10. 监测链接交付 v1 既查询已有链接，也支持创建新链接。
11. 指标解释资料以后台结构化录入为主，知识库作为补充来源。
12. 自动报表 v1 先支持文本模板和 Excel 模板，截图识别后置。
13. 采集 MCP 采用统一采集网关 + 内部分 adapter。
14. 巨量自动联调 v1 按最小后台配置打透，用户侧只指定项目、媒体和联调目标。
15. 监测链接创建 v1 采用有权限用户直接创建，不先做审批流。
16. 指标解释首批覆盖核心投放、转化、回传、BI 差异指标。
17. 自动报表 v1 支持文本模板和标准二维 Excel 模板，复杂 Excel 和截图识别后置。

## 4. 开发计划

### Phase 1：运行时协议闭环

目标：统一思维链、Skill、MCP、来源、确认表单、评测事件。

任务：

- T1.1 核查当前 `AgentProcessEvent` 类型。
- T1.2 补齐缺失事件：`capability.checked`、`skill.failed`、`ui.component_rendered` 等。
- T1.3 后端所有回复统一输出 `process_event`。
- T1.4 前端按事件渲染思维链和调用来源。
- T1.5 刷新后从事件恢复状态。
- T1.6 连弩评测接口返回同一事件链。

验收：

- 页面、刷新后页面、评测接口三处事件一致。
- 思维链和来源不会在结果输出后消失。

### Phase 2：Skill Contract 与后台配置

目标：所有核心服务有可管理、可评测、可调用的 Skill contract。

任务：

- T2.1 建立 Skill contract 存储。
- T2.2 后台 Skill 管理支持查看、编辑、保存、启停。
- T2.3 Prompt 管理与 Skill 绑定。
- T2.4 MCP 管理展示工具 schema、权限、调用日志。
- T2.5 联调配置支持媒体账号、事件资产、回传位置、设备、游戏账号。

验收：

- Skill 状态刷新后不丢。
- 配置能被 Agent runtime 读取。
- 编辑默认查看态，避免误操作。

### Phase 3：P0 闭环实现

目标：质量保障能力可用。

任务：

- T3.1 实现 `ads_health_monitor_skill`。
- T3.2 实现 `metric_diff_diagnosis_skill`。
- T3.3 实现 `auto_debug_skill`。
- T3.4 实现 `preflight_quality_check_skill`。
- T3.5 打通真实 MCP，不可用时显式返回缺口。
- T3.6 完成巡检卡、排查卡、联调步骤、质量结论组件。
- T3.7 为监控巡检补齐 7 层数据链路观测：媒体报表指标资产、媒体报表采集、媒体点击日志、自归因结果、运维调度、平台数据上报报告、数据质量监控。
- T3.8 新增统一采集网关 MCP，用于承载媒体报表、点击日志、自归因、调度、上报报告、数据质量等链路观测；内部按数据源分 adapter。
- T3.9 巨量自动联调按最小后台配置打透：媒体账号、事件资产、回传位置、应用共享账号、渠道包、抖音端、关键词、游戏包、游戏登录、设备。

验收：

- “昨天投放有什么异常”能先巡检。
- “激活数和 BI 不一致”能先取系统实际值。
- “巨量立即联调”能走自动联调组件。
- “这个包能不能投”能给可投放结论。
- 未接入的监控层明确标记为未接入或不可用，不使用 mock 巡检结果。
- 自动联调 v1 以巨量为首个完整验收媒体。
- 用户发起巨量联调时只需项目、媒体和联调目标。

### Phase 4：P1 闭环实现

目标：高频服务直接交付。

任务：

- T4.1 实现 `knowledge_answer_skill`。
- T4.2 实现 `metric_explainer_skill`。
- T4.3 实现 `tracking_link_delivery_skill`。
- T4.4 实现 `report_template_builder_skill`。
- T4.5 实现 `scheduled_report_skill`。
- T4.6 完成指标解释器、监测链接卡、报表模板、数据预览组件。
- T4.7 监测链接交付支持查询已有链接和创建新链接。
- T4.8 指标解释资料后台结构化录入，知识库仅作为补充。
- T4.9 自动报表先支持文本模板和 Excel 模板，截图识别后置。
- T4.10 监测链接创建做权限校验、创建前预检、参数确认、创建日志、来源记录和失败原因。
- T4.11 指标解释首批录入消耗、展示、点击、激活、注册、付费、ROI、自然量、媒体回传和 BI 差异指标。
- T4.12 自动报表 Excel 模板 v1 只支持标准二维表，复杂合并单元格、多区域透视表、跨 sheet 公式后置。

验收：

- 指标解释结构化展示。
- 监测链接直接交付。
- 自动报表从模板确认到定时任务闭环。
- 缺少指标解释结构化资料时明确提示资料缺失，不用知识库泛化内容伪装完整口径。
- 无创建权限时，监测链接交付明确提示无权限，不做假创建。

### Phase 5：评测准入

目标：每个核心 Skill 上线前必须通过连弩评测。

任务：

- T5.1 建立意图路由评测集。
- T5.2 建立 P0 Skill 评测集。
- T5.3 建立 P1 Skill 评测集。
- T5.4 连弩评测结果返回事件链、来源、失败原因。
- T5.5 发布前运行评测。
- T5.6 后台只展示连弩准入状态、最近结果和失败原因链接。

验收：

- 每个核心 Skill 至少 5 条评测用例。
- 失败能定位到路由、表单、MCP、来源、组件或结果环节。

## 6. 当前不做

v1 暂不做：

- 自动投放。
- 复杂审批流。
- 创意洞察。
- 预测模型。
- 策略决策自动执行。
- 多媒体统一接入全量打透。

这些能力只保留接口和路线，不进入当前开发验收。
