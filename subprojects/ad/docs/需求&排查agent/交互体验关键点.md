数据排查类 Agent 交互与体验关键点
（精炼、可直接写进 PRD / 评审用）
一、入口与发起体验
支持两种发起方式，自由切换
自然对话：直接说 “为什么转化掉了”“帮我查下消耗”
快捷菜单：指标、时间、计划一键点选
做到：老手用对话、新手用菜单，互不干扰
入口简洁，不堆砌功能
只保留：输入框 + 常用指标快捷按钮
不做复杂面板、多级菜单，降低使用门槛
支持上下文追问
业务说 “再查昨天的”“换另一个计划”
能继承上一轮条件，不用重新说一遍
二、提问引导与信息收口
模糊提问必须 “结构化兜底追问”
业务说不清楚 → AI 只补问缺失项，不乱发散
必问 4 件事：
查什么指标（曝光 / 转化 / 消耗 / 成本…）
时间范围
哪个计划 / 账户 / 版位
异常表现（突增 / 突降 / 没量 / 对不上）
一次只问一个问题，不连环轰炸
避免一次抛 5 个问题让业务不知所措
信息自动补全与联想
输入计划号、账户名时自动联想匹配
时间支持口语化识别：今天、昨天、近 7 天、这周三
三、结果呈现体验
先给一句话结论，再给细节
业务第一眼看到：
转化下降 27%，主要原因为头部素材 CTR 下滑
而不是先甩一堆数据表
数据可视化优先于纯文本
波动趋势、维度占比用简单图表展示
表格只放关键维度，不堆砌全量数据
异常必须高亮标注
波动幅度、异常维度、贡献度突出显示
让业务不用自己找问题在哪
结论必须带 “可执行建议”
不说 “请观察”“建议优化”
要说：
建议替换素材
建议小幅提价恢复竞争力
建议检查时段投放是否被关停
四、对话节奏与表达体验
回复长度克制，不写小作文
关键信息 3–5 行讲完，细节可展开查看
统一专业口吻，不口语化、不卖萌
数据类场景信任第一，语气专业、稳定、一致
拒绝甩锅式表达
禁用：
数据正常
属于正常波动
请以后台为准
必须给出：波动值 + 对比 + 可能原因
错误与兜底要 “安抚 + 明确预期”
不说 “我查不了”“系统出错”
统一表达：
这个问题涉及较复杂的数据逻辑，我已整理完整背景提交产品分析，有结果第一时间同步你
五、闭环与后续体验
每次排查结束自动输出标准小结
包含：对象、时间、结论、原因、建议
方便业务转发、留存、同步
支持一键重新查询 / 调整条件
不用重新打字，可直接改时间、改指标再查
历史对话可回溯、可继续
昨天查的问题今天接着问，能定位到原记录
不强制登录、不打断流程
业务进来就能用，减少验证、跳转、等待
六、底线体验（必须卡死）
绝不出现幻觉式解释
不确定原因 → 走兜底，不乱编原因
同一问题多次回答逻辑一致
同样条件今天和明天结论必须一样
加载状态明确，不僵死不空白
查询中给明确状态提示，避免以为卡住
