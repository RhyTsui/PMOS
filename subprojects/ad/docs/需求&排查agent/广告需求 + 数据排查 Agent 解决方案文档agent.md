广告需求 + 数据排查智能 Agent 完整解决方案文档
一、项目整体概述
1.1 项目背景
针对广告业务侧（发行、投放、运营同学）需求沟通效率低、数据排查等待周期长、人工对接成本高的核心痛点，结合已搭建的知识库、标准需求工作流程、产品 Agent 基础能力，打造一体化智能 Agent，实现需求自助处理、数据自动排查、结果即时反馈，异常场景不转接人工，自动生成 Case 通知单推送产品端，同时依托成功 Case 实现系统自进化。
1.2 核心定位
以自然对话范式为主、固定菜单范式为辅，覆盖广告需求收集沟通、数据排查分析双核心场景，兼具灵活性与稳定性，替代 80% 人工常规对接工作，打造「即时闭环、柔性沟通、自动归档、自进化迭代」的智能需求处理工具。
1.3 核心原则
常规需求即时结论型反馈，拒绝仅记录不处理
否定答复柔性表达 + 替代方案，杜绝生硬沟通
人工兜底不转人工对话，仅自动生成 Case 推产品
成功 Case 自动沉淀，实现系统自主进化
双场景架构复用，降低落地成本与开发难度
二、核心需求梳理
2.1 广告需求收集沟通需求
引导式收集投放、素材、审核、排期、预算等结构化需求信息，必填项无遗漏
依据知识库规则，实时校验需求可行性，3 秒内输出能否做、排期、风险提示
自动生成标准化《广告需求确认单》，业务一键确认归档
超预算、非标、政策敏感等异常需求，自动生成完整需求 Case 通知单推送产品
统一柔性话术，保障业务沟通体验
2.2 数据排查需求
支持自然对话提问，也可通过固定菜单快速选择排查指标、时间、计划等维度
自动拉取数据、多维度下钻分析（版位 / 素材 / 人群 / 时段 / 大盘），直接输出排查结果、波动原因、优化建议
数据口径异常、复杂无法归因、跨模块数据问题，自动生成排查 Case 通知单推送产品
排查结果精准无歧义，统一数据口径，避免错误解读
2.3 通用兜底与自进化需求
明确兜底触发边界，异常场景自动识别，不中断业务对话
成功 Case（无异议、正常闭环、结果准确）自动沉淀为规则库、样本库
产品处理后的兜底 Case，回填系统转化为新规则，实现系统越用越聪明
所有需求、对话、Case 结构化归档，可检索、可复盘
三、核心决策结论
3.1 交互范式决策
采用「自然对话为主 + 固定菜单为辅」的混合范式，为数据排查与广告需求场景最优解
主范式：自然对话
适配业务模糊提问、灵活诉求，体验贴近真人对接
支持业务直接表述需求，无需繁琐操作，提升使用意愿
辅范式：固定菜单
提供常用指标、时间、维度快捷按钮，新手可快速上手
规避自然语言理解偏差，保障核心场景稳定性
决策原因：兼顾业务灵活使用需求与系统准确性，新手靠菜单、老手靠对话，80% 常规场景高效处理，20% 复杂场景兜底可控。
3.2 人工兜底决策
绝对不实时转接人工，仅执行「异常识别→自动打包 Case→推送产品→业务告知等待结果」流程，全程由 Agent 承接交互，不中断业务体验，产品集中处理异常需求，提升双方效率。
3.3 系统能力决策
AI 仅做自然语言理解、字段抽取、话术输出，核心决策（可行性、排期、排查逻辑）由规则引擎执行，杜绝 AI 自主决策失误
数据排查具备自动下钻、同比 / 环比计算、归因分析能力，不做表面数据查询
搭建自进化模块，成功 Case 与产品回填 Case 自动转化为系统规则，无需人工频繁迭代开发
3.4 落地路径决策
先搭建 MVP 最小版本，覆盖高频 30% 场景，小范围试用后收集数据优化规则与话术，再逐步开放全场景，降低上线风险，快速验证效果。
四、整体解决方案
4.1 业务需求框架
4.1.1 广告需求模块框架
基础信息层：需求人、部门、提交时间、联系方式
需求类型层：投放、素材、审核、排期、策略、非标需求
业务核心层：预算、版位、时段、素材要求、KPI、截止时间
处理规则层：合规红线、预算门槛、排期分级、可行性判定规则
输出层：需求确认单、可行性结论、排期信息、柔性话术
兜底层：非标 / 超预算 / 政策敏感需求→Case 通知单→推送产品
4.1.2 数据排查模块框架
排查条件层：指标（曝光 / 转化 / 消耗等）、时间范围、计划 ID、版位、操作记录
排查执行层：数据拉取、波动计算、多维度下钻、归因分析
输出层：排查结论、波动数值、原因分析、优化建议、排查报告
兜底层：复杂 / 口径异常 / 无法归因→Case 通知单→推送产品
4.2 系统能力框架（四层架构）
第一层：交互层
对话交互：自然语言问答、引导式提问、柔性话术应答
菜单交互：固定快捷按钮（指标、时间、维度），一键生成查询条件
信息校验：必填项缺失提醒、信息格式自动格式化
第二层：核心引擎层
意图识别引擎：解析自然语言，映射为结构化需求 / 排查参数
规则决策引擎：执行可行性判定、排期计算、合规校验、异常识别
数据排查引擎：自动取数、维度下钻、同比环比分析、波动归因
第三层：执行与输出层
单据生成：自动生成广告需求确认单、数据排查报告、异常 Case 通知单
推送执行：Case 通知单实时推送产品指定端口（群聊 / 系统 / 邮箱）
归档存储：所有需求、对话、结果、Case 结构化存档
第四层：自进化层
成功 Case 沉淀：提取规则、话术、排查逻辑，自动入库
兜底 Case 回填：产品处理结果同步系统，转化为新规则
规则库迭代：定期更新优化，减少兜底触发率
4.3 双场景标准化执行流程
4.3.1 广告需求处理流程
业务发起需求（对话 / 菜单）
Agent 引导收集完整结构化信息，实时校验
规则引擎判定可行性
可执行：输出结论 + 排期→生成确认单→业务确认归档
不可执行：柔性告知 + 替代方案
异常需求：自动生成 Case→推送产品→告知业务等待结果
4.3.2 数据排查处理流程
业务发起排查（对话 / 菜单）
Agent 确认排查条件，补全必要信息
数据引擎自动执行排查、归因分析
输出排查结果 + 建议→生成排查报告
复杂异常：自动生成排查 Case→推送产品→告知业务等待结果
4.4 异常兜底与 Case 通知单规范
4.4.1 兜底触发范围
广告需求：超预算阈值、非标定制、政策敏感、规则无匹配
数据排查：数据口径异常、复杂无法归因、跨模块数据、结果存疑
4.4.2 异常 Case 通知单标准内容
plaintext
【Agent自动归档】异常需求/数据排查Case通知单
1. 基础信息：提交人、时间、场景类型、核心诉求
2. 详细内容：需求/排查完整信息、Agent处理过程
3. 异常原因：无法自动处理的核心问题
4. 对话纪要：业务与Agent交互核心内容
5. 推送对象：产品对应负责人
4.5 成功 Case 自进化机制
成功 Case 定义：需求正常闭环、业务无异议、排查结果准确、未触发兜底
自动沉淀流程
提取成功 Case 的需求参数、决策规则、排查逻辑、优质话术
存入规则库、样本库、话术库
后续同类需求直接复用，无需产品干预
兜底 Case 转化：产品处理完成后，回填处理结果与规则，系统自动学习，转化为可执行规则
五、核心成功关键点
需求结构化：所有业务需求必须转化为结构化字段，杜绝无边界闲聊，保障后续可执行、可归档
规则前置：核心决策由规则引擎执行，AI 仅做理解与交互，避免决策失误
即时闭环：常规需求 3 秒内输出确定性结果，拒绝拖延与模糊答复
柔性沟通：所有否定性回复必附替代方案，保障业务体验
兜底边界清晰：明确异常场景判定标准，不滥触发、不遗漏，Case 自动推送产品
混合范式互补：对话保体验，菜单保稳定，兼顾灵活与可控
自进化闭环：成功 Case 与回填 Case 持续迭代规则，减少人工介入，系统持续优化
收口标准化：每一次交互必输出对应单据，实现流程闭环与信息沉淀

七、落地执行步骤
阶段一：业务梳理：梳理双场景高频需求、规则、兜底边界，准备训练 & 测试 Case
阶段二：MVP 开发：搭建四层系统架构，优先实现高频场景，完成话术库、规则库搭建
阶段三：小范围试用：定向业务侧试用，收集对话数据，优化规则与话术
阶段四：全量上线：开放全场景使用，完善自进化机制
阶段五：持续迭代：基于成功 Case 与产品回填，定期更新规则库，降低兜底率
