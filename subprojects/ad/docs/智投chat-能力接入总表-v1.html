目标是把“解答、排查、联调、监控、监测链接、自动报表”从聊天回复升级为端到端直接交付。此页用于确认还需要哪些 Agent、Skill、MCP、Workflow 和知识资料，方便团队按模块补齐。

当前结论

已有报表 MCP、归因 MCP、运维调度 MCP、配置管理 MCP 的部分能力。当前最大缺口不是前端组件，而是“可被 Agent 稳定调用的真实 MCP 契约”和“Skill 级流程验收集”。下一步应优先把 P0 质量保障链路打通，再做 P1 提效能力。

五条 Workflow

1

投放前质量保障

新包或配置变化 → 获取包和媒体状态 → 自动联调 → 可投放结论。
2

投放异常巡检

项目/媒体/时间 → 采集、质量、调度、报表检查 → 异常转排查。
3

指标差异排查

查两边数据 → 查口径 → 查采集/归因/回传/入库 → 输出根因。
4

监测链接交付

查询已有链接 → 无链接则预检 → 创建 → 返回可用链接。
5

自动报表

模板确认 → 指标校验 → 拉数预览 → 创建定时任务。

下一步顺序

  1. 先补齐采集状态 MCP 和数据质量 MCP。
  2. 把巨量自动联调 Demo 封装成标准 MCP。
  3. 把监测链接查询/创建/校验映射到现有配置 MCP 工具。
  4. 录入指标解释结构化资料。
  5. 为每个 Skill 建连弩准入评测。

验收标准

  • 每个 Skill 至少 5 条真实业务 case。
  • 每条 case 都能看到 AgentProcessEvent。
  • 每次引用数据都能看到来源。
  • MCP 不可用时不生成假结论。
  • 用户得到的是结果、链接、任务或明确阻塞项。
