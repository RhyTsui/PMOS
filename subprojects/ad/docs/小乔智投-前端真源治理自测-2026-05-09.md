## 0. 自测目标

确认 `ad` 当前有效真源里，哪些前端约束此前没有真正生效，哪些已经通过本轮治理补齐。

---

## 1. 自测口径

本轮只检查会直接影响前端输出的有效文档，不检查：

1. `废弃或不用`
2. 乱码旧文档
3. 单次补丁说明

本轮首页治理只看一件事：

`首页是否被真源约束为“围绕当前任务推进”，而不是“能力平铺展示页”。`

---

## 2. 发现的问题

### 2.1 没有真正生效的约束

1. `统一入口` 被误读成首页平铺所有能力
2. `单对话工作台` 被误读成单屏收纳所有模块
3. `结果区` 被误用成说明文案容器
4. `历史任务区` 被误用成首页常驻主模块
5. `持续执行验证` 被误抬到业务首页主区
6. `rollout / 会话支撑 / 压测方案` 越界影响了首页信息架构

### 2.2 根因判断

1. 主真源虽然禁止文档页，但没有把首页层级关系写死
2. 补充文档对运行治理与会话支撑写得更细，容易反向压过页面主真源
3. 实现时把“系统能力都接上”误当成“首页更像系统”

## 4. 现行生效规则

1. 首页只有一个主操作焦点：发起并推进当前任务
2. `结果区` 是当前任务检查器，不是说明板
3. `历史任务区` 默认二级承接，不常驻平铺
4. `持续执行验证` 默认二级承接，不属于业务首页主区
5. `rollout / 会话支撑 / 压测方案 / 分层方案` 不直接定义首页信息架构
6. 任何把会话、任务、结果、运行验证、治理信息同层平铺的实现，都视为违背真源
