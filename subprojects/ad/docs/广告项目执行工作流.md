# 广告项目执行工作流

- 版本：v0.1
- 日期：2026-04-20
- 状态：Active
- 适用范围：`subprojects/ad`

---

## 一、目的

本文件是广告项目当前唯一执行顺序。

它的作用不是重写已有业务规划，而是把广告项目从“多份文档并行漂移”收敛成统一工作流，确保：

1. `月底 Demo` 与 `长期 PRD/蓝图` 不再混跑
2. 项目产品经理可以并行推进，但必须按同一提交契约工作
3. 主管只做收敛和评审，不替代首稿执行

---

## 二、当前项目判断

广告项目当前处于：

`阶段 3：产品方案收敛 + Demo First 执行`

原因：

1. 已有较完整的长期产品蓝图和 PRD
2. 已有明确的 `2026-04-30` 月底 Demo 收敛方向
3. 当前主要问题不是“从零研究领域”，而是“把长期构想收敛成月底可验证的最小闭环”

因此当前允许：

1. 围绕月底 Demo 做范围收敛
2. 输出 Demo 对应任务拆解
3. 输出页面/接口/Mock 级执行定义

当前不允许：

1. 回到“大而全 Ad OS”继续无限扩写
2. 把后续 MVP 或平台化能力塞进月底 Demo 范围
3. 用零散文档推进而不经过统一评审

---

## 三、当前阶段的唯一目标

在 `2026-04-30` 前完成一套可演示、可讲故事、可验证方向的广告 Agent Demo，并形成 Demo 后续的 MVP 任务输入。

当前阶段只验证三件事：

1. 用户是否接受“对话式广告问题处理工作台”
2. 团队是否能在 Mock 数据下跑通最小业务闭环
3. 后续 MVP 应优先投在哪些能力，而不是继续做无效铺量

---

## 四、当前阶段的收敛范围

### 4.1 月底必须做

月底 Demo 只保留 3 个演示模块：

1. `诊断 Agent Demo`
2. `对话式分析 Demo`
3. `统一工作台入口`

### 4.2 月底可选做

仅当主闭环提前完成时，才允许补：

1. 静态联调流程演示
2. 更完整的步骤日志或动画
3. 更细的 drill-down 视图

### 4.3 月底明确不做

以下内容不进入 `2026-04-30` Demo 范围：

1. 真实媒体 API 接入
2. 真正自动化联调执行
3. 素材 Agent 全链路
4. 知识库/规则变更监控系统化建设
5. 大而全控制台、权限、审计、生产化治理

## 五、执行阶段顺序

### 阶段 1：问题与目标确认

输出：

1. 月底 Demo 目标说明
2. 成功标准
3. 范围内 / 范围外清单

当前已基本存在于：

- `月底Demo重规划-2026-04-20.md`

### 阶段 2：Demo 方案收敛

输出：

1. 演示故事线
2. 统一工作台信息架构
3. 两个核心场景的用户流
4. 结果卡片/执行状态/分析表格结构

### 阶段 3：执行任务拆解

输出：

1. 产品任务拆解
2. 前端任务拆解
3. 后端 Mock API 任务拆解
4. 数据与内容任务拆解
5. 验收清单

### 阶段 4：Demo 验证

输出：

1. 演示检查清单
2. 遗留问题清单
3. 月底 Demo 结论
4. 后续 MVP 需求池输入

## 六、角色分工

### 真实产品主管

负责：

1. 判断月底 Demo 是否继续收敛
2. 判断哪些内容必须砍掉
3. 对跨项目、跨平台边界问题拍板

### 虚拟产品主管 Agent

负责：

1. 判断当前阶段
2. 分发任务
3. 审核产品经理提交
4. 标记 accepted / rejected / blocked

### 广告项目产品经理 Agent

负责：

1. 输出 Demo 执行相关首稿
2. 整理现有 PRD、蓝图、Demo 重规划文档
3. 主动同步阶段性产出
4. 不擅自扩大范围

---

## 七、提交规则

当前阶段允许提交的文档类型：

1. `demo-scope-note`
2. `demo-flow-note`
3. `demo-task-breakdown`
4. `demo-acceptance-checklist`
5. `mvp-input-note`

每次提交必须包含：

1. 核心结论
2. 依据来源
3. 当前范围
4. 不确定项
5. 请求上级决策的问题

---

## 八、主动同步规则

广告项目产品经理 Agent 必须主动同步：

1. 接到任务后，先同步这轮要交哪些文档
2. 每完成一份首稿，立刻同步一次
3. 如果 30 分钟内没有形成首稿，也必须同步当前卡点

同步内容至少包含：

1. 已产出什么
2. 还缺什么
3. 当前卡点是什么
4. 下一步要谁配合

未同步的工作不视为正式提交。

---

## 九、主管评审结论

每次评审只允许以下结果：

1. `accepted`
2. `accepted-with-constraints`
3. `needs-more-clarity`
4. `blocked-by-scope`
5. `rejected`

广告项目当前阶段最常见的打回原因：

1. 范围又变大
2. 把长期能力混进月底 Demo
3. 只给方向，不给任务拆解
4. 没有主动同步

---

## 十、当前唯一优先交付

广告项目产品经理当前优先交付顺序是：

1. `月底 Demo 执行任务拆解`
