# 小乔智投-v0.7-rollout-to-aiocoding-2026-05-09

- 版本：v0.1
- 状态：active
- 日期：2026-05-09
- 目标：把平台 `v0.7` 已优化的交付链、评审链、Hermes 治理链先落到 `ad` 子项目，作为 `aiocoding` 前的实战测试包

---

## 0. rollout 目标

这次 rollout 不是抽象同步，而是为 `ad` 子项目提供一套可以直接进入开发的真源组合：

1. 新交付链已经落到子项目
2. 多角色评审已经落到子项目
3. 前端页面口径已经从示意页切到交付页
4. 开发 task 已经结合系统现状拆开
5. 旧口径不再继续作为默认真源

---

## 1. 当前已同步到 `ad` 的平台能力

### 1.1 新交付链

已同步：

`调研文档 -> 规划文档 -> 需求文档 -> 功能文档 -> 设计文档 -> 前端页面 -> 数据表 -> 后端接口 -> 联调与验收`

### 1.2 新评审链

已同步：

`self-check -> multi-role review committee -> Hermes governance -> human final approval`

### 1.3 前端反漂移约束

已同步：

1. 禁止 `hero` 首屏
2. 禁止 `summary-card` 工作台
3. 禁止 `explanation-first`
4. 禁止示意页和文档页冒充前端页面

### 1.4 开发评审要求

已同步：

1. 必须结合系统现状
2. 必须拆到开发 task
3. 必须指出风险和回归面

## 3. 当前不建议再作为默认真源的内容

1. 旧 `前端简单版 / UIUX版` 口径
2. 乱码文档
3. 单次补丁说明文档
4. 旧 PoC Android workbench 说明
5. 旧静态 HTML/SVG 示例输出

这些内容可以保留，但默认视为：

- `superseded`
- `archived`

---

## 4. 当前 `ad` 子项目的验证目标

这次 rollout 主要验证 4 件事：

1. 平台新交付链能否在真实子项目里稳定使用
2. 多角色评审是否能真正约束页面、contract 和 task
3. Hermes 是否能为子项目提供可执行的治理闭环
4. 前端页面是否已经能从“模型默认壳”切到“交付级用户页面”
