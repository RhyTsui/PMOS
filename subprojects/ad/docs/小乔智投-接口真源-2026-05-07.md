## 3. 接口清单

### 3.1 创建会话

`POST /api/v1/xiaoqiao/conversations`

请求：

```json
{
  "user_id": "user-001",
  "title": "可选标题"
}
```

返回：

`Conversation`

### 3.2 获取会话列表

`GET /api/v1/xiaoqiao/conversations`

返回：

会话摘要列表

### 3.3 获取单个会话

`GET /api/v1/xiaoqiao/conversations/{conversation_id}`

返回：

会话基础信息、最近任务摘要和最后状态

### 3.4 发送消息

`POST /api/v1/xiaoqiao/conversations/{conversation_id}/messages`

请求：

```json
{
  "content": "为什么这个计划激活比 BI 少了 28%",
  "attachments": []
}
```

返回：

```json
{
  "message": {},
  "routing": {},
  "task": null,
  "assistant_reply": {}
}
```

说明：

这条接口是第一阶段最关键入口，负责：

1. 收用户输入
2. 生成消息
3. 生成路由判断
4. 必要时创建任务
5. 返回首轮助手反馈

### 3.5 查看路由判断

`GET /api/v1/xiaoqiao/routing/{routing_id}`

返回：

1. 是否业务相关
2. 当前业务域
3. 当前诉求类型
4. 当前工作流层级
5. 是否需要追问
6. 判定原因

### 3.6 获取任务列表

`GET /api/v1/xiaoqiao/tasks`

支持过滤：

1. `task_type`
2. `workflow_level`
3. `status`
4. `conversation_id`

### 3.7 获取单个任务

`GET /api/v1/xiaoqiao/tasks/{task_id}`

返回：

1. 任务基础信息
2. 上下文
3. 结果摘要
4. 证据摘要

### 3.8 更新任务状态

`PATCH /api/v1/xiaoqiao/tasks/{task_id}`

请求：

```json
{
  "status": "running"
}
```

### 3.9 获取任务结果

`GET /api/v1/xiaoqiao/tasks/{task_id}/results`

返回：

一个或多个 `WorkflowResult`

### 3.10 写入任务结果

`POST /api/v1/xiaoqiao/tasks/{task_id}/results`

请求：

```json
{
  "result_type": "diagnosis_report",
  "summary": "缺口主要集中在首登回传链路",
  "structured_payload": {},
  "confidence": "high",
  "next_action": "继续核对 SDK 回传与媒体映射"
}
```

### 3.11 获取 Case 列表

`GET /api/v1/xiaoqiao/cases`

### 3.12 从任务生成 Case

`POST /api/v1/xiaoqiao/tasks/{task_id}/case`

### 3.13 获取工作台首页视图

`GET /api/v1/xiaoqiao/workspace`

返回：

1. 顶部状态摘要
2. 快捷任务模式
3. 最近任务
4. 应用支持状态摘要
