
# 小乔智投Agent - 产品需求文档 (PRD) v1.1.0

版本：v1.1.0  
更新日期：2026-04-15

---

## 0. 概要
在 v1.0.0 基础上，本版本补充用户画像（Persona）、关键指标（KPI）、详细里程碑计划、各模块验收标准与主要风险及缓解方案，目标把概念化 PRD 推向可交付、可执行的产研计划。

---

## 1. 目标回顾
产品愿景与核心目标沿用 v1.0.0：赋能优化师、提升产研效率、AI-First 体验。本文档目标是将高层愿景拆解为可量化交付项与风险控制点，便于实施与验收。

---

## 2. 目标用户画像（Persona）
1. Persona A — 初级优化师（小李）
   - 角色/资历：投放执行，1–2 年经验
   - 目标：快速获得投放报表、定位简单问题（上报漏数据/回传失败）
   - 使用频率：每日多次（常查日报、异常告警）
   - 权限：查看报表、提交对接需求、不能触发自动回推/暂停
   - 痛点：看不懂复杂归因、联调耗时、对接沟通成本高

2. Persona B — 中级优化师/策略师（小王）
   - 角色/资历：3–6 年，负责策略调整与预算配置
   - 目标：获取多维洞察、回本预测、策略建议并执行
   - 使用频率：每日/每周
   - 权限：查看全部报表、执行策略建议需审批；可申请手动暂停广告
   - 痛点：跨媒体数据不一致、预测不确定性、缺少一键回放工具

3. Persona C — 资深/运营负责人（小陈）
   - 角色/资历：6+ 年，决策层，关注 ROI 与风险
   - 目标：衡量投放总体回报、审批自动优化策略、审计操作记录
   - 使用频率：每日/定期报告审阅
   - 权限：全权限（审批自动化策略、历史审计查看）
   - 痛点：自动化误判带来成本、合规与审计需求

---

## 3. 关键成功指标（KPI，样例指标与目标）
1. 平均故障定位时间（MTTR） ≤ 1 小时（从告警到定位具体环节）
2. 对接需求从提交到完成平均周期 ≤ 5 个工作日
3. 异常检测召回率 ≥ 90%，精确率 ≥ 85%
4. 自动化策略误判率（误暂停/误回推） ≤ 2%
5. 自动化执行覆盖率：可自动执行的场景中，达到 60% 自动执行率（含人工审核/免审路径）
6. 系统可用率 SLA ≥ 99.9%
7. 平均查询响应时延（交互式问答关键请求）P50 ≤ 1s，P95 ≤ 3s
8. 优化师满意度（内部 NPS）≥ 40 分（上线后 3 个月评估）
9. 数据延迟：实时链路数据延迟 P95 ≤ 2 分钟，回推延迟 P95 ≤ 15 分钟（目标）

## 4. 里程碑与时间线（建议）
起始日期假设：2026-04-20。以下为建议迭代计划与可交付成果。

- M0：准备与设计（Week 0–2，2026-04-20 至 2026-05-03）
  - 成果：详细需求说明、用户故事、数据源清单、合规与权限方案、API 列表草案

- M1：问答 Agent + 对接需求池（Week 3–8，2026-05-04 至 2026-06-14）
  - 成果：全屏对话流框架、知识库接入（巨量/腾讯 文档至少各一套）、指标解释模块、对接需求自动标准化与需求池通知流程、基础权限控制
  - 验收重点：问答准确率、对接需求字段满足率、UI/交互通过可用性测试

- M2：监控 Agent + 链路观测（Week 9–18，2026-06-15 至 2026-08-23）
  - 成果：链路拓扑可视化、实时指标面板、延迟/异常告警规则、自动比对（媒体 vs 平台）报表、AI 初步告警建议
  - 验收重点：告警召回率、延迟监控覆盖率、降级与告警传递机制

- M3：诊断 Agent（Week 19–26，2026-08-24 至 2026-10-18）
  - 成果：自动联调流程（白名单预设、一次联调通过日志展示）、指标诊断与日志回溯、异常排查工具
  - 验收重点：自动联调成功率、诊断定位准确率、日志链路回放能力

- M4：分析 Agent + 策略 Agent（Week 27–40，2026-10-19 至 2027-01-31）
  - 成果：聚合报表、多维分析模板、回本预测模型（首日 ROI、LTV 预测）、自动回推策略、自动暂停/预算出价模块（带金丝雀和审批流）
  - 验收重点：预测误差范围、自动策略误判率、回本提升率（A/B 测试）

- M5：生产化 & 合规（并行）与移动驾驶舱 MVP（Week 41–52）
  - 成果：审计日志、RBAC 完整实现、合规审查通过、移动端简版监控与语音交互 MVP

## 5. 各 Agent 验收标准（Acceptance Criteria）
注：下面按 Agent 列出最低可交付（MVP）验收项与量化阈值。

1. 问答 Agent（知识问答、指标解释、需求收集）
   - 功能：
     - 支持自然语言提问，返回知识库或指标解释（文本 + 可选链接）
     - 能引导并收集对接需求（必填字段校验）
   - 性能与质量：
     - 问答准确率（人工抽样评估）≥ 85%
     - 指标计算示例正确率 100%（关键原子指标）
   - 交互：
     - 对接需求标准化后能自动写入需求池并向产研发送通知（邮件/系统消息）
   - 安全：
     - 对接敏感字段（Key/Secret）不在明文日志内显示，采用遮罩

2. 监控 Agent（链路观测、风险熔断、数据比对）
   - 功能：
     - 实时获取链路关键指标（上报延迟、回推延迟、采集延迟、异常流量）
     - 自动生成媒体 vs 平台日/小时对比差异报表
     - 支持告警阈值与 AI 建议阈值
   - 性能：
     - 数据延迟 P95 ≤ 2 分钟（实时链路）
     - 告警触达成功率 ≥ 99%
   - 可用性：
     - 控制台展示负载与告警状态，支持 drill-down 至日志

3. 诊断 Agent（自动联调、指标诊断、异常排查）
   - 功能：
     - 自动联调流程能在 1 次触发中完成必要步骤并返回联调日志
     - 针对复合指标能拆解并追溯原始事件/明细到小时级或行级（依据数据可用性）
   - 质量：
     - 自动联调一次成功率 ≥ 80%（白名单账号）
     - 指标诊断定位准确率 ≥ 85%（已标注问题样本集）

4. 分析 Agent（聚合取数、多维分析、回本预测）
   - 功能：
     - 生成广告标签汇总报表，支持按媒体/渠道/广告位/设备等维度切分
     - 支持首日 ROI、LTV/ROI 倍率预测，并能给出不确定性区间
   - 性能/质量：
     - 预测模型 MAPE（平均绝对百分比误差）在可接受范围（首日 ROI MAPE ≤ 15%；LTV 视数据量设阈）
     - 报表查询 P95 响应 ≤ 5s（交互型）

5. 策略 Agent（智能回推、自动暂停、预算出价）
   - 功能：
     - 根据模型输出生成策略建议，支持模拟回放与金丝雀下发
     - 自动暂停规则支持审批链与强制人工复核模式
   - 安全/风险：
     - 默认首次部署为“建议/待审批模式”（不执行自动操作），经过小范围金丝雀与回滚验证后逐步放开自动执行
     - 自动执行误判率 ≤ 2%（上线后 30 天内监控）

## 6. 权限、合规与审计（要点）
- RBAC：定义 Viewer / Operator / Approver / Admin 四级权限
  - Viewer：只能查看报表与监控
  - Operator：可发起联调、执行非破坏性操作
  - Approver：审批自动化策略/暂停
  - Admin：管理账号、配置与审计权限
- 审计日志：所有自动/手动操作、策略变更与审批链必须保留审计记录（包含操作者、时间、变更前后内容），日志保存 1 年（或依合规要求）
- 敏感数据处理：所有 Key/Secret、用户凭证需加密存储并屏蔽展示，日志中做遮罩
- 合规检查：遵循所在地域相关隐私法规（数据最小化、加密、访问审计）

---

## 7. 数据与技术依赖（最小清单）
- 媒体接入：巨量、腾讯、快手（优先）— 每家需明确 API 文档、鉴权方式、限流策略
- 数据存储：实时流计算（Kafka / Flink）+ OLAP（ClickHouse / Doris）+ 元数据仓库
- 日志与追溯：集中化日志系统（Elasticsearch / Loki）
- 模型服务：在线预测 API（低延迟）、离线训练平台（数据科学平台）
- 运维：告警（PagerDuty/企业微信/邮件）、SRE SLA 支持
- 安全：密钥管理（KMS）、传输加密（TLS）

## 8. 风险与缓解（Risks & Mitigations）
1. 风险：媒体 API 变更/限流导致数据丢失或延迟
   - 缓解：适配层抽象、重试与退避机制、降级策略（缓存/历史回填）、监控指标与自动告警

2. 风险：自动化策略误判造成经济损失（误暂停/误降低出价）
   - 缓解：默认人工审批/金丝雀发布、自动回滚策略、模拟环境回放、KPI 门控（阈值与额度限制）

3. 风险：数据质量差影响模型输出与诊断准确性
   - 缓解：引入数据质量校验（Completeness/Timeliness/Uniqueness/Timestamp 合法性），异常数据隔离与报警，模型输入校验

4. 风险：敏感信息泄露或审计不合规
   - 缓解：密钥管理、访问控制、审计日志与角色分离（SoD），定期安全评审

5. 风险：AI 误导（回答不准确/产生幻觉）
   - 缓解：知识库优先、来源标注、置信度展示与人工复核路径、限制对高风险操作的直接执行权限

6. 风险：SLA 不达标导致使用者流失
   - 缓解：分级容灾方案、预估负载能力、自动降级 UX（提示当前能力限制）、关键路径冗余

## 9. 验收测试与示例场景（端到端）
1. 场景 A：新媒体对接请求
   - 步骤：优化师通过问答 Agent 提交媒体对接请求（含开发地址、Key、账号） → Agent 标准化字段并入库 → 产研收到通知
   - 验收：对接请求入库字段完整率 100%，产研通知成功率 100%

2. 场景 B：链路异常发现与定位
   - 步骤：监控 Agent 检测到媒体上报延迟异常 → 自动告警 → 诊断 Agent 自动触发初步排查并返回异常环节与日志 → 人工确认并执行回溯修复
   - 验收：从告警到定位所需时间 ≤ 1 小时；诊断指向正确环节 ≥ 85%

3. 场景 C：自动策略金丝雀发布
   - 步骤：分析 Agent 给出预算出价建议 → 策略 Agent 在金丝雀目标上小比例下发 → 观测 24 小时效果 → 自动回滚或扩大
   - 验收：金丝雀窗口内无重大损失（误判率 ≤ 2%），回滚流程可在 30 分钟内完成
