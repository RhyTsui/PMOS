## 0. 文档目的

本文档承接需求真源与原型真源，明确第一阶段的交互层口径，作为前端实现的直接输入。

---

## 1. 整体交互原则

### 1.1 统一入口原则

用户优先通过 `单对话主工作台` 进入系统，不要求用户先理解系统架构。

### 1.2 意图先行原则

系统先识别用户当前任务类型，再决定展示哪些结构化结果与动作入口。

### 1.3 结果可视原则

对话结果不能只停留在聊天文本里，必须同步展示为结构化卡片、表单、证据链、日志或图表。

### 1.4 专项能力分层原则

长驻观察型、重分析型、重图表型能力不强塞进主聊天区，而是进入对应专项页承接。

### 1.5 支撑能力内聚原则

个人知识库、跨部门 MCP、定时任务、小闪通知、全局记忆、网络搜索、快捷短语、技能、常规设置、图片文档上传等能力不直接散落在业务页里，而是通过应用支撑页统一管理或展示。

## 2. 主工作台交互

## 2.1 首屏交互

用户进入主工作台后，首先看到：

1. 顶部状态条
2. 当前任务上下文条
3. 快捷意图入口
4. 主对话输入区
5. 右侧结果面板空态

空态下应显示：

1. 欢迎提示
2. 四个快捷任务入口
3. 最近任务
4. 系统状态摘要

## 2.2 意图进入方式

意图进入有两种方式：

1. 用户直接输入一句话
2. 用户点击快捷任务入口

两种方式最终都进入同一套意图识别逻辑。

## 2.3 四类任务态交互

### 2.3.1 使用帮助态

#### 输入

- 用户提问指标、术语、流程、系统位置

#### 系统动作

1. 调用知识库
2. 匹配示例
3. 给出系统跳转建议

#### 结果区展示

1. 标准答案卡
2. 指标解释卡
3. 示例卡
4. 跳转按钮

### 2.3.2 需求沟通态

#### 输入

- 用户提出对接、需求、配置、回传相关诉求

#### 系统动作

1. 识别需求类型
2. 追问缺失字段
3. 调用媒体文档与知识库补充上下文
4. 生成结构化需求单
5. 展示依赖项与风险

#### 结果区展示

1. 结构化需求单
2. 缺失字段提醒
3. 依赖项清单
4. 媒体文档引用
5. 下一步动作

### 2.3.3 问题排查态

#### 输入

- 用户描述异常、gap、归因不一致、回推问题

#### 系统动作

1. 拉取各 MCP 数据源
2. 形成排查路径
3. 生成结论、证据链、置信度和建议

#### 结果区展示

1. 一句话结论卡
2. 证据链时间线
3. 置信度
4. 风险等级
5. 下一步建议

### 2.3.4 广告联调态

#### 输入

- 用户发起联调、查看联调状态或联调失败原因

#### 系统动作

1. 带入默认账号与设备
2. 展示联调准备项
3. 推进浏览器与 App 自动化
4. 实时输出日志
5. 必要时联动问题排查
6. 生成联调报告

#### 结果区展示

1. 联调准备项
2. 执行状态
3. 实时日志
4. 异常节点
5. 联调报告

## 2.4 历史任务交互

用户可以通过历史任务抽屉：

1. 查看最近会话
2. 重新打开某次需求任务
3. 查看某次排查记录
4. 查看某次联调记录

历史任务不作为独立导航主页面。

---

## 3. 监控大屏页交互

### 3.1 页面角色

监控大屏页负责全局运行态感知，不负责主任务发起。

### 3.2 核心交互

1. 查看全局状态摘要
2. 查看模块级风险卡片
3. 查看告警流
4. 下钻单项监控维度

---

## 4. 素材分析页交互

### 4.1 页面角色

素材分析页负责把创意内容理解、聚类、报表和沉淀做成专项分析面。

### 4.2 核心交互

1. 查看创意脚本解析
2. 查看创意相似度匹配结果
3. 查看脚本 ID 归并结果
4. 查看创意报表
5. 查看优质创意沉淀

---

## 5. 广告预测页交互

### 5.1 页面角色

广告预测页负责承接大盘、媒体、应用、广告、素材维度的预测与测算。

### 5.2 核心交互

1. 选择预测对象
2. 查看 LTV 预测
3. 查看 ROI 预测
4. 查看回本周期
5. 查看首日 ROI 倒推结果

---

## 6. 应用支撑页交互

### 6.1 页面角色

应用支撑页负责承接运行底座能力，不替代外部知识库平台或测试平台。

### 6.2 核心交互

1. 查看个人知识库挂载状态
2. 查看和配置跨部门 MCP 挂载
3. 查看和管理定时任务
4. 查看小闪通知目标与开关
5. 查看全局记忆状态
6. 查看网络搜索开关
7. 管理快捷短语
8. 查看已启用技能
9. 管理常规设置
10. 上传图片与文档并触发业务任务

---

## 7. 当前阶段前端交互要求

第一阶段前端应满足：

1. 主工作台可在同一页内切换四类任务态
2. 右侧结果面板随任务态动态变化
3. 可从主导航进入监控、素材、预测专项页
4. 可从主导航进入应用支撑页
5. 整体结构对 6 月广告 BI 分析、8 月广告执行、9 月创意生产能力可扩展
