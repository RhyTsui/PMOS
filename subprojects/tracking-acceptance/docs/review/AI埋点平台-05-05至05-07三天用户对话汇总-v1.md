## 4. 关于设计工具的对话与判断

用户主动讨论并追问了：

- Figma 是否更专业
- Figma 是否收费
- Figma 是否值得引入
- UI 设计师如何介入
- Pixso 是否可替代 Figma
- Pixso MCP 是否能接
- Pixso 是否支持 md 导入导出

用户明确表达的判断包括：

- 正常产品设计阶段应在视觉/UI确认阶段有设计图产出
- 因为设计图容易漂，所以希望用 HTML 来确认和修改
- Figma/Pixso 只是设计流程中的一环
- 不应让 UI 重头重做
- 产品自己也可能承担 UI 设计职责

用户对工具能力的具体要求包括：

- 需要有 UI 协作但不是必须每页都引入设计师
- 设计稿应便于后端一比一还原
- 希望 `md / Figma(or Pixso) / HTML / 交接包` 有清晰权责
- 希望工具过程统一，而不是有 UI 一套、无 UI 一套

用户后来又明确切换方向：

- Figma 太贵，先废弃
- 接入 Pixso MCP
- 但 Pixso 不能稳定承担写入
- 先手动导入，再通过 MCP 回读
