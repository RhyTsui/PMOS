# AI埋点平台历史导入与版本继承 v1

日期：2026-05-05

## 1. 目的

本文件把“导入历史埋点文档”和“从旧版本继承到新版本”从零散操作说明提升为正式产品合同。

它要解决的问题是：

- 项目接入时不知道旧 Excel、旧版本、旧接入单元该落到哪里
- 版本差异只能靠人工比对
- 导入完成后与规则、日志、报告之间没有稳定回链

## 2. 用户场景

- 新项目首次接入，上传一份历史埋点 Excel，快速生成当前版本草案
- 已有项目从 `vN` 继承到 `vN+1`，只处理差异，不重建全量事件
- 研发确认某个历史字段仍然有效，保留继承链
- QA 在报告里看到问题后，回查该字段来自哪一版导入

## 3. 与 7 个正式模块的关系

- `埋点需求` 是历史导入和版本继承的主入口
- `数据字典` 承接导入时带入的字典映射和差异
- `校验规则` 在继承后可按差异重新生成或补齐规则
- `日志查询` 需能回链到版本来源和导入批次
- `验收报告` 需定位到具体版本差异
- `概览` 展示当前版本从何继承、差异规模多大
- `AI助手` 解释导入差异并建议如何处理冲突

## 4. 页面与入口归属

- `埋点需求` 提供“新建版本并导入”主动作
- `埋点需求` 提供“从历史版本继承”主动作
- `埋点需求` 提供“查看差异摘要 / 差异详情”入口
- `概览` 只展示继承来源、差异数量、待处理状态摘要

## 5. 关键业务对象

### 5.1 导入任务

- `import_task_id`
- `project_id`
- `source_file`
- `source_type`
- `target_version_id`
- `status`
- `parse_summary`

### 5.2 版本继承关系

- `inheritance_id`
- `from_version_id`
- `to_version_id`
- `inherit_mode`
- `created_by`
- `created_at`

### 5.3 版本差异记录

- 新增事件
- 删除事件
- 新增字段
- 删除字段
- 语义修改
- 步骤结构变化

### 5.4 差异处理动作

- 保留继承
- 覆盖为新定义
- 标记待人工确认
- 暂不接入

## 6. 状态流 / 生命周期

1. 创建目标版本
2. 选择历史文件或历史版本
3. 发起导入任务
4. 解析预览
5. 生成差异记录
6. 人工确认差异处理动作
7. 写入目标版本
8. 联动更新规则、字典、验收范围摘要

## 7. 角色作用域与权限边界

- 数据分析师可发起导入、确认差异、提交继承结果
- 研发可查看与自身端侧相关的差异，不直接提交全局继承结果
- QA 查看继承范围与差异，便于解释验收偏差
- 项目经理 / 运营查看继承摘要和高风险变动，不直接编辑差异明细

## 8. 与主真源的衔接点

本文件承接的是 `埋点需求` 模块中“版本化需求编排”的实现细化，不改变主真源对 7 个模块和页面合同的定义。

## 9. 明确不做什么

- 不自动合并所有历史项目的冲突
- 不自动判定所有语义修改都可安全继承
- 不直接把历史 Excel 视为正式真源
- 不绕过人工确认直接覆盖当前版本

## 10. 本版结论

从现在起，历史导入与版本继承不再是“上传文件”的附属动作，而是 `创建版本 -> 解析差异 -> 人工确认 -> 写入版本` 的正式产品链路。
