## 3. 页面映射与开发动作

### 3.1 P0-1 验收版本成为一级治理对象

#### 当前页面落点

- `概览`
- `埋点需求 -> 验收版本`
- `数据字典`
- `校验规则`
- `实时查询`
- `验收报告`

#### 当前对象落点

- `验收版本`
- `验收周期`
- `版本目标`
- `验收范围`

#### 当前前端状态

- `埋点需求` 已把 `验收版本` 作为默认打开页签
- `概览`、`数据字典`、`验收报告` 已存在版本上下文表达
- 但跨页版本联动仍偏“页面各自带版本”，尚未形成统一版本主上下文

#### 开发动作为

1. 冻结 `验收版本` 对象字段
2. 统一版本切换上下文
3. 统一版本切换后的页面刷新规则
4. 统一“当前版本 / 历史版本”归档与查询范围

#### 最小交付顺序

1. 对象冻结
2. 页面统一引用
3. 报告与规则归档绑定

## 4. 推荐开发顺序

建议排期顺序：

1. `P0-1 验收版本成为一级治理对象`
2. `P0-5 模板输入统一为 xlsx`
3. `P0-3 报告区分简报与详细报告`
4. `P0-4 简报暴露核心未通过事件`
5. `P0-2 概览页只做同步`
6. `P0-6 失败批次支持一键重试`

原因：

- 版本对象不稳，后续所有页面都会继续串
- 模板不稳，需求与数据字典输入就不稳
- 报告结构不稳，后续同步与分享就不稳
- 风险不进简报，报告就失去同步价值
- 概览页的最终收口依赖报告与风险口径稳定
- 一键重试必须建立在前面对象链已经闭合的前提上
