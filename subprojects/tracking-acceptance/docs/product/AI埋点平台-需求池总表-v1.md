# AI埋点平台-需求池总表-v1

日期：2026-05-07

状态：active

## 1. 文档定位

本文件是 `tracking-acceptance` 项目的常设需求池总表。

它不替代：

- 主真源
- 迭代需求拆解
- 开发任务单

它的职责是承接：

- 会议纪要反馈
- 用户纠偏
- 需求方建议
- 页面收口过程中新增的产品关切
- 尚未正式升级到主真源或任务单的候选需求

正式链路为：

`反馈 / 纪要 / 评审建议 -> 需求池总表 -> 主真源 / 迭代需求 / 任务单 / parked / rejected`

## 2. 当前来源入口

当前已正式接入的来源包括：

1. [AI埋点平台-05-06会议反馈转需求池-v1.md](/E:/AI/ai-os/subprojects/tracking-acceptance/docs/product/AI埋点平台-05-06会议反馈转需求池-v1.md)
2. 当前页面收口过程中新增的产品反馈与结构纠偏

后续任何新增反馈，不允许只停留在：

- 会话记录
- 会议原文
- 页面修改口头说明

都应先进入本总表，再决定后续去向。

## 3. 状态定义

- `accepted`
  - 已确认进入当前产品主线，后续应继续回写主真源、页面或任务单
- `candidate`
  - 有价值，但仍待边界、依赖或优先级进一步确认
- `parked`
  - 明确暂缓，不进入当前主线
- `rejected`
  - 明确不做，保留原因，避免重复讨论

## 5. 当前建议改动的地方

基于当前会议反馈与页面收口，建议继续修改的重点是：

1. 验收版本、渠道包、事件、步骤、参数、数据字典、规则、报告之间的版本化关系继续统一
2. 继续把“通用模板 + 项目版本实例”作为平台底座
3. 报告继续作为最终结论源，查询只做证据层
4. 动态规则生成继续围绕验收标准、终端、渠道包等条件表达展开
5. 渠道包继续作为一级业务维度，而不是只在页面右上角挂一个筛选

## 6. 当前不建议继续扩散的地方

当前不建议继续推进的方向包括：

1. 实时监控大盘
2. 概览页继续堆更多明细
3. 数据字典在线逐条编辑
4. 规则页继续堆更多静态字段，而不先解决动态规则结构
5. 先按旧系统容器去反向约束当前新系统规划设计

## 7. 使用规则

1. 新反馈先入需求池，再决定是否升级
2. 已进入主真源的条目不从需求池删除，而是保留状态与去向
3. `parked / rejected` 必须保留原因，避免重复讨论
4. 需求池总表是项目级持续对象，不是某次会议的一次性产物
