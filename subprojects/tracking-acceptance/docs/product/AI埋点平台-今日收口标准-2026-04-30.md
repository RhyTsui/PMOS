# AI埋点平台今日收口标准 - 2026-04-30

> 状态说明（2026-05-05）：
> 本文档保留为 2026-04-30 的阶段性收口记录，不再定义当前唯一真源口径。
> 当前生效链路请以 `AI埋点平台-实现真源包索引-v1.md`、`AI埋点平台-详细需求真源-v2.md` 和 `AI埋点平台-真相源治理-v1.md` 为准。

## 1. 目的

本文档不新增产品范围，只定义：

- 今天这条项目主线什么情况下可以算“完成”
- 哪些事项必须今天闭合
- 哪些事项可以明确转入后续迭代
- 哪些状态不能再被误报为完成

目标是避免今天结束时出现多套口径。

## 2. 今日接受目标

今天不是要把 AI 埋点平台所有能力做完，而是要完成以下收口：

1. 明确当日阶段的一期产品真相源
2. 明确历史观点、会议纪要、SVG、研究稿、设计图、前端现状的分层权限
3. 明确当前一期的核心模块口径
4. 明确今天之后继续推进的优先级主线

如果以上四项闭合，今天就可以判定为“产品口径收口完成”。

## 3. 今日必须完成的事项

### 3.1 真相源治理闭合

必须完成：

- 当日阶段判断中，`docs/product/AI埋点平台-详细需求真源-v1.md` 被确认为当时的一期产品真相源
- `docs/product/AI埋点平台-真相源治理-v1.md` 已正式定义文档分层与改动顺序
- 团队后续默认遵循“先改真源，再改设计，再改前端”

今日完成判据：

- 不再允许把会议纪要、研究图、设计图或前端现状直接当产品真源

### 3.2 核心模块口径闭合

必须完成：

- 统一采用 `4 个业务模块 + 1 个底座协同层` 口径

统一口径为：

1. 版本化需求编排
2. 规则化验收
3. 证据查验
4. 放行报告
5. AI 协同与知识挂载底座

今日完成判据：

- 不再把原始 5 个点当作 5 个平级业务模块

### 3.3 下一优先级闭合

必须完成：

- 今天要把“下一步先做什么”压成统一顺序，而不是继续平铺推进

统一优先级为：

1. `SSOT 治理回挂主真源`
2. `标准底座平台化`
3. `历史导入 / 版本继承`
4. `成熟看板与查询方案`

今日完成判据：

- 后续主线不再发散到更多新模块定义

## 4. 今日允许不完成的事项

以下事项今天可以不做完，但必须明确归档为后续项，不能伪装成已完成：

1. 前端角色化过滤细化
2. 接入单元详情抽屉
3. 日志与接入单元的强绑定联查
4. 查询方案保存 / 共享 / 绑定角色
5. 环境上下文在日志 / 验收 / AI 中的统一贯穿
6. 验收进度完整泳道化
7. 节点维度的结构化视图
8. 标准事件库 / 默认字段透视 / 项目裁剪模型的完整前端表达

这些项的状态应统一标记为：

- `后续主线`
- 或 `部分满足`

不能标为：

- `完成`
- `已收口`

## 5. 今日不能再模糊处理的事项

### 5.1 不能再混用真源

不能再出现：

- 主真源一套
- 设计图一套
- 前端实现一套
- 会议纪要口头理解再来一套

### 5.2 不能再混层定义模块

不能再把以下三类内容并列：

- 业务模块
- AI 能力
- 平台底座

### 5.3 不能再把“有草稿”当“已完成”

以下状态不等于完成：

- mock 已存在
- 页面已有卡片
- SVG 已画
- AI 能生成草稿
- 文档里提到过

只有回到主真源并形成统一执行口径，才算正式成立。

## 6. 今日结果判定方法

### 6.1 可判定为完成

当以下条件同时满足时，今天可判定为完成：

1. 唯一 SSOT 已明确
2. 文档分层与改动顺序已明确
3. 核心模块口径已统一
4. 下一优先级已统一
5. 未完成项已明确归档，而不是继续模糊挂起

### 6.2 只能判定为部分完成

若出现以下任一情况，今天只能判定为 `partial`：

1. 主真源仍未与治理规则接通
2. 核心模块口径仍可被不同人各自解释
3. 覆盖检查中的后续主线仍没有统一优先级
4. 设计与前端仍可绕过主真源继续改口径

## 7. 当前判断

截至本文落盘时：

- `唯一真相源治理文档`：已完成
- `核心模块口径收敛`：已完成
- `今天的统一优先级收口`：已完成
- `治理规则正式回挂主真源`：进行中

因此当前总状态应判定为：

- `partial`

原因：

- 今天的产品收口标准已经建立
- 但主真源尚未完全完成治理规则回挂，最后一个硬连接点还在执行中

## 8. 今日结论

今天这条项目主线的完成定义不是“功能全做完”，而是：

把 AI 埋点平台的一期产品口径收成一个可持续执行的单一真相源体系，并把下一步优先级压实。

只要主真源回挂治理规则完成，今天就可以从 `partial` 提升为：

- `solved`

限定语义：

- 这里的 `solved` 指“今天的产品口径收口任务完成”
- 不指“整个平台功能开发完成”
