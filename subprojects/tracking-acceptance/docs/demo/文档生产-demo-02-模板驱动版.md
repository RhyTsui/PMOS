# 埋点需求规格文档

版本：模板驱动 + AI 生成  
样例：花花世界

## 1. 基本信息

- 项目：花花世界
- 文档范围：新手引导漏斗链、支付与订单关联链、渠道分包与设备透传链
- 目标角色：产品、客户端/SDK、服务端、数分、验收

## 2. 业务目标

### 2.1 新手引导漏斗链

- 用于分析新用户新手引导步骤完成情况
- 用于支持新手转化漏斗、流失点分析

### 2.2 支付与订单关联链

- 用于分析支付流程漏斗
- 用于关联支付成功、发货通知、订单信息

### 2.3 渠道分包与设备透传链

- 用于渠道分包效果分析
- 用于客户端到服务端设备上下文透传

## 3. 埋点规格

### 3.1 新手引导漏斗链

#### 事件

- `108 / TutorialStep`
  - 来源：前端/SDK
  - 接口：`GSSdk.analytics.track()`
  - `type = "2"`
- `20001 / Tutorial`
  - 来源：后端
  - 说明：新手引导开始与完成
  - 映射：对应原 `10001`

#### 字段

- `stepId`
  - 含义：步骤 ID
  - 要求：必填，数字字符串，按流程递增
- `stepName`
  - 含义：步骤名称
  - 要求：必填

#### 风险

- 新手引导打点在接入文档中为“推荐”
- `stepId` 需要人工确认

### 3.2 支付与订单关联链

#### 事件

- `106 / PayFlow`
  - 来源：前端/SDK
- `221 / PayFlow`
  - 来源：后端
- `205 / Pay`
  - 来源：后端

#### 字段

- `order_id`
- `channel_order_id`
- `pay_channel`
- `recharge_channel`

#### 风险

- 存在 `不发送，未获取`
- 存在“必填但部分步骤不发送”

### 3.3 渠道分包与设备透传链

#### 字段

- `cpsid`
- `device_id`
- `c_oneid`
- `s_oneid`

#### 来源

- SDK 上报结构
- `getTrackingInfo` 回调
- 客户端透传给服务端

#### 风险

- 透传一致性待验证
- 新增链路仍在演进

## 4. 验收口径

### 4.1 新手引导

- 校验是否存在 `108 / TutorialStep`
- 校验 `stepId / stepName` 是否完整
- 校验步骤顺序是否可比

### 4.2 支付链

- 校验 `106 / 221 / 205` 是否串联
- 校验关键订单字段是否缺失
- 校验发货通知映射是否完整

### 4.3 透传链

- 校验 `cpsid / device_id / c_oneid / s_oneid` 是否打通
- 校验客户端与服务端字段是否一致

## 5. 待确认项

- 新手引导是否升级为必接
- 前置事件是否升级为必接
- 支付链关键字段未获取的原因
- 设备字段是否已在服务端链路稳定使用

## 6. 业务影响

- 新手引导链异常会影响漏斗分析
- 支付链异常会影响付费分析和订单回查
- 渠道透传异常会影响分包分析和归因串联

## 7. 版本说明

- 存在事件迁移：
  - `20001 -> 10001`
  - `20002 -> 10002`
  - `20004 -> 10004`

该模板适合继续生成规则和报告，但阅读体验偏规格化，业务背景和协作语义不够自然。
