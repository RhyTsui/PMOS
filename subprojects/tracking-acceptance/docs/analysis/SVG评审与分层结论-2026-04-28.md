# SVG 评审与分层结论

日期：2026-04-28

## 1. 结论

`docs/svg` 里的图整体有效，但用途不同，不能继续混用。

后续应按四类使用：

1. 一期主真源图
2. 一期辅助设计 / 架构图
3. 背景研究图
4. 中长期参考图

如果继续把全景图、行业研究图、AI-native 拼装图直接当一期页面真源，会再次把范围做重。

## 2. 一期主真源图

这些图可以直接约束当前一期产品边界、信息架构和页面结构。

### `01-背景与问题.svg`

- 有效
- 作用：定义为什么现在必须做、三条问题主轴、一期开场判断
- 当前利用情况：已进入 `PRD前言`、总览页、需求分析主轴

### `02-一期方案与主链路.svg`

- 有效
- 作用：定义一期 8 步主链路
- 当前利用情况：已进入总览页、P0 主链定义

### `03-需求范围与产品骨架.svg`

- 有效
- 作用：约束 P0 / P1 / P2 边界
- 当前利用情况：已进入需求池收口口径

### `04-工作台原型.svg`

- 有效
- 作用：定义总览页的入口职责，不让首页重新变成模块堆叠区
- 当前利用情况：已进入当前前端总览页结构

### `16-最小AI-native闭环.svg`

- 有效
- 作用：提醒一期不是大而全平台，而是最小闭环
- 当前利用情况：已进入“AI 主提效，人做确认，系统承接闭环”口径

## 3. 一期辅助设计 / 架构图

这些图对当前方案有帮助，但不应直接替代主真源图。

### `05-全景流程图.svg`

- 有效
- 定位：全景说明图
- 风险：容易把文档、规则、检查、报告、多端输出一次性说成一期全部做深

### `06-干系人协同图.svg`

- 高价值
- 定位：协作真源辅助图
- 当前不足：
  - 角色粒度还不够细
  - 还没显式放入问题总表 / 角色待办分发层

### `07-文档自动优化模块.svg`

- 有效
- 定位：文档结构化与标准化辅助模块
- 当前边界：
  - 一期可做：Excel / Word / 纪要转结构化基线、差异摘要、待确认项
  - 一期不承诺：AI 自动写出无需人工审阅的最终文档

### `09-规则生成与自动检查模块.svg`

- 有效
- 定位：规则框架图
- 当前不足：
  - 规则分类不够细
  - 没有足够具体的规则样例
  - 规则适用范围和结果等级还不清楚

### `10-人工确认与版本记录模块.svg`

- 有效
- 定位：人工拍板与历史留痕辅助图
- 当前不足：
  - 还需补明确的结论状态枚举
  - 还需显式承接例外确认和问题闭环

### `11-多端调用架构图.svg`

- 有效
- 定位：多端消费同一套核心资产的辅助架构图
- 当前价值：支持平台、研发本地、机器人同步等同源复用

### `14-delay根因与AI切入点.svg`

- 高价值
- 定位：AI 价值切入真源辅助图
- 当前作用：说明 AI 该减少什么 delay，而不是做炫技 Agent 编排

### `17-真实样例推演.svg`

- 有效
- 定位：真实数据验证辅助图
- 当前价值：提醒原型必须落回真实文档和真实链路

## 4. 背景研究图

这些图对统一认知有帮助，但不应直接驱动一期页面与功能设计。

### `12-行业研究-业务价值链.svg`

- 有效
- 定位：背景研究图
- 作用：说明埋点不是采字段，而是服务业务目标、分析和协作闭环

### `13-行业研究-方案演进趋势.svg`

- 有效
- 定位：背景研究图
- 作用：用于说明后续演进方向，不用于压一期交付边界

## 5. 中长期参考图

这些图不应再作为一期页面和功能真源使用。

### `15-AI-native方案拼装图.svg`

- 有研究价值
- 但应降级为中长期路线参考图
- 原因：
  - 容易把系统重新带向更重的控制面
  - 容易把 P2 能力提前压进一期

## 6. 还需补细节的图

### `08-SVG目录生成模块.svg`

- 当前对一期产品价值较低
- 更偏文档生产辅助链
- 暂不作为当前产品核心能力真源

## 7. 后续使用原则

后续如果继续出图、改文档或改前端，应遵守：

1. 先看 `01 / 02 / 03 / 16`
2. 再参考 `04 / 06 / 07 / 09 / 10 / 11 / 14 / 17`
3. `12 / 13 / 15` 只用于背景和路线说明

这样能避免再次把研究图或全景图直接误用成一期交付边界。
