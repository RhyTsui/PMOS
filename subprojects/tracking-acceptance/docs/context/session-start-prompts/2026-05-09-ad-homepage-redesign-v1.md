## 2. 续跑目标

下次进入时，不要做以下事情：

1. 不要在当前临时首页样稿上继续补区块
2. 不要把旧展示态首页再包装成“操作台”
3. 不要沿用旧 `frontend` 思路回退

下次要直接做的事情是：

1. 重新定义企业操作台首页的信息结构
2. 用三段主结构重建页面骨架
3. 围绕任务流和处理流组织首页，而不是围绕宣传展示组织

## 3. 首页主结构

首页只保留以下三段主结构：

### 3.1 任务队列

用于展示：

1. 当前待处理任务
2. 优先级
3. 状态
4. 来源
5. 截止时间或阻塞情况

### 3.2 当前处理区

用于展示：

1. 当前正在处理的任务
2. 核心上下文
3. 关键动作入口
4. 处理进度
5. 最近变更或执行记录

### 3.3 右侧检查器

用于展示：

1. 当前选中对象详情
2. 风险检查项
3. 规则提示
4. 异常与提醒
5. 需要确认的约束
