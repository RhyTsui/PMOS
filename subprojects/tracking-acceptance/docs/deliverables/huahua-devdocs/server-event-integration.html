服务端负责补齐业务上下文

角色链、支付链、资源链和大部分玩法事件都依赖服务端上报，不能只看客户端。

自定义事件

服务端自定义事件覆盖成长、解锁、货币、种植、制作、售卖、订单、场景、任务、换装、社交、交易、排行、公会、公会竞赛、内嵌游戏、活动等。

范围：20001-20027，其中 20019 / 20020 字段设计当前仍不完整。

支付链要求

核心冲突：order_id、channel_order_id、pay_channel、recharge_channel 在规范层是关键字段，但部分步骤存在“未获取 / 不发送”。服务端文档必须保留这个事实。

  • 221 负责支付后端步骤和关键字段补齐。
  • 205 负责最终支付成功 / 发货成功结果。
  • 验收时必须按 106 -> 221 -> 205 串查，而不是只查 205。
