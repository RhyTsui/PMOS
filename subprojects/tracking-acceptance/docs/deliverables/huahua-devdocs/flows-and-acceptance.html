Flows & Acceptance

不要按页面验收，要按链路验收

花花世界埋点的验收对象不是单个事件，而是多条端到端链路。

关键流程

前置链

  1. 启动
  2. 版本获取 / 资源下载
  3. SDK 登录
  4. 设备信息获取
  5. 请求服务器列表
  6. 配置文件加载
  7. 网关地址获取 / 网络连接
  8. 客户端角色登录

支付链

  1. 106 记录前端支付流程
  2. 221 记录后端支付流程
  3. 205 记录支付成功 / 发货成功

新手链

  1. 108 记录前端新手步骤
  2. 20001 记录后端新手状态

验收检查项

检查项通过标准
客户端 SDK 调用track / getTrackingInfo 调用结构正确
步骤版本step_version 明确且未混淆
支付闭环106 -> 221 -> 205 可串查
透传字段device_id / c_oneid / s_oneid / cpsid 可对齐
新手链108 + 20001 同时可解释
灰区留痕所有未获取 / 不发送字段都体现在验收报告中

常见排障

症状：支付最终成功，但 106 / 221 中关键字段缺失。
处理：不要只补 205，要回到 106 / 221 查字段来源和步骤时序。

症状：前置漏斗数据断层。
处理：先确认是不是不同 step_version 被合并统计。

症状：服务端无法关联渠道或设备。
处理：优先检查 getTrackingInfo 回调消费和透传路径。
