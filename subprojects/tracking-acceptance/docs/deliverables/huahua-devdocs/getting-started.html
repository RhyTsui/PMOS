Quick Start

五步完成接入

如果你是第一次接入花花世界埋点，先按这个顺序做，不要一开始就看全量事件表。

步骤 1：确认接入范围

  1. 基础账号链：111 / 114 / 102 / 104 / 105 / 201 / 203 / 204 / 206
  2. 前置链：107
  3. 新手链：108 + 20001
  4. 支付链：106 + 221 + 205
  5. 透传链：getTrackingInfo + 服务端字段消费

步骤 2：客户端先通 SDK 调用

统一调用：GSSDK.getInstance().call("analytics", "track", info)

  • 前置事件：type = "1"
  • 新手引导：type = "2"
  • 客户端自定义事件：type = "0"
  • 支付 / 激励视频：type = "9"

步骤 3：服务端补齐业务上下文

  • 角色链、支付链、资源链、自定义玩法链由服务端补齐关键字段。
  • 特别是支付链，不能只靠客户端。
  • 服务端统一产出 game_uuid、start_time、role_id、server 等公共头。

步骤 4：明确版本与字典

  • 前置 / 支付 / 激励视频链都必须带 step_version。
  • 枚举字段必须对应字典。
  • 内容对象 ID 必须可回溯到内容 ID 字典。

步骤 5：按链路验收

  • 支付链：106 -> 221 -> 205
  • 前置链：按 step_version 重建漏斗
  • 新手链：108 + 20001 同时可解释
  • 透传链：device_id / c_oneid / s_oneid / cpsid 端到端一致

不要忽略：资料里写明的 未获取 / 不发送 是开发与验收重点，不是应当从文档里删掉的瑕疵。
