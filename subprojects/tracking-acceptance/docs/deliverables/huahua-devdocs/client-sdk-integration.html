SDK Contract

客户端通过 SDK analytics 模块上报

当前源资料能证明的客户端接入面是 GSSDK 调用，而不是项目组直连 REST API。

统一入口：GSSDK.getInstance().call("analytics", "track", info)

事件类型

类型type适用事件说明
前置事件1107启动到进入游戏前的步骤链
新手引导2108新手引导步骤链
客户端自定义事件020011 / 20022 / 20023 / 20026 等玩法或平台特性级上报
支付/激励视频流程9106 / 110支付式流程链

请求示例

前置事件

{ "type": "1", "extra": { "stepId": "100", "stepName": "开始播放CG" } }

新手引导

{ "type": "2", "extra": { "stepId": "101", "stepName": "点击进入游戏按钮" } }

客户端自定义事件

{ "type": "0", "event": "20011", "eventName": "GameStory", "extra": { "story_id": "123", "operation_type": "skip" } }

支付流程

{ "type": "9", "event": "106", "extra": { "step_version": "20250724", "step_id": "501", "step_name": "支付成功", "result": "0" } }

设备透传

同步读取：GSSDK.getInstance().callString("analytics", "getTrackingInfo", new HashMap())

异步读取：getTrackingInfoAsync 回调返回 code / data / message

字段说明
c_oneid客户端设备标识
s_oneid服务端设备标识
device_id设备 ID
cpsid渠道分包 ID
channel用户渠道
app_version / app_package_name / os_type环境字段

接入要求

  • stepId / step_id 必须为数字字符串或整型，项目内保持一致。
  • stepName / step_name 必须为可读文本。
  • 自定义事件 event 必须大于 10000。
  • 支付链和激励视频链必须带 step_version。
  • 失败步骤建议带上错误信息，不要只保留成功路径。

注意：当前资料同时存在 20250122 与 20250724 两套前置步骤版本，客户端不得把它们混成一套漏斗。
