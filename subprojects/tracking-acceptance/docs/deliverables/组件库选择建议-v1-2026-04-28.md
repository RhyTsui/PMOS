# 组件库选择建议 v1

日期：2026-04-28

## 1. 当前结论

`tracking-acceptance/frontend` 当前没有引入成熟组件库，仍然是：

- React
- Vite
- 自定义 CSS

这也是当前页面容易长得像“结构化文档页”的原因之一。

## 2. 现有可复用方向

相邻子项目 `chokonu/frontend` 已明确使用：

- `antd@5`

并且已经在这些页面中使用：

- `Button`
- `Card`
- `Drawer`
- `Select`
- `Tabs`
- `Tag`
- `Table`

这说明仓内已经存在一条实际可复用的 B 端组件方向。

## 3. 推荐方案

本项目后续 UI 重构，优先推荐：

### 方案 A：沿用 `antd`

优点：

- 现有子项目已验证
- B 端后台表格、抽屉、表单、Tabs、Tag 都成熟
- 可快速把“像文档页”往“像业务工作台”拉
- 与当前项目的中后台信息密度更匹配

适用范围：

- 总览页的指标卡、表格、Tag、Tabs
- 文档基线 / 规则中心 / 验收中心
- AI 助手中的消息列表、侧栏卡片、操作区

### 方案 B：继续用轻量自定义组件

优点：

- 不引依赖
- 改动更可控

缺点：

- 需要自己补大量业务交互细节
- 很容易继续停留在“结构化页面”而不是“成熟产品工作台”

## 4. 当前建议

如果下一轮目标是：

- 快速把 UIUX 拉到更像生产态
- 提升表格、抽屉、页签、输入框、对话区的一致性

建议优先走：

- `antd` 复用路线

## 5. 当前约束

本轮尚未直接引入 `antd`，原因是：

- 当前前端主任务先是接真实数据、跑通主链、压掉说明感
- 组件迁移应在下一轮作为明确的 UI 重构步骤单独执行

## 6. 执行顺序建议

1. 先完成当前页面信息结构和主交互重构
2. 再引入 `antd`
3. 优先替换：
   - Table
   - Drawer / Split Panel
   - Tabs
   - Input / Select
   - Tag / Badge
   - Card

## 7. 一句话结论

当前最合适的组件库方向不是重新发散，而是：

- 优先复用仓内已被验证的 `antd@5`
