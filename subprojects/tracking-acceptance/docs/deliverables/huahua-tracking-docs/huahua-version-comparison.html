一、版本说明

版本 定位 适合对象 优点 不足 地址
V1 概览版 高层总览 老板、项目方、第一次了解项目的人 快、清楚、适合先读 不是完整需求文档 SVG 总览板
V2 接入版 开发者接入文档 客户端、SDK、服务端、测试 补了接入方式、payload、职责、时序、校验 仍然偏“规范”而不是“业务清单” 开发者版接入总册
V3 需求版 业务完整需求总册 产品、数据 BP、研发交接 更接近《代号Q》那种“清单 + 自定义事件 + 流程”结构 还没展开到每个事件独立一节 人读需求总册
V4 AI 结构版 结构化规格 AI、规则生成、自动验收 Fact / Conflict / Assumption 清晰 不适合作为业务方首读版本 AI 结构化规格总册

三、使用建议

如果你要拿给业务和研发一起看：优先看 V3，再补 V2。

如果你要拿来继续扩成最终正式文档：以 V3 为主线，把 V2 的接口契约内容并进去。

如果你要交给 AI 或后续自动验收：看 V4。
