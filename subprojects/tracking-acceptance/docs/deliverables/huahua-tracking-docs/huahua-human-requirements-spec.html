业务完整需求中的关键问题：当前资料已经明确表明，这条链存在“字段定义为必填，但部分步骤不发送 / 未获取”的冲突。这不是文档错误，而是业务完整需求的一部分，必须保留给研发和测试看。

八、灰区与补充说明

1. 107 / 108 的定位冲突：资料中写成“是 / 必接”，但执行中常被当增强项。本文按核心链处理。

2. 支付链字段冲突：关键字段在规范层和实现层不一致，因此研发交接时必须把“字段目标状态”和“当前发送状态”分开写。

3. Demo 边界：本文会尽量完整，但不会伪造不存在的 API 网关、环境地址和鉴权策略。

4. 对标《代号Q》的写法：当前版本已经按“埋点清单 + 服务器自定义事件 + 客户端自定义事件 + 前置流程 + 新手引导”的方式重写，但花花世界实际复杂度高于《代号Q》。
