# 前端设计规格 v1

日期：2026-04-28

## 1. 当前问题

当前前端虽然已经接入真实数据，但仍存在三类明显问题：

1. 页面仍有“说明文案块”，看起来像产品文档而不是工作台
2. 右侧抽屉使用过重，很多信息被做成说明面板，操作主次不清
3. AI 助手不是对话工作台，而是静态信息页

## 2. 本轮设计目标

本轮不再新增功能范围，重点只做 UIUX 重构：

1. 去掉首页说明块和闭环流程块
2. 强化页面的状态、问题、动作三层
3. 把 AI 助手改成真正的对话工作台
4. 把日志样本与详情改成双栏，不再默认依赖抽屉
5. 保持真实数据承载，不回退到 mock

## 3. 页面级设计原则

### 3.1 总览

- 不讲背景
- 不讲方法论
- 不讲闭环流程
- 只保留：
  - 当前状态
  - 当前问题
  - 当前任务
  - 当前动作

### 3.2 文档基线 / 规则中心 / 验收中心

- 仍然保留“列表 + 详情”范式
- 但详情只承载当前选中对象，不放解释性大段文案
- 抽屉只用于强对象详情，不用于讲方案

### 3.3 验收与协作

- 主区看日志样本和结果表
- 右区直接看字段样本、异常摘要、知识链接
- 优先双栏，不优先抽屉

### 3.4 AI 助手

- 必须是对话工作台
- 至少要有：
  - 推荐问题
  - 对话消息区
  - 输入框
  - 建议动作
- 不能只是一组卡片和说明块

## 4. 布局策略

### 4.1 总览

- 上：指标卡
- 中：问题 / 状态 / 快捷动作
- 下：任务列表 + 责任路径 + 本周目标

### 4.2 日志样本页

- 左：日志查询与样本表
- 右：当前样本详情

### 4.3 AI 助手页

- 左：推荐问题 / 知识源
- 中：对话区 + 输入区
- 右：建议动作

## 5. 视觉层级

### 一级

- 指标卡
- 页面标题
- 主操作按钮

### 二级

- 表格
- 对话区
- 详情区

### 三级

- 辅助状态
- 风险标签
- 解释性短说明

## 6. 明确不做

本轮不做：

- 重型设计系统建设
- 暗色化重主题切换
- 复杂拖拽编排
- 大规模交互动效

## 7. 当前落地顺序

1. 总览页去文档化
2. AI 助手工作台化
3. 验收与协作双栏化
4. 再看文档基线、规则中心是否要继续减轻抽屉感
