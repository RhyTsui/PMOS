# AI埋点平台-页面整改清单-v1

- 本清单用于把 05-07 UI 设计反馈转成逐页整改执行表
- 其作用是承接主真源、前端交互文档、通用设计规范
- 后续页面整改默认先查本清单，再进入页面实现修改

上游依据：
- `docs/product/AI埋点平台-详细需求真源-v2.md`

## 1. 执行规则

所有页面整改默认遵守：

1. 先做差异诊断，再改页面
2. 优先复用内部组件库与 YokaUI 语义
3. 不自由创造新视觉语言
4. 不通过补说明性文案来解决结构问题
5. 筛选区、表格区、操作区、卡片区保持统一节奏

## 2. 概览页

整改目标：
- 保持同步页定位
- 强化版本目标、核心指标、最近更新、趋势、风险摘要
- 不回加大面积说明文案

必须保持：
- 版本目标
- 核心指标
- 最近更新
- 验收趋势
- 不通过埋点
- 未上报埋点

不建议继续改：
- 再扩成明细堆砌页
- 再扩成实时监控页
- 再补解释性废话

状态：`已形成规则，页面可继续精修`

## 3. 埋点需求

整改目标：
- 强化版本化治理入口
- 让 验收版本 / 渠道包 / 事件 / 步骤 / 参数 成为统一结构
- 保持模板导入与查询管理并存

必须保持：
- 默认从验收版本打开
- 版本下挂渠道包、事件、步骤、参数
- 模板导入导出仅保留在渠道包/事件/步骤/参数
- 渠道包成为一级业务维度

继续整改点：
- 统一筛选区节奏
- 统一按钮组件语义
- 统一列表状态口径
- 保持上传覆盖更新入口稳定

状态：`持续整改中`

## 4. 数据字典

整改目标：
- 固定为 参数主表 + 枚举值展开 + 参数详情抽屉
- 保持查阅、联动、上传覆盖更新定位
- 不走在线 Excel 编辑器路线

必须保持：
- 参数主表
- 枚举值展开子表
- 参数详情抽屉
- 最近变更可视化
- 上传或覆盖更新

继续整改点：
- 局部滚动只挂内容区
- 枚举值展开列宽稳定
- 详情抽屉中的关联事件/规则/变更记录层级更稳

不建议继续改：
- 页面内在线编辑
- 把枚举值硬塞回主表单元格

状态：`结构已定，细节持续精修`

## 5. 校验规则

整改目标：
- 固定为规则治理页
- 服务版本内动态规则生成与人工修正
- 承接渠道包 / 事件 / 步骤 / 参数维度筛选

必须保持：
- 规则ID 基础增删改查
- AI 规则生成
- 生效状态开关固定在右侧
- 渠道包/事件/步骤/参数筛选

继续整改点：
- 列宽稳定
- 固定列与局部滚动稳定
- 验收标准与上报示例列的阅读节奏

状态：`结构已定，表格继续精修`

## 6. 实时查询

整改目标：
- 固定为查询执行页
- 不扩成实时监控大盘
- 强化筛选、分布、明细三段式证据链

必须保持：
1. 筛选条件
2. 分布区域
3. 日志明细

必须保持的交互：
- 事件筛选
- 字段筛选
- 接收时间筛选
- 日志接收筛选
- 查询/重置/导出统一在筛选区底部
- 时间分布 / 事件分布 标题切换
- 单事件查询 / 多事件查询

不建议继续改：
- 再扩成监控首页
- 再加解释性模块

状态：`结构已定，样式与密度持续精修`

## 7. 验收报告

整改目标：
- 固定为 简报卡片列表 + 详细报告详情 双层结构
- 报告结论作为平台主要统计来源
- 不再混入分布图类监控表达

必须保持：
- 列表页承载简报卡片
- 详情页承载详细报告
- AI 报告配置拆分为简报提示词与详细报告提示词
- 推送简报到小闪
- 分享链接

详情页必须保持：
- 摘要
- 验收概况
- AI报告统计
- 未上报明细
- 未通过明细
- 已通过明细
- 未通过规则
- 校验结果明细

不建议继续改：
- 再回退成普通表格列表
- 再塞入事件/时间分布图

状态：`主结构已定，视觉层级继续精修`

## 8. AI助手

整改目标：
- 固定为会话页
- 强调能力入口，不堆说明页

必须保持：
- 自写 chat
- 多轮会话
- 文件上传
- 内置MCP
- 内置知识库

不建议继续改：
- 再堆说明型文案
- 再拆回多个说明模块页

状态：`主结构已定，交互与布局可继续精修`
