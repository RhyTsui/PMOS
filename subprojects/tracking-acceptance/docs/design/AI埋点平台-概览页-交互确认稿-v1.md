# AI埋点平台概览页交互确认稿 v1

日期：2026-05-05

## 1. 文档定位

本文档位于 `原型确认稿` 与 `视觉/UI确认稿` 之间。

它负责确认：

- 页面怎么用
- 切换后怎么变
- 区块怎么刷新
- 状态怎么反馈
- 点击后去哪里

它不负责确认：

- 最终视觉样式
- 最终 HTML 表达
- 真实接口实现

## 2. 交互目标

概览页交互层的目标不是提供复杂操作，而是保证用户能在最短时间内：

1. 完成上下文切换
2. 看懂当前态势
3. 理解偏差来源
4. 进入正确的后续处理页面

因此本页交互遵循 3 条原则：

- 先切换上下文，再刷新内容
- 先给结论，再给入口
- 先区块级反馈，不做整页级打断

## 3. 全局切换交互

### 3.1 项目切换

- 切换后重置到该项目的默认验收版本
- 保留合法视角；若当前视角在新项目下不适用，则回退到 `全部`
- 触发全页业务区块重载，但不清空顶部上下文

刷新优先顺序固定为：

1. 项目切换器选中态
2. 验收版本切换器内容
3. 视角切换器合法状态
4. 态势区
5. 目标区
6. 进度区
7. 更新区
8. 趋势区
9. 风险区

### 3.2 验收版本切换

- 切换后保留当前项目和当前视角
- 目标区、进度区、更新区、趋势区、风险区全部重算
- 若当前版本无验收数据，进入空态，不清空页面上下文

刷新优先顺序固定为：

1. 验收版本切换器选中态
2. 态势区
3. 目标区
4. 进度区
5. 更新区
6. 趋势区
7. 风险区

### 3.3 视角切换

- 切换后不改变项目和版本
- 仅改变当前责任口径下的数据呈现
- 指标卡、趋势图、TOP5 按同一责任口径重算

刷新优先顺序固定为：

1. 视角切换器选中态
2. 态势区
3. 进度区
4. 趋势区
5. 风险区

说明：

- 目标区默认不随视角切换改变业务描述
- 更新区默认保留版本级更新摘要，不按责任视角重写文案

## 4. 区块交互

### 4.1 态势区

- 只承接查看，不承接编辑
- 态势标签支持 hover 查看补充解释
- 一句话原因支持点击进入对应问题详情或报告详情

点击落点约束：

- 若原因源于不通过事件，进入对应事件详情
- 若原因源于批次结论，进入对应验收报告
- 若原因源于规则变更，进入对应规则详情

### 4.2 目标与门槛区

- 默认按摘要阅读，不在概览页直接编辑
- 验收范围支持点击查看范围详情
- 当前状态不可在本页直接修改
- 验收通过门槛只查看，不支持在本页改写

### 4.3 指标卡区

- 每张卡片都应可点击
- 点击后进入对应模块或对应明细页

默认跳转目标：

- `埋点数量` -> 埋点需求页的当前版本全集
- `已验收` -> 验收报告页的已通过明细
- `待验收` -> 验收报告页的待验收明细
- `通过率` -> 验收报告页的当前版本总览
- `P0 埋点数量` -> 埋点需求页的 P0 过滤结果
- `P0 待验收` -> 验收报告页的 P0 待验收结果

### 4.4 更新影响摘要区

- 每条更新支持点击进入变更详情
- 概览页默认仅展示最近 3 条
- “查看全部变更”进入完整变更列表

默认跳转目标：

- 版本信息变更 -> 版本详情
- 埋点需求变更 -> 埋点需求差异详情
- 数据字典更新 -> 数据字典更新详情
- 校验规则变更 -> 规则详情
- 验收批次报告更新 -> 批次报告详情

### 4.5 趋势区

- 支持 hover 查看每日数值
- 不支持在本页修改图表维度
- 点击趋势点只查看，不进入编辑

点击行为约束：

- 点击具体日期点后，仅进入对应日期或批次的结果详情
- 不允许从趋势图进入复杂配置面板

### 4.6 风险区

- 不通过事件 TOP5 点击进入事件详情或规则详情
- 待验收埋点 TOP5 点击进入待验收详情
- 不在概览页展开完整问题参数和完整规则详情

点击优先规则：

- 点击事件名：进入事件详情
- 点击规则摘要：进入规则详情
- 点击状态：进入对应验收结果页

## 5. 状态反馈

### 5.1 加载态

- 切换项目 / 版本 / 视角时采用局部骨架屏
- 顶部上下文优先更新
- 不允许整页白屏重刷

优先级：

- 上下文区先于业务区块更新
- 态势区优先于进度区
- 进度区优先于趋势区和风险区

### 5.2 空态

- 保留上下文区和目标区
- 进度区、趋势区、风险区显示“暂无验收数据”

空态不允许替代：

- 无权限态
- 异常态

### 5.3 异常态

- 趋势区失败：仅趋势区显示错误和重试
- 更新区失败：仅更新区显示错误和重试
- 风险区失败：仅风险区显示错误和重试
- TOP5 数据不足：按真实数量显示，不补伪数据

异常优先级：

- 区块级异常优先于区块级空态
- 无权限态优先于异常态

### 5.4 无权限态

- 明确显示无权限原因
- 不使用空态替代无权限态
- 若仅部分区块无权限，允许局部无权限提示，不扩大为整页无权限

## 6. 禁做动作

概览页明确不提供以下动作：

- 批量处理
- AI 回写
- 查询方案保存
- 复杂筛选器配置
- 规则编辑
- 版本目标编辑

## 7. 与其他稿件的关系

- 上游依赖：`原型确认稿`
- 下游承接：`视觉/UI确认稿`、`静态HTML确认稿`

## 8. 本轮确认结论

本稿当前判断为：`solved`

原因：

- 切换联动、区块交互、状态反馈、点击落点和禁做动作都已锁定
- 评审时最容易产生歧义的刷新顺序、局部异常和跳转目标都已显式定义
- 本稿已经可作为概览页交互层的正式确认物继续向视觉/UI和静态 HTML 承接
