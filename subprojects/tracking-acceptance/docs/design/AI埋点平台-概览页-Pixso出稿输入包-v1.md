# AI埋点平台-概览页-Pixso出稿输入包-v1

日期：2026-05-05

## 1. 文档定位

本文档是给 `Pixso高保真视觉稿` 使用的统一出稿输入包。

它不替代原稿，只做结构化汇总，确保 `Pixso` 出稿时不丢重要约束。

## 2. 输入组成

本输入包由以下 3 份上游共同组成：

1. `AI埋点平台-概览页-原型确认稿-v2.md`
2. `AI埋点平台-概览页-交互确认稿-v1.md`
3. `AI埋点平台-概览页-CodexUI设计初稿-v1.md`

使用原则：

- 不得只看 `CodexUI设计初稿`
- 必须同时尊重 `原型确认稿` 和 `交互确认稿`
- 若三者冲突，以 `原型确认稿`、`交互确认稿` 为上位约束

## 3. 页面角色

- 页面名称：概览页
- 页面定位：`版本验收态势页`
- 页面目标：让用户快速判断当前版本态势、偏差、进度和风险

## 4. 不可改边界

- 不新增菜单结构
- 不改页面主链
- 不引入 AI 助手建议
- 不引入复杂编辑与批量执行动作
- 不把页面改成大屏、导航页或工作台

## 5. 固定结构

页面固定结构如下：

1. 全局上下文
2. 版本验收态势
3. 版本目标与门槛
4. 当前进度摘要
5. 更新影响摘要
6. 进度趋势
7. 当前主要风险

## 6. 固定交互约束

- 项目切换优先刷新版本与上下文，再刷新业务区块
- 版本切换重算态势、目标、进度、变更、趋势、风险
- 视角切换不改项目与版本，只改责任口径下内容呈现
- 概览页所有点击都应落到详情或对应模块，不在本页深度处理

## 7. 固定视觉约束

- 态势区必须是首屏第一视觉层
- 页面风格为企业级数据平台
- 页面为中密信息密度、浅底深字、弱装饰强结论
- 指标区固定 `4 + 2`
- `P0` 卡必须显式区别于普通指标
- 趋势区固定为 `目标线 + 实际线 + 口径提示`
- 风险区固定为双摘要表

## 8. 文案约束

- 使用真实产品短文案
- 使用接近真实业务的示例数据
- 不使用解释型长文案
- 不带章节编号
- 不写“为什么这样设计”类说明句

## 9. Pixso 出稿提示词

```text
为 AI 埋点平台设计一个企业级数据平台风格的概览页高保真视觉稿。
页面定位是“版本验收态势页”，不是导航页，不是工作台，不是营销官网页。

出稿必须同时遵守三份上游输入：
1. 原型确认稿
2. 交互确认稿
3. CodexUI设计初稿

固定结构：
全局上下文、版本验收态势、版本目标与门槛、当前进度摘要、更新影响摘要、进度趋势、当前主要风险。

关键要求：
- 态势区必须是首屏第一视觉层
- 指标区固定为 4+2
- P0 风险卡必须显式区别于普通指标
- 更新影响摘要必须降权
- 趋势区必须是目标线 + 实际线，并附带口径说明
- 风险区必须是两个摘要表，不扩成详情中心

整体风格：
企业级数据平台、中密信息密度、浅底深字、弱装饰强结论、稳定克制、可扫读。

禁止：
- 区块标题带编号
- 大段解释性页面文案
- 说明稿式页面
- 数据大屏风格
- 运营导航页风格
- 复杂执行动作入口
```

## 10. Pixso 回写要求

Pixso 冻结后，至少反馈以下内容给 Codex：

- 区块主次是否调整
- 组件表达是否调整
- 状态样式是否调整
- 哪些建议未采纳
- 对静态HTML确认稿的强约束是什么
