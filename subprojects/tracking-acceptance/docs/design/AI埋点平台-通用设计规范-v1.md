# AI埋点平台通用设计规范 v1

日期：2026-05-06

## 1. 文档定位

本规范用于统一 AI 埋点平台各页面的最终交付级设计口径，直接约束：

- `Pixso高保真视觉稿`
- `视觉/UI确认稿`
- `静态HTML确认稿`
- `产品设计工程包`

本规范优先级高于单页自由发挥。页面如需偏离，必须先在单页确认稿里显式说明。

## 1.1 当前吸收来源

本规范当前已吸收以下输入：

- 当前项目已确认的概览页设计口径
- `docs/sources/inbox/【UI设计规范文档】-设计规范.docx`

若两者冲突：

- 当前项目已确认口径优先
- 通用规范保留为平台默认基线

## 1.2 规范层级

统一采用以下层级：

- 全局说明：设计语言、色板、字体、布局、通用框架
- 基础组件：按钮、输入、筛选、表格、表单、页签
- 业务组件：项目切换、页面筛选、图表组合、场景化列表
- 特殊场景：复杂业务态、异常态、无权限态、动态交互态

## 1.3 规范使用角色

- 设计：统一视觉风格与组件细节
- 开发：作为实现参数与组件复用基线
- 产品：统一页面口径与交付标准

## 2. 适用范围

适用于：

- 概览
- 埋点需求
- 数据字典
- 校验规则
- 日志查询
- 验收报告
- AI 助手

不适用于：

- 营销官网风格页面
- 大屏展示页
- 杂志化视觉实验稿

## 3. 总体风格

- 风格类型：常规 B 端企业平台
- 视觉基调：蓝白主体系
- 设计目标：清晰、稳定、克制、可还原
- 默认倾向：先信息效率，后装饰表达
- 页面气质：生产级业务平台，不做概念稿、示意稿、宣传稿

## 4. 字体与字号

### 4.1 字体

- 全局统一使用：`Microsoft YaHei` / `微软雅黑`

### 4.2 字号层级

- 品牌主标题：`20`
- 一级导航 / 一级区块标题：`16`
- 正文 / 普通指标说明：`14`
- 表格正文 / 次级说明：`12`
- 标签 / 轻量辅助信息：`10`

### 4.3 字重

- 默认不使用加粗强调
- 正文、标题、指标值统一优先使用 `400`
- 不通过字重拉开主次，主次改用字号、颜色、间距、位置表达

### 4.4 文字颜色分层

- 主信息：深色
- 次信息：中灰
- 辅助信息：浅灰

禁止：

- 用大面积加粗制造重点
- 用高饱和颜色代替信息层级

## 5. 颜色体系

### 5.1 主色体系

- 主色：蓝
- 背景：白 / 浅灰白
- 常规边框：浅灰蓝

### 5.2 状态色

- 失败：红
- 告警：黄
- 提示：蓝
- 成功：绿

推荐主品牌蓝：

- `#3B86F9`

推荐主文字色：

- `#252934`

推荐辅助说明色：

- `#6B7981`

### 5.3 使用原则

- 状态色只用于状态表达
- 默认不做大面积高饱和底色块
- 标签、提示、徽标优先使用低饱和浅底色
- 指标卡默认使用中性底色，不因 `P0` 单独切成红底

禁止：

- 用大红底、大黄底作为普通信息背景
- 同一页面同时出现过多高饱和标签色

## 6. 圆角、边框、阴影

### 6.1 圆角

- 全局默认圆角：`2`

### 6.2 边框

- 默认细边框
- 边框颜色以浅灰蓝为主

### 6.3 阴影

- 仅允许轻阴影
- 不做浮夸悬浮卡片效果

## 7. 图标规范

- 图标库统一：`Ant Design Icons`
- 图标尺寸默认与当前文本行高协调
- 非主视觉图标默认保持轻量，不可压过文本
- 图标仅做识别辅助，不做装饰中心

禁止：

- 图标尺寸明显大于相邻文本视觉高度
- 同一区域混用风格不一致的图标
- 使用占位图标提交最终交付稿

## 8. 布局与密度

### 8.1 页面布局

- 默认后台平台布局
- 顶部操作区固定
- 左侧导航固定
- 主内容区按页面类型展开

### 8.2 信息密度

- 采用中密到中高密
- 保证业务判断效率优先
- 不做大面积留白型视觉稿

### 8.3 间距

- 以稳定、规则、可对齐为原则
- 同层级区块使用统一间距
- 标题、卡片、表格、图表之间保持稳定节奏

### 8.4 页面页眉

平台型页面默认页眉规则：

- 仅标题页眉：最小高度 `60px`
- 含切换和筛选项页眉：最大高度 `110px`
- 页面名称建议字号：`18px`
- 页面辅助说明建议字号：`12px`

当前项目覆盖：

- 概览页最终交付稿不在正文重复展示解释型辅助说明

## 9. 组件表达规则

### 9.1 顶部操作

- 只放已确认的业务操作
- 不额外附加解释性文字
- 默认直接展示选择值或结果值

### 9.2 左侧导航

- 文案顺序必须锁死
- 不得因视觉原因随意增删重排

### 9.3 指标卡

- 默认中性卡片表达
- 不使用大色块抢占页面重心
- 重点通过数字位置、标题顺序、文字颜色表达

### 9.4 标签与徽标

- 默认浅底、低饱和
- 优先表达状态，不做视觉装饰
- 避免同屏过多彩色标签

### 9.5 表格

- 表格是核心阅读器之一
- 表头轻、正文清晰、状态可扫读
- 不通过粗字和重底色制造压力

推荐表格基线：

- 基础表格行高：`40px`
- 素材类或高信息量表格行高：`60px`
- 分页默认值：`20 / 50 / 100`
- 表头固定：表格超出一屏时顶部固定
- 表头背景：`#F9FAFC`
- 行悬停背景：`#FAFAFA`

列规则：

- 日期列左对齐
- 维度列左对齐
- 指标列右对齐
- 最小列宽：`100px`

标题规则：

- 默认单行按 `4` 个中文字符适配
- 允许最多 `2` 行
- 超出后省略并支持悬停查看完整内容

当前项目覆盖：

- 不使用表头加粗作为主强调手段
- 通过颜色与间距区分层级

### 9.6 图表

- 图表只表达业务判断所需信息
- 不做大屏化装饰
- 图例、坐标、口径说明必须清楚

通用可视化规则：

- 单指标多日期、多指标多日期、多维度对比应使用统一色板
- 图表颜色先服从主品牌蓝及系统状态色，再扩展辅助色
- 环形图、构成图不使用高饱和彩虹配色

### 9.7 筛选与切换

筛选输入默认状态：

- 默认
- 悬停
- 点击
- 禁用

默认推荐色值基线：

- 默认边框：`#E7E7E7`
- 激活高亮：`#3B86F9`
- 禁用背景：`#F9FAFC`

单选与复选规则：

- 单选按钮尺寸：`12 * 12`
- 多选按钮尺寸：`12 * 12`
- 单组选项建议间距：`30px`

输入下拉尺寸建议：

- 最小宽度：`150px`
- 中等宽度：`200px`
- 最大宽度：`250px`
- 默认高度：`30px`

页面切换类组件应优先收敛为：

- 项目切换
- 页面切换
- 报表页切换

不混入解释性文案

## 10. 文案规则

- 页面只展示业务必要文案
- 禁止解释型长文案落到最终交付页面
- 描述型上下文不重复展示

例如：

- `当前项目`
- `当前版本`
- `当前视角`

这类文案如果已由切换器本身承担，不再重复写在正文区。

## 11. 状态表达规则

- 状态是交互的一部分，不单独做成“状态稿模块”挂在页面正文里
- 正常态、空态、加载态、异常态、无权限态应作为交互态存在
- 静态HTML确认稿允许保留实现承接所需的状态模板，但不得破坏页面主稿

## 12. 页面交付规则

- `静态HTML确认稿` 是最终交付级产品前端
- 它是交给后端一比一还原的直接依据
- 它不是示意稿，不是演示稿，不是说明稿

提交前必须满足：

- 文案锁死
- 导航锁死
- 顶部操作锁死
- 图标可交付
- 组件语义明确
- 状态语义完整

## 13. 组件库对齐规则

最终交付默认按 `YokaUI` 组件语义对齐。

至少覆盖：

- 页面容器：`YkContainer`
- 栅格布局：`FlexGrid`
- 图表：`YkCharts`
- 表格主题：`TableTheme`
- 提示：`YkTooltip`
- 空态：`Empty`
- 图标：`Ant Design Icons`

若无现成组件：

- 标记为 `待补组件确认`
- 不凭空发明内部组件名

## 13.1 YokaUI 之外的通用组件设计规则

按钮类：

- 主按钮仅用于主流程动作
- 默认、悬停、禁用态必须完整

输入类：

- 单行输入
- 密码输入
- 多行输入
- 数字输入
- 键值输入

日期类：

- 年月日范围
- 时分秒范围
- 年月日时分秒组合
- 快捷时间筛选
- 时间间隔筛选

筛选类：

- 单选可输入
- 多选可输入
- 多级联选
- 报表筛选下拉
- 自定义列筛选

这些属于平台默认可复用能力，不需要每页重新发明交互。

## 14. 当前已锁定的专项约束

- 重点优先使用黑 / 灰 / 浅灰拉层级
- 标签用色偏淡
- 图标与文本高度必须协调
- 描述型文案不重复展示

## 16. 05-07 UI设计反馈落地补充

本节用于把 2026-05-07 的 UI 设计反馈正式沉入通用设计规范，作为后续页面整改默认遵守的补充规则。

### 16.1 页面整改执行顺序

后续页面整改默认采用以下顺序：

1. 现状诊断
2. 与通用设计规范比对
3. 明确差异项
4. 再进入页面修改

不再默认直接改样式。

### 16.2 页面风格边界

继续统一采用：

- B 端企业后台风格
- 蓝白主色体系
- 高信息密度但可读优先
- 组件优先复用，不重新创造一套局部风格

不建议继续扩散：

- 监控大盘式视觉
- 营销官网式卡片堆叠
- 说明性长文案回流到页面正文
- 为视觉装饰牺牲表格可读性

### 16.3 概览页规范补充

概览页继续保持同步页定位，页面层不回退为明细堆砌页。后续整改默认遵守：

- 版本目标、核心指标、最近更新、验收趋势、风险摘要作为主骨架
- 不回加大段解释文案
- 不把风险区重新扩为明细中心
- 不按实时监控大盘方式扩展

### 16.4 实时查询规范补充

实时查询页固定为三段式：

1. 筛选条件
2. 分布区域
3. 日志明细

页面优化默认围绕：

- 筛选器一致性
- 区块紧凑度
- 局部滚动
- 证据导向图表

不再向监控首页方向扩展。

### 16.5 验收报告规范补充

验收报告固定为双层承载：

- 列表页默认承载简报卡片
- 详情页承载详细报告

因此后续报告整改默认遵守：

- 列表卡片暴露核心结论与同步信息
- 详情页暴露结构化明细
- 报告页不回退为普通纯表格列表

### 16.6 数据字典规范补充

数据字典继续按以下结构表达：

- 参数主表
- 枚举值展开子表
- 参数详情抽屉

并继续保持：

- 页面用于查阅、联动、变更可视化
- 上传覆盖更新为正式更新入口
- 不做页面内在线 Excel 式编辑
