# AI埋点平台-概览页-Pixso高保真视觉稿输入附件-v1

日期：2026-05-05

## 1. 文档定位

本文档是 `Pixso高保真视觉稿` 的输入附件。

它只负责给设计责任人或 AI 提供出稿上下文，不单列为正式阶段，不承担确认稿职责。

概览页正式设计链路固定为：

1. `原型确认稿`
2. `交互确认稿`
3. `Pixso高保真视觉稿`
4. `视觉/UI确认稿`
5. `静态HTML确认稿`
6. `实现承接稿`

## 2. 已确认上游

出稿时必须以上游已确认产物为准：

- `AI埋点平台-概览页-原型确认稿-v2.md`
- `AI埋点平台-概览页-交互确认稿-v1.md`

## 3. 页面边界

页面定位固定为：

- `版本验收态势页`

页面不承载：

- 其他页面引导集合
- AI 助手建议入口
- 复杂编辑动作
- 批量执行工作台

## 4. 固定结构

页面固定结构为：

1. 全局上下文
2. 版本验收态势
3. 版本目标与门槛
4. 当前进度摘要
5. 更新影响摘要
6. 进度趋势
7. 当前主要风险

不允许改动：

- 不改变结构顺序
- 不让指标卡上升为首屏第一视觉层
- 不把更新摘要提升成页面主区
- 不把风险区扩成详情中心

## 5. 关键视觉约束

- 态势区必须是首屏第一视觉层
- 页面保持企业级数据平台风格
- 使用中密信息密度、浅底深字、弱装饰强结论
- 指标区固定为 `4 + 2`
- `P0` 卡必须显式区别于普通指标卡
- 趋势区固定为 `目标线 + 实际线 + 口径提示`
- 风险区固定为 `不通过事件 TOP5 + 待验收埋点 TOP5` 两个摘要表

## 6. 文案与内容约束

- 使用真实产品短文案
- 使用接近真实业务的示例数据
- 不使用大段解释性页面文案
- 不把文档章节标题原样带进页面
- 不在区块标题前加编号
- 不把“为什么这样设计”直接写在页面里

## 7. Pixso 出稿提示词

```text
为 AI 埋点平台设计一个企业级数据平台风格的概览页高保真视觉稿。
页面定位是“版本验收态势页”，不是导航页，不是工作台，不是营销官网页。

必须基于已确认的原型确认稿和交互确认稿出稿，不新增菜单，不改页面主链，不引入 AI 助手建议。

固定结构：
全局上下文、版本验收态势、版本目标与门槛、当前进度摘要、更新影响摘要、进度趋势、当前主要风险。

关键要求：
- 态势区必须是首屏第一视觉层
- 指标区固定为 4+2
- P0 风险卡必须显式区别于普通指标
- 更新影响摘要必须降权
- 趋势区必须是目标线 + 实际线，并附带口径说明
- 风险区必须是两个摘要表，不扩成详情中心

整体风格：
企业级数据平台、中密信息密度、浅底深字、弱装饰强结论、稳定克制、可扫读。

禁止：
- 区块标题带编号
- 大段解释性页面文案
- 说明稿式页面
- 数据大屏风格
- 运营导航页风格
- 复杂执行动作入口
```
