# AI埋点平台-概览页-CodexUI设计初稿-v1

日期：2026-05-05

## 1. 文档定位

本文档是概览页的 `CodexUI设计初稿`。

它的职责是：

- 在已确认的 `原型确认稿` 与 `交互确认稿` 基础上，给出第一版 UI 视觉方向
- 为 `Pixso高保真视觉稿` 提供可直接参考的布局、层级、组件表达和文案方向

它不是：

- 最终视觉定稿
- 视觉/UI确认稿
- 静态HTML确认稿

## 2. 已吸收的上游约束

本稿已吸收以下已确认产物：

- `AI埋点平台-概览页-原型确认稿-v2.md`
- `AI埋点平台-概览页-交互确认稿-v1.md`

核心继承结果：

- 页面定位固定为 `版本验收态势页`
- 页面主链固定为 `上下文 -> 态势 -> 目标 -> 进度 -> 变更 -> 趋势 -> 风险`
- 概览页不承载 `AI 助手建议`、复杂编辑、批量执行和其他页面引导

## 3. 初版视觉方向

### 3.1 总体气质

- 企业级数据平台
- 中密信息密度
- 浅底深字
- 弱装饰、强结论
- 稳定、克制、可扫读

### 3.2 首屏策略

- 第一视觉层必须是 `当前态势`
- 第二视觉层是 `版本目标`
- 第三视觉层是 `4 + 2` 进度摘要

不允许：

- 指标卡抢占态势区
- 更新摘要前置到首屏第一层
- 风险区做成详情页中心

## 4. 页面结构建议

### 4.1 全局上下文区

- 顶部保留产品标识、项目、版本、视角
- 项目 / 版本 / 视角使用同一组浅底卡片式切换器
- 不带编号标题

### 4.2 当前态势区

- 使用大卡或主结论卡
- 必须同时容纳：
  - 当前态势判断
  - 一句话原因
  - 目标偏差 / 时间偏差 / 责任聚焦

### 4.3 版本目标区

- 使用规整字段卡
- 字段固定：
  - 版本号
  - 验收范围
  - 时间范围
  - 通过门槛

### 4.4 验收进度区

- 固定为 `4 + 2`
- 前四卡为：
  - 埋点数量
  - 已验收
  - 待验收
  - 通过率
- 后两卡为风险卡：
  - P0 埋点数量
  - P0 待验收

### 4.5 最近变更区

- 只做轻量摘要
- 仅保留影响当前结论的 3 条
- 视觉权重低于进度区与风险区

### 4.6 趋势区

- 固定使用 `目标线 + 实际线`
- 图旁常驻口径说明
- 整体背景克制、干净

### 4.7 风险区

- 双摘要表并排
- 左侧：`不通过事件 TOP5`
- 右侧：`待验收埋点 TOP5`
- 不展开成完整详情中心

## 5. 初版组件表达建议

### 5.1 切换器

- 浅底
- 圆角
- 边框清晰
- 选中态同时用底色和边框变化表达

### 5.2 态势卡

- 高识别度
- 可用蓝底主卡承载主结论
- 风险标签和偏差指标需有明显层次

### 5.3 指标卡

- 普通指标卡统一体系
- `P0` 风险卡使用风险语义色，与普通指标明显区分

### 5.4 摘要表

- 表头轻
- 行高中密
- 优先级、状态标签一眼可扫

## 6. 文案约束

- 使用真实产品短文案
- 使用接近真实业务的示例数据
- 不写解释型长段落
- 不把文档章节号和说明句搬进页面

## 7. 禁止项

- 禁止区块标题带编号
- 禁止线框稿式占位表达
- 禁止说明稿式页面
- 禁止数据大屏化
- 禁止营销官网化
- 禁止 AI 建议模块进入概览页
