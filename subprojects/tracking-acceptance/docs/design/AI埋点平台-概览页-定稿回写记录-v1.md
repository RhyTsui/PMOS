# AI埋点平台-概览页-定稿回写记录-v1

日期：2026-05-06

## 1. 文档定位

本文档用于承接概览页在最终交付前端定稿后，相对历史 `原型确认稿`、`交互确认稿`、`CodexUI设计初稿` 的生效差异。

本文档的作用是：

- 作为当前概览页的补充真源
- 避免历史稿继续与当前前端交付结果漂移
- 为后续 `产品设计工程包` 和后端一比一还原提供当前口径

当前概览页的实际交付基线为：

- `subprojects/tracking-acceptance/frontend/src/App.jsx`
- `subprojects/tracking-acceptance/frontend/src/styles/app.css`
- `subprojects/tracking-acceptance/frontend/src/data/overviewData.js`

## 2. 当前定稿结论

概览页当前已按“无后端接入的最终交付前端页面”定稿。

这意味着：

- 页面结构已锁
- 视觉风格已锁
- 前端交互已锁
- mock 数据口径已锁
- 页面用于产品确认、业务评审与后端一比一还原

## 3. 顶部区域定稿口径

当前顶部只保留 4 个入口：

1. 项目切换
2. 验收版本
3. 用户视角
4. 个人中心

补充约束：

- `项目切换` 使用内部统一组件 `YkPorjectSelect`
- `YkPorjectSelect` 当前页面已去掉 `在营 / 关服` 展示层
- `验收版本`、`用户视角` 使用原生 `LabelSelect`
- 两个 `LabelSelect` 的左侧是组件标签，右侧仅展示真实值
- 顶部为深蓝一体化头带，logo 区与顶部栏已合并为一体

## 4. 页面结构定稿口径

当前正式页面结构为：

1. 顶部上下文切换区
2. `版本目标` 复合块
3. `最近更新`
4. `验收趋势`
5. `不通过埋点`
6. `未上报埋点`

其中：

- 原历史稿中的 `验收态势` 独立模块已删除
- 原历史稿中的独立 `验收进度` 模块已并入 `版本目标` 复合块

## 5. 版本目标复合块定稿口径

当前首屏正式结构不是“目标块 + 单独指标区”，而是一个复合块：

- 左侧：
  - 验收版本
  - 验收周期
  - 版本目标
  - 验收范围
- 右侧：
  - 埋点数量
  - 已验收
  - 不通过
  - 未上报

补充约束：

- 右侧 4 个指标卡是页面当前正式指标集合
- `未上报` 使用紫色语义
- `不通过` 使用黄色指标卡语义，但表格状态中 `不通过` 仍为红色风险态

## 6. 中下区域定稿口径

### 6.1 最近更新

- 当前正式命名：`最近更新`
- 固定展示 5 行
- 每行单行省略
- 超出内容使用 tooltip 展示完整文案
- `更多` 跳转至：`埋点需求 -> 版本管理`

### 6.2 验收趋势

- 当前正式命名：`验收趋势`
- 使用真实前端图表组件，不再使用示意 svg
- 展示 X / Y 轴
- Y 轴显示整数百分比，不保留小数
- 展示 4 条横向网格线

### 6.3 不通过埋点

- 当前正式命名：`不通过埋点`
- 去掉 `TOP5` 文字
- `更多` 跳转至：`日志查询 -> 不通过事件`

### 6.4 未上报埋点

- 当前正式命名：`未上报埋点`
- 去掉 `TOP5` 文字
- `更多` 跳转至：`日志查询 -> 待验收事件`

## 7. 状态与优先级定稿口径

### 7.1 状态

- `不通过`：红色
- `未上报`：蓝色

### 7.2 优先级

- `P0`：红色
- `P1`：黄色
- `P2`：蓝色
- `P3`：绿色

当前页面默认示意排序：

- 前 4 条为 `P0`
- 第 5 条为 `P1`

## 8. 与历史稿的主要差异

相对历史 `原型确认稿`、`交互确认稿` 和 `CodexUI设计初稿`，当前生效差异包括：

- 删除 `验收态势` 独立模块
- 删除独立 `验收进度` 模块
- `通过率趋势` 收敛为 `验收趋势`
- `最近变更 / 版本更新 / 变更通知` 最终收敛为 `最近更新`
- `不通过事件 TOP5` 收敛为 `不通过埋点`
- `待验收埋点 TOP5` 收敛为 `未上报埋点`
- 状态稿不再作为正文独立模块展示
- 当前最终交付物从“静态示意 HTML”升级为“真实前端交付页”
