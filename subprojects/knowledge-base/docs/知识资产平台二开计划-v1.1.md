# 知识资产平台二开计划 v1.1

- 版本：v1.1
- 日期：2026-04-21
- 状态：规划中
- 适用范围：v1.0 基础能力上线后的正式产品化改造阶段

## 1. 背景与目标

v1.0 的重点是把基础能力搭起来，验证知识处理、检索、模型接入、MCP 接入等底层链路可用。  
v1.1 不再是补单点功能，而是正式进入产品化二开阶段。

v1.1 的核心目标不是“继续做一个知识库”，而是把现有能力改造成一个真正可用的知识资产平台，并形成统一的工作入口。

v1.1 要解决的核心问题：

1. 让企业知识和个人知识都能在同一个平台里被管理和调用
2. 让知识资产和提示词资产都能成为 AI 工作流里的可用资源
3. 让主对话框成为统一工作入口，而不是单独的问答页
4. 让现有 WeKnora 从“可运行产品”升级为“适配当前业务的产品底盘”

## 2. 平台定位与产品结构

### 2.1 平台定位

平台名称：`小荀 · 知识资产平台`

一句话定位：

面向企业与个人双场景，统一管理知识资产与提示词资产，并通过主对话框、业务 Chat、MCP、工作流等入口，把这些资产持续供给给 AI 使用。

这里不再区分“企业优先、个人补充”或“个人优先、企业扩展”，而是明确：

- 企业知识和个人知识并重
- 企业场景解决业务补强和问答支持
- 个人场景解决长期上下文沉淀和随处调用

### 2.2 产品主结构

v1.1 的产品结构按一级并列菜单表达：

1. 主对话框
2. 知识资产
3. 提示词资产
4. 接入中心
5. 知识广场
6. 权限管理

说明：

- 主对话框是统一工作入口
- 知识资产和提示词资产是并列核心能力
- 接入中心负责持续把内容收进来
- 知识广场负责发现、订阅和分发
- 权限管理负责统一账号、访问与调用范围

### 2.3 首页与整体入口叙事

v1.1 的首页和登录后整体叙事，采用“工作入口优先”。

也就是：

- 首页不是先讲平台宣传
- 首页也不是先讲知识库概念
- 首页应该直接回答“我进来能做什么”

首屏应突出：

1. 主对话框可以做什么
2. 可以调用哪些知识与提示词
3. 可以接入哪些内容
4. 可以支持哪些业务分析和问答场景

## 3. WeKnora 二开策略

### 3.1 二开路线

v1.1 采用：`基于 WeKnora 现成产品做改造`

这不是把 WeKnora 仅仅当成一个底层 SDK，也不是完全重做一套平台，而是：

- 以 WeKnora 现成产品能力为基础
- 保留其已有的知识库、文档解析、检索、问答、模型配置等现成能力
- 在其之上改造产品结构、业务表达、资产模型和工作入口

### 3.2 直接沿用的能力

v1.1 可以直接沿用 WeKnora 的能力：

1. 知识库基础能力
2. 文档导入与解析能力
3. 切片、索引、检索、引用返回
4. 模型配置与基础推理链路
5. MCP 接入和外部能力调用框架
6. 基础权限与组织管理结构

### 3.3 重点改造的能力

v1.1 重点改造的不是底层检索，而是产品层和业务层：

1. 登录后的整体信息架构
2. 首页和主对话框入口
3. 企业 / 部门 / 项目 / 个人知识库组织方式
4. 提示词资产管理入口与管理方式
5. 接入中心和来源管理
6. 知识广场和知识分发
7. 权限表达与按库配置方式
8. 主对话框中的模型 / MCP / Skill / 知识挂载体验

### 3.4 v1.1 不做的事

v1.1 明确不做：

1. 不做 Agent / Workflow 编排平台
2. 不做完整的录音转写产品
3. 不追求 NotebookLM / Notion 级知识洞察效果
4. 不重写 WeKnora 整个底层能力体系
5. 不把测试平台、评测平台整合进主平台

## 4. v1.1 范围与交付物

### 4.1 核心业务范围

v1.1 的核心业务范围包括：

1. 企业知识库
2. 部门知识库
3. 项目知识库
4. 个人知识库
5. 提示词资产管理

支持接入的来源范围：

1. 会议纪要
2. 外部知识库
3. 小闪文档
4. 小闪聊天记录转发
5. 外部订阅
6. 自然语言爬虫任务

支持的内容格式：

1. PDF
2. Word
3. Excel
4. 图片
5. 视频
6. URL
7. 脚本包

### 4.2 主对话框能力范围

主对话框在 v1.1 里是核心工作入口，需要支持：

1. 直接与知识对话
2. 跨个人、他人、部门、公司知识库联合分析
3. 调用 Skill 做专项分析
4. 调用 MCP 做外部能力分析
5. 访问外部网络和外部知识
6. 支持文本、图片、文件、视频、语音、会议 URL 输入
7. 提供最近会话
8. 提供使用建议与引导

配置策略：

- 默认关闭高级配置
- 默认使用系统内置模型和系统分配 MCP
- 打开高级配置后，再允许手动切换模型、MCP 和策略

### 4.3 提示词资产范围

v1.1 明确把提示词资产纳入平台主计划，但不再写成知识库下二级附属能力。

提示词资产在 v1.1 的定位：

1. 作为一级并列菜单出现
2. 作为平台的并列核心资产能力
3. 可以供主对话框、业务 Chat、MCP、后续 Agent 使用

首版重点是：

1. 分类管理
2. 基础编辑
3. 版本管理
4. 调用挂载
5. 与知识资产的组合使用

### 4.4 交付物

v1.1 二开的主要交付物：

1. 一级并列菜单信息架构
2. 首页与登录后工作入口重构
3. 主对话框增强版
4. 知识资产管理页
5. 提示词资产管理页
6. 接入中心骨架
7. 知识广场骨架
8. 权限管理与按库配置能力
9. 企业 / 部门 / 项目 / 个人知识库闭环

## 5. 研发拆分与阶段安排

### 5.1 前端

前端二开重点：

1. 登录页与首页改造
2. 主对话框能力增强
3. 知识资产列表、详情、管理页重构
4. 提示词资产页面搭建
5. 接入中心与知识广场搭建
6. 弹窗、抽屉、分类、表单等交互细化

### 5.2 后端

后端二开重点：

1. 资产组织方式扩展
2. 按知识库的权限模型扩展
3. 接入任务抽象
4. 消费接口标准化
5. MCP / 模型 / 外部能力白名单与调用链路稳定化
6. 会议纪要等特殊资产的结构化接入

### 5.3 联调优先顺序

优先打通三条链路：

1. 企业 / 部门 / 项目 / 个人知识库创建与消费
2. 主对话框挂载知识与提示词进行分析
3. 模型、MCP、文档解析、URL 解析链路跑通

### 5.4 验收标准

v1.1 的验收重点不是功能点数量，而是结构是否搭对。

验收标准：

1. 页面结构已经体现一级并列菜单
2. 主对话框已经成为统一工作入口
3. 知识资产和提示词资产都能被管理与调用
4. 企业和个人两类知识场景都能被覆盖
5. WeKnora 原有能力被接住，且产品层改造方向清晰

## 6. 结论

v1.1 的二开不是简单“在 WeKnora 上加几个页面”，也不是“重做一套新系统”，而是：

1. 在 WeKnora 现成产品基础上做面向当前业务的改造
2. 把平台从知识问答产品升级为知识资产平台
3. 把知识资产、提示词资产、主对话框、接入中心、知识广场和权限管理整成一套统一产品结构

一句话结论：

`v1.1 = 基于 WeKnora 现成产品做平台化改造，形成企业与个人并重、知识与提示词并列、主对话框优先的知识资产平台。`
