## 一、v0.1 MVP 版本目标

### 1.1 版本定位

知识库 v0.1 是一个**最小可行产品**，聚焦核心能力：
- **权限管理**：支持团队、用户、Agent 的权限控制
- **知识接入**：支持文档导入和自动采集
- **Agent API**：支持 Agent 注册和知识查询
- **后台管理**：知识库管理页面（无小闪 Chat 前台）

### 1.2 不做范围（v0.1）

- ❌ 小闪 Chat 前台（v1.0 规划）
- ❌ 知识图谱可视化（v1.0 规划）
- ❌ 混合检索/Rerank（v1.0 规划）

### 1.3 知识构建目标

| 知识类型 | 具体内容 | 接入方式 | 负责人 |
|----------|----------|----------|--------|
| **媒体文档** | 巨量/腾讯/快手 API 文档 | 爬虫自动采集 | TBD |
| **广告业务流程** | 投放流程、需求流程文档 | 手动录入 | TBD |
| **游戏数据上报** | 游戏项目数据上报需求 | 手动录入 | TBD |

## 二、用户场景

### 场景 A：管理员创建团队并邀请成员
管理员在后台创建团队，邀请成员加入，设置管理员/编辑/查看者角色。

### 场景 B：用户上传业务文档
编辑角色用户在后台上传广告业务流程文档，系统自动建立向量索引。

### 场景 C：Agent 注册并查询知识
广告优化 Agent 通过 API 注册，声明需要访问媒体文档和广告业务知识，调用查询 API 获取投放策略。

### 场景 D：用户按权限查看知识库
普通用户登录后，只能看到有权限的知识库和文档，无权限的文档不可见。

### 场景 E：媒体文档自动采集
系统定时爬取巨量引擎/腾讯广告官方 API 文档，自动更新到知识库。

## 三、核心功能

### 3.1 权限管理（P0）

**功能描述**：支持团队、用户、Agent 的权限控制。

**核心机制**：
- **应用可见权限**：控制用户可以看到哪些应用/知识库
- **数据分类权限**：控制用户可以查看/编辑/删除哪些数据
- **角色体系**：管理员（Owner）、编辑（Editor）、查看者（Viewer）

**权限矩阵**：

| 角色 | 查看 | 编辑 | 删除 | 邀请成员 | 权限管理 |
|------|------|------|------|----------|----------|
| 管理员 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 编辑 | ✅ | ✅ | ❌ | ❌ | ❌ |
| 查看者 | ✅ | ❌ | ❌ | ❌ | ❌ |

**Agent 权限**：
- Agent 使用统一权限控制
- 注册时声明知识范围（knowledge_scope）
- 查询时自动匹配权限范围

### 3.2 团队管理（P0）

**功能描述**：支持创建团队、邀请成员、管理角色。

**核心机制**：
- 创建团队：填写团队名称、描述
- 邀请成员：通过链接或邀请码
- 角色分配：管理员可修改成员角色
- 成员列表：查看团队成员和角色

**数据结构**：
```json
{
  "team_id": "team-001",
  "name": "广告业务组",
  "description": "负责广告业务自动化",
  "owner_id": "user-001",
  "members": [
    {"user_id": "user-001", "role": "owner"},
    {"user_id": "user-002", "role": "editor"},
    {"user_id": "user-003", "role": "viewer"}
  ],
  "created_at": "2026-04-14T00:00:00Z"
}
```

### 3.3 知识接入（P0）

**功能描述**：支持文档导入和自动采集。

**接入方式**：
- **手动录入**：支持 Markdown/DOCX 上传
- **自动采集**：爬虫采集媒体官方文档

**知识分层**：
```
L1 - 原始文档层：媒体官方文档、API 文档、政策文件
L2 - 结构化层：提取的知识点、规则、参数定义
L3 - 语义层：统一术语、概念关系、业务语义
L4 - 应用层：问答对、场景方案、最佳实践
```

**向量索引**：
- 文档上传后自动建立向量索引
- 支持语义检索

### 3.4 文档管理（P0）

**功能描述**：支持文档的增删改查。

**功能清单**：
- 上传文档：支持 Markdown/DOCX
- 编辑文档：在线编辑
- 删除文档：软删除（进入回收站）
- 查看文档：按权限查看
- 知识分层：设置文档的 L1-L4 层级

**非功能要求**：
- 查看者只能查看，不能编辑/删除
- 编辑可以编辑，不能删除
- 管理员可以查看/编辑/删除

### 3.5 术语表（P1）

**功能描述**：统一管理广告行业术语。

**核心机制**：
- 术语录入：名称、定义、分类、同义词
- 术语查询：支持搜索
- 术语编辑：管理员/编辑可修改

**示例**：
```json
{
  "term_id": "term-001",
  "name": "oCPM",
  "definition": "优化千次展现出价（Optimized Cost Per Mille）",
  "category": "出价",
  "synonyms": ["智能 oCPM", "自动 oCPM"],
  "related_terms": ["oCPC", "oCPA", "CPM"]
}
```

### 3.6 Agent API（P0）

**功能描述**：支持 Agent 注册和知识查询。

**核心接口**：
- `POST /api/v1/agents/register` - Agent 注册
- `GET /api/v1/agents/:id` - 获取 Agent 信息
- `POST /api/v1/agents/:id/query` - Agent 查询知识
- `DELETE /api/v1/agents/:id` - 注销 Agent

**注册数据结构**：
```json
{
  "agent_id": "agent-001",
  "name": "广告优化 Agent",
  "description": "负责广告投放优化",
  "capabilities": ["投放策略", "创意优化"],
  "knowledge_scope": ["媒体文档", "广告业务"],
  "api_key": "sk-xxx",
  "status": "active"
}
```

### 3.7 检索 API（P0）

**功能描述**：支持语义检索和精确查询。

**核心接口**：
- `POST /api/v1/search` - 语义检索
- `GET /api/v1/terms/:name` - 术语精确查询

**检索请求**：
```json
{
  "query": "oCPM 是什么意思",
  "type": "auto",
  "limit": 10,
  "layers": ["L1", "L2", "L3", "L4"]
}
```

**检索响应**：
```json
{
  "results": [
    {
      "id": "term-001",
      "type": "term",
      "title": "oCPM",
      "content": "优化千次展现出价...",
      "score": 0.95,
      "source": "广告行业术语表"
    }
  ],
  "suggested_questions": ["oCPM 和 oCPC 有什么区别？"]
}
```
