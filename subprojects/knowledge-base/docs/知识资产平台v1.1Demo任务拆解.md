# 知识资产平台 v1.1 Demo任务拆解

- 版本：v1.1 Demo
- 日期：2026-04-20
- 状态：执行拆解

## 1. 拆解原则

Demo 任务围绕 4 个目标组织：

1. 接入成立
2. 治理成立
3. 消费成立
4. 演示成立

所有任务按“能支撑主场景演示”为标准，不做额外扩张。

## 2. 任务分组

### A. 底座准备

#### A1. WeKnora 部署与可用性验证

目标：

- 跑通 WeKnora 最小可用环境

产出：

- 本地可用底座
- 检索链路可验证

#### A2. 检索与引用验证

目标：

- 验证知识检索与引用片段返回

产出：

- 检索样例
- 引用返回样例

### B. 平台适配层

#### B1. 统一资产对象定义

目标：

- 定义 `asset`、`asset_type`、`source`

产出：

- Demo 层对象模型

#### B2. 权限对象定义

目标：

- 定义 `permission_policy`

产出：

- 可见/可引用/可编辑权限结构

#### B3. 消费绑定对象定义

目标：

- 定义 `consumer_binding`

产出：

- Chat 与资产绑定结构

### C. 接入样例

#### C1. 文本文档接入

目标：

- 支持文本导入并生成资产

#### C2. 会议纪要文本接入

目标：

- 支持会议纪要文本解析成资产

#### C3. 内部文档样例接入

目标：

- 支持企业文档类样例导入

#### C4. 项目系统样例接入

目标：

- 支持项目信息样例导入

### D. 资产展示

#### D1. 资产列表样例

目标：

- 能浏览和筛选资产

#### D2. 资产详情样例

目标：

- 能展示原文、结构化内容、来源、权限

#### D3. 会议纪要详情样例

目标：

- 能展示纪要字段与结构化结果

### E. 消费样例

#### E1. 平台内 Chat 样例

目标：

- 平台内 Chat 能使用知识资产

#### E2. 数据分析 Chat 样例

目标：

- 验证数据分析 Chat 的知识补强

#### E3. 广告业务 Chat 样例

目标：

- 验证广告业务 Chat 的知识补强

### F. 知识广场

#### F1. 广场首页样例

目标：

- 展示推荐、热门、专题

#### F2. 订阅动作样例

目标：

- 验证资产加入个人范围

### G. 修正与治理

#### G1. 权限配置样例

目标：

- 演示不同权限对结果的影响

#### G2. 修正样例

目标：

- 演示摘要/标签/纪要字段修正

#### G3. 引用记录样例

目标：

- 展示资产被谁消费、如何被引用

## 4. 必测任务

必须验证：

1. 不同来源都能进入统一资产模型
2. 不同权限拿到不同结果
3. 业务 Chat 能成功调用知识补强
4. 会议纪要能完成结构化入库
5. 知识广场能完成发现与订阅

## 5. 拆解结论

这份任务拆解优先保证：

- Demo 可跑
- 场景可讲
- 平台框架可被验证

不把精力消耗在首版非核心深能力上。
