# Dataki v1.2 Prompt 知识资产管理补充需求文档
- 版本：v1.2
- 日期：2026-04-28
- 状态：Draft
- 适用范围：Dataki Prompt 知识资产管理补充需求
- 读者对象：产品、设计、前端、后端、测试、AI

## 1. 版本定位

本文件是 Dataki 在现有系统基础上，对 `Prompt 知识资产管理` 的补充需求定义。

本次版本直接按 `v1.2` 管理，不再继续挂在 `v1.1` 补充口径下。

`v1.2` 的目标不是新增一套 Prompt 中台，而是：
1. 把 Prompt 正式纳入 Dataki 统一资产体系。
2. 让 Prompt 在产品结构上成为与 `知识库`、`智能体` 并列的一级菜单。
3. 让 Prompt 的共享方式、权限方式、调用方式与知识库保持一致。
4. 让 Prompt 可被主对话框、业务 Chat、外部 Agent 在受控条件下使用。

## 2. 背景与当前策略

当前 Dataki 已经形成以下真实策略：
1. 以 `空间` 作为协作载体。
2. 以 `分享到空间` 作为资产共享动作。
3. 以 `邀请 / 加入空间` 作为成员进入协作关系的方式。
4. 以 `空间内角色` 与 `共享权限` 共同决定成员实际权限。

因此 Prompt 的补充需求必须遵守两条：
1. 菜单结构上，Prompt 与知识库、智能体并列。
2. 权限机制上，Prompt 与知识库复用同一套空间共享逻辑。

这也意味着：
1. Prompt 不再挂在知识库下面做二级资产入口。
2. Prompt 不再单独走“部门权限模型”。
3. 如果业务有部门协作诉求，应通过部门对应的空间来承载，而不是给 Prompt 发明另一套规则。

## 3. 产品定位

### 3.1 定位结论

Prompt 在 Dataki v1.2 中定位为：
1. 一级菜单模块
2. 一种正式资产
3. 可共享、可绑定、可调用的受控内容对象

Prompt 不定位为：
1. Prompt 中台
2. Prompt 评测平台
3. Prompt 工作流平台
4. Prompt 运营平台

### 3.2 产品结构口径

1. 一级菜单至少包含：`知识库`、`Prompt`、`智能体`。
2. Prompt 与知识库、智能体为并列关系。
3. Prompt 页面沿用原系统后台布局，不另起新的产品框架。
4. Prompt 在资产模型、共享机制、绑定调用机制上复用现有能力。

## 4. 权限与共享策略

### 4.1 统一策略

Prompt 必须复用知识库现有权限策略：
1. 可创建在个人名下。
2. 可共享到一个或多个空间。
3. 空间成员通过加入空间获得访问前提。
4. 成员实际权限由“共享给空间的权限”与“成员在空间内角色”共同决定。

### 4.2 不再采用的策略

以下口径不再作为 Prompt 主权限模型：
1. 单独按部门配权限
2. 单独按公司层级配权限
3. 为 Prompt 发明不同于知识库的另一套协作规则

### 4.3 空间共享规则

Prompt 资产需支持：
1. 共享到指定空间
2. 查看已共享到哪些空间
3. 更新共享给空间的权限
4. 取消共享

规则要求：
1. 同一 Prompt 可共享到多个空间。
2. 每个空间的共享关系独立配置。
3. 共享到空间不等于成员自动可编辑。
4. 最终权限仍受空间角色限制。

## 5. 核心需求目标

### 5.1 业务目标

1. 让可复用 Prompt 不再散落在聊天、文档和个人本地。
2. 让 Prompt 与知识库一样具备归属、共享、权限、状态和追踪能力。
3. 让 Prompt 在多人协作中可被安全复用，而不是通过复制粘贴扩散。

### 5.2 产品目标

1. 在现有 Dataki 架构内补齐 Prompt 管理闭环。
2. 在现有共享空间策略内补齐 Prompt 协作闭环。
3. 在现有调用链路内补齐 Prompt 绑定与调用闭环。

## 7. 核心用户与场景

### 7.1 核心用户

1. Prompt 维护人
2. 空间管理员
3. 空间成员
4. 业务负责人
5. 主对话框使用者
6. 业务 Chat 维护人
7. 外部 Agent 接入方

### 7.2 典型场景

#### 场景 A：个人沉淀 Prompt

用户把常用 Prompt 录入 Dataki，作为个人资产维护。

#### 场景 B：共享到空间协作

用户把某个 Prompt 共享到指定空间，空间成员根据共享权限和空间角色查看、使用或编辑。

#### 场景 C：绑定到业务 Chat

业务 Chat 只允许调用已绑定、且当前空间有权访问的 Prompt。

#### 场景 D：供外部 Agent 调用

外部 Agent 仅能调用已绑定且具备共享与权限前提的 Prompt。

## 8. Prompt 资产定义

Prompt 至少满足：
1. Prompt 是正式资产类型。
2. Prompt 具备名称、用途说明、正文、归属、状态、负责人。
3. Prompt 可被查看、编辑、发布、共享、绑定 / 调用。
4. Prompt 可像知识库一样共享到空间。
5. Prompt 在导航上独立可见，而不是藏在知识库内页里。

## 9. Prompt 基础字段

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 名称 | Prompt 资产名称 | 是 |
| 用途说明 | 描述使用场景与目标 | 是 |
| 正文内容 | Prompt 主体内容 | 是 |
| 变量说明 | 参数、占位符、可替换变量说明 | 否 |
| 适用模型或场景 | 适用模型、业务场景或任务类型 | 否 |
| 归属 | 个人所属或空间共享归属 | 是 |
| 负责人 | 主要维护人 | 是 |
| 状态 | 草稿 / 启用 / 停用 | 是 |
| 版本说明 | 当前版本变更摘要 | 否 |
| 最近更新时间 | 最近修改时间 | 是 |

## 10. 权限动作

Prompt 至少区分以下权限动作：

| 权限动作 | 说明 |
|----------|------|
| 查看 | 可见、可查看详情 |
| 编辑 | 可修改内容、说明、状态 |
| 发布 | 可让 Prompt 进入可用状态 |
| 共享 | 可共享到空间、取消共享、调整共享权限 |
| 绑定 / 调用 | 可挂载到消费端并被实际调用 |

补充规则：
1. 查看权限不等于绑定 / 调用权限。
2. 编辑权限不等于发布权限。
3. 共享权限不等于成员最终编辑权限。
4. 最终权限仍受空间角色约束。

## 11. 共享、绑定与调用

### 11.1 共享

Prompt 需支持：
1. 共享到指定空间
2. 获取共享空间列表
3. 更新共享权限
4. 取消共享

### 11.2 绑定

Prompt 需支持挂载到：
1. 主对话框
2. 指定业务 Chat
3. 外部 Agent

### 11.3 调用规则

调用前必须依次校验：
1. 是否已共享到当前可访问空间
2. 是否已绑定到当前消费端
3. 当前主体是否具备绑定 / 调用权限

未共享、未绑定或无权限时，不得调用。

### 11.4 调用记录

调用后需记录：
1. 调用主体
2. 调用时间
3. 调用来源类型

至少可识别：
1. `user`
2. `chat`
3. `agent`
4. `app`

## 12. 页面范围

本次至少覆盖以下页面：
1. Prompt 一级菜单入口页 / 列表页
2. Prompt 资产详情页
3. Prompt 资产新增 / 编辑页
4. Prompt 共享弹层
5. Prompt 绑定弹层或右侧抽屉

页面原则：
1. 基于原系统界面布局出图。
2. 不额外新增新的全局框架。
3. 页面必须体现空间共享，而不是重新发明部门权限。
4. 菜单层级上必须体现 Prompt 与知识库、智能体并列。

## 13. 验收标准

### 13.1 需求验收

1. Prompt 已被定义为正式资产类型。
2. Prompt 已与知识库统一使用空间共享策略。
3. Prompt 可共享到空间。
4. Prompt 可挂载到主对话框、业务 Chat、外部 Agent。
5. Prompt 调用前可正确校验共享、绑定和权限。
6. Prompt 在信息架构上是一级菜单，而不是知识库下属入口。

### 13.2 产品验收

1. Prompt 与知识库、智能体并列。
2. 页面结构仍然是原 Dataki 系统，不是新起 Prompt 平台。
3. 共享口径与知识库一致。
4. 文档中不再把部门作为主权限口径。

## 14. 一句话结论

Dataki v1.2 的 Prompt 知识资产管理，不是做一个新的 Prompt 中台，而是让 Prompt 在产品结构上与知识库、智能体并列为一级菜单，同时在权限与协作机制上继续复用知识库现有的“空间 / 分享 / 邀请 / 角色权限”规则完成管理与调用闭环。
