# Dataki v1.1 Prompt 知识资产管理需求文档
- 版本：v1.1
- 日期：2026-04-28
- 状态：Draft
- 适用范围：Dataki v1.1 补充需求
- 读者对象：产品、设计、前端、后端、测试、AI

## 1. 需求背景

Dataki v1.1 原有知识资产管理能力覆盖知识库、知识详情、分享与挂载，但未把 `Prompt` 作为一种正式知识资产纳入统一管理。

当前需要补齐的不是独立 Prompt 中台，而是：
- 让 Prompt 作为知识的一种进入现有 Dataki 资产体系
- 让 Prompt 能在现有知识库菜单下被管理
- 让 Prompt 能被主对话框、业务 Chat 与外部 Agent 调用
- 让 Prompt 的查看、编辑、发布、绑定 / 调用进入统一权限链路

本次需求以最小可用闭环为目标，不额外重构 Dataki 整体框架。

## 2. 需求目标

### 2.1 业务目标

1. 让业务可沉淀可复用的 Prompt 不再散落在聊天记录、文档和个人本地。
2. 让 Prompt 与知识库一样具备归属、权限、状态和可追踪能力。
3. 让 Prompt 能以受控方式挂载到指定消费端，而不是被任意复制使用。

### 2.2 产品目标

1. 在不新增一级系统框架的前提下，补齐 Prompt 资产管理闭环。
2. 与现有知识资产管理风格一致，优先复用原系统页面布局、筛选结构和详情交互。
3. 把 Prompt 纳入 Dataki 当前的统一资产、统一权限、统一挂载口径。

## 3. 产品定位

### 3.1 定位结论

`Prompt` 在 Dataki v1.1 中定位为一种知识资产，不定位为独立 Prompt 中台产品。

### 3.2 结构口径

1. 入口挂在知识库菜单下。
2. 页面布局沿用现有知识资产页的列表 + 详情 / 编辑逻辑。
3. 不新增新的全局导航框架。
4. 不单独扩张出 Prompt 工程平台、评测平台、工作流平台。

## 4. 需求范围

### 4.1 本次要做

1. Prompt 资产列表
2. Prompt 资产筛选
3. Prompt 资产详情
4. Prompt 资产新增 / 编辑
5. Prompt 资产启停状态管理
6. Prompt 资产版本说明展示
7. Prompt 资产挂载到主对话框 / 指定业务 Chat / 外部 Agent
8. Prompt 资产权限控制
9. Prompt 调用主体记录

### 4.2 本次不做

1. 重型 Prompt 中台
2. Prompt 自动评测体系
3. Prompt A/B 实验平台
4. Prompt 工作流编排平台
5. Prompt 自动优化引擎
6. Prompt 市场 / 社区化分发体系

## 5. 用户与场景

### 5.1 核心用户

1. 知识库管理员
2. 业务负责人
3. Prompt 维护人
4. 主对话框使用者
5. 业务 Chat 维护人
6. 外部 Agent 接入方

### 5.2 典型场景

#### 场景 A：沉淀标准 Prompt

业务负责人把常用分析 Prompt、新人指导 Prompt、报表解释 Prompt 录入 Dataki，作为正式资产维护。

#### 场景 B：挂到主对话框

用户在主对话框中选择指定 Prompt 作为当前任务模板，提高提问一致性。

#### 场景 C：挂到业务 Chat

广告、数据分析等业务 Chat 只允许调用已绑定的 Prompt，不允许自由越权读取其他 Prompt。

#### 场景 D：供外部 Agent 调用

外部 Agent 通过绑定关系使用指定 Prompt，并在调用前经过权限校验。

## 6. 核心需求

### 6.1 Prompt 资产定义

Prompt 必须进入统一知识资产模型。

最小定义要求：
1. Prompt 是一种正式资产类型。
2. Prompt 具备名称、说明、正文、归属、状态、负责人。
3. Prompt 可被查看、编辑、发布、绑定 / 调用。
4. Prompt 可被指定消费端使用。

### 6.2 Prompt 资产管理

Prompt 资产至少支持以下能力：
1. 列表浏览
2. 关键词搜索
3. 按范围筛选
4. 按状态筛选
5. 查看详情
6. 新增
7. 编辑
8. 启用 / 停用
9. 查看版本说明
10. 查看最近更新时间

### 6.3 Prompt 资产基础字段

最小字段如下：

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 名称 | Prompt 资产名称 | 是 |
| 用途说明 | 用来描述使用场景与目标 | 是 |
| 正文内容 | Prompt 主体内容 | 是 |
| 变量说明 | 参数、占位符、可替换变量说明 | 否 |
| 适用模型或场景 | 适用的模型、业务场景或任务类型 | 否 |
| 归属范围 | 个人 / 部门 / 公司等归属 | 是 |
| 负责人 | 主要维护人 | 是 |
| 状态 | 草稿 / 启用 / 停用 | 是 |
| 版本说明 | 当前版本变更摘要 | 否 |
| 最近更新时间 | 最近修改时间 | 是 |

### 6.4 Prompt 挂载与调用

Prompt 资产需支持以下挂载对象：
1. 主对话框
2. 指定业务 Chat
3. 外部 Agent

调用规则：
1. 调用前必须先校验绑定关系。
2. 调用前必须先校验权限。
3. 未绑定或无权限的消费端不得调用。
4. 需要记录调用主体，至少可识别用户、Chat、Agent 或应用来源。

### 6.5 权限要求

Prompt 资产沿用知识资产权限模型，但至少要区分以下四类动作：

| 权限动作 | 说明 |
|----------|------|
| 查看 | 可见、可查看详情 |
| 编辑 | 可修改内容、说明、状态 |
| 发布 | 可让 Prompt 进入可用状态 |
| 绑定 / 调用 | 可挂载到消费端并被实际调用 |

权限要求：
1. 查看权限不等于绑定 / 调用权限。
2. 编辑权限不等于发布权限。
3. 绑定到外部 Agent 时，必须明确绑定主体。
4. 调用记录需可追踪。

## 7. 页面范围

### 7.1 页面原则

1. 基于原系统界面布局出图。
2. 不额外新增新的全局框架。
3. 优先沿用现有知识资产列表、筛选区、详情区、编辑页结构。

### 7.2 必出页面

1. Prompt 资产列表页
2. Prompt 资产详情页
3. Prompt 资产新增 / 编辑页
4. Prompt 绑定弹层或侧栏

### 7.3 页面展示要求

#### Prompt 资产列表页

必须体现：
1. 列表标题
2. 搜索框
3. 范围筛选
4. 状态筛选
5. 最近更新时间
6. 负责人
7. 新增按钮

#### Prompt 资产详情页

必须体现：
1. 名称
2. 用途说明
3. 正文内容
4. 变量说明
5. 适用模型或场景
6. 归属范围
7. 负责人
8. 状态
9. 版本说明
10. 最近更新时间
11. 已挂载对象

#### Prompt 新增 / 编辑页

必须体现：
1. 表单结构清晰
2. 正文内容编辑区
3. 变量说明区
4. 状态设置
5. 负责人选择或展示
6. 保存与发布动作

## 8. 交互与视觉边界

1. 不改 Dataki 当前系统级布局骨架。
2. 不新增独立 Prompt 平台首页。
3. 不增加复杂工作流节点图、实验图、评测大盘。
4. 列表、详情、编辑、挂载四类页面即可支撑本次设计。
5. 风格必须延续 Dataki 现有品牌语言与登录页视觉语义。

## 9. 验收标准

### 9.1 需求验收

1. Prompt 已被定义为正式知识资产类型。
2. Prompt 可在知识库菜单下被管理。
3. Prompt 列表、详情、新增 / 编辑、挂载入口完整。
4. Prompt 可挂到主对话框、业务 Chat、外部 Agent。
5. Prompt 权限至少区分查看、编辑、发布、绑定 / 调用。

### 9.2 设计验收

1. 页面基于原系统布局，不额外起新框架。
2. 页面结构与知识资产管理风格一致。
3. 页面能直接指导后续前端落地。

## 10. 一句话结论

Dataki v1.1 的 Prompt 能力不是另起一个 Prompt 平台，而是在原有知识资产体系和原系统布局基础上，补齐 Prompt 作为知识资产的管理、权限与调用闭环。
