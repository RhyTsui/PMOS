## 三、数据模型设计

### 3.1 核心数据表

#### 文档表 (documents)
```sql
CREATE TABLE documents (
    id INTEGER PRIMARY KEY,
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    layer TEXT CHECK(layer IN ('L1', 'L2', 'L3', 'L4')),
    category TEXT,  -- 知识类型：媒体文档/业务流程/数据上报/埋点/指标
    source TEXT,    -- 来源
    tags TEXT[],    -- 标签数组
    status TEXT DEFAULT 'draft',  -- draft/published/archived
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 术语表 (terms)
```sql
CREATE TABLE terms (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    definition TEXT NOT NULL,
    category TEXT,  -- 分类：出价/定向/创意/数据/指标
    synonyms TEXT[],  -- 同义词数组
    parent_term TEXT,  -- 上位词
    related_terms TEXT[],  -- 相关术语
    examples TEXT[],  -- 使用示例
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 查询日志表 (query_logs)
```sql
CREATE TABLE query_logs (
    id INTEGER PRIMARY KEY,
    query TEXT NOT NULL,
    user_id TEXT,
    agent_id TEXT,
    results TEXT[],  -- 返回的结果 ID 列表
    feedback TEXT,  -- up/down
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### Agent 表 (agents)
```sql
CREATE TABLE agents (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    api_key TEXT UNIQUE NOT NULL,
    capabilities TEXT[],  -- 能力列表
    knowledge_scope TEXT[],  -- 知识范围
    status TEXT DEFAULT 'active',  -- active/inactive/suspended
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_active_at TIMESTAMP
);
```

### 3.2 向量数据（ChromaDB）

```python

### 4.1 检索 API

```
POST /api/v1/search
{
    "query": "oCPM 是什么意思",
    "type": "auto",  // auto/term/vector/keyword
    "limit": 10,
    "layers": ["L1", "L2", "L3", "L4"]
}

Response:
{
    "results": [
        {
            "id": "term-001",
            "type": "term",

            "content": "优化千次展现出价...",
