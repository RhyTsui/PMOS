## 新增关键需求

- 个人知识库必须默认“绝对加密”
- 除用户主动分享的部分外，平台超管无权直接查看明文
- 如需访问，必须走审批 + 审计路径，而不是隐式后台权限
- 产品服务介绍中需要明确对外表达“知识内容是加密的”

## 对产品的约束含义

- 这不是普通 RBAC 权限项，而是隐私边界与密钥边界要求
- “超管不可见”意味着后台管理端默认只应见到密文、元数据或审批态信息
- “分享部分可见”意味着分享对象、分享范围、分享时效需要成为独立模型
- “审批审计路径”意味着必须补访问申请、审批决策、访问留痕、事后审计能力

## 后续分析时需重点覆盖

- 加密边界：哪些数据端到端加密，哪些只做服务端加密
- 密钥策略：用户侧持有、托管解锁、组织托管、恢复机制
- 分享例外：被分享内容如何脱离“超管不可见”的默认规则
- 审批审计：谁能申请、谁能批准、审批后能看多久、记录如何追溯
- 对外文案：服务介绍如何承诺“加密”，同时避免过度承诺
