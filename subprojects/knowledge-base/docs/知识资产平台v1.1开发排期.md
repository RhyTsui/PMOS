# 知识资产平台 v1.1 开发排期

- 版本：v1.1
- 日期：2026-04-20
- 状态：规划中

## 1. 排期目标

v1.1 排期围绕“框架搭建完成”组织，不按功能堆砌排期。

排期主线：

1. 底座接通
2. 平台骨架
3. 场景样例
4. 演示与验收

## 2. 阶段排期

### 阶段 1：底座接通

周期：第 1 周

目标：

- WeKnora 部署可用
- 检索与引用链路跑通

交付：

- 底座环境
- 检索样例
- 引用样例

### 阶段 2：平台骨架

周期：第 2 周

目标：

- 完成资产模型、权限模型、接入/消费接口外壳

交付：

- 统一资产对象
- 权限对象
- 接入 API 壳
- 消费 API 壳

### 阶段 3：场景样例

周期：第 3 周

目标：

- 跑通三个主场景

交付：

- 个人知识接入闭环
- 数据分析 Chat 样例
- 广告业务 Chat 样例
- 会议纪要资产样例

### 阶段 4：演示层与验收

周期：第 4 周

目标：

- 形成可演示 Demo
- 完成汇报材料所需展示面

交付：

- 页面骨架
- 知识广场样例
- 修正与治理样例
- 验收清单

## 3. 周维度任务

### 第 1 周

- 底座部署
- 检索验证
- 引用返回验证

### 第 2 周

- 资产模型落地
- 权限模型落地
- 接入与消费接口外壳

### 第 3 周

- 接入样例
- 资产页样例
- 业务 Chat 样例
- 会议纪要样例

### 第 4 周

- 广场页样例
- 修正与治理样例
- 验收回归
- 演示整理

## 4. 并行建议

可并行的工作：

1. WeKnora 底座准备 与 平台对象设计
2. 接入样例准备 与 页面骨架设计
3. 数据分析 Chat 样例 与 广告业务 Chat 样例

不建议并行的工作：

1. 在底座未跑通前做大量页面细化
2. 在权限模型未明确前绑定业务 Chat
3. 在资产模型未明确前推进连接器深接入

## 5. 风险点

主要风险：

1. WeKnora 底座超出预期复杂度
2. 连接器样例准备不充分
3. 页面过早复杂化
4. Demo 范围膨胀

对应策略：

1. 底座只验证最小可用链路
2. 连接器允许以导入结果模拟
3. 页面先做骨架，不做视觉细化
4. 所有开发围绕三大主场景收敛

## 6. 排期结论

v1.1 排期建议按 4 周组织：

- 第 1 周打通底座
- 第 2 周搭起骨架
- 第 3 周跑通场景
- 第 4 周完成演示与验收

这个排期适合“框架搭建版”目标，不适合承载过多深能力扩张。
