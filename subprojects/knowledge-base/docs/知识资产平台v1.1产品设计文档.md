# 知识资产平台 v1.1 产品设计文档

- 版本：v1.1
- 日期：2026-04-20
- 状态：规划中
- 版本关系：v1.0 开发中，v1.1 进入正式规划

## 1. 版本目标

### 1.1 版本目标定义

v1.1 的目标是完成知识资产平台的框架搭建，形成“可接入、可治理、可消费”的平台骨架。

v1.1 不是单点功能迭代，而是平台框架版本。

### 1.2 成功标准

版本成功标准：

1. 平台能承载至少 4 类知识来源
2. 平台能服务至少 3 类消费方
3. 平台形成统一权限治理闭环
4. 平台形成知识资产类型框架
5. 数据分析 Chat 与广告业务 Chat 可接入知识补强链路
6. 会议纪要能以资产类型进入平台

## 2. 核心用户与场景

### 2.1 核心用户

核心用户分为两类：

1. 个人知识使用者
2. 企业业务知识使用者

企业业务知识使用者重点包含：

- 数据分析人员
- 广告业务人员
- 运营与产品人员
- 业务 Chat 的维护与运营方

### 2.2 核心场景

场景 A：个人知识接入闭环

- 用户导入会议纪要、文档、笔记、订阅内容
- 平台自动解析并沉淀为知识资产
- 用户在平台内 Chat 或外部 Chat 中直接使用

场景 B：数据分析 Chat 知识补强

- 接入数据口径、指标定义、报表说明、案例
- 数据分析 Chat 在回答时引用知识资产
- 返回可引用来源，降低口径歧义

场景 C：广告业务 Chat 知识补强

- 接入广告规则、投放策略、媒体文档、项目资料
- 广告业务 Chat 在问答时调用平台知识
- 返回来源与引用片段，提高业务可信度

场景 D：知识发现与订阅

- 用户在知识广场发现可用资产
- 将资产加入个人知识范围或订阅列表
- 后续在 Chat 中消费

## 3. 产品结构

### 3.1 一级模块

建议 v1.1 一级模块：

1. 接入中心
2. 知识资产
3. 知识消费
4. 知识广场
5. 资产治理

### 3.2 接入中心

目标：

统一定义知识进入平台的入口、来源和处理方式。

v1.1 支持的来源类型：

- 手动文本
- 文档上传
- 会议纪要文本
- 公司文档工具接入结果
- 项目系统接入结果
- 外部网页/订阅导入结果

接入中心核心能力：

- 新建接入任务
- 选择来源类型
- 配置资产归属与权限
- 查看接入状态
- 查看解析结果

### 3.3 知识资产

目标：

形成统一知识资产目录与详情体系。

首批知识资产类型：

- 正式知识
- 过程知识
- 会议纪要资产
- 业务规则/口径
- FAQ/案例
- 外部订阅内容

知识资产列表页能力：

- 按类型筛选
- 按来源筛选
- 按标签筛选
- 按权限范围筛选
- 按业务域筛选
- 搜索与排序

知识资产详情页能力：

- 查看原文与结构化内容
- 查看摘要、标签、来源
- 查看权限与责任人
- 查看版本与更新时间
- 查看引用关系
- 查看修正记录

### 3.4 会议纪要资产

会议纪要资产作为独立知识类型进入 v1.1。

当前定位：

- 只支持文本接入
- 平台负责文本解析、结构化抽取和资产化治理
- 不内建长时录音转写

最小字段：

- 标题
- 时间
- 来源
- 参与方
- 原始文本
- 摘要
- 决策项
- 待办项
- 待确认问题
- 标签
- 权限归属

### 3.5 资产治理

v1.1 不再采用“公开/不公开”模型，统一使用权限设置。

权限模板首版：

1. 仅自己
2. 指定成员
3. 指定团队
4. 全员

权限维度至少区分：

- 可见
- 可引用
- 可编辑

治理动作包含：

- 权限设置
- 版本管理
- 发布管理
- 生命周期管理
- 来源与责任人记录
- 引用关系管理
- 质量状态标记

### 3.6 知识消费

v1.1 明确的消费对象：

- 平台内 Chat
- 外部 Chat
- 数据分析 Chat
- 广告业务 Chat
- MCP / 模型调用

消费规则：

- 所有消费都要经过权限控制
- 返回结果必须支持引用片段
- 返回结果必须保留来源信息
- 消费时可限定知识范围和知识类型

### 3.7 知识广场

v1.1 的知识广场不是社区，而是发现与订阅入口。

首版能力：

- 浏览公共/团队知识
- 查看推荐资产
- 查看热门资产
- 查看专题或类型集合
- 一键加入个人知识范围或订阅清单

## 4. 关键能力设计

### 4.1 多源接入框架

首版目标不是全量连接器落地，而是把接入框架搭起来。

框架需要统一定义：

- 来源类型
- 接入任务
- 解析方式
- 资产归属
- 权限策略
- 更新策略

### 4.2 结构化解析

平台导入后不只保存原文，还要做最小结构化抽取。

首版抽取目标：

- 摘要
- 主题
- 标签
- 决策项
- 待办项
- 待确认问题

### 4.3 检索与引用

这是 v1.1 的核心用户价值。

平台必须支持：

- 按权限检索
- 按资产类型过滤
- 返回可引用片段
- 返回引用来源
- 支持被业务 Chat 挂载

### 4.4 修正与确认

知识资产平台不是只进不管。

v1.1 最小修正闭环：

- 修改摘要
- 修改标签
- 修改纪要结构化字段
- 标记内容错误
- 标记内容过时

## 5. 页面与用户流

### 5.1 页面清单

v1.1 页面清单：

- 接入列表页
- 新建接入页
- 资产列表页
- 资产详情页
- 会议纪要详情页
- 权限与引用页
- 知识广场页
- 消费配置页

### 5.2 用户流 A：个人知识接入

1. 用户创建接入任务
2. 选择文本/文档/会议纪要来源
3. 平台解析并生成知识资产
4. 用户设置权限、标签、归属
5. 用户在 Chat 中调用

### 5.3 用户流 B：业务 Chat 知识补强

1. 平台导入业务规则、口径、案例等资产
2. 绑定给数据分析 Chat 或广告业务 Chat
3. Chat 在问答时调用知识资产
4. 返回引用片段与来源

### 5.4 用户流 C：知识广场订阅

1. 用户进入知识广场
2. 浏览推荐或专题资产
3. 查看资产摘要与权限范围
4. 加入个人知识范围
5. 在 Chat 中直接使用

## 6. 范围边界

### 6.1 v1.1 内

- 多源接入框架
- 知识资产模型
- 权限治理框架
- 知识消费框架
- 知识广场最小版本
- 会议纪要资产类型
- 修正与确认最小闭环

### 6.2 v1.1 外

- 完整录音转写产品
- 平台级 AI 测试中台
- Agent / Workflow 编排平台
- 深度知识洞察系统
- 视频内容深解析
- 重度 Prompt 中台产品化

## 7. 版本结论

v1.1 产品设计的关键不是“多做几个页面”，而是建立一套未来可持续扩展的知识资产平台骨架。

这套骨架要保证：

- 个人和企业业务都能使用
- Chat 和业务 Chat 都能消费
- 会议纪要、文档、订阅等不同来源都能进入统一资产模型
- 后续可扩展到知识洞察、Prompt 资产和 Agent 汇总层
