# Dataki v1.1 页面清单与页面流
- 版本：v1.1
- 日期：2026-04-23
- 状态：Draft
- 适用范围：Dataki v1.1 首版 IA 与页面流
- 读者对象：产品、设计、前端、后端、测试、AI

## 1. 页面结构原则

Dataki v1.1 当前只围绕两条最终需求展开：

1. 进入系统
2. 品牌替换
3. 权限与系统配置治理

## 2. 一级导航建议

登录后一级导航建议收敛为：

1. 当前工作入口页
2. 设置
3. 用户管理
4. 模型 / MCP 管理

说明：

- `当前工作入口页` 保留 WeKnora 现有壳层，只做 Dataki 品牌替换
- `设置` 收敛超管高级设置与统一配置可见项
- `用户管理` 承载小闪组织结构映射与用户维护
- `模型 / MCP 管理` 只对指定超管开放

## 3. 页面清单

### 3.1 进入系统

1. 登录页
2. 首次超管注册页
3. 小闪登录回调页
4. 未授权 / 无组织映射页
5. 登录失败态页 / 提示区

### 3.2 当前工作入口页

1. 当前首页 / 工作入口页
2. 保留壳体下的品牌替换区
3. 用户当前组织态展示区

### 3.3 权限与系统配置

1. 用户管理页
2. 用户详情 / 组织信息页
3. 组织映射页
4. 超管系统配置页

### 3.4 MCP 与模型

1. MCP 管理页
2. MCP 外挂配置页
3. 默认数据部 MCP 详情页
4. 系统模型展示页

## 4. 核心页面流

### 4.1 首次超管初始化流

1. 首次进入系统
2. 检查系统是否已有超管
3. 无超管时进入首次注册页
4. 注册成功后落超管身份
5. 进入系统配置引导

### 4.2 普通用户登录流

1. 进入登录页
2. 在登录页选择小闪扫码登录或小闪手机号登录
3. 后端换取 access_token
4. 通过第三方账号查询小闪用户
5. 同步组织字段
6. 进入当前工作入口页

补充约束：

- 登录页只允许保留小闪扫码登录与小闪手机号登录
- 其他登录方式全部去掉

### 4.3 超管统一配置流

1. 超管进入系统配置页
2. 配置统一大模型服务
3. 配置统一 MCP 服务
4. 保存后向普通用户生效

### 4.4 用户与组织同步流

1. 普通用户通过小闪登录
2. 系统读取小组 / 部门 / 公司 / 集团信息
3. 用户进入系统后按统一配置使用

### 4.5 用户管理流

1. 超管进入用户管理页
2. 查看用户基础信息与组织结构
3. 执行必要的用户管理动作

## 5. 页面优先级

### P0

1. 登录页
2. 首次超管注册页
3. 当前工作入口页
4. 超管系统配置页
5. 用户管理页
6. 组织映射页
7. 系统模型展示页
8. 默认数据部 MCP 展示页

### P1

1. MCP 外挂配置页
2. 用户详情页
3. 登录失败 / 未授权态

## 6. 旧范围回收说明

下列内容不再作为首版一级页面主线：

1. 项目知识库
2. 知识广场
3. 提示词资产独立主导航
4. 通用接入中心大而全入口
5. 三类知识库页面体系
6. 以知识挂载为中心的主对话框重构

## 7. 测试关注点

1. 首次注册与普通登录是否正确分流
2. 登录页是否只保留小闪扫码登录与小闪手机号登录
3. 超管是否为唯一系统配置入口
4. 普通用户是否只能消费统一模型与 MCP 配置
5. 小闪组织结构是否同步正确
