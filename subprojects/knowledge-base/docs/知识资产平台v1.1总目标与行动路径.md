# 知识资产平台 v1.1 总目标与行动路径

- 版本：v1.1
- 日期：2026-04-21
- 状态：Active
- 定位：本项目当前唯一的 v1.1 总体目标与执行基线

## 1. v1.1 总目标

v1.1 不再只是知识库功能迭代，也不只是产品方案输出。

v1.1 的总目标是：

1. 先把企业 AI 应用与知识资产权限需求讲清楚。
2. 再完成知识资产权限设计与 WeKnora 二开边界设计。
3. 再完成本地 Demo 级实现与测试闭环。
4. 最后把已经评审通过、已本地验证的实现包交给开发做生产实现。

一句话定义：

`v1.1 = 需求澄清 -> 权限设计 -> 改造方案 -> 工程方案 -> 本地实现验证 -> 生产交接`

## 2. 已确认的核心原则

### 2.1 统一权限中心的边界

公司已有统一权限中心，但当前只负责：

1. 鉴权
2. 身份可信输入
3. 可选基础组织信息同步

不负责：

1. 知识资产权限管理
2. AI 应用挂载权限管理
3. 个人/部门/项目/公共知识资产控制面
4. 工具与数据源授权策略

结论：

`v1.1` 里权限管理控制面由本项目负责定义和落地，统一权限中心只作为上游鉴权输入。

### 2.2 实施顺序

当前不直接做生产实现。

必须先完成：

1. 需求与边界评审
2. 权限设计评审
3. 改造方案评审
4. 工程方案评审
5. 本地 Demo 实现与测试通过

然后才进入：

6. 交给开发做生产实现

### 2.3 生产推进方式

生产实现默认不做大包交付，而是按任务包逐批推进：

1. 小批实现
2. 观察效果
3. 明确影响
4. 需要时回退
5. 再推进下一批

## 3. 双层门禁结构

### 3.1 外层：项目总门禁

沿用现有 6 阶段治理框架：

1. 问题定义
2. 研究分析
3. 总体架构确认
4. 产品方案
5. PRD / 原型
6. 交付规划

### 3.2 内层：权限专项门禁

在 v1.1 主流程中嵌入 5 步权限专项流程：

1. 权限需求澄清
2. 权限设计
3. 权限设计改造方案
4. 工程实现方案
5. 本地实现与测试闭环

规则：

1. 任一权限专项阶段未通过，不得进入对应外层下一阶段。
2. 不允许把未定设计问题直接丢给开发或 AICoding 临场判断。
3. 不允许在本地验证前直接启动生产实现。

## 4. v1.1 正式交付物

### 4.1 评审类交付物

每一阶段都必须产出评审包，最少包含：

1. 结论
2. 依据
3. 未决问题
4. 决策影响
5. 风险
6. 回退点
7. SVG 或结构图
8. 下一阶段准入条件

### 4.2 设计类交付物

v1.1 设计类交付物至少包括：

1. 企业 AI 应用与知识资产权限架构图
2. 现有 WeKnora 权限架构解读
3. 权限主体模型
4. 知识资产模型
5. AI 应用挂载模型
6. 工具与数据源授权模型
7. 本地 Demo 验证范围
8. 生产任务包定义方式

### 4.3 实现类交付物

v1.1 实现类交付物至少包括：

1. 本地 Demo 工程骨架
2. 权限模型接口骨架
3. 本地验证清单
4. 任务包模板
5. 生产交接包

## 5. 本地验证目标

v1.1 的本地验证目标不是只验证一个接口，而是验证完整 Demo 闭环。

必须至少验证：

1. 个人库、部门库、项目库、公共库的统一资产模型成立。
2. AI 应用支持内置挂载和外挂挂载。
3. 权限能影响可见、检索、引用、编辑、绑定结果。
4. 数据部 AI 应用同时受知识权限和工具权限控制。
5. 关键页面、关键接口、关键测试在本地可联通。
6. 本地验证结果可直接转化为生产任务包。

## 6. 生产交接目标

生产交接不是“把方案发给开发”，而是把以下内容交完整：

1. 已评审通过的设计结论
2. 已本地验证通过的实现基线
3. 逐批任务包
4. 每个任务包的验收标准
5. 每个任务包的观察指标
6. 每个任务包的回退条件

## 7. 当前默认优先级

v1.1 当前默认优先级如下：

1. 企业统一权限主模型
2. 本地 Demo 闭环
3. 数据部与全公司共用的知识资产能力
4. AI 应用挂载权限
5. 工具与数据源权限

## 8. 评审结论集

v1.1 全流程继续使用统一评审结论：

1. `accepted`
2. `accepted-with-constraints`
3. `needs-more-research`
4. `blocked-by-architecture`
5. `rejected`

本文件生效后，所有 v1.1 推进都以这份目标与行动路径为基线，不再把 v1.1 仅理解为“方案版”或“排期版”。
