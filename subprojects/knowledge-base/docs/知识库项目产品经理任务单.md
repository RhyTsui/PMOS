# 知识库项目产品经理任务单

- 版本：v0.1
- 日期：2026-04-19
- 状态：Active
- 负责人：虚拟产品经理 Agent
- 评审人：虚拟产品主管 Agent

---

## 一、任务目标

你当前不负责输出正式业务方案，也不负责继续扩写 PRD。

你当前只负责完成知识库项目进入下一阶段前必须具备的研究包首稿。

---

## 二、必须交付的 4 份文档

### 1. `WeKnora 适配性评估`

必须回答：

1. WeKnora 解决的核心问题是什么
2. 它更像底座、组件，还是产品外壳
3. 它适不适合支撑当前知识库项目
4. 接入成本、维护成本、改造成本分别是什么
5. 采用它的主要风险是什么

### 2. `知识库竞品与开源方案对比`

至少包含：

1. 竞品 / 开源方案清单
2. 定位和适用场景
3. 优势
4. 局限
5. 与当前项目的匹配度
6. 为什么不能直接定案

### 3. `内部能力盘点`

必须说明：

1. 当前平台已有的公共能力
2. 哪些能力可以复用
3. 哪些能力缺失
4. 哪些边界必须由上层架构确认
5. 哪些内容不应由知识库项目自行重复建设

### 4. `待主管拍板问题清单`

至少包含：

1. 知识库、Prompt、Chat、Agent 的上层归属
2. 权限中心是否复用统一平台能力
3. 发布、版本、评测是否属于本项目范围
4. WeKnora 在整体架构中的层级
5. 首版做平台层还是业务层应用

---

## 三、提交格式

每份文档必须包含：

1. 核心结论
2. 证据来源
3. 不确定项
4. 影响范围
5. 请求上级决策的问题

如果缺少以上字段，视为未提交完成。

除此之外，你必须主动同步提交状态。

同步最少包含：

1. 当前已完成的文档
2. 当前正在写的文档
3. 还缺哪些输入
4. 当前卡点是什么

未主动同步的内容，不视为主管已收到。

---

## 四、禁止事项

当前阶段禁止出现以下行为：

1. 把候选方向写成正式方案
2. 写“最稳路线是”
3. 默认上层架构已确认
4. 直接推进 PRD、页面清单、用户流
5. 用个人判断替代证据

---

## 五、评审标准

你的提交会按以下标准被评审：

1. 是否有真实调研和明确来源
2. 是否把事实、判断、待确认项分开
3. 是否存在越级拍板
4. 是否准确提出了需要主管拍板的问题
5. 是否足以支撑进入“总体架构确认阶段”

主管评审结果只会是：

1. `accepted`
2. `accepted-with-constraints`
3. `needs-more-research`
4. `blocked-by-architecture`
5. `rejected`

---

## 六、一句话要求

先把研究做扎实，再提候选方向；在主管拍板前，不得把任何方向写成定案。

补充要求：

每形成一份首稿或关键更新，必须主动同步一次，不得默默持续工作而不汇报。
