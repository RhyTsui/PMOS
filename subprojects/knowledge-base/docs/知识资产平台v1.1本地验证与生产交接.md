# 知识资产平台 v1.1 本地验证与生产交接

- 版本：v1.1
- 日期：2026-04-21
- 状态：Active

## 1. 本地验证目标

本地验证不是做一个静态原型，而是要形成“能跑、能测、能评审、能拆包”的 Demo 闭环。

本地必须验证通过的内容：

1. 统一鉴权输入接入成功
2. 统一主体模型成立
3. 统一知识资产模型成立
4. AI 应用挂载模型成立
5. 权限过滤影响检索与引用结果
6. 工具权限影响数据能力调用结果
7. 关键页面与关键接口联通
8. 回归测试可重复执行

## 2. 本地验证范围

### 2.1 场景

至少覆盖 4 类场景：

1. 个人知识管理
2. 数据部公共支持库
3. 项目成员维护的 AI 应用挂库
4. 数据部 AI 应用调用知识与工具

### 2.2 资产

至少覆盖 5 类资产：

1. 个人库资产
2. 部门库资产
3. 项目库资产
4. 公共库资产
5. 应用绑定资产

### 2.3 测试

至少覆盖 5 类测试：

1. 鉴权输入测试
2. 权限判定测试
3. 检索与引用差异测试
4. 应用挂载测试
5. 工具权限测试

## 3. 本地验证完成定义

只有同时满足以下条件，才算通过本地验证：

1. Demo 主链路可运行
2. 测试结果可复现
3. 关键权限差异可观察
4. 风险点与限制已记录
5. 可拆成生产任务包

## 4. 生产交接方式

本地验证通过后，不整体甩给开发，而是按任务包交接。

默认任务包结构：

1. 任务标题
2. 目标
3. 改动边界
4. 输入依赖
5. 输出接口
6. 测试要求
7. 观察指标
8. 回退条件
9. 回退步骤

## 5. 默认任务包顺序

建议默认顺序如下：

1. 鉴权输入接入包
2. 主体与资产模型包
3. 权限判定链包
4. AI 应用挂载包
5. 工具与数据源权限包
6. 页面与治理能力包

## 6. 观察指标

每个任务包至少要有：

1. 功能通过率
2. 权限过滤准确率
3. 关键接口错误率
4. 关键回归用例通过率
5. 是否出现权限越权或漏权

## 7. 回退原则

1. 回退实现，不回退已确认的设计结论。
2. 回退按任务包边界执行，不做大面积回退。
3. 回退后必须恢复到上一个稳定任务包状态。
4. 回退原因必须写入评审记录。
