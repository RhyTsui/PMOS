# Dataki v1.1 前后端任务拆分表
- 版本：v1.1
- 日期：2026-04-23
- 状态：Draft
- 适用范围：Dataki v1.1 首版开发拆分
- 读者对象：产品、前端、后端、测试、AI

## 1. 拆分原则

1. 先改壳层与关键链路，再扩展边缘能力
2. 先完成品牌、登录、超管配置、用户管理，再进入管理侧深化
3. 所有任务都要能回挂到首版保留需求

## 2. 前端任务

| 模块 | 任务 | 优先级 | 说明 |
| --- | --- | --- | --- |
| 品牌壳 | 替换 Dataki Logo、命名、主色与必要设计资源 | P0 | 以品牌包为准 |
| 登录 | 登录页只保留小闪扫码登录与小闪手机号登录 | P0 | 去掉其他登录方式 |
| 初始化 | 保留首次超管落位能力 | P0 | 只用于系统初始化 |
| 首页 | 保留 WeKnora 现有壳结构下完成必要品牌替换 | P0 | 不做首页结构重构 |
| 设置 | 增加超管系统配置页 | P0 | 承接高级设置权限 |
| 用户管理 | 增加用户管理、组织映射、用户详情展示 | P0 | 对齐小闪字段 |
| 模型 / MCP | 新增统一模型展示与 MCP 管理页面 | P0 | 普通用户只读，超管可配 |

## 3. 后端任务

| 模块 | 任务 | 优先级 | 说明 |
| --- | --- | --- | --- |
| 认证 | 增加首次超管注册判定 | P0 | 系统初始化分流 |
| SSO | 接入小闪 token 获取与用户查询接口 | P0 | 依赖明天的密钥 |
| 用户 | 增加小闪字段持久化与组织映射 | P0 | 支持重试同步 |
| 权限 | 增加超管角色与高级设置权限控制 | P0 | 收敛到指定管理员 |
| 模型 | 收敛为超管统一配置、普通用户只读 | P0 | 控制面后置 |
| MCP | 默认内置数据部 MCP，支持超管管理 | P0 | 普通用户不开放配置 |
| 登录清理 | 去掉其他并行登录方式 | P0 | 保留小闪扫码 / 手机号两种形态 |

## 4. 测试任务

| 模块 | 任务 | 优先级 | 说明 |
| --- | --- | --- | --- |
| 登录链路 | 首次注册 / 普通登录 / 无权限登录 | P0 | 核验分流 |
| 登录方式 | 只保留小闪扫码 / 手机号登录 | P0 | 防止旧入口残留 |
| 统一配置 | 超管配置后普通用户只读生效 | P0 | 权限核心 |
| 用户管理 | 用户信息、组织映射、组织层级显示正确 | P0 | 对齐小闪返回 |
| 模型 / MCP | 模型显示、MCP 默认服务、超管可配置 | P0 | 用户主入口 |
| 小闪接口 | token 缓存、过期续期、失败兜底 | P1 | 依赖联调 |
| 回退 | 分支回退与容器回退验证 | P1 | 首版上线前必须走通 |
