## 2. 结构前提

本次出图必须先满足以下结构口径：
1. `Prompt` 是一级菜单。
2. `Prompt` 与 `知识库`、`智能体` 并列。
3. `Prompt` 不是知识库下的二级入口。
4. `Prompt` 的共享与权限继续复用知识库现有的空间机制。

## 4. 出图总原则

1. 基于原系统界面布局，不额外加东西。
2. 不新增新的系统框架。
3. Prompt 作为一级菜单出现，但页面骨架仍沿用原系统后台。
4. 不新增独立 Prompt 中台首页。
5. 不画复杂工作流、实验平台、评测大盘。
6. 只把原有知识资产管理页面换成 Prompt 管理语境。
7. 共享与权限表达必须使用空间 / 分享 / 邀请口径，不再使用部门权限口径。

## 5. 页面范围

本次 image2 至少出 5 张图：
1. Prompt 一级菜单列表页
2. Prompt 资产详情页

4. Prompt 共享弹层
5. Prompt 绑定弹层或右侧抽屉

## 6. 页面级要求

### 6.1 Prompt 一级菜单列表页

要求：
1. 左侧导航明确出现 `知识库`、`Prompt`、`智能体` 三个并列菜单。
2. 主内容区仍是原系统后台列表布局。
3. 不要改成卡片瀑布流。
4. 列表上方只保留必要筛选与新增按钮。

页面内容应体现：
1. 页面标题：Prompt 资产管理
2. 搜索框
3. 空间或共享范围筛选
4. 状态筛选
5. 列表列：名称、用途说明、归属、负责人、状态、最近更新时间
6. 新增 Prompt 按钮

### 6.2 Prompt 资产详情页

要求：
1. 沿用原系统详情页结构。
2. 信息区块清楚，适合前端直接切分。
3. 详情页要体现已共享空间与已绑定对象。

页面内容应体现：
1. 名称
2. 用途说明
3. 正文内容
4. 变量说明
5. 适用模型或场景
6. 归属
7. 负责人
8. 状态
9. 版本说明
10. 最近更新时间
11. 已共享空间
12. 已挂载对象

### 6.3 Prompt 新增 / 编辑页

要求：
1. 基于原系统表单页样式。
2. 以单页表单为主，不要做复杂多步骤。
3. 保持企业后台风格，避免花哨表达。

页面内容应体现：
1. 名称输入
2. 用途说明输入
3. 正文内容编辑区
4. 变量说明区
5. 适用模型或场景输入
6. 负责人
7. 状态切换
8. 保存按钮
9. 发布按钮

### 6.4 Prompt 共享弹层

要求：
1. 沿用知识库共享弹层风格。
2. 以空间为共享对象。
3. 不要设计部门树权限面板。

页面内容应体现：
1. 空间选择
2. 已共享空间列表
3. 共享权限选择
4. 确认共享按钮
5. 取消共享动作

### 6.5 Prompt 绑定弹层 / 抽屉

要求：
1. 沿用原系统弹层或右侧抽屉结构。
2. 不单独新开一整页。
3. 强调绑定对象与权限校验，而不是复杂流程。

页面内容应体现：
1. 绑定对象类型
2. 可选对象列表
3. 当前绑定状态
4. 权限说明
5. 确认绑定按钮

## 7. 文案要求

页面文案统一使用以下口径：
1. `Prompt 资产管理`
2. `Prompt 详情`
3. `新建 Prompt`
4. `编辑 Prompt`
5. `共享到空间`
6. `绑定到主对话框`
7. `绑定到业务 Chat`
8. `绑定到外部 Agent`

不得擅自扩写成：
1. Prompt 中台
2. Prompt 工程平台
3. Prompt 运营平台
4. Prompt 实验平台
