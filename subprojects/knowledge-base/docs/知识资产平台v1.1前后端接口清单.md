# 知识资产平台 v1.1 前后端接口清单

- 版本：v1.1
- 日期：2026-04-20
- 状态：开发定义

## 1. 设计原则

接口清单服务于 `v1.1` 框架搭建，不追求一次性定义全部生产接口。

原则：

- 先定义平台层接口，不直接暴露底座细节
- 接口围绕接入、资产、消费、治理四条链路组织
- 接口以 Demo 可跑和主场景可用为标准

## 2. 核心对象

### 2.1 asset

字段建议：

- `id`
- `name`
- `type`
- `source_id`
- `owner_id`
- `scope`
- `tags`
- `summary`
- `status`
- `updated_at`

### 2.2 asset_type

首批值：

- `formal_knowledge`
- `process_knowledge`
- `meeting_note`
- `business_rule`
- `faq_case`
- `subscription_content`

### 2.3 source

字段建议：

- `id`
- `source_type`
- `source_name`
- `import_mode`
- `last_sync_at`

### 2.4 permission_policy

字段建议：

- `visibility`
- `reference_access`
- `edit_access`
- `members`
- `teams`

### 2.5 consumer_binding

字段建议：

- `consumer_type`
- `consumer_id`
- `asset_scope`
- `asset_types`

## 3. 接入接口

### 3.1 创建接入任务

`POST /api/v1/intakes`

用途：

- 创建一个知识接入任务

请求字段：

- `source_type`
- `payload_type`
- `asset_type`
- `name`
- `content`
- `tags`
- `permission_policy`

返回字段：

- `intake_id`
- `status`

### 3.2 查询接入任务列表

`GET /api/v1/intakes`

用途：

- 获取接入任务列表

支持参数：

- `source_type`
- `status`

### 3.3 查询接入任务详情

`GET /api/v1/intakes/:id`

用途：

- 查看单次接入任务的结果

返回字段：

- 原始输入
- 结构化结果
- 入库结果
- 错误信息

## 4. 资产接口

### 4.1 查询资产列表

`GET /api/v1/assets`

支持参数：

- `keyword`
- `type`
- `source_id`
- `tag`
- `business_domain`

### 4.2 查询资产详情

`GET /api/v1/assets/:id`

返回内容：

- 基础信息
- 原文
- 摘要
- 标签
- 来源
- 权限
- 引用记录

### 4.3 更新资产摘要与标签

`PATCH /api/v1/assets/:id`

用途：

- 修正摘要、标签和基础信息

### 4.4 查询会议纪要详情

`GET /api/v1/meeting-notes/:id`

返回内容：

- 原始纪要
- 摘要
- 参与方
- 决策项
- 待办项
- 待确认问题

### 4.5 更新会议纪要结构化字段

`PATCH /api/v1/meeting-notes/:id`

用途：

- 修正会议纪要结构化内容

## 5. 治理接口

### 5.1 更新权限策略

`PATCH /api/v1/assets/:id/permissions`

用途：

- 更新可见/可引用/可编辑权限

### 5.2 查询引用记录

`GET /api/v1/assets/:id/references`

用途：

- 查询资产被哪些消费方使用

### 5.3 标记资产错误或过时

`POST /api/v1/assets/:id/flags`

用途：

- 标记 `error` 或 `outdated`

## 6. 消费接口

### 6.1 平台知识检索

`POST /api/v1/knowledge/search`

请求字段：

- `query`
- `asset_scope`
- `asset_types`
- `consumer_type`
- `consumer_id`

返回字段：

- `results`
- `references`
- `sources`

### 6.2 平台内 Chat 问答

`POST /api/v1/chat/query`

用途：

- 平台内 Chat 调用知识资产

返回要求：

- 回答正文
- 引用片段
- 来源信息

### 6.3 业务 Chat 绑定

`POST /api/v1/consumer-bindings`

用途：

- 创建消费方绑定关系

### 6.4 查询消费方绑定

`GET /api/v1/consumer-bindings`

用途：

- 查询某个 Chat 的知识范围

## 7. 广场接口

### 7.1 查询广场首页

`GET /api/v1/plaza/home`

返回内容：

- 推荐资产
- 热门资产
- 专题集合

### 7.2 查询广场资产详情

`GET /api/v1/plaza/assets/:id`

### 7.3 加入个人知识范围

`POST /api/v1/plaza/assets/:id/subscribe`

用途：

- 把公共/团队资产加入个人知识范围

## 8. 前端联调点

前端必须联调的关键点：

1. 接入任务状态轮询
2. 资产筛选条件传递
3. 权限控制导致的结果变化
4. Chat 返回中的引用片段展示
5. 会议纪要字段编辑回写

## 9. 接口边界说明

当前接口清单不包含：

- 底座原生管理接口
- 深度连接器同步协议
- 录音转写接口
- Agent 编排接口
- 测试平台评测接口

这些接口属于后续版本能力，不进入 v1.1 首版定义。

## 10. 结论

`v1.1` 接口清单重点不是求全，而是明确：

- 平台层怎么接收知识
- 平台层怎么治理资产
- 平台层怎么把知识供给给 Chat 和业务 Chat

只要这三类接口定义清楚，v1.1 框架就具备可开发性。
