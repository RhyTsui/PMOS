当前重点

这一版先把企业、部门和项目知识的接入、整理、调用和权限控制做扎实。提示词先放在知识库下面，用菜单占位，不独立做一级入口。
One Platform / Many Knowledge Flows

把企业、部门和项目知识整理好，让 AI 随时拿来用。

这不是一个只放文档的库，而是一个统一的知识服务平台。它先承接企业、部门和项目知识，同时兼容个人知识补充；既能给人查，也能给 Chat、Agent、工作流和 MCP 调用。

平台提供的服务

用户能听懂的表达

知识库服务

支持企业知识库、部门知识库、项目知识库，个人知识作为补充层，同时预留提示词资产管理。

  • 企业、部门、项目是主知识范围
  • 提示词先挂在知识库下面

知识接入服务

把会议纪要、小闪文档、小闪聊天记录、外部知识库和订阅内容持续接进来。

  • 支持文档、图片、视频、链接、脚本包
  • 支持会议 URL 与文本纪要接入

知识使用服务

让知识和提示词在 Chat、Agent、工作流、MCP 和知识洞察里真正被用起来。

  • 支持跨知识库引用分析
  • 支持按场景补强业务问答

权限服务

以小闪账号为主做登录和权限控制，每个知识库都能独立设可见、可用和可编辑范围。

  • 小闪扫码 / 手机号登录
  • 按库名与使用范围配置权限

主对话框能力

统一 AI 工作入口
主对话框 默认系统模型 / 默认 MCP 分配
在这里直接提问，或者粘贴图片、文件、视频、会议 URL。也可以引用团队知识、项目知识、公司知识，必要时再打开高级配置切换模型、MCP 和提示词。
  • 支持跨知识库分析、Skill 调用、MCP 调用和网络补充
  • 支持最近会话、建议引导、语音输入与附件上传
  • 高级配置默认关闭，默认走系统模型和系统 MCP
分析广告投放波动原因 引用公司知识库 + 项目知识库 调用 MCP 做 BI 查询 挂载提示词模板

资产类型

先立框架，再逐步扩展

业务知识

数据字典、指标说明、报表说明、埋点文档、规则说明、FAQ。

过程知识

会议纪要、讨论记录、项目记录、复盘、研究笔记和周报月报。

个人补充知识

个人收藏、研究资料、日周月报、经验总结，作为补充背景使用。

提示词资产

聊天提示词、分析模板、业务 Prompt、Agent Prompt，当前先放在知识库下面管理。

接入来源

内部内容小闪文档、小闪聊天记录转发、企业知识库、项目系统、会议纪要。
外部内容外部知识库、外部订阅、网络资料、自然语言采集任务。
多种格式PDF、Word、Excel、图片、视频、URL、脚本包。

使用去向

对人使用主对话框、业务问答、知识洞察、知识广场。
对系统使用Agent 补强、工作流、MCP、知识分发调用。
高级开关默认关闭，打开后才允许自定义模型、MCP 和提示词策略。
