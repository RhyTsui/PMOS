Layer 2 / Platform Layer

统一平台治理层

不按“公开/不公开”分，而是统一做权限、版本、发布、引用关系、生命周期、责任人和质量状态管理。

统一登录 权限控制 版本/发布 引用关系

接入与消费编排层

平台自己定义接入协议、知识域、消费绑定和业务场景路由，不直接让底座产品形态反向控制上层平台。

接入中心 知识消费配置 知识广场 修正闭环
Layer 3 / Core Asset Domains

知识资产域

核心做知识接入、结构化解析、切片、语义/逻辑分层、索引、检索、证据片段返回和资产化治理。

会议纪要资产 业务规则 正式知识 过程知识

提示词资产域

与知识资产并列，统一纳入平台，但保留独立处理链路，供 Chat、MCP、模型和后续 Agent 使用。

Chat Prompt MCP Prompt Agent Prompt 模板/变量
Layer 4 / Engine Foundation

WeKnora 中二开底座

承担知识处理与检索能力，不承担最终产品壳。重点复用文档解析、索引、检索、RAG、引用返回和部分连接器能力。

解析 Chunking Retrieval RAG

平台自建能力

你们自己掌握平台资产模型、权限体系、业务知识类型、消费接口、知识广场和运营修正，不深改 WeKnora 主系统。

资产模型 业务知识域 消费接口 广场/运营
Layer 5 / Consumers

平台内 Chat

提供平台内使用入口和验证入口，但不等于完整测试平台。

外部 Chat

知识资产平台外的 Chat 独立存在，通过权限和绑定引用平台知识。

业务 Chat

优先支持数据分析 Chat 和广告业务 Chat，作为 v1.1 的企业业务闭环。

MCP / 模型调用

面向服务和模型调用链路提供统一知识供给，后续可扩到更多自动化场景。

Layer 6 / Future Aggregation

后续 Agent 编排汇总层

不是 v1.1 主线。等 Agent 编排成熟后，知识、提示词、Chat、MCP 等能力再自然汇总到运行时编排层。

Agent 编排 运行时聚合 服务汇总

AI 测试与洞察平台

保持独立边界。知识资产平台内只做资源级验证，不吞并平台级评测、回归和统一质量门禁。

资源级验证 平台级评测独立 洞察后续扩展

当前 v1.1 重点

  • 不是做完整 AI 平台，而是把知识资产平台框架先立起来。
  • 不是只服务个人知识，也要服务数据分析 Chat 和广告业务 Chat。
  • 不是直接拿 WeKnora 当产品壳，而是把它放到底座层。

v1.1 边界

做 多源接入、知识资产模型、权限治理、消费绑定、会议纪要资产、最小知识广场。
不做 完整录音转写、深度 Agent/Workflow 编排、完整 AI 测试平台、深度多模态生产化。

一句话定位

知识资产平台统一管理个人与企业业务知识资产，为 Chat、业务 Chat、MCP 与模型调用提供可权限控制、可检索引用、可持续沉淀和修正的知识供给能力。
