# 二、📊 覆盖率评估系统（Coverage Evaluation System）

## 2.1 系统目标

量化当前系统采集能力：

> 我到底抓到了多少“真实抖音游戏广告市场”？

---

## 2.2 核心问题

当前系统最大问题：

- ❌ 不知道漏了多少广告
- ❌ 不知道覆盖是否稳定
- ❌ 无法优化策略

---

## 2.3 覆盖率定义（核心）

```
Coverage = 已采集广告 / 估计广告总量
```

---

## 2.4 三种估算方法

---

### 方法①：关键词空间估计

```text
覆盖率 ≈ 已命中关键词 / 总关键词空间
```

---

### 方法②：embedding去重估计（推荐）

```text
如果新广告相似度 > 0.9 → 已覆盖
否则 → 新增覆盖
```

---

### 方法③：广告簇估计（高级）

```text
广告 = cluster
统计 cluster 覆盖比例
```

---

## 2.5 数据结构

```json
{
  "ad_id": "xxx",
  "embedding": [],
  "cluster_id": "cluster_12",
  "source": "douyin_search",
  "is_new": true
}
```

---

## 2.6 覆盖率计算（核心算法）

```python

def coverage_rate(total_clusters, covered_clusters):
    return covered_clusters / total_clusters
```

---

## 2.7 全局覆盖评估模型

```
关键词覆盖 + 推荐流覆盖 + 相似扩展覆盖
```

最终公式：

```
Final Coverage =
  0.4 * keyword_coverage
+ 0.3 * feed_coverage
+ 0.3 * similarity_coverage
```

---

## 2.8 可视化指标（必须）

- 每日新增广告数
- 重复率
- 新增cluster数量
- 覆盖增长曲线

---

## 2.9 预警机制

```text
如果 24h 新增广告 < 阈值 → 说明漏采
```
