# Ant Design Family Frontend Acceptance Checklist

- Is `Ant Design X` treated as the frontend host baseline?
- Are foundational `Ant Design` bindings explicit?
- Are target roles, route, shell, and data refs explicit?
- Is the page a final delivery page instead of a demo shell?
- If `Pro` appears, is it explicitly marked as `reference only`, `isolated adapter layer`, or `blocked by compatibility`?
- Are dynamic surfaces typed first and A2UI second?
- Is raw HTML agent rendering prohibited?
- Are automated browser verification and final-state validation expected before user review?
