## 0. 文档目标

这份文档聚焦“提示词管理”专项，目标是把当前散落在代码和文档中的 prompt 规则，收成正式的管理能力。

---

## 1. 专项范围

### 1.1 当前专项解决的问题

1. 提示词散落在代码和临时文档里
2. 缺版本管理
3. 缺适用范围配置
4. 缺启停与回滚

### 1.2 当前不做的事情

1. 不自建评测平台
2. 不自建知识库平台
3. 不在第一批做复杂 AB 实验体系

---

## 2. 专项需求

### 2.1 目标用户

1. 产品经理
2. 运营 / 管理角色
3. 技术配置维护角色

### 2.2 核心需求

#### R-PROMPT-01 Prompt 列表与详情

需要能查看：

1. Prompt 名称
2. 分类
3. 适用业务流
4. 当前生效版本
5. 状态

#### R-PROMPT-02 Prompt 编辑与版本管理

需要支持：

1. 新建
2. 编辑
3. 版本记录
4. 发布
5. 回滚

#### R-PROMPT-03 适用范围与开关管理

需要支持：

1. 适用业务流
2. 适用 agent
3. 适用模型
4. 启停开关

---

## 3. 专项功能

### 3.1 功能分组

#### F-PROMPT-01 列表与检索

1. 列表展示
2. 分类筛选
3. 状态筛选

#### F-PROMPT-02 详情与编辑

1. 查看详情
2. 编辑内容
3. 编辑变量说明

#### F-PROMPT-03 版本管理

1. 查看历史版本
2. 发布版本
3. 回滚版本

#### F-PROMPT-04 绑定与开关

1. 绑定业务流
2. 绑定 agent
3. 配置开关

### 3.2 后端承接对象

1. `PromptConfig`
2. `PromptVersion`
3. `PromptBinding`
4. `FeatureSwitch`

### 3.3 接口建议

1. `GET /api/v1/xiaoqiao/admin/prompts`
2. `GET /api/v1/xiaoqiao/admin/prompts/{prompt_id}`
3. `POST /api/v1/xiaoqiao/admin/prompts`
4. `PUT /api/v1/xiaoqiao/admin/prompts/{prompt_id}`
5. `GET /api/v1/xiaoqiao/admin/prompts/{prompt_id}/versions`
6. `PUT /api/v1/xiaoqiao/admin/prompts/{prompt_id}/binding`
7. `GET /api/v1/xiaoqiao/admin/feature-switches`
8. `PUT /api/v1/xiaoqiao/admin/feature-switches/{switch_key}`

---

## 4. 页面与设计承接

### 4.1 管理页结构

管理页至少分成三块：

1. 左侧筛选导航
2. 中间列表区
3. 右侧详情 / 编辑区

### 4.2 列表区展示

1. Prompt 名称
2. 分类
3. 业务流
4. 当前版本
5. 状态

### 4.3 详情 / 编辑区展示

1. Prompt 正文
2. 变量说明
3. 适用范围
4. 版本历史
5. 发布 / 回滚操作

---

## 5. 实施顺序

1. 先定义对象和接口 mock schema
2. 先做列表页与详情页承接
3. 再做版本管理
4. 最后再做绑定与开关管理
