- 版本：v0.1
- 状态：初版
- 日期：2026-05-07
- 负责人：Codex + 用户

---

## 0. 文档目的

本文档用于把小乔第一阶段的核心接口正式收敛出来，作为后续后端实现与前端接入基线。

---

## 1. 接口设计原则

### 1.1 先支持运行时，不先追求全量平台化

第一阶段接口优先保证：

1. 能发起对话
2. 能形成任务
3. 能读取任务上下文与结构化结果
4. 能查看历史与 Case

### 1.2 前台统一以小乔入口承接

接口层可以有多个资源，但前端主要通过“统一对话入口 + 任务与结果读取”来组合体验。

### 1.3 轻重工作流都走统一任务框架

帮助类请求和排查类请求不应是完全不同协议。

---

## 2. 第一阶段资源总览

建议接口资源分为七组：

1. 会话 `conversations`
2. 消息 `messages`
3. 路由 `routing`
4. 任务 `tasks`
5. 结果 `results`
6. Case `cases`
7. 工作台视图 `workspace`

---

## 3. 会话接口

## 3.1 创建会话

`POST /api/v1/xiaoqiao/conversations`

### 请求

```json
{
  "user_id": "user-001",
  "title": "可选标题"
}
```

### 返回

返回 `Conversation` 基础信息。

## 3.2 获取会话列表

`GET /api/v1/xiaoqiao/conversations`

### 返回

返回当前用户的会话摘要列表。

## 3.3 获取单个会话

`GET /api/v1/xiaoqiao/conversations/{conversation_id}`

### 返回

返回会话基础信息、最近任务摘要和最后状态。

---

## 4. 消息与入口接口

## 4.1 发送消息

`POST /api/v1/xiaoqiao/conversations/{conversation_id}/messages`

### 请求

```json
{
  "content": "为什么这个计划激活比 BI 少了 28%",
  "attachments": []
}
```

### 返回

建议返回统一入口响应：

```json
{
  "message": {},
  "routing": {},
  "task": null,
  "assistant_reply": {}
}
```

### 说明

这条接口是第一阶段最关键入口。

它负责：

1. 收用户输入
2. 生成消息
3. 生成路由判断
4. 必要时创建任务
5. 返回小乔第一轮回复

---

## 5. 路由接口

## 5.1 查看某条消息的路由判断

`GET /api/v1/xiaoqiao/routing/{routing_id}`

### 返回

返回：

1. 是否业务相关
2. 当前业务域
3. 当前诉求类型
4. 当前工作流层级
5. 是否需要追问
6. 判定原因

---

## 6. 任务接口

## 6.1 获取任务列表

`GET /api/v1/xiaoqiao/tasks`

### 支持过滤

1. `task_type`
2. `workflow_level`
3. `status`
4. `conversation_id`

## 6.2 获取单个任务

`GET /api/v1/xiaoqiao/tasks/{task_id}`

### 返回

建议返回：

1. 任务基础信息
2. 上下文
3. 结果摘要
4. 证据摘要

## 6.3 更新任务状态

`PATCH /api/v1/xiaoqiao/tasks/{task_id}`

### 请求

```json
{
  "status": "running"
}
```

### 说明

第一阶段先允许内部服务更新状态，不急着开放复杂外部写入。

---

## 7. 结果接口

## 7.1 获取任务结果

`GET /api/v1/xiaoqiao/tasks/{task_id}/results`

### 返回

返回该任务的一个或多个 `WorkflowResult`。

## 7.2 写入任务结果

`POST /api/v1/xiaoqiao/tasks/{task_id}/results`

### 请求

```json
{
  "result_type": "diagnosis_report",
  "summary": "缺口主要集中在首登回传链路",
  "structured_payload": {},
  "confidence": "high",
  "next_action": "继续核对 SDK 回传与媒体映射"
}
```

---

## 8. Case 接口

## 8.1 获取 Case 列表

`GET /api/v1/xiaoqiao/cases`

## 8.2 从任务生成 Case

`POST /api/v1/xiaoqiao/tasks/{task_id}/case`

### 说明

第一阶段需要有“任务升级为 Case”的明确入口。

---

## 9. 工作台视图接口

## 9.1 获取工作台首页视图

`GET /api/v1/xiaoqiao/workspace`

### 返回

建议聚合：

1. 顶部状态摘要
2. 快捷任务模式
3. 最近任务
4. 应用支撑状态摘要

### 说明

这个接口主要服务前端首页装配，减少前端一次性拼太多独立请求。

## 9.2 获取专项页视图

建议保留：

1. `GET /api/v1/xiaoqiao/views/monitor`
2. `GET /api/v1/xiaoqiao/views/material`
3. `GET /api/v1/xiaoqiao/views/forecast`
4. `GET /api/v1/xiaoqiao/views/support`

---

## 10. 第一阶段最小接口集

如果先只做最小业务运行时，建议至少实现：

1. `POST /api/v1/xiaoqiao/conversations`
2. `POST /api/v1/xiaoqiao/conversations/{conversation_id}/messages`
3. `GET /api/v1/xiaoqiao/tasks`
4. `GET /api/v1/xiaoqiao/tasks/{task_id}`
5. `GET /api/v1/xiaoqiao/tasks/{task_id}/results`
6. `GET /api/v1/xiaoqiao/workspace`

然后第二批补：

1. `GET /api/v1/xiaoqiao/cases`
2. `POST /api/v1/xiaoqiao/tasks/{task_id}/case`
3. 专项页视图接口
4. 路由与证据读取接口
