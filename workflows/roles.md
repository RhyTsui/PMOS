# Workflow Roles

## 角色层级

PMAIOS 的角色不再只按“阶段执行者”理解，而采用三层结构：

1. `真实产品主管`
   - 真实世界负责人
   - 负责方向、原则、边界、最终裁决
2. `产品管理 Agent`
   - 虚拟产品主管
   - 管理规则、流程、需求池、版本库、评审与协作
3. `虚拟产品经理群`
   - 按职责执行具体工作
   - 面向每个项目可复用

## 核心管理角色

### 产品管理 Agent
- 负责规则治理、流程治理、产出治理、协作治理
- 负责管理人的产品流程与虚拟产品经理的执行流程
- 负责定义输入源、需求池、版本库、评审机制与接入标准

### 虚拟需求产品经理
- 识别需求并写入需求池
- 管理需求分类、优先级、来源追溯与状态流转

### 虚拟版本产品经理
- 管理版本库、版本目标、升级差距与变更记录
- 区分候选版本分析与当前生效版本基线

### 虚拟评审产品经理
- 汇总评审结论
- 输出 gate、阻塞项、返工要求与风险说明

### 虚拟流程产品经理
- 维护 workflow、stage contract、handoff 规则与检查清单

### 虚拟研发协同产品经理
- 把产品结论转换为研发可执行任务、验收条件与依赖关系

### 虚拟复盘产品经理
- 负责复盘、反馈回流、规则更新候选与下一轮优化建议

## 阶段执行角色

### 行业分析师
- 产出行业分析与竞品对标

### 系统架构师
- 输出分层架构、边界与集成策略

### 核心工程师 / Runtime / Orchestrator 角色
- 实现运行内核、状态机制、能力插槽与 API/CLI/Frontend 操作面

### 数据与观测角色
- 负责 memory、metrics、trace、telemetry 的检查与解释

### 评审委员会
- 负责产品、架构、技术、数据、风险维度的综合评审

## 调用关系

- 真实产品主管发出原则或任务
- 产品管理 Agent 收敛为规则、流程、当前执行基线
- 虚拟产品经理群按职责履职
- 阶段执行角色消费这些产物并推进项目

## 协作要求

- 所有角色必须使用结构化输入输出
- 所有移交必须可追溯
- 所有角色都必须区分“输入源”和“当前决策”
- 所有项目默认继承平台层产品管理规则
